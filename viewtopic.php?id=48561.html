I use Linux on several systems.
I have a Pelican PL-3669 XBox wireless access thingy.
Supposedly the 3669 is the same as ZCom XG-1020.
Neither company lists firmware.

I can connect to the 3669 via my laptop (Kubuntu 13.10) using the laptop wired ethernet port.
When I login, I click the wireless settings in the original firmware.
I see the screen first load one way (call this version 1), then pop, it changes to a second screen that only allows configuring the device as an access point (screen version 2). If I am quick enough, and only when I use Firefox, I can stop that page load at a certain point (at version 1), and there, I see there is an ability to change the device to any of 4 modes, Bridge, Access Point, and whatnot. Problem is, this version 1 screen isn't going to reconfigure the 3669 in that mode, as when I click the apply button, the screen returns to version 2.

According to one Youtube video (http://www.youtube.com/watch?v=3hDdvh3htoY), this device is the same as a ZCom XG-1020, which device offers those options which the Pelican screen version 2 is not allowing.

I see this issue as a firmware option. ZCom no longer lists the XG1020, so I don't think I can get that firmware.
Pelican doesn't offer a firmware update, and probably wouldn't offer a firmware that gives the user features that aren't documented.

The wiki seems to say OpenWRT offers support for the ZCom XG-1020's chipset.
I have updated networking firmware correctly a few times before.
I have compiled once before. And that compiling was successful.
I have telnetted about a thousand times to remote servers, I can probably get into the PL-3669.

I do not mind that the hardware is limited to Gplus, my intended use is to allow cellphones to get a wireless data connection in a home that only has wired ethernet - plug the 3669 into a wired port on their non-wifi router, and use the 3669 to provide data to the cellphones via wireless. You know, those pesky cellphones have no hardwire ethernet ports. The device isn't doing anything right now, and if I'm not mistaken, the XBox 360 could still use the device after a firmware adds missing features.

Anyone think this is sensible device to put OpenWRT onto?
