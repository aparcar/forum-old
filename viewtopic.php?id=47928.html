At first Hi to All, this is my first time on this forum.

I have technical problem related to WLAN overall and to be honest i can't find better than this forum to ask about it.

As topic subject says: Random data (preferably) tool for traffic generation (ideal case with parametrization possibility) for Access Point (any brand, any SoC, available trough normal retail channels) is highly desired.

Purpose for this question is quite simple. I'm involved in tests that verifying WLAN (802.11n 2.4GHz) and other 2.4GHz transmission (like Bluetooth, ZigBee etc.) interoperability. As i struggle to have particular WLAN channel occupancy (utilization) from power/time/spectrum point of view and as i don't have easy access to WLAN simulators, I'm trying to replace WLAN simulator with normal AP (or combination of AP's).

In other words i'm searching for software solution that can be run on modified FW AP and it have capability to occupy (transmit) particular WLAN channel (channels for 40MHz) and for example i can set remotely some parameter like channel, mode, traffic (for example as Mbps or 0 - 100%) etc.
All test are performed on shielded room (Faraday chamber thus no problem with pollution open space).
Currently i'm using ZAP form Ruckus (and zperf based on ZAP) however there some combination's for channels (like two 40 MHz non overlap) where overall spectrum utilization is relatively low as devices frequently drops to 20MHz mode.

Thank You in Advance.

Not really knowing anything about that but I will point out that there are many researchers using OpenWrt for research.
That research often involves generation of traffic and then compare/research/evaluate protocols etc...
That research has to be replicabable so there should be some tools that are used there ...

via scholar.google.com there are papers available like: Do You Trust Your Software-Based
Traffic Generator? ( http://wpage.unina.it/a.botta/pub/COMMAG_GENERATION.pdf )
mentioning and comparing traffic generators

There is an OpenWrt package: "Click" ( http://www.read.cs.ucla.edu/click/ ) - maybe thats of some use for you
(there is a list of research projects using it: http://www.read.cs.ucla.edu/click/projects )
