davidc502
2 Jan 2016, 19:32

If you have the 1900acs, then you want to install "shelby" image.
If you have version 1 (v1) then install "mamba" image.
If you have version 2 (v2) then install "cobra" image.

I'm not sure if trunk now includes the "nand" patch or not, but if you have problems rebooting, just post to the 1900ac forum, and I'll send you to a link for a image that does include the nand patch.

Actually, I have found trunk to be very stable, and haven't heard of anyone's router crashing. However! I always have to make the statement that your experience may differ. However, with good communication with this forum a lot of newbie issues can be resolved fairly quickly.

Best Regards,

David

P.S. If you haven't done ANY research about OpenWrt, and you install trunk, you're not going to get a web front end. You have to SSH and install LuCi (#opkg update ; opkg install luci)  << example to get a web front end.

If you don't know anything about ssh or Linux, I recommend you do a lot of reading first.

It doesn't appear that Kaloz images have been updated since the 27th of November -- Recommend for the latest updates (New wifi drivers etcl) that you go with trunk.

**EDIT**

You are welcome to install my image build (David's) which does include the latest wifi drivers and nand patch for acs.  The kernel is 4.1.13.

http://personalpages.tds.net/~davidc502/mvebu/

Post #5
davidc502
2 Jan 2016, 23:41

If trunk doesn't have the nand patch yet, then I definitely don't recommend it. The reason being is as soon as you go to boot, it's likely to revert to the Linksys Stock firmware after 3 failed boots.

EDIT -- BTW -- The risk of bricking is pretty low. Fortunately the 1900ac(s)/v1/v2 have a secondary flash system. So, if you blow it up somehow, there's a procedure to recover.

Let me know if you have further questions.

PS: I found the procedure:

1. Assuming the router is on, turn it off.
2. Turn it back on... the power light will illuminate and then go out briefly as it prepares to boot. As soon as it goes out, turn it off again. Repeat this step three times, leaving the router on after the third time you turn it off.
3. Let it boot. It should return to stock after you let it boot.

davidc502
3 Jan 2016, 00:51

Currently trunk is on 4.1.13.

These are the recovery instructions off the wiki.

"Backup Firmware Recovery

contributed by edgeman
Manually switch the WRT1900ac from a trashed flash to the secondary [backup] flash:

    Reset router by pressing reset button until PWR light starts to flash {should take ~ 15 seconds}
        Once Power LED stops flashing, power off router with power switch
    Turn power back on and Power LED will light.
        As soon as any other LED turns on, power off router with power switch
    Turn power back on and Power LED will light.
        As soon as any other LED turns on, power off router with power switch
    Turn power back on and Power LED will light.
        As soon as any other LED turns on, power off router with power switch
    Turn power back on and Power LED will light.
        Let router power up completely; it should now be on the alternate firmware "
