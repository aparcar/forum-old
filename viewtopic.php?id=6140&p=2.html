For those of you working with OpenWrt micro on these devices:

There really needs to be more tweaking of OpenWrt to allow for running on these devices. As I mentioned before, if you want *ANY* non-default packages, you need to package your own custom image so that they are on the SquashFs partition, which offers better compression than does JFFS2.

Additionally, since there's no room to write to the device after pre-include any decent number of packages, and OpenWrt doesn't make exclusive use of the nvram area for configuration, you end up with a box that is stuck largely in a 'static' configuration. So, be sure to include any non-nvram configuration you want in the custom package as well.

This is currently one of my areas of interest, but it will take a lot of work to make OpenWrt useful, and will sacrifice many of the design goals of OpenWrt.

FWIW, I found the hard way that the ".bin" file produced by the dd-wrt project is the equivalent of the ".trx" file produced by the openwrt buildroot-ng project.  In order to put my custom image created with buildroot-ng on my WRT54Gv6 router, I had to tftp over the openwrt-brcm-2.4-squashfs.trx file created by the buildroot, not the .bin file created.

Hopefully this will keep someone else from denting their forhead against the wall.
