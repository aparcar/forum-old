So, recently I bricked my wr1043nd v1 but debricked succesfully (or not-so..)

but
if I am connecting the WAN cable the router just freezes and 192.168.1.1 doesn't resport
the LAN/WAN LEDs are blinking indicating some activity though...

I managed to debrick it using @phukit method, w/ the Tx Rx GND pins and USB/TTL board on a Windwos 7 x64.
Debricking commands were:

  • using just the LAN1 connector to PC (pc adressed as 192.168.0.5 w/ 255.255.255.0)
    naming openwrt-15.05.1-ar71xx-generic-tl-wr1043nd-v1-squashfs-factory.bin to code.bin
    using putty connected COM3 @ 115200 8n1, no flow
    typed tpl and then:
    erase 0xbf020000 +7c0000
    tftpboot 0x81000000 code.bin
    cp.b 0x81000000 0xbf020000 0x7c0000
    bootm 0xbf020000

    logged in to router's fw successfully

    then again, using the same configuration also
    renamed openwrt-15.05.1-ar71xx-generic-tl-wr1043nd-v1-squashfs-sysupgrade.bin to code.bin
    using putty connected COM3 @ 115200 8n1, no flow
    typed tpl and then:
    erase 0xbf020000 +7c0000
    tftpboot 0x81000000 code.bin
    cp.b 0x81000000 0xbf020000 0x7c0000
    bootm 0xbf020000

    successfully logging into fw as root

How, trying to connect to the internet, plugged WAN cable but 192.168.1.1 goes away, not responding... Same as the router

I have read the "gotchas" the the wikidev page but I can't download stuff and then flashing then directly from the router
