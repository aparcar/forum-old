Which one of those:

AA's openvpn is using openssl and therefor equivalent to BB's openvpn-openssl, only with AA there were no alternatives requiring distinguishion in package names. openvpn-polarssl will work just as fine, it's only using a different SSL library. Don't go for openvpn-nossl, it will (obviously) not support SSL connections.

bouwew
9 Mar 2015, 10:41

As Arokh describes in his topic, first post.
I use Putty (http://www.chiark.greenend.org.uk/~sgtatham/putty/).
First, connect and login to your router. At the command prompt, use the follwing commands, one by one: (text between{ } are extra info, don't copy)

cd /tmp
wget http://enduser.subsignal.org/~trondah/wdr4900/r44590/openwrt-mpc85xx-generic-tl-wdr4900-v1-squashfs-factory.bin
{this URL is an example}
wget http://enduser.subsignal.org/~trondah/wdr4900/r44590/md5sums
{I always check for a bad download}
md5sum -c md5sums 2> /dev/null | grep OK 
{If the result is "OK", then continue with:}
sysupgrade -n openwrt-mpc85xx-generic-tl-wdr4900-v1-squashfs-factory.bin
{and then you wait some time...}

Here (http://wiki.openwrt.org/doc/techref/sysupgrade) some more info on the sysupgrade command.

Happy flashing
