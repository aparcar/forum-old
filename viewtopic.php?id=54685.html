Some more information related to GRC and stealth:

- https://forum.openwrt.org/viewtopic.php … 56#p244156
- https://forum.openwrt.org/viewtopic.php?id=17949


- https://forum.openwrt.org/viewtopic.php … 94#p238894
"As for GRC, this is a ridiculous test. Responding as per how the internet is specified to work and how it should work is not a failure and is not frowned upon.

Designing systems around GRC's idea of "passing" with resulting violations internet specifications and breaking standard practices is frowned upon.
http://www.wilderssecurity.com/threads/ … ud.216892/ "


- https://forum.openwrt.org/viewtopic.php … 75#p238975 and the rest of this thread
"Well, no, as jow mentioned earlier, serious hackers aren't going to be scanning you anyway.

Secondly, there's a big flaw in that argument. It assumes any would-be attacker is scanning ports one at a time and waiting for a response, which is *never* the case. Thus, it doesn't make anything more time consuming. Whether you have all stealthed ports, closed ports, or open ports, the amount of time it takes to scan you is exactly the same."


- http://wiki.openwrt.org/doc/uci/firewal … _vs_reject
- http://www.chiark.greenend.org.uk/~pete … -vs-reject
"Conclusion
DROP offers no effective barrier to hostile forces but can dramatically slow down applications run by legitimate users. DROP should not normally be used."

-> Don't rely too much on this GRC testing...
