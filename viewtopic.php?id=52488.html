Post #6
metai
31 Aug 2014, 16:47

CMIIW, but in a "mass deploying" scenario* this would actually be more work. You would have to get the new machine to a point where you can use scp or wget to transfer the cloned image (Edit: Deletia, you don't need mtd). Meaning you'd have to either have a router you can somehow get shell access to, or, more likely, install OpenWrt the "regular" way. At which point you could already be installing your own preconfigured image.

That being said I believe that, yes, it is possible to clone OpenWrt between completely identical machines. You'd obviously not clone all partitions (especially not the ARP tables) and you should really know what you're doing though. I am not familiar with what dd/mtd does with bad blocks, though, so someone should comment on that.

*) Obviously not on a manufacturing scale.

Assuming you have access via a terminal

cd /tmp
cat /dev/mtd0 of=firmw.bin

where mtd0 is the location of the firmware
