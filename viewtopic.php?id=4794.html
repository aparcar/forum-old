If you are deathly afraid of the command-line, then OpenWRT is probably not the best distribution to use.  I'm not trying to be snippy here, nor am I an expert, but the FAQ is one clear indicator. See item 2.2:

http://wiki.openwrt.org/Faq#head-62ba2f â€¦ 58c72245ef

Is there an effort to add some web-interface stuff? Yes.  Should it supplant the CLI? IMHO, no.

I am not an expert, but if you want a lot of point-and-clickiness, then many other firmware do that already.


The nature of OpenWRT includes a lot of user-contributed packages. By the nature of this process there will be some overlap, duplication of effort, old packages versus new ones, etc.  There is less of a single-person deciding "this is the One True Way, we will not offer other packages that perform this function."  I see this as a choice that allows much freedom. The whole thought-process behind making GUI your god, means you boil things down to as few choices as possible.  For example OpenVPN.  Another firmware project thinks it's too complicated to have "server" setup screens so they cripple the OpenVPN implementation to client mode only.  With OpenWRT you can run OpenVPN with whatever configuration file you like and it is fully functional.  You can use dnsmasq, or you can use some other package to perform DHCP functions.  Other firmware often pick their "right" package and leave you out in the cold if you want other choices.  If you have PPTP, why do you need OpenVPN, IPSec, or other choices?  I suppose if I were a masochist I could run all 3 at one time with OpenWRT.  Look at Nico's testing directory, quite a lot of stuff in there eh?  Not everyone uses it though. The basic package set seen with RC4 is relatively better. It could use better documentation to be sure,  some well-written contributions would be welcome.  Jump in!

A few are valid I think, dropbear vs OpenSSH server for example. Although, dropbear is just a lightweight SSH server. So if your needs exceeded what it can do, I would probably just load OpenSSH server and disable dropbear not run both at once. Although you certainly could, just run them on different ports.
