Post #15
mk24
28 Apr 2015, 22:51

The /tmp directory is in RAM, not persistent storage.  It is normal that anything you place there will get wiped out on a reboot.

The WRT54 lacks the RAM space to do a file conversion in place.  You will need to use another computer like Zajec said.  If you don't have a computer formatted with Linux there are "live CD" projects that let you temporarily boot up Linux on your x86 without altering anything on the hard drive.

Or you can use a hex editor to remove the first 32 bytes from the bin file and save it as a trx file.  That is all that the dd command is doing.

I would also think that you could dd to standard output and pipe it to mtd on standard input but that is not for beginners.
