you need to edit /etc/config/wireless file. there is entry like "option disabled 1", you have to change value 1 to 0 or comment this line at all. in order to edit this file you have to logon using f.e. putty/ssh and type vi /etc/config/wireless , when you enter the editor, press "i", edit desired values, press "esc" and type ":wq" - it will exit vi editor. after all type "wifi". it should be all.

You may be suffering from this https://forum.openwrt.org/viewtopic.php?id=38650.  Assuming that this is the case:

If you can ssh to the box, you can try flashing it with this (1) firmware using the command

"dd if=wr1043nv1_de_3_9_17_up_boot_100331.bin of=tplink.bin skip=257 bs=512"

If that unlocks the WAN then you can either stick with that firmware or upgrade to the one that you link to in the opening post with the sysupgrade command. 

Alternatively, the TP-Link just has a single switch divided up using a vlan so there is nothing preventing you from using one of the LAN ports as your WAN port.  This would require some alterations to /etc/config/network.

1) http://rghost.net/36040799
