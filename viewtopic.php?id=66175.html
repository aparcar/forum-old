Hello Everyone.

I have installed OpenWrt firmware "Chaos Calmer 15.05-rc1" with the intent of using it as a hotspot router with the service provided by HotspotSystems.com.

The firmware flashing went fine, the splash screen prompting me to change my password appeared and configuring the router trough SSH also worked fine, but for the most part I configured all basic settings trough the graphical user interface.

The problem: After rebooting the router, all settings are just gone, the router acts as if it was never configured it in the first place. It acts as if it was the first time booting OpenWrt.

I have searched the forum and tried out various commands to try and solve the problem with no success.

Here is the link to the FTP where I downloaded the firmware from: http://www.hotspotsystem.com/firmware/openwrt/UBNT/

I also tried to go to the "System > Flash Firmware" tab on the GUI and load up the firmware i found here: http://downloads.openwrt.org/barrier_br … x/generic/

After doing so, nothing seems to have changed, and when I log on trough SSH the "HotspotSystem" domain and splash still appears, meaning the firmware flashing didn't work?

I have tried searching for hours for a solution, any help would really be appreciated.

I have 2 AirRouters, but I only need 1 up and running right now. However, I upgraded the running one from 15.05 to 15.05.1 and it worked fine. I reconfigured it after the upgrade and it saved fine.

https://downloads.openwrt.org/latest/ar … actory.bin

and/or

https://downloads.openwrt.org/latest/ar … pgrade.bin

Use the sysupgrade if you're upgrading from a previous version of OpenWRT. Us the other if your flashing from the stock firmware.

It is because of a new(er) U-Boot installed onto the Ubiquiti devices.  You mush downgrade AirOS before installing OpenWRT.  Someday it will be fixed in OpenWRT, but for now, this is the solution.

See:

https://forum.openwrt.org/viewtopic.php?id=67094
https://forum.openwrt.org/viewtopic.php?id=64326

https://wiki.openwrt.org/toh/ubiquiti/airmaxm
