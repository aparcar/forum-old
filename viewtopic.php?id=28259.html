The reason why I do this is that I want to insert my bridge module.

so  I didn't built-in bridge in kernel when kernel compiling.

I made a bridge.ko  file used in openwrt the same as bridge open source and I insert the bridge module like this.

brctl addif br0 eth0.2
brctl addif br0 eth0.3
brctl addif br0 eth0.4
brctl stp br0 0
ifconfig br0 up

When I use ping command (ping 192.168.2.3) in PC1,  PC2 receives broadcast ARP and send unicast ARP reply to PC1.

However PC1 cannot receive the ARP reply.

I use printk debug and  I see  the bridge 'br0' receive the ARP reply in eth0.4 and forward the ARP reply to PC1 using eth0.3.

But actually eth0.3 does not forward the ARP reply.

I think eth0.3 store PC1's MAC address and it cannot forward the address.
I wonder why  the bridge does not work.
