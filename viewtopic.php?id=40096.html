depends.. if it ain't broke don't fix it. After that decision you'll need to know what you want to do with your router. Now what exactly do you want to hear? Better or not is a personal opinion. Some like OpenWRT more for it's versatility, some like DD-WRT more for its rich out-of-the-box configuration. It's all up to you. If your router is supported by either of the two firmwares, basic router and wifi functionality is ok for both. So it all comes down on what you want to do with it. Need VPN? It's not up to any community to tell what's better for you. Make a decent backup of your DD-WRT setup, install OpenWRT, play with it for a week or 2 and decide what's better. (and for me that's OpenWRT)

DD-WRT is not based on OpenWrt and the two projects are completely unrelated.

OpenWrt is more heavily developed and is rather bleeding-edge. DD-WRT is old and stale, but is pretty damn stable and everything seems to work out of the box. OpenWrt has more community contributions than DD-WRT.

OpenWrt supports more platforms and has a larger collection of extra software.
The devs read the forums and are helpful. The IRC channel is also very helpful.

DD-WRT is based on WhiteRussian architecture build (Kernel 2.6) of OpenWRT which development from 2004-2006. The latest OpenWRT trunk codename Barrier Breaker (Kernel 3.6). DD-WRT also updated theirs to 3.6 but impossible to compile by the ordinary user and also there's a lot of bugs to be fix.

DD-WRT only have packages for Broadcom Chips, but if you have router with different processor like ramips your out of luck.
OpenWRT has 2000+ packages and counting which you can install from different or any router that is supported by OpenWRT. You can also create/build your own custom image with packages and configuration that you wanted in OpenWRT. While DD-WRT has packages installed only selected by their Pope. So OpenWRT is customizable in many more levels.

I would say DD-WRT web interface is easy to understand, easy to play with it but if you want more complex configuration you can't do it on just web interface so you have to use SSH the problem is their SSH is total crap and it's relies from the old architecture of the NVRAM and JFFS2. NVRAM is a storage of your configuration which is only limited to 64KB which is bad. Not only that it doesn't have a standard way of inputting configuration unlike OpenWRT it has UCI and you can install nano (it's an text editor for shell) then modify your configuration files all located in /etc/config which is easy to understand. OpenWRT is not relying on NVRAM anymore and they use more reliable SquashFS.

OpenWRT is the most optimized distro for router's and most develop than any other. So if your saying should I give it a try? I say FUCK YES! but beware you'll have to use wiki most of the time as people only have few spare time to replying on threads.  Just use your brain much better ROFL.

They say you can buy $200 dollars router making to industrial router using DD-WRT, I say it can only do basic things that ordinary firmware has but in OpenWRT I can run server type application like lightppd, Apache, PHP Server etc.etc.

If your familiar with Linux distro like Ubuntu or Fedora. OpenWRT can do it also having a PC on your router. That how powerful OpenWRT is.

See how OpenWRT is heavily develop.
