I have run into a lot of controversial information and abandoned topics on that matter..
To my understanding, a MIB in the side of the SNMP Manager (let's say PC) is a translation of the OID numbers to variables with meaningful name, and it is definitely something that can be loaded. But it is not strictly necessary.
The MIB on the side of the SNMP Agent (let's say Router) is an implementation of how the firmware will provide the queried information or use the provided values of the set/get commands. So this is something necessary to be compiled.

More than 4 years after this topic was created, I still see the folders ferolim mentioned empty or not existing. The compilation of the bleeding edge returns output messages of including the 802.11 mib, but how to access it remains a mystery.
