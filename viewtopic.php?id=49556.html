Hello, earlier today I successfully updated a TP-LINK TL-WR710N from its stock firmware to OpenWrt (openwrt-ar71xx-generic-tl-wr710n-v1-squashfs-factory.bin). Unfortunately I made a mistake when configuring the network settings so the device was unreachable afterwards. So I entered failsafe mode, mounted the partition holding 'etc' (there is no mount_root script!) and corrected my mistake.

Well, now the problem is that the device seems to be dead since the reboot. The LED blinks as follows:
- Off for ~1,5s
- Dimmed on for maybe ~200ms
- On for ~1s

It is not possible to enter failsafe mode any more, the device keeps on blinking forever. Does anybody know what the blink code means? Is there any chance of recovering the device via Ethernet? I successfully opened the case but it seems to be impossible to remove the board in order to reach the UART pins without damaging anything (glue!). In case someone did that, please tell me how.

Thanks in advance.

Post #2
rfk
22 Mar 2014, 20:44

Seems like the flash lost its content. After some cutting and glue-removing I actually managed to get the board out of the case without damaging both. With access to the serial interface it was easy to restore OpenWrt. Problem solved.
