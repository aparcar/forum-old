Hi,
maybe sb can help me out, since I cant figure it out for days now.

My setup is as follows:
clients (192.168.1.x) <--> openwrt LAN Port (192.168.1.92) || open wrt WAN Port (192.168.1.93) <--> router 1 (192.168.1.1) <----VPN & InternetGW----> router 2 (192.168.2.1) <--> clients (192.168.2.x)


The VPN between router 1 and router 2 is working fine. My clients from 192.168.1.x can ping 192.168.2.1 and clients at 192.168.2.x. Also I have internet access when 192.168.1.x clients set 192.168.1.92 as gateway and 192.168.1.92 sets 192.168.1.1 as gateway.

The Problem is, that no client from 192.168.1.x can ping 192.168.1.1. Clients from 192.168.2.x can ping 192.168.1.1 but not 192.168.1.92 or clients at 192.168.1.x.

When I add the WAN port to the VLAN1 (where the LAN Ports are active) at openwrt switch configuration everything works as expected (obviously). So my guess is that it is a firewall issue. The point I'm not getting is: even if I allow everything in the openwrt firewall  (LAN & WAN: accept input, accept output, accept forward, allow forward to, allow forward from) the problem persists.
I would think, if I allow everything, the firewall shouldn't interfere? Any ideas?

Thanks in advance.
