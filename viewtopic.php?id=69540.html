I have following two UID 0 users in my OpenWRT Backfire 10.03 /etc/passwd file:

root@backfire:~# cat /etc/passwd
root:$1$d2QcFY2J$ePAHYiqDitoxjcshASe9P2:0:0:root:/root:/bin/ash
testuser:*:0:0:testuser:/root/testuser:/bin/ash
nobody:*:65534:65534:nobody:/var:/bin/false
daemon:*:65534:65534:daemon:/var:/bin/false
root@backfire:~#

Both /root/testuser/.ssh/ directory and /root/testuser/.ssh/authorized_keys file are owned by UID 0 and GID 0. However, I'm not able to SSH into this router with username "testuser".

Is it somehow possible to enable SSH key-based authentication for multiple users with the same UID?
