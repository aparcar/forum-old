I'm running Gargoyle Open-WRT on my WNDR3700v2.  I was recently forced to update to Barrier Breaker to fix some bugs on the router, and had to reinstall Transmission as a result. 

Well I've got Transmission up and running just fine, there's just one problem - it loses 90% of its settings every time the router reboots.  I say 90%, because a lot of the settings are still saved - my entire config file is intact, the default download directory is correct, and the user name and password for the web UI and remote CLI are still there.  But the download/upload limits are reset, and most importantly, the entire list of torrents is gone.  That never used to happen.  Now every time I have to reboot the router, I lose every torrent, whether it was downloading, seeding, or completed.

Any ideas as to why this is happening?

k4i
22 Mar 2016, 04:14

Take a look at the configuration files if anything is pointing to /var, since in openwrt /var is symlinked to /tmp which in turn is mounted as tmpfs, nothing there will survive a reboot.

Post #3

k4i wrote:

Take a look at the configuration files if anything is pointing to /var, since in openwrt /var is symlinked to /tmp which in turn is mounted as tmpfs, nothing there will survive a reboot.

Yup, that was it, thank you!  In /etc/config/transmission, option config_dir was set to 'tmp/transmission', so I changed it to '/usr/transmission', and now it should save.
