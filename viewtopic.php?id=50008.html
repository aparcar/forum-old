Hi all,

I'm new here so I don't know exactly where to post this.
I have 3 rare routers that are not supported officially by OpenWRT but I would like to see what I could do about it.
As a matter of fact most of the components are already supported:
Topex Bytton HSDPA (not to be mistaken with that other bytton that is actually a asus w500gl)
CPU: Infineon adm5120
RAM: :Samsung K4S281632K-UC75 x2 (2 x 8 MB)
Flash memory: M29W320EB70N6 x2 (2 x 4 MB)
USB: VT6212L
Wireless lan: xg-623
Wireless telecom: MC8790V
Ethernet:40SS1041AXLF, TS8121CXLF

There are a lot of other chip on it because it can do a lot (video recording, ipsec, usb (2 ports!), all kind of phone function, etc...)
I don't need that much anyway. I don't even need the wireless or usb, I would like to use them as openvpn bridge.

- My questions so far would be:
1) How fast is the hardware as to compare to the linksys wrt54gl, this has obviously more memory but i'm not familiar with the cpu.
2) Can someone give me the right to write a page for that router? I have also photos, log and different command return
3) After all infos I have (see following remarks and logs/command), what should I try/do next?


- Extra info I gathered by trying different stuff:
1) it is Linux based (2.4.32) I can ssh in it but not as root
2) the web interface would let me update one of the flash with whatever file as long as I provide the right control sum for it
3) I can also update the same and the other flash mem by ssh (using the /dev/mtd/* files)
4) on boot the device first tries the first flash mem. At some point if something goes wrong it then go on with the second one. The second flash also provide a minimal web interface that allows me to re flash the first flash and the second (yes I know this is all weird I never thought a device could have 2 flash mem, and I'm not sure I got it right on the way it boots... see the log below)
5) That minimal web interface also provide a BOOTLOG!!! So I can check the way the boot goes with different openwrt images
6) I didn't find ANY file as original firmware on the internet! A sticker inside the router give the name of the two firmware installed:
TOPEX-HSUPA-1.1.1-IRDSU-T.TRX
TOPEX-1.0.9-ROOT-IEA-P.IMG
7) I successfully recover a bad flash on the first flash mem by sshing the minimal “root” firmware and dding those /dev/mtd/0 1 2 3 files back from one of the others I have.

I got them for 3 dollars each so I'm pretty ok to have fun with them and learn...
