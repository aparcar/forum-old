The easiest way to actually see if it's up or not is take another computer and check if it can see the AP or install an app on your phone that displays the whole frequency spectrum.

Examples are for Android:
https://play.google.com/store/apps/deta … ifiscanner (Wifi Overview 360)
https://play.google.com/store/apps/deta … i.analyzer (Wifi Analyzer)

I am not affiliated with them in any way.

The second one is easier to use and looks better in my opinion where the first is better at actually displaying the whole spectrum. Sometimes the second one refuses to show channels on 5GHz that are above 48 which is sad as I run my WRT1900AC at channel 108 using 80MHz and DFS. (With a DFS-patch obviously).

EDIT: Note that if you run the same SSID on both and check from a computer, most default tools won't show which band it's on, taht's why I just take my cellphone out when I want to check frequencies.

The bonus is that those apps will show you where there are good holes for you to run your wireless on.

// Stefan

l3333 wrote:
camrxr wrote:

I'm trying to upgrade to the latest snapshot that kaloz put up through the normal method I usually do (flash to factory linksys firmware and then flash back to openwrt.  Some reason though when I'm flashing back, its retaining settings from the previous version of openwrt.  Its also showing 4 different radios and I can't get anything to work.

Any suggestions on how to get it so its a fresh install and not retaining anything?

Did you uncheck 'keep settings' when flashing to stock Linksys?
Did you use openwrt_wrt1900ac_snapshot.img to get back to Openwrt?

You may try just to press and hold reset button under Openwrt to default settings.

Kaloz's latest image is kernel 3.18.11

If you have 4 radios this is probably because your previous settings are from kernel 4.0. Edit /etc/config/wireless file and delete radios 2 and 3 but also delete word 'platform' in radio 0 and 1 paths.

It should look
config wifi-device 'radio0'
    option path 'soc/soc:pcie-controller/pci0000:00/0000:00:02.0/0000:02:00.0'


That did the trick, thanks for the help.

Operating frequency
Mode: AC
Channel: 44
Width: 80

Interface configuration
Access point

Save and apply.
