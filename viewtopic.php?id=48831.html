Hi!

some time ago I've built a custom OpenWRT firmware and I'm successfully using it for some time. However, now I need to install some kernel modules. I tried to compile the kernel modules using the same build environment used for building the firmware, but when I try to install package I get an error about wrong kernel version (md5 is different for running kernel and package). So it seems I've changed something in the build environment after flashing the firmware. I don't want to reflash as there are many things set up on the router. So my question is - it there any way to get the config of the kernel on the running openwrt? The only options I've found (for general linux) are: /proc/config.gz, /boot/config* and /usr/src/kernel. I've found none of that on the router

Thanks in advance!

7 Feb 2014, 14:27

If you are using the same build environment as you did for the firmware itself, then you should be able to just use the "--force-depends" option in opkg to install the new module anyway. The new module is most likely quite compatible. There is no practical way to recompile with the same option hash.

(The kernel version checksum check is overtight.)
