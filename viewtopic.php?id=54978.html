I have fakeidentd running on Barrier Breaker and answering to all incoming IPv4 auth requests without issue.

Is there some way to configure the firewall to force all incoming IPv6 auth requests, regardless of internal LAN destination, to be answered by the router - effectively, DNAT over IPv6?

While I'm sure many would argue that IPv6 NAT is a terrible idea in the general case, for fakeident, it's the only way to fly.

Recipe:

# opkg update
# opkg install ip6tables-mod-nat

then add something like the following to /etc/firewall.user:

# Force all incoming IPv6 traffic (interface 6in4-wan6) on port 113 to 2001:a:b::c
ip6tables -t nat -A PREROUTING -i 6in4-wan6 -p tcp --dport 113 -j DNAT --to-destination [2001:a:b::c]

(Obviously, replace the interface name, port, and destination address as needed.)

Voila!
