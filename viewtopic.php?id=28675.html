Post #6
sador
16 Feb 2011, 15:42

well, that is quite simple. check for the chipset of your wireless. then check in the imagebuilder whether a driver has been ported to your target platform already. If so, it should show up unter kernel->modules->wireless devices

you can find out what chipset you have by attaching it to a linux computer and having a look at the last lines of dmesg. or if its listed on wireless.linux.org the chipset should be listed there as well.

For the how to use part - you use it exactly the same way you would use your internal wifi. create a config entry, and you are ready to go.

So far I realized:
- audio works well (still there is lack of a good mini console player like shell-fm that knows all streams, weget|madplay works for mp3 and ogg streams)
- printer server solved, works well if you disable Enable bidirectional support in windows/linux and in wrt. (raw)
- bluetooth works fine without starting some applciations (hciconfig hci0) and anyremote are all you need to have remote control
- vsftps is ok, still need to configure it, using mountd for mounting automatically flash drive
- dropbear ssh and scp works ok
- web-cam, incredbile, it works fine (Microsoft Lifecam VX-1000) with modules: kmod-usb-video kmod-i2c-core kmod-video-core kmod-video-gspca-core kmod-video-gspca-sonixj and mjpg-streamer
- 3G usb modem didn't try but usb-modeswitch, chat and comgt should be fine

- the broadcom Wi-Fi does not work, if it is enabled, and is set with some security level, it will crash the kernel ==> very sad
- I will try the USB dongle TEW-429UB C1, with Zydas druver (zd1211rw), hope it will work

All this on kernel 2.6 without any other partition for kernel.
