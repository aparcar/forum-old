I'm running an image built from the latest Backfire source, Backfire (10.03.1-RC5, r26663) to be exact. I've noticed by default that Network is started before Firewall:

root@alix:~# head /etc/init.d/network
#!/bin/sh /etc/rc.common
# Copyright (C) 2006 OpenWrt.org

START=40
STOP=90

root@alix:~# head /etc/init.d/firewall
#!/bin/sh /etc/rc.common
# Copyright (C) 2008-2010 OpenWrt.org

START=45

This leaves a small period of time where the network is running but the firewall is not. This allows things like ipv6 autoconfiguration ICMP messages to slip through, DHCP messages to come through, and all connections seem to be allowed for a small period of time.

Obviously its relatively easy to reverse the order of these init.d scripts to correct the problem. However, I'm curious to know if there's a reason why Network is started before Firewall?

Thanks!
