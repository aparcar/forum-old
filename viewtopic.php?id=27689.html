Hi,

I've spent the better part of 2 months fighting with OpenWRT in an attempt to set up a multi hop wireless link. I'd really appreciate it if someone could confirm that what I want to do is possible.

I have 3 wireless routers:

The first router is a PC Engines WRAP 2C x86 board with 2 AR5413 cards on it.
One card provides ap services for the clients on the network. Clients use 2.4 GHz. The second card forms part of the multi hop wireless link. It uses 5.8 GHz. This works fine.

The second router is a Ubiquiti RouterStation with 2 AR5413 cards on it. One card talks to the first router while the other talks to the third router. Both use 5.8GHz on different channels. This has never worked successfully with 2 cards at the same time. I've tried with and without WDS, with and without WPA2-PSK and using the ath5k and madwifi kernel modules.

The third router is a Ubiquiti RouterStation with 1 AR5413 card on it. It talks to the second router on a 5.8 GHz channel. This connects to a DSL line via a DSL modem in bridge mode.

I've tried 10.03 (stable release) and 10.03.1-rc3 on all 3 routers, keeping the version the same throughout the network each time.

The cards are all Mikrotik R52 or R52H.

Can anyone confirm that using 2 Atheros cards on a Ubiquiti Routerstation at 5.8GHz works?
