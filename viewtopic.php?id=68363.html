bolvan wrote:

luci-ssl and uhttpd-mod-tls are dummy packages for compatibility. SSL support is now builtin into uhttpd

uhttpd-mod-tls is still required for TLS ciphers... hence the uhttpd-mod-tls

LuCI itself does not implement SSL/TLS, the webserver does.

  • The only purpose for luci-ssl is it auto installs px5g, which then auto creates a ssl cert & key per the px5g commands contained with the uhttpd init.d script.

    • px5g should not be utilized, as unless OpenWrt also incorporated LEDE's backport for PolarSSL, Chrome will not display pages of any ssl webserver using libustream-polarssl, as it lacks additional GCM ciphers required as of Chrome v51.

I personally recommend utilizing an openssl.cnf, creating a self signed CA, then using that CA to issue and sign the ssl cert for uhttpd.  My github link in my signature has a pre-built openssl.cnf, which contains all information and commands you'll need to issue at the bottom of the config.
