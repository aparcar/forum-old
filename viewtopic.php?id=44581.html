I just installed OpenWRT 12.09 on my NSLU2.   I am having troubles configuring IPv6.  I referenced the following page:

http://wiki.openwrt.org/doc/uci/network6

12.09 is the latest, build on the website.  So what in the world is this documenting referencing as > 12.09?

In any case there does not appear to be an ipv6-support package, so this document must be referencing things that are not yet available...

So I go to:

http://wiki.openwrt.org/doc/howto/ipv6

None of the options listed are what I want.  I want the nslu2 to just read the radvd broadcast and configure it's IPv6 address, just like my desktop computers, laptops, tablets, etc do.   I could configure a static IPv6 address.  But currently I have two IPv6 tunnels available (one for each internet connection).  Only one of them is advertised from a router at any given time.  I would like my NSLU2 be on the current one I have advertising.

Any suggestions how I can achieve this?

Is there any way to make my IPv6 configuration appear in the web interface?


Bill

Turns out this was trivially easy.

opkg install kmod-ipv6

reboot

It works.  No additional configuration necessary.  There appears to be a bug in the web interface where it then reports IPv6 WAN as not connected.  But when I look at the network device I see it has indeed configured itself with a correct IPv6 address, and when I go to a commandline prompt, I see I can successfully do "ping6 ipv6.google.com".
