You CAN run OpenWRT on a DIR-300, and it works much better than the default firmware.
use this instructions to install it:
http://wiki.openwrt.org/toh/d-link/dir-300


this is my "network" file:

config 'interface' 'loopback'
    option 'ifname' 'lo'
    option 'proto' 'static'
    option 'ipaddr' '127.0.0.1'
    option 'netmask' '255.0.0.0'

config 'interface' 'lan'
    option 'type' 'bridge'
    option 'proto' 'static'
    option 'ipaddr' '192.168.1.1'
    option 'netmask' '255.255.255.0'
    option 'ifname' 'eth0.1'

config 'switch' 'eth0'
    option 'name' 'eth0'
    option 'reset' '1'
    option 'enable_vlan' '1'

config 'switch_vlan' 'eth0_1'
    option 'device' 'eth0'
    option 'vlan' '1'
    option 'ports' '0 1 2 3 5t'

config 'switch_vlan' 'eth0_2'
    option 'device' 'eth0'
    option 'vlan' '2'
    option 'ports' '4 5t'

config 'interface' 'wan'           
        option 'ifname' 'eth0.2'   
        option 'proto' 'pppoe'
        option 'username' 'myusername'
        option 'password' 'mypassword'
        option 'peerdns' '0'
        option 'dns' '8.8.8.8 8.8.4.4'



as you can see, my wan router is in bridge mode.

This is the "wireless" file to enable wireless support. don't worrry, the status led won't work for wireless. there are several workarounds, but I just didn't need them. Remember to edit macaddr, ssid and key to suit your needs.

config wifi-device  radio0
    option type     mac80211
    option channel  11
    option macaddr    00:00:00:00:00:00
    option hwmode    11g

    # THIS LINE DISABLES WIFI:
    # option disabled 1

config wifi-iface
    option device   radio0
    option network  lan
    option mode     ap
    option ssid     MyNetworkSSID
    option encryption psk2
    option key MyWirelessPasswordInASCII


and that's all. My DIR-300 rev A1 is working fine after that changes.
