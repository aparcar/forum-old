Post #5
jow
6 May 2009, 17:39

The only driver that supports client/sta with bridging is the proprietary broadcom driver - it's called wet (wireless ethernet) mode there.
All other chipsets would need arpnat patches for that which are not present in vanilla OpenWrt.

The only way of getting client mode working reliably on all chipsets is using routing and nat (or no nat and configure a back route on the ap).

~ JoW
