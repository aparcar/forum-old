I've flashed the router with Backfire 10.03.1-rc5, and installed kmod-ath9k, as suggested on the router's wiki page.  Even after a reboot, wifi detect gives me no output.  Can someone help me figure out what I've done wrong?

The ethernet seems to work fine, I just can't get the wifi working.

It's the DG834GT as provided by Sky in the UK.  To flash it, I followed the instructions on the wiki page, except I used a sky-specific firmware image that  found through Google (the file name is "DG834GT-1SKUKS.V1.03.87.CU3.04.img") instead of the one suggested, and using the broadcom firmware tool, set the target firmware to "DG834GT-1SKUKS", the same as in the downloaded img file.  Using the stock firmware .img caused invalid file errors (unfortunately, I've forgotten the specific error I got).

After flashing, I've connected via ethernet, disabled DHCP, given it a static IP address while I configure everything else, and connected it, again via ethernet, to my current router (a virgin superhub).  I can use that to connect to the netgear's openWrt ssh and web interface, and the netgear can connect to the internet, so I'm confident nothing's wrong with the firmware.  The "opkg install kmod-ath9k" completed successfully - or at least it gave no errors.

I've tried manually entering the /etc/config/wireless values found in a different thread on this forum, but running the "wifi" command with those values brought up "interface not supported".  I'm lost on what else to try.

This is my first foray into openWrt, so any help is greatly appreciated.  I hope I've not missed something glaringly obvious.
