Bridging is not supported in adhoc mode.  That explains your loss of connection. If you want to use adhoc mode you will have to route instead.  One way is to deactivate the WAN port and move the wireless to the WAN zone.  That will allow you to connect via wireless or a via any one of the wired LAN ports.

Another option is to use 802.11s open source mesh standard.  It supports bridging.

Ad-hoc was designed to connect multiple wireless clients without any provision to connect to other interfaces.  In otherwords, you can only connect to an adhoc interface via another ad-hoc wireless client or by routing to a different network.

Starting from default, change your LAN interface from a bridge to a single interface with eth0. That will make the router wired only.

If you then want to create a mesh network with adhoc, change your WAN interface from eth1 to ad-hoc wireless with a private static IP of say for example. 10.61.0.1. Do the same with each router using 10.61.0.2 and then 3 etc...

If you then want to connect your mesh network to the internet, configure one router with the LAN interface configured to adhoc wireless and the WAN interface connected to the internet.

Bottom line, if you try to bridge ad-hoc wireless to another interface it will stop working.  You have to use routing to connect to and from your ad-hoc network with any other interface. That is the purpose of configuring it as either the WAN or LAN interface with the other network configured for the remaining.
