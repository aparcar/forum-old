Hello there!
After several years without OpenWrt I'm back: Using a TP-Link Archer C7 v2.0 (aka AC1750) with the latest release from last week OpenWrt Chaos Calmer 15.05.1. For about ten days I was using OpenWrt Chaos Calmer 15.05 before.

The described issue was there at 15.05, too, but I wanted to test 15.05.1 to see if "my bug" got fixed. So it's not, it's still there.

About the issue:
My 5ghz wifi is running in 802.11ac mode, auto channel, 40 Mhz channel width, 100mW transmition power as access point with wpa2-psk aes/ccmp and country code DE (Germany).

When connecting to the wifi everything is fine (got a Nexus 5 and a Nexus 5X Android smartphone). After a while, just a few minutes, on the table, the phones seem to lose their connection. Getting them back from "sleep mode" both phones show the configured wifi, but they do not connect until I tap it, but they do show the wifi as "saved".

This issue doesn't show up on the second wifi I got running at 2.4ghz 802.11n. They stay connected even in "sleep mode" on the smartphones.

The last try days I tinkered a little with channel width and 802.11n mode on 5ghz, but the issue doesn't go away.

Do you have any advice for debugging? I want to exclude smartphone misconfiguration - but software bugs still possible ;-).

Best regards
ocram

I have an Archer C7 v2 as well, running Chaos Calmer. My wife recently got a Nexus 5X and she complained about connectivity issues, too. I had looked into it, but didn't find anything I could change on the router's end to avoid that. Wifi problems with the Nexus 5X seem to be rather common, though, according to various reports on the web.

The problem always occured when the Nexus woke up from sleep mode. The workaround was to reboot the Nexus and then it worked again for a while. However - and that is the good news (at least for my wife) - last week or so, she installed the latest OTA update for her phone and after that, the issue seems to have been resolved. She doesn't experience any connectivity problems anymore.

ocram wrote:

I would like to try your wifi config, would you share it?

Sure, here is my wireless config:

config wifi-device 'radio0'
    option type 'mac80211'
    option hwmode '11g'
    option path 'platform/qca955x_wmac'
    option htmode 'HT20'
    option channel '6'
    option country 'DE'
    option txpower '20'

config wifi-iface
    option device 'radio0'
    option network 'lan'
    option mode 'ap'
    option ssid 'PUT_YOUR_WIFI_NAME_HERE'
    option key 'PUT_YOUR_KEY_HERE'
    option encryption 'psk2+ccmp'

config wifi-device 'radio1'
    option type 'mac80211'
    option hwmode '11a'
    option path 'pci0000:01/0000:01:00.0'
    option htmode 'VHT80'
    option country 'DE'
    option txpower '20'
    option channel '40'

config wifi-iface
    option device 'radio1'
    option network 'lan'
    option mode 'ap'
    option ssid 'PUT_YOUR_WIFI_NAME_HERE'
    option key 'PUT_YOUR_KEY_HERE'
    option encryption 'psk2+ccmp'

There is two things I removed from that configuration - but neither of these should affect the 5GHz connectivity:
1) I use the mac filtering options to only allow known hosts
2) I have a second ssid for a guest network on the 2.4GHz radio
