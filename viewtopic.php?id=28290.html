Hello. I have a weird problem here.

I have a WR1043nd running 10.03.1 rc4 set up with a ext3 formatted hard drive, shared with samba3.

My smb.conf contains "unix charset = ISO-8859-1"

Whenever I copy a file with accented characters to a folder inside a share on the router, depending on the folder and depending on the file, I hear the hard drive starting to seek like crazy (I assume it's trying to swap to the swap partition) and then, after a few moments, I get a "failed to transfer file " message from my OS, I have lots of "I ran out of memory" messages in dmesg and eventually, the router reboots.

To make things stranger, not all files with accents or target folder exhibit these symptoms, but if I find such a file/folder combination, I can reproduce the problem on every time. Also, if I rename the file to use unaccented characters, I can move it to the share and then rename it on the share with the original accented character.

Any ideas?
