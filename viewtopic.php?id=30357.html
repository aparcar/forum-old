Post #5
eleon216
9 Jun 2011, 12:34

should the "heavy bittorrent-usage" happen on the router or should the router just be capable of handling bittorentclients behind it?
If the bittorrentdaemon should run on the router you will need a lot of ram. If it should just route bittorrenttraffic every decent router should be enough.

For the webserver: if you just run a static webpage with a lightweight webserver like lighttpd or uhttpd,  64MB should be more than enough. But if you want to run apache, php and mysql etc. you would need a lot of ram.

So a AR7161@680MHz with 64MB RAM is more than enough for serving a static webpage and routing traffic (the cpu is fast enough for at least 60Mbit/s, even with heavy iptables-rules), and you can expect a usb-storage-performance of about 10-15MByte/s.
