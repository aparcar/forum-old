semenzato
2 Oct 2008, 20:06

As a previous poster reported, the ASUS WL-520Gu is a good choice. I have one.  It runs Kamikaze r12259.  I had to compile that Kamikaze release from source, which was slow but painless.  I use Linux 2.4. There are some more hints in the SupportedHardware wiki.  At first I tried top-of-trunk and Linux 2.6, but that resulted in a nonresponsive router.  Conservatively, I then picked something that someone else had already tried.

I don't use the wireless part, but it should work.

I connected a Phidgets InterfaceKit 0/0/8 to its USB port.  It's a card with 8 USB-controlled relays.  I had to cross-compile the library but that was also relatively painless.  The only glitch was a missing pthread library on the router, which I found in the Kamikaze tree after compilation and copied over manually.  The card works fine.

However, it's unclear that USB 2.0 works well enough on this device.  It's also unclear if it's a SW or HW problem.  I think that by default only USB 1.1 is enabled, which works well enough for my purposes, but turning relays on and off is not exactly a high-bandwidth application.

The ASUS costs $50 at NewEgg, including shipping.  What we really need is a WRT54GL with a USB port.
