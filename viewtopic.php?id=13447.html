jef2000
22 Nov 2007, 02:09

Yes, you have to compile also pthsem-2.0.7-2 and argp-1.3-2
Linknx should compile succesfully with the 2 packages above, but there is a bug in uclibc++-0.1.11 that makes it crash, so I recommend also compiling and installing uclibc++-0.2.2-1

If you want to send e-mails from Linknx, you also need libesmtp-1.0.4-2
And if you want to send SMS via clickatell.com, you need libcurl (curl-7.14.0-1)

The makefiles for all these packages are in
http://ouaye.net/linknx/OpenWRT-WhiteRussian-RC6/build/
Just untar-gz them in OpenWrt-SDKxxxx/package and rebuild

Post #11
jef2000
22 Nov 2007, 22:59

I just tried to compile it for Kamikaze and have some additional remarks:
1) For curl and uclibc++, you have to remove  the following line from the makefile (it was needed for WhiteRussian but fails on Kamikaze):
TAR_OPTIONS:=-xf
2) I had package dependency problems to build everything at once. I had to remove linknx and eibd from the "package" directory, build everything else then add them again and rebuild (There must be a better way to handle package dependencies but I don't know it)
3) After I add or remove things in the package directory, I sometimes have build errors like:
Collecting package info: merging...cat: tmp/info/.packageinfo-curl: No such file
In that case I just re-start the build and the second try is successfull

jef2000

The OpenWrt SDK is not made to work this way. The way it works is:
1) You create a directory OpenWrt-SDK-xxx/package/eibd
2) You put a Makefile in OpenWrt-SDK-xxx/package/eibd
3) You go to OpenWrt-SDK-xxx and execute "make V=99"
This latest command will do everything for you. Fetch the sources from sourceforge, call ./configure with the right parameters, call make, create the ".ipk" file with the resulting build(s).

Normally, you never need "./configure" or "make" yourself, the OpenWrt-SDK does it for you.
And the .tar.gz files you can find in http://ouaye.net/linknx/OpenWRT-WhiteRussian-RC6/build/ only contains the Makefile I speak about in step 2)
Since there are some slight differences between WhiteRussian-RC6 SDK and the one you are using, the makefiles need some adaptations, that's what I described a couple of messages above.
