Does anyone have any suggestions how I might setup OpenWrt for bridged Ethernet / Ethernet pass-through so I can take advantage of some features available with OpenWrt but not use OpenWrt as my wireless solution?

For instance, I know I can setup:
- Double NAT, but that can cause some issues with port forwarding
- I could try setting up static route statements between OpenWrt and a router or firewall and only do NAT (IP masquerading) on one device
- I see there is a "bridge" package that can be installed on OpenWrt but am not certain if that will do what I need

This question stems from my desire to not be limited by slow CPU, limited memory capacity, or slow Wireless capabilities, but still be able to take advantage of unique features available through OpenWrt packages. If I had my choice, I would prefer to be able to install OpenWRT on a small computer or machine with 2 or more Gigabit Ethernet interfaces and a faster CPU with more memory but there seems to be limit of choices for beefier machines.

Since, I want to use an 802.11ac meshed network solution instead with seperate multiple WAP's and with wireless controller software. I also have a separate router I want to use for different firewall zones. What I am thinking is something like this for my home:
Cable modem --> OpenWRT wireless router (with Wifi disabled) --> (Beefier) Router or firewall --> Managed Ethernet switch --> LAN with different Zones:
- DMZ with public server(s)
- LAN devices, printers, private server(s), media server, & handheld devices, security cameras with DVR
- Private Wi-Fi, meshed, multiple WAP's
- Guest Wi-Fi, meshed, multiple WAP's

Alternatively, If WiFi doesn't work well on a Wireless router for OpenWrt with i.e. close to 1GHz CPU and 128Mbps or more memory, can I just disable the Wifi on it and use my other, faster wireless solution(s)?

Any suggestions would be most helpful. Thanks.
