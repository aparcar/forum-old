I've actually never done this, but it should be possible for you to telnet into the router running stock firmware using user "nexxadmin", password "y1n2inc.com0755" (all without quotation marks of course). Even on a very limited shell, it should give an output to "df -h", and you should immediately see if you've got a device with 4MB or 8MB. (8MB seem to be a lot less common, fwiw.)

Post #312
hackru
26 Aug 2015, 10:33

DRAM_SIZE: 128 Mbits
TOTAL_MEMORY_SIZE: 16 MBytes

this is your problem - uboot detects incorrect memory size. this probably could happen because of your arduino have ttl level of 5v and nexx has 3.3v. you can try to connect serial console after powering up router, waiting for about second, maybe this could help to detect right memory size. anyway, i recommend to buy usb2ttl with correct voltage.
