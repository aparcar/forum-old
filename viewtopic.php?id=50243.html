Nope, the device still works on it's IP, it has just all the interfaces bridged together and no nat is done. It is better to set static IP, but dynamic shoukd work too. I have my wr1043nd v2 as samba server and dumb ap, vpn router, ftp server, web server and many other things at the same time with no problem I use dumb ap mode, because openwrt does not support hardware nat, so it cannot nat gigabit traffic on a soho router.

Dump AP has only one disadvantage, you cannot forward ports from one to another, or listen to one port and send it to another ip. But it is possible to listen on ports and forward them anywhere using gnu netcat.
