Hi all,

At some point the wiki says: "if there is no WebUI (Web user interface) in the image, you can swiftly install one following this HowTo: LuCI"

How do I know whether the firmware I'm about to flash has a WebUI built in? I want to flash OpenWRT on a TP Link 1043 v1 using this file: openwrt-ar71xx-generic-tl-wr1043nd-v1-squashfs-factory

Second question, if it does come with a WebUI, can I simply access it after flashing to OpenWRT without using Telnet to set up a password?

I'm going to be honest, my knowledge of flashing does not go any further than uploading a .bin file via a WebUI. I haven't got a clue about Telnet and SSH connections so if it is not possible to simply point and click my way into a fully functional OpenWRT that comes with a WebUI I am not sure I will attempt it. The reason I want to use OpenWRT is because I've read numerous articles on how it would improve my wireless signal reach/performance, something I desperately need right now. I once flashed Tomato on my old WRT54G and that was peanuts, after uploading the .bin file I was immediately prompted with a WebUI and a default user/pw.
