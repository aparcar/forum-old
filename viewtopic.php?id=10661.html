For about two weeks I've been kicking around if I should post this or not, but since the topic has already come up in this thread I decided that this thread might be appropriate.  So...

Myself and others have a good deal of time and effort invested in getting WhiteRussian working, and with Kamikaze approaching Milestone 1 (based on the small number of open bugs tied to that release), I know that there will be many folks wanting to upgrade from the various RC versions of WhiteRussian.  What is needed at this point is some sort of guide to assist in the transition, how to make basic services work, moving settings and scripts to the new firmware, how things differ under Kamikaze, etc.  One thing I understand has changed a bit is how iptables are handled (based on this bug), and I'm sure there's plenty else that's different.


An incredibly helpful thing I can suggest right away is a web based directory listing of the files and folders, so that someone could click on etc, then open init.d, and then view the contents of the various files.  This could go a long way in helping folks know what's under the hood and what might need to be done to adapt existing setups, for those that don't have a spare router to work with.

I'm not asking the developers to spend time on this, their energy is best spent on the code, but I know that a lot of folks are running Kamikaze now and I'm sure that a few could take a stab at getting some docs in the wiki to help the rest of us make the transition.

vincentfox wrote:

I'm still very confused over the situation with 2.6 kernel and the Broadcom drivers.  It seems like that branch is kind of on hold?

The situation for Broadcom derived devices in Kamikaze is very easy:

* There are 2 targets for the same kind of hardware: brcm-2.4 and brcm47xx-2.6.
* brcm-2.4 just exists because the open source drivers for the wireless chip (bcm43xx) don't work, so we need the proprietary driver (wl.o) which only work with Linux kernel 2.4.
* brcm47xx-2.6 is the way to go and the future, and it works except for the wireless chip.
* When the bcm43xx driver (open source for the wireless chip) work on Broadcom devices (like the WRTs) the brcm-2.4 will be dropped.
* If you want to use Kamikaze in WRT like devices _and_ want to use the wireless chip, you need the brcm-2.4 target, if you don't need the wireless chip I would recommend using the brcm47xx-2.6 target.

(This should be part of a FAQ?)
