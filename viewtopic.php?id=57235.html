bagsummer wrote:

So in the general case how does one get from only knowing a package version number to the appropriate changeset number(s)?

For OpenWrt-maintained packages: searching dev.openwrt.org (by way of the search function or simply using Google). I am not aware of a more convenient way.

Other packages not maintained by OpenWrt (a good example may be OpenVPN or OpenSSL): Look at wherever the code is maintained. I believe there are changesets/revisions whenever a package is bumped to a certain version of the source, but that doesn't necessarily mean that the changes are listed in the changeset remark.

Since security vulnerabilities fixes are of particular importance, is there at least some kind of centralized information feed for OpenWrt security fixes?

No, there is no pre-processed "news feed" as far as I am aware.

As you can tell, I am somewhat puzzled how an OpenWrt admin can best accomplish this area of his responsibilities.

I'm going out on a limb and say: Either an administrator is putting the effort in himself, regularly checking for updates and following the developer mailing lists ... Or devices remain in a less well maintained state for a long time.
