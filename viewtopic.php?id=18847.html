Hi all,

I installed kamikaze 8.09 RC2 on my Asus WL-500gp a few days ago and got most of the important stuff working in the meantime.

There is one thing I realized, but I'm not sure now if it is RC2 specific or if it was there already on RC1:
I can't reach some websites which are up when connecting from different networks. My first guess was that the correct MTU (1492 in my case) was set incorrectly in the pppoe part of RC2. But checking the corresponding scripts and the commandline in ps ax proved me wrong.

After some searching I found that sometimes the MTU set by the dhcp server can cause problems, too - and that seems to be the case here.
My clients have an MTU of 1500 when they receive their settings from the kamikaze box. If I set the MTU to 1492 manually with an ifconfig command (on the client), everything works.

Now I've been looking for a way to influence the MTU which is set by dhcp in /etc/config/dhcp but I couldn't find one. Does anyone have a hint here?

Cheers,
hol9000

Ok, to directly add to my above post:
dnsmasq --help dhcp
shows the known dhcp options.
So I went on and added the following line to /etc/config/dhcp in the config 'dhcp' 'lan' section:
option 'mtu' '1492'

Unfortunately neither
/etc/init.d/dnsmasq restart
nor rebooting the router (and the client to start from a default network setup there as well) didn't help.

Any ideas?

Cheers,
hol9000
