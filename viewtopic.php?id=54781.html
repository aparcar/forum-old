I'm trying to get sshfs to work on BB14.07, however, it's not that simple as described in

http://wiki.openwrt.org/doc/howto/sshfs.client
http://wiki.openwrt.org/doc/howto/sshfs.server

Client side:

root@DIR-505-3:~# sshfs root@192.168.178.36:/etc /mnt/sshfs_dir5052 -o sshfs_debug,ssh_command="ssh -p 222"
SSHFS version 2.2
read: Connection reset by peer

Server side:

Sat Dec 27 23:01:40 2014 auth.err sshd[27613]: error: Could not load host key: /etc/ssh/ssh_host_ecdsa_key
Sat Dec 27 23:01:40 2014 auth.err sshd[27613]: error: Could not load host key: /etc/ssh/ssh_host_ed25519_key
Sat Dec 27 23:01:40 2014 auth.crit sshd[27613]: fatal: Read from socket failed: Connection reset by peer [preauth]

OK, there seem to be some keys missing, so let's create them:

root@DIR-505-2:/etc# ssh-keygen -t ecdsa -f /etc/ssh/ssh_host_ecdsa_key
Generating public/private ecdsa key pair.
Segmentation fault

root@DIR-505-2:/etc# ssh-keygen -v -t ecdsa -f /etc/ssh/ssh_host_ecdsa_key
Generating public/private ecdsa key pair.
Segmentation fault

root@DIR-505-2:/etc# ssh-keygen -A
ssh-keygen: generating new host keys: RSA1
ECDSA Segmentation fault

Any hints why the keygeneration gives a segfault?
Any hints how to get sshfs to work?

Edit: The missing keys were not the reason for the "Connection reset by peer" error, mounting sshfs works now.
However, I still have those error messages "Could not load host key", which I would like to get rid of.
