I just made a ipv6 tunnel networking testing,
the topology just like this:

Windows PC1------Router-------------Internet-------------Windows PC2
                                  |
                                  |
                                HE.NET-----------IPv6 Internet

The router connects to ipv6 internet through HE tunnel broker, and radvd runs on it, windows pc1 directly connects to the router and get ipv6 address through radvd, and ping6 to google properly.

Windows PC2 has been connected to internet, and connects to the router though tunnel, and get the ipv6 address properly, but it cannot ping to ipv6 google.

the router interface as below:
br0: 2001:470:c:1101::100/64
he-ipv6: 2001:470:c:1101::2/64
tun0: 2001:470:c:1101:a::1/80 (tunnel to other clients)

the route as:
2001:470:c:1101:a::/80 dev tun0  proto kernel  metric 256  mtu 1500 advmss 1440 hoplimit 0
2001:470:c:1101::/64 via :: dev he-ipv6  proto kernel  metric 256  mtu 1480 advmss 1420 hoplimit 0
default dev he-ipv6  metric 1000  mtu 1480 advmss 1420 hoplimit 0

the PC2's ip: 2001:470:c:1101:a:ac10:164:e1af

and PC2 can ping to 2001:470:c:1101:a::1, 2001:470:c:1101::100 and 2001:470:c:1101::2, but it cannot ping to internet ipv6.
