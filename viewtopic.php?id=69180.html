In a normal boot, one of the LEDs will start flashing rapidly, then slowly, then stay on.  If you press the reset button at the stage where it is flashing rapidly, it should start flashing very rapidly instead of changing to slowly.  This is failsafe mode.

stangri wrote:

I was under impression that failsafe is triggered by rapidly pressing/releasing hardware buttons after power on, not a single press.

Not true, but several pushes may work in practice ;-)

Only one button press/release is needed, but that needs to be inside a defined two second period in the boot process. And it can be rather hard to know the correct moment. Like mk24 says, the preinit process tries to adjust LED blinking speed to indicate the "look for failsafe trigger" moment.

Ps. I have lengthened the trigger window time to 5 seconds in my own build. There is a preinit option for it in menuconfig.
