I've used 802.11s.  The advantages include that it is built in, and has good interoperability between different hardware and software versions.

Realize that all three of the station types start by setting up a mesh interface that connects to the other routers. 
The "mesh access point" configuration adds a standard AP interface for non-mesh aware users to use.
The "mesh portal" configuration has a connection (usually wired Ethernet) to another network, which is usually a path to the Internet.
These functions are all built in to OpenWrt and can be activated by editing the config/wireless and config/network files. 

Optionally you can make a mesh portal advertise that it is the portal on the mesh, which improves routing performance.   When a mesh is set up to provide Internet access, almost all traffic on that mesh will be to or from the Internet.

You can run DHCP over a mesh if you don't want to administer the addresses of clients, for example if you want to allow third parties to bring their own hardware and join your mesh.  In most small networks you will usually want to use static IP for the backbone mesh nodes so you can always find each router to administer it.

xrated wrote:

what is the advantage of batman-adv over 802.11s ? does it provide also dhcp to the clients?

@xrated - this are great questions. I suspect the answer is that a) the science of mesh networks isn't settled, so there are *lots* of experiments going on (this is a good thing), and b) that it's "just a little too hard" to learn about and set it up.

I'm on a team to improve the OpenWrt documentation, and I would encourage all of you who have experience with a particular protocol to write it up in the Wiki (http://wiki.openwrt.org).

We have people who're using the following:

B.A.T.M.A.N
batman-adv
cjdns
80211s
babel

If each had a HOWTO (see http://wiki.openwrt.org/doc/howto/start ), with a short description of what the facility would do, and simple steps to get it going, we'd all be better off.
