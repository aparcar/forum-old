It's very strange, but i have found a solution: I use a cheap USB-TTL 3.3V CP2102 and Windows 10. I always ends up with 4Mb above. The solution is: Start Putty. Disconnect the Ground pin from CP2102 board. Power on the modem and after 1 second connect the Ground pin to the CP2102 board. Than you see the text scrolling en everything is working fine. I can enter commands. The Ground pin will hang the boot process, don't know why. If this will not work, connect a 10k resistor between 3.3v and TX-pin on the CP2102 interface. Than try again the Ground-pin trick.

Hello ...

You can use "TP-Link’s tftp emergency firmware recovery mode"

So how do you do it?

1) "wr1043ndv2_tp_recovery.bin"  Download a stock firmware file or download from here #http://www.dd-wrt.com/phpBB2/viewtopic.php?t=85237&postdays=0&postorder=asc&start=30   the tl-wr1043NDv2_webrevert.rar   whatever file you choose rename it to wr1043ndv2_tp_recovery.bin

2) Get a tftp server software, in Windows I use tftpd32 in Linux use dnsmasq

3) Use a UTP-cable just an ethernet cable

4)  While powered off, connect your router to your network card thru a network cable,
PUT YOUR LAN CABLE IN THE LAN (yellow) PORT or WAN try them both

5) Change your computer ip in 192.168.0.66 and subnet mask in 255.255.255.0

6) Open Tftpd, and browse to where your recovery firmware (wr1043ndv2_tp_recovery.bin) is.
Hit show dir button to make sure that your recovery file is listed.

7) Select from the dropdown box, your network card, where is showing 192.168.0.66

8) Press and hold the RESET button on your router, and power it on. After about 20 seconds, the firmware will upload to the router...don't release RESET button before about 20 seconds.

If you change to the Log Viewer TAB, you’ll see that the file was uploaded succesfully to the router.


9) The router will reboot, and you will be able to log in to the web interface  – http://192.168.0.1
