Hi there,

We are planning to develop a WiFi Positioning System for the Openwrt community. And we would like to know if anyone here will be interested in using.

For now, we call it LoFi (LOcation enabled WiFi) which works as follows.

1. You will hack your WiFi APs to run Openwrt, of course :-)

2. You will need to install the opkg that we'll develop.

3. You will need to create a free account in the cloud system that we are going to create. In this cloud system, we will ask you to provide the MAC addresses of smartphones/Laptops/etc that you wish LoFi to track.

4. You will need to train LoFi, i.e., you will stand at locations that you want LoFi to know and press a 'training' button in the cloud system. LoFi will then generate fingerprints for these locations and save in the cloud.

5. Now, LoFi is ready to be used anyone.

As a summary, LoFi is the WiFi based Location/Positioning system in the cloud. It will be free (unless you need awful a lot of storage in the cloud in which case a small fee will be charged, say you have < 5 smartphones and 5 APs).

I know the explanation is over simplified but hopefully you get the idea. As we haven't started doing it yet, changes are likely but the basic logic will remain.

We are planning to have a demo in < 3 months. And if you are interested, please email skyahead#gmail.com.

Comments are warm welcomed, but I may or may not visit this forum frequently.

Thanks very much
Tianji

Since the openwrt part is opensource, it doesn't matter that much that the cloud portion isn't.
You could (simply) replace it with a solution of your own, because you can see how LoFi communicates with the server and what it expects back.

Since SkyAhead mentioned that you can decide what to upload to the cloud, my guess would be that it will have a comprehensible config that makes finding out the communication easier as well.

But what I think is interesting about this project is this approach involves checking the wifi signal router side instead of device side. and thereby overcomes the problems with the iphone since there isn't an api to access the wifi information for an app on that device. that's why there aren't any (simple installable) indoor positioning apps for the iphone as far as I know.

But the hardest part of this project would be triangulating the devices and not the cloud part, so the most important part of this project would be opensource probably.

However if the calculations are done in the cloud, that complicates things a bit and on the openwrt router in this case then runs a rather simplistic app that only reports signal strengths and probably the mac address of the device to the server cloud.


(the mac address is most likely always sent to identify the device or a hash of it).
