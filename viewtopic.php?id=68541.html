Set up your network with LAN -> WAN as you want it. Then duplicate it for WLAN to WAN. That's it.

To keep LAN separate from WLAN don't make any forwards/bridges between them.

greybeard

I'd make all forward = reject.

If in doubt, try reject and either make explicit rules to accept (better practice but can be time consuming to configure and probably over kill for domestic use) or change to accept.

Zone rules are the sledge hammers of firewall config, traffic rules getting down to small hammer, port forward pretty much for nat stuff, custom rules are for masochists

Best security practice is to deny everything and only allow bare minimum both in/out and to/from a firewall. For domestic type firewalls reality is to usually allow anything to exit the firewall to the wan, Only allow specific ports back into the lan from the wan if you have servers that require it.
