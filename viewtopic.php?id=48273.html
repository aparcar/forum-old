I just bought a used TP-LINK TL-WDR3600 locally. It already has been flashed with OpenWrt Attitude Adjustment 12.09 / LuCI 0.11.1 Release (0.11.1). The router is up and running. I've have several devices, both wired and wireless connecting to the internet just fine.

I want to change the WAN DNS to 8.8.8.8 & 8.8.4.4. A Google search mentioned editing /etc/config/network. So I tried to fire up a telnet session, but it says connection refused. I can ping 192.168.1.1 just fine from a terminal session. But telnet 192.168.1.1 returns the following error:

Connection refused
telnet: Unable to connect to remote host

I am running a Mac w/ OS X 10.9.1. The connection is over through a direct ethernet connection.

Interestingly, I've already run OpenWrt x86 on VMWare and had no problem with telnet or LuCI.

Hope I don't have to re-flash the firmware. The previous owner has already done that.

Thanks in advance.

Steve

You probably have set a password for the router this disables telnet access.

To connect to the router cli, you have to use ssh. since you are on a mac.
just open up the terminal and type: ssh root@192.168.1.1 and hit enter

Additionally, you don't need to edit network settings via the console if you have LuCI on the machine. Go to Network, "Edit" on the WAN interface, Advanced Settings, uncheck "Use DNS servers advertised by peer" and fill in your own DNS servers right below.
