Hi all

The doc says that the option "wpa_group_rekey" is only relevent for Entreprise WPA.
However, after having lost hours of experiments, I tryed setting this variable to 0 in the configuration, for my home network (WPA2-PSK).
It solved my problem immediately for my 2 devices (Sagem myradio 700) that lost the connection every several minutes, and that could lose it for minutes, and even forever (some blacklisting, probably) !  They definitely don't like the rekey... Actually, on the original TPLink firmware, the corresponding option ("Group Key Update Period") was set to 0 also.

I haven't checked the code to see the impact of this variable. But it definitely has an impact on PSK also.
I'd like therefore to move the line describing this option out of the Entreprise section, to the general section.

Your opinion on this ?
