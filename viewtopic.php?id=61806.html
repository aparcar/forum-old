I've been adding CLI tricks in a text file that sits in my /etc/config directory. Misc things and Wiki, just to keep commands in a easy to reach file that I can grep. I am open to other suggestions/tricks for any one that would like to contribute.

Here is my list:

Restore from CLI
-------------------------
tar -C / -xzf /mnt/16GB/openwrt-backup-CC/backup-OpenWrt-2015-05-19.tar.gz


sysupgrade error
----------------------------
rm /lib/upgrade/merakinand.sh

#WIFI

iwinfo wlan0 scan
iwinfo wlan1 scan
iw phy0 info
iw phy1 info

Generate WIFI Config:
rm -Rv /etc/config/wireless
wifi detect > /etc/config/wireless

Openwrt reset defaults via CLI
-------------------------------------
mtd -r erase rootfs_data

#Misc
cat /proc/version
cat /proc/meminfo
cat /proc/cpuinfo
cat /proc/mtd
cat /proc/partitions

#Printer
/etc/init.d/p910nd start
/etc/init.d/p910nd enable

One nice additional trick I found to find packages with opkg without the desription, see below:

1. create a file in /usr/bin and name it whatever you like (I named mine opkg2), and used touch to create the file.

2. Use nano or any text editor to add the code:
opkg list  | cut -f 1 -d ' '

3. chmod + /usr/bin/opkg2 (or whatever you name it).

4. Lastly, as an example, from any directory you can execute:
opkg2 | grep top

And get this package name output:

getopt
htop
iftop
kamailio3-mod-textops
kamailio4-mod-textops
kamailio4-mod-topoh
kmod-ata-artop
luabitop
lxc-stop
perlbase-getopt
procps-ng-slabtop
procps-ng-top
