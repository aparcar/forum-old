Hi all, I finally got around to building and installing OpenWRT for myself and have a few questions I'd like to ask. Of course I've searched the wiki etc but since I'm new to this I haven't fully answered my questions properly yet.

I've built r32643 which was the latest when I did this yesterday. It's running on my Buffalo WHR-G301N and I'm using luci for all configuration until I learn more.

1. From what I can see uhttpd is set to listen on 0.0.0.0:80 by default, which I haven't changed. This means it's accepting connections from the internet right? not just from my lan? However when I did a port scan from grc.com and also telnetted to port 80 (and also port 22 to check), the connection was refused in all cases.
Further reading led me to believe I'd need to change some firewall rules to actually allow connections from outside my lan?
Just to be clear, I actually DON'T want anyone to be able to connect from outside my lan. So if I don't do anything it's OK as it is? Should I change uhttpd (and dropbear) to listen only to connections from 192.168.1.* ?? or there's no point and it's fine as it is now?

2. When I first installed, in the interfaces section of luci there was a LAN (br-lan) and WAN (eth1) interface. I added one more called 'modem' to allow me to connect to the internet through my cable modem (using pppoe). Do i still need the WAN interface? Both the LAN and MODEM interfaces have a constantly updating uptime, but WAN stays at 0 all the time. However all of them show numbers for the amount of bytes/packets sent and received.

3. When the router starts up the pattern of LEDs flashing is different to what it was with the default buffalo firmware. The "power" and "diag" leds come on first and stay on for a while, then the "router" light comes on. After another short while the "router" light flashes for a while, then stays solid again. At first I thought the flashing meant there was an error but everything is working fine as far as I can tell (also there's nothing obviously wrong in the system or kernel logs). Is there any info on what the normal LED sequence should be and what will happen if there really is an error? Is the sequence specific to my router, or the same for all?

That's all for now, thanks in advance!
