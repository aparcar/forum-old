I'm having speed issues on Freifunk Firmware. Let me explain you - my network setup like this:

net <-> adsl router (static ip: 192.168.1.254) with DHCP server enabled (192.168.1.x) <-> WRT54g with Freifunk Firmware (FF) via LAN #1 port (static ip: 192.168.1.1), wireless ip same as LAN #1 (192.168.1.1) and olsr routes: 192.168.1.0/30. I don't use WAN port because that WAN didn't get DHCP from adsl router. I tried to set WAN port on static ip (192.168.1.1) but it didn't work, so I changed back to LAN #1.

Then I tested desktop computer hooked up with FF box via LAN #2 port and able to get ip address from adsl router quickly, no problem. Desktop computer able to access the net and get higher speeds (via speed test sites) no problem.

When I tested laptop computer with Wifi card, it able to connect to FF box and get DHCP from adsl router, able to access the net - no problem at all. But.... when I put that laptop on speed tests,  the download speeds are less than 80kbps AND upload speeds are much higher than that!

Methink this speed issues have something to do with S45firewall script and LAN & WAN. I've been trying to figure out what rules to change. Do I have to break the br0?  Anyone have updated their S45firewall or other S scripts on their FF boxes to improve the WiFi speeds via LAN instead of WAN?

Thanks!
