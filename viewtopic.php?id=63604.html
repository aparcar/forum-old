In that capabilities list, I am pretty sure "mesh point" means that is supports IEEE802.11s mesh networking.

You could configure batman-adv to use an 802.11s interface, however that would be pretty much as klugey as forcing batman-adv to work with a "master" mode interface like you have already done.  Perhaps *slightly* more useful, in that the 802.11s mesh would be more flexible configuration-wise than you having to set up AP/STA links, though still pretty klugey.  Essentially, you'd have batman-adv mesh network on 2.4GHz, and having batman-adv essentially act as a bridge to a separate 802.11s mesh on 5GHz.  It would probably work (though there might be some perverse interactions between 802.11s and batman-adv which might bring the whole thing down), but even if it does work it'll still be pretty klugey.

I have to say I am pretty surprised that your wireless card driver does not support "adhoc", I assumed that it ought to be supported by virtually any chipset there....?  Guess I am wrong!
