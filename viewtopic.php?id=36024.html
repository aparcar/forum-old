your eth0 iface is actually 'wan' so better clarify with declarations

config interface 'wan'
    option ifname 'eth0'
    option type bridge
    option proto dhcp


you dont have 'lan'  .. so kill all references to it

easy way is just bridge 'wifi' to 'wan'

  config 'wifi-iface'
        option 'device'     'radio0'
        option 'network'    'wan'
        option 'mode'       'ap'
        option 'ssid'       'OpenWrt'
        option 'encryption' 'none'

or else edit you firewall setup according to 'wifi' and 'wan'

Your config is incorrect.
Maybe webui is easier for you.

Bridge mode:  (no nat and routing)

/etc/config/wireless
config wifi-iface
    option device 'radio0'
    option network 'wan'  <-------
    option mode 'ap'
    option ssid 'name'
    option encryption 'psk2'
    option key 'bla'

/etc/config/network
config interface 'wan'
    option type 'bridge' <------ optional if you need multi-ssid
    option ifname 'eth0'
    option proto dhcp

If you 703N behind a main router, your wifi devices will get IP from main router instead.

jow
8 Apr 2012, 15:49 
uci set network.wan=interface
uci set network.wan.proto=dhcp
uci set network.wan.ifname=eth0
uci del network.lan.ifname
uci del wireless.radio0.disabled
uci commit network
uci commit wireless
reboot
