Has anyone done any work on compiling OpenWRT for Apple's Time Capsule gen4?

SoC: Marvell 88F6281 (1.5GHz, 2x GBIC MAC, 2x USB2; lots more) http://www.marvell.com/embedded-process … Source.pdf
Flash: Winbond 25Q128BVFG (128Mbit) 16MB Flash
Memory: Hynix H5PS1G83EFR (1Gbit) 128MB DDR2 Ram (x2)
Switch: Marvell Link Street 88E6350R 7-Port GBIC Switch
WiFi: Broadcom BCM94331PCIEDUAL

I realize that at this point it's not really just an "Access Point" any more; but this thing just seems like a paperweight with proprietary software on it

The main issue I see is; there are no JTAG/UART ports; so DEV work would be very tricky (the pins on the cpu are .... small...  Any insight?

I have datasheets for every chip used on it; from what I can tell everything is connected in a standard method..

From Apple, the devices run iOS; which *IS* a *NIX environment; but it is based off of OSX..

Marvell, on the other hand, does have an SDK for the processor, but I have not had any success in getting it yet (http://www.marvell.com/embedded-process … 4_ver1.pdf)

The toolchain (for Linux 2.6.24) can be had at http://sourceforge.net/projects/dsgpl/f … %202.6.24/

Assumed functional Kernel patches can be found at http://www.xilka.com/sheeva/kernel/2/2. … /2.6.30.9/ (has nearly every kernel available)

The issue is- there is no real "safety" with this- There is no UART Port, no JTAG port... uploading custom firmware is pretty much a shot in the dark- if you get one thing wrong, you have a $400 doorstop

V/r,
Conjur
