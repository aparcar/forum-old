Hello. I'm using OpenWrt Attitude Adjustment (r30770) on TP-Link WR741ND v1.3. My ISP doesn't uses any type of VPS, only ethernet and DHCP (via a cable modem).
Sometimes WAN connection suddenly becomes lost. It may happen after several hours of work or ever at start. LAN. SSH, WiFI, LuCI are working OK. Ping requests sent from the router to internet IPs are not being delivered. Looks like all necessary routes persist.
It is possible to fix it by running /etc/init.d/network restart or pressing "Reconnect" near WAN interface in LuCI. Restarting the modem doesn't helps. Original firmware worked OK. PC, connected directly to modem, works OK too.
I thought that this is a bug of udhcpc. I installed dhcpcd, but this didn't helped. I tried to run dhcpcd when WAN becomes lost, it prints "broadcasting for a lease" and nothing more.

Yes, I solved it by adding "ifup wan" in /etc/rc.local. Also, I recently have another problem - a lot of dropped packets, it was fixed by adding "ifconfig eth1 txqueuelen 500", where eth1 is your WAN interface, in the same file.
