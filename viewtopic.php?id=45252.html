I have been building recently my images with dnsmasq-dhcpv6 just to be "future ready" though I never used DHCPv6 services on my network, but now I am no longer sure it actually makes any sense with 6relayd coming with DHCPv6 support as well.

I understand there could be only one DHCP(v6) authority so what is the recommended approach (or foreseen evolution) regarding those two modules? Are they supposed to coexist? Or is 6relayd going to replace DHCPv6 functionality of dnsmasq-dhcpv6?

Post #2
CyrusFF
14 Jul 2013, 12:44

There is no coexistence. You can use either one to your liking. The size of both features is nearly identical.

6relayd is better integrated into OpenWrt (picks up prefixes automatically and so on, has webui-support) and provides some additional features like Prefix Delegation server and NDP-proxying and / or DHCPv6 relaying. It is also integrated by default in trunk.

dnsmasq-dhcpv6 needs to be configured manually, doesn't support Prefix Delegation or proxying but can provide an additional feature called ra-names for hostname detection and is a little bit more customizable.
