pedropt wrote:

First if you want to setup a different lan ip address with DHCP activated is not possible , only static .

If you want to setup a specific IP address to the LAN port, then it has to be static; that is what "static" means here. Then you can also add a DHCP server on that port, so other computers can auto-configure.

Or you can leave the LAN port as a DHCP client, and then you must configure the DHCP server (somewhere else in your LAN network) to give your OpenWrt router a specific IP address; but you cannot configure the port both as a DHCP client and a server.

Most people has a fixed IP address and a DHCP server on then LAN network, and changing the IP address is not uncommon.

pedropt wrote:

Configuring the wan port with static ip and then configuring lan with dhcp it will not work .

Stock firmware in its default configuration works exactly the same as OpenWRT's default configuration:
The LAN has a static address 192.168.1.1 and runs a DHCP server to give clients addresses in the 192.168.1.x range.
The WAN is a DHCP client to acquire its address from the modem and /or ISP.  This can be changed if necessary when the ISP does not run DHCP but instead expects you to configure a static address.

Checking the box near the bottom of the lan network configuration page turns OFF the DHCP server.  This then requires that either you have another DHCP server, or that every user on the LAN is configured with static addresses.  In a typical simple home network where you have a single router and want it to be the DHCP server on the LAN--  leave the box UNchecked.
