Hi all. I have a certain network in mind, but the hardware my ISP gave me is giving me trouble, and I'm wondering whether there is some solution for it using OpenWRT's many many functions. I have only very basic knowledge of networking, so most things I read are over my head. That's why I came here.

I recently rewired my whole home network because we upgraded to this new ISP. I have installed OpenWRT (barrier breaker) on my TL-WDR3600 router, which is working great. Let's call this "Main Router". From the ISP we got a Modem and another router (Let's call it "Sub Router"), 2 seperate devices. Here is my setup in mind (all wired through ethernet cables):

Modem -> Main Router
               -> IP Phone
Main Router -> Sub Router (which is 50m away)
                       -> PC 1 (Home Media Server)
Sub Router  -> PC 2
                       -> IPTV

The problem lies with mr. Sub Router. The ISP has locked down the device so it can only perform a few functions. All LAN ports are pre-configured and I can't change it: LAN 1-3 is for normal internet using VLAN 500, while LAN 4 is for IPTV using VLAN 600. Furthermore, it can only support PPPoE for the WAN connection (5th port, beside the 4 LAN ports). And I believe only the WAN port can act as the uplink port.
What I want to know is, can OpenWRT on my Main Router be configured in some way that it can deal with the PPPoE connection? Maybe let it act as if it is the PPP server?
