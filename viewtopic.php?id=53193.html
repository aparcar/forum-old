I hate to be the guy that starts another 'bricked' thread, but I've gone about as far as I can go without someone elses help. I'll start from the beginning, and will also post up some screenshots in a write-up I've been working on.

Initially I soft-bricked it by botching some switch settings in the web ui. The fix is to crack the device open and plug into the JTAG interface. Unbrick by fixing the config in vim via console. Cool.

Step 2 was to upgrade the device to latest release (14.07/ar71xx/generic/openwrt-ar71xx-generic-ubnt-uap-pro-squashfs-factory.bin). I opted for the factory because it'd been quite a while since I updated - I figured that a fresh flash would do the WAP good (and nobody said anywhere that I saw not to use the factory BIN).

I tried to flashing via TFTP but the connection kept breaking part way through. So then I downloaded the .bin into /tmp, verified the sig (it matched), and then tried sysupgrade -v but it failed due to "platform_check_image_failed". So following the guide (http://wiki.openwrt.org/doc/howto/generic.sysupgrade), I used MTD which worked just fine.

However, I now get a kernel panic when ever I boot, with the following message repeated over and over again.
