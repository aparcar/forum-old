In my work i use both.

In some routerboard "433, 411" i've succesfully installed openwrt "Compiled from source CPU: Atheros AR7241 400MHz is fully supported.

But for doing this i connect to board through the serial port for change some settings in his bios like dhcp boot protocol instead bootp and boot source to ethernet instead the nand.

And i ran dnsmasq for start the netboot process.


The mikrotik groove hasn't the serial port, for reinstall his software  there is a program called "Netinstall" and for to put the card in netboot mode you press the reset button when you power up this board until the led flash. But for default use the bootp protocol. I honestly do not know if there is a possibility to boot the kernel openwrt with that protocol.

After a little email chit-chat with the Mikrotik-support I got the following answer which seems very promising for OpenWRT-support on the Grooves:

On those routers you can use Metarouter to run openwrt, and then run batman or
OLSR from there.
http://wiki.mikrotik.com/wiki/Manual:Me … al_machine

Metarouter seems to be a virtual machine in RouterOS. There's even a simple OpenWRT-image provided by Mikrotik.

Example application mentioned by Mikrotik:

The MetaRouter function is useful for allowing clients or lower-privilege users access to their own 'router' and config to configure as they like, without the need for a complete second router, or giving them access to the main router configuration.

For example; a WISP can create a virtual router for the clients ethernet port allowing them to define their own firewall settings, while leaving the WISP's wireless settings untouched.
