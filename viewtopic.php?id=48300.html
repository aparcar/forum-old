This is a follow up to another topic on my Windows VM. The VM is running in a Mac using Parallels Desktop virtualization software.

I can NOT get either of my 2 Macs to ping the VM by host name, though pinging by ip address now works.

There is a work around that fixes the issue on each Mac. Add the following line to /etc/hosts:

192.168.1.223    win8x64propd.local

But I prefer to use DNS in the OpenWrt router to handle this. The same line above is also entered in /etc/hosts on the router. The line should NOT be needed in /etc/hosts on the 2 Macs. For some reason the DNS lookup through the router is failing.

Any ideas?

Steve

It seems I fixed my own problem. I did a full reset with LuCI, under Flash Operations > Reset to Defaults. And now my LAN universe is happy. And I only had to change a handful of defaults after the reset. Apparently some settings previously that I played with were messing things up.
