Hello,

Currently I am running Barrier Breaker r41181 on a TP-Link 1043 V2.1. Whenever trying to install OpenVPN OpenSSL (2.3.2-3) via either Luci or command line I get the following message: 

Collected errors:
 * satisfy_dependencies_for: Cannot satisfy the following dependencies for openvpn-openssl:
 *     kernel (= 3.10.36-1-dd0494e7c599544e99d1297b2b20740c) * 
 * opkg_install_cmd: Cannot install package openvpn-openssl.

It is a fresh install, in which I mean that I bought the router a few days ago and flashed it with OpenWRT yesterday.
Is there something I am completely missing or did not think of?
Or is it a known problem with the current release?

Thanks a million in advance.

When opkg complains about kernel dependencies not being satisfied it is because in the meantime, a new snapshot has been built with a new kernel signature. It actually doesn't matter if your installation is (like in your case) only one day or several months old, the signature does not match anymore.

openvpn-openssl, for whatever reason, requires the kernel version to match.

Now, updating the kernel usually calls for a sysupgrade with the current snapshot. This would be the "proper" way. However, with openvpn-openssl, you cannot really break anything if it fails, and most probably it will work with yesterday's kernel, too. So you can look into the "--force-depends" option that ignores dependency warnings.

Or, you could try openvpn-polarssl. It works the same and just as well as the openssl version.
