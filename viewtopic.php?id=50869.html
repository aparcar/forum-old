You can set the default policy to DROP

iptables -P FORWARD DROP

And then you make one rule for each destination IP you want  to allow.

iptables -A FORWARD -d xx.xx.xx.xx  -j ACCEPT

duni wrote:

How can I add more IPs?

Well, if I understood the question correctly, 'how to enter several IPs in address option'
or better 'how to manage IP list in just one place, not copying firewall rules'

It's not a common practice, but this can be done by using "ipset"

prereq - you gonna need modules ipset (and its dependencies iptables-mod-ipset, kmod-ipt-ipset)

1. define ipset in /etc/config/firewall
this two sections defines one ipset for later use

config ipset
    option storage 'hash'
    option name 'ip-whitelist'
    option match 'dest_ip'

config ipset
    option storage 'hash'
    option external 'ip-whitelist'
    option match 'dest_ip'

2. instead of 'dest_ip' option use 'ipset' in your /etc/config/firewall

config rule
    option name 'Whitelist-Allowed'
    option src 'lan'
    option dest 'wan'
    option proto 'all'
    option ipset 'ip-whitelist'
    option target 'ACCEPT'
    option family 'ipv4'

3. add ipset contents (your whitelist) in /etc/firewall.user

#when ipset name is undefined (for example - a typo in firewall config) this line creates it
ipset create ip-whitelist hash:ip

#every time the firewall is restarted - ipset still exists, better clean it up
ipset flush ip-whitelist

#now add ip addresses
ipset add ip-whitelist 11.22.33.44
ipset add ip-whitelist 22.33.44.55
ipset add ip-whitelist 33.44.55.66

4. to apply changes

/etc/init.d/firewall restart

i think it is easy enough both to configure and to administer, although this is not the way ipsets were designed for.

I had the same problem and found the solution, Id copied the same DDWRT script as you across but hadn't noticed that the `nvram get lan_ifname` is not enumerated in OpenWRT in the same way.

if you actually specify the lan interface name (br-lan on my device), it works, but you also need to include the router itself in the allowed list otherwise you'll have to do a hard reset!

i.e.

# Set up the chain
iptables -N wanout
iptables -I FORWARD -i br-lan -j wanout

# Allow everyone access to these sites
iptables -I wanout -d www.bbc.co.uk -j ACCEPT
iptables -I wanout -d www.telegraph.co.uk -j ACCEPT

# Everything else gets blocked
iptables -A wanout -i `nvram get lan_ifname` -j DROP
