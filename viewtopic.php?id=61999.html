Hello!

I have TP-Link 3220 v2 with Barrier Breaker installed on flash card.

After looking through repository, i found that busybox version there is newer than installed one.

So i did the following:

root@OpenWrt:/tmp# opkg install busybox_1.22.1-3_ar71xx.ipk
Upgrading busybox on root from 1.19.4-5 to 1.22.1-3...
Collected errors:
 * check_data_file_clashes: Package busybox wants to install file /etc/init.d/sysntpd
        But that file is already provided by package  * base-files
 * check_data_file_clashes: Package busybox wants to install file /sbin/mkswap
        But that file is already provided by package  * swap-utils
 * opkg_install_cmd: Cannot install package busybox.

I did:

root@OpenWrt:/tmp# opkg install busybox_1.22.1-3_ar71xx.ipk --force-overwrite
Upgrading busybox on root from 1.19.4-5 to 1.22.1-3...
Removing obsolete file /sbin/lsmod.
Removing obsolete file /sbin/logread.
Removing obsolete file /sbin/klogd.
Removing obsolete file /sbin/rmmod.
Removing obsolete file /sbin/insmod.
Removing obsolete file /sbin/watchdog.
Removing obsolete file /usr/bin/arping.
Removing obsolete file /sbin/syslogd.
Removing obsolete file /sbin/init.
Configuring busybox.

For some reason it deleted the above mentioned files (i suppose symlinks) from /sbin/ and after that device doesn't boot with usb inside.

But without it it boots just fine.

Is there a way to restore symlinks to busybox without whole system reinstallation?
