Post #2
mk24
2 Apr 2017, 17:59

192.200.x.x is not an IP range that is reserved for private LAN use.  You should be using 192.168.x.x

After changing the LAN IP to a different range, your PC needs to re-run DHCP so it isn't trying to use the old range.  Unplugging the Ethernet cable and plugging it back in usually works.

eduperez
5 Apr 2017, 23:03

As mk24 pointed out, you should not use 192.200.x.x for your internal network.

From your description, I understand that your WDR has one WAN interface, connected to the main router and configured as a DHCP client, and a LAN interface, configured as a DHCP server. In that case, you cannot have the same network configuration on both interfaces, you have to bridge them together before.

If the WDR device is connected to the main router using wifi, you should definitively read the client mode wireless guide.

WDS support is needed at the main router to use WDS modes.  That generally means running OpenWrt on it.  If your main router is an ordinary router with stock firmware, you can only connect to it as a regular client.

That means you use routed client or maybe relayd.
