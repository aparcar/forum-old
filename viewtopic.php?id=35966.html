Hi,

I installed Openwrt/Backfire on my Linksys WRT54GL v1.1 last week and it works nicely in AP mode.  However, I was trying to set it up as client (192.168.1.10) to access my verizon DSL router (192.168.1.1) by closely following the instructions in this article http://wiki.openwrt.org/doc/recipes/bridgedclient.  I could not make it work.  It appears that it can connect to the DSL router but it's not getting the IP address needed. Therefore "ping 192.168.1.1" from Openwrt never returns.

I tried many different things but nothing helped.  One idea was that perhaps it has to use DHCP instead of static IP on the client side.  I tried it and almost lost access to the router because once I make it DHCP, the router's IP is lost.  Fortunately I made the Wan port static IP so I was able to access using cable.  But still no luck with getting IP from the DSL router.  I tried the LUCI interface and couldn't make it work either.  When I scan and join the DSL router, it will create the "wwan" interface (not sure why) which doesn't connect.  If I use the "lan" interface, it connects but not getting the IP address.  Any suggestions would be appreciated.
