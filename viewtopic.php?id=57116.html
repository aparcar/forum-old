To get around my school's throttling, on OS X I ssh into a certain computer (using command below) and set up a SOCKS proxy on my computer to connecto to 127.0.0.1 port 8080.  This gets around the block.

ssh command:  ssh user@host -D 8080

How can I implement this on the router itself so that when it boots up, it executes that kind of SSH command to connect to the computer, then uses itself as a proxy so all clients connected to the router are unlimited too?

Post #2
makro
2 May 2015, 00:25

Redsocks claims to work as a transparent SOCKS proxy. It's available as a package, you could look into it. You would set your router up to open the SSH tunnel to the remote host with -D option, then set redsocks to redirect all connections through the SSH tunnel.
