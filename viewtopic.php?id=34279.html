So it goes in Luci (Network -> Firewall) like:

Source: wan:0.0.0.0/0:9999

If I try to connect from inside LAN to the dropbear instance running in the port 22 it works.

Check your settings. I had the same problem with backfire 10.03.1, but on TP-Link TL-WR1043ND. I have two active instances of dropbear, one on port 22 for the internal network and one on port 2222 for access from outside. Port 2222 is opened in the firewall. If your dropbear instances are bind to specific interface, try to set them like this: System/Administration/SSH Access, interface set to unspecified. Do this for all dropbear instances. It worked for me. 5 days and no access problems so far. And just for precaution, I have this:

30 0 * * * /sbin/wifi down && sleep 5 && /sbin/wifi up && sleep 5 && /etc/init.d/dropbear restart

in my Scheduled Tasks. This line restart the WiFi interface and all dropbear instances once a day (at 0:30 AM).
