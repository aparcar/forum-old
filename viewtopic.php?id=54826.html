I believe you only have to start dropbear (/etc/init.d/dropbear start) and copy the files using WinSCP or some other SCP client you use. If you don't have dropbear in your currently used image, you might try using telnet or load the image through TFTP using the bootloader. I'm pretty sure you can only do this if you have serial access to the device, though.

Hope this helps!

If the running kernel has the necessary USB and filesystem modules built in you can mount a USB drive:
mount -t msdos /dev/sda1 /mnt
Then you can access the files on the drive under /mnt directory.
ls -l /mnt      (confirm your file is there)
sysupgrade -n /mnt/<image file name>

geryhun
31 Dec 2014, 13:51

Try this after entering failsafe mode:

root@openwrt# mount_root
root@openwrt# echo -e "12345\n12345\n" | passwd
root@openwrt# dropbear -R -P /var/run/dropbear.1.pid

This will set root's password to "12345". This is needed as there are no TTYs enabled in failsafe mode therefore some processes are unable to read user input. Using echo works this around.
