Whether an OEM firmware gives you better throughput then OpenWrt is a matter of SOFTWARE (since both run on the same hardware…). So ask yourself, what Operating System is used for the OEM firmware. Probably some sort of GNU/Linux. In Linux, drivers are part of Kernel. In OpenWrt we use quite new Kernel with even newer wireless drivers. So it IS possible, that OEM firmware gives you better throughput, but since both, OEM and OpenWrt, use Linux kernel and Linux drivers, well, …

well,... what? At the most important part of the sentence you quit. Still not an explanation why OEM firmware give faster wireless speeds...


The real reason (atleast, what i found) is that the WNDR3700 stock firmware is using a slightly modified wireless driver compared to one used in OpenWRT. That modified part is closed source, so OpenWRT dev's need to reverse engineer this. This takes alot time and is probably also the reason of the slower speeds with OpenWRT (as the modified closed source part is not implemented in OpenWRT yet).
