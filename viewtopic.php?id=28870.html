Post #3
MarkMentovai
5 Mar 2011, 00:41

The firmware seems to have the same magic number as the WNDR3700v2. You can try flashing that via the bootloader, using TFTP. You might not be able to flash it via the factory web UI because it has a different device name in the DNI header.

The magic number matching isn’t necessarily a strong indicator of compatibility with an unreleased product, though. If you can grab some boot messages (either from a serial console or by using dmesg—look around for telnetenable to gain access via telnet), you could compare those to what a shipping WNDR3700v2 produces and make a better determination.
