http://downloads.openwrt.org/snapshots/trunk/ar71xx/ provides r40565 factory images for the NETGEAR WNR2200 but you can't install them from the original NETGEAR interface.

The problem seems to be that the original NETGEAR firmware wants line 1 of the image file openwrt-ar71xx-generic-wnr2200-squashfs-factory.img to be "device:wnr2200" instead of "device:WNR2200".

(Editing the .img file to downcase the device name is fine, but causes a server-side message "The image has bad check-sum!".)

Who is responsible for generating factory images for http://downloads.openwrt.org/snapshots/trunk/ar71xx/ ?

After some more experimentation, I got the WNR2200 factory image http://downloads.openwrt.org/snapshots/ â€¦ actory.img to boot:

  1. Edit the first line of the .img file to change "device:WNR2200" to "device:wnr2200"

  2. Change the file checksum (the second last byte of the file) to 0x77

  3. Delete the last byte of the file

The file length should now be 2,949,253  bytes.
