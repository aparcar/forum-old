I'm looking for the mkimage binary for my OpenWRT device.

A mkimage for the OpenWrt device is not available.

It does not make sense for most devices because it implies that you built an image for your device on your device. With 4-8M flash this is not possible and will take days to compile on most platforms.

You have to modifiy the build system to cross compile the u-boot tools (mkimage is part of these)

I think that you are contradicting yourself:

I'm not building a image for my device on my device.

I want to create uImage with an embbeded DTB

I want to execute .... "mkimage "

normal mkimage in Image Builder/Image Generator / OpenWrt buildroot:
- runs on x86-64 or other system.
- creates/manipulates/convert images that can run on target (kirkwood/pogoplug)

you want:
- run mkimage on pogoplug
- create/manipulate/convert images that can run on target pogoplug

running mkimage on pogoplug means building images for pogoplug on your pogoplug ("native")
As I said before for that you have to edit the build system to:
- cross compile a mkimage package for your target (its already done for the cross compile host)
OR
- compile mkimage on your target yourself

PS: read about http://en.wikipedia.org/wiki/Cross_compiler
