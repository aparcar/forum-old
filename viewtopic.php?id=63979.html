I created a .sh script that create a reverse tunnel from OpenWRT to and external server running Ubuntu. The login is done via private/public key in background. The script runs well if launched from ssh (./openwrttunnel.sh) but I'm not able to run it as a cron job. In the Scheluded Task I've put this line

5,10,15,20,25,30,35,40,45,50,55,0**** /bin/sh /root/openwrttunnel.sh

I want to run the script every 5 minutes but it never starts...

Anyone can give me an advice?

speusippo wrote:

In the Scheluded Task I've put this line

5,10,15,20,25,30,35,40,45,50,55,0**** /bin/sh /root/openwrttunnel.sh

I want to run the script every 5 minutes but it never starts...

If that is verbatim quote from your crontab file, you are missing the spaces between the timing fields:

5,10,15,20,25,30,35,40,45,50,55,0 * * * * /bin/sh /root/openwrttunnel.sh

And wiki contains a simple example for "every five minutes:
https://wiki.openwrt.org/doc/howto/cron

*/5 * * * * /bin/sh /root/openwrttunnel.sh

OpenWrt wiki wrote:

Cron is not enabled by default, so your jobs won't be run. To activate cron:

/etc/init.d/cron start
/etc/init.d/cron enable
