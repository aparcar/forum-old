When upgraded my Archer C7 from CC rc2 to rc3, two sqm configs files appears:

One of them:

# cat /etc/config/sqm-opkg
config queue 'eth1'
        option enabled '0'
        option interface 'eth1'
        option download '85000'
        option upload '10000'
        option qdisc 'fq_codel'
        option script 'simple.qos'
        option qdisc_advanced '0'
        option ingress_ecn 'ECN'
        option egress_ecn 'NOECN'
        option qdisc_really_really_advanced '0'
        option itarget 'auto'
        option etarget 'auto'
        option linklayer 'none'

And the other one (this appears to be the used one):

# cat /etc/config/sqm
config queue 'eth1'
    option qdisc 'fq_codel'
    option script 'simple.qos'
    option qdisc_advanced '0'
    option linklayer 'none'
    option enabled '1'
    option interface 'eth0'
    option download '12000'
    option upload '1000'

Why the difference (in name and content) between both? It is safe to delete the first one (sqm-opkg)? In which interface SQM must be applied: eth0 (wan) or eth1 (internal uplink switch)?

Thanks in advance.

/etc/config/sqm is used.
if you delete sqm package the config file is not deleted if changed.
If you install/reinstall sqm package and a config already exists then opkg copy the file from the package to /etc/config/sqm-opkg not overwriting the existing one.
That's the way opkg is working.

About sqm itself I cannot help. Sorry don't know the package.
