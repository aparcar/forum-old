Hi,

I am working on a command line script to automate certain actions on radio1 (a second wifi adapter, radio0 provides the connection/AP to the router).

When using Luci to make changes to radio1 and doing Save & Apply, it never drops connection on radio0 (the radio I am connected with to access the router/luci).

When I use UCI and/or edit /etc/config/wireless directly (only modifying radio1), when I issue "wifi" to reload the settings it essentially restarts all of the wireless services which causes me to be disconnected from the router and requires reconnecting.

I tried looking in the source of Luci but it seems like it also issues the "wifi" command, so I'm not sure if there is a method I'm overlooking to apply changes to radio1 only via UCI without causing the entire wireless stack to be reloaded (specifically radio0).

Any ideas?

Maybe "uci commit"?

You could use iw to poke settings directly into the radio1 driver, but if you're doing anything involving encryption it needs to be coordinated with hostapd / wpa_supplicant.  The uci system generates a file in /var/run (which is a RAM disk) for each encrypted interface then launches an instance of wpa_supplicant to run it.  You could try to modify those files then restart the supplicant processes.
