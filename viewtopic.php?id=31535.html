Hi all,

I'm hoping someone can help properly configure my setup. I have a WRT54G v2.0 flashed with the latest OpenWrt backfire (10.03.1-rc5). I have been using the LuCI web interface to configure my setup. I unbridged the WAN/LAN interfaces, and enabled the wifi in client mode. I can successfully connect to a wireless network with WPA2-PSK encryption, however when I try to connect to a wireless network with WPA2-EAS TLS encryption it does not work.

I have a valid certificate for the network I'm trying to connect to, which I downloaded to my local machine (netcert.p12). The certificate came in .p12 format, and I have the password. I am unsure how I am supposed to properly setup the certificate for use with my WRT54G/OpenWrt configuration. The LuCI interface asks for the 'Path to CA-Certificate' and 'Path to Private Key' and 'Password of Private Key'.

I have seen some instructions for extracting the certificate and key which I tried to follow, namely:

openssl pkcs12 -clcerts -nokeys -in netcert.p12 -out cacert.pem
openssl pkcs12 -nocerts -in netcert.p12 -out key.pem
*(the 2nd command here asks me to set a password for the key.pem, which let's say I set as 'password')

I then go to the LuCI interface, and after inputting the basic network settings I direct to the 'Network' -> 'Wifi' -> 'Wireless Security' tab, upload the two files I just generated, and use 'password' as the 'Password of Private Key'. However no luck in connecting. Am I generating the .pem files incorrectly? Is there some other step I'm missing here? I have seen some references to OpenVPN but unsure if I need this or not.

Any help is appreciated! Thanks!
