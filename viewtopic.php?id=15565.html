Hello,

I have built the latest Trunk / SVN for the Fon / Atheros and have ran into a strange problem.  I have a serial cable and am able to see that the firmware loads successfully without issue.  I have also removed all the packages and custom scripts I usually include to rule out that they are not causing the issue.

The firmware built is the basic barebones of what is necessary.  The 2100 boots successfully (verified via serial cable console) and I can ping the ethernet port with the IP 192.168.1.1.  Using the serial console I have turned off the firewall to rule out it is causing the problem.

When I telnet into 192.168.1.1, it connects successfully, meaning no 'could not open connection / could not connect' error messages.  However, no banner page or command prompt is seen in the telnet window.  Just a cursor sits at the telnet screen.  For fun I typed the command (even though I can not see what I am typing or what the output is) 'reboot' and the fon router restarted successfully.  I connected to telnet again, and again from a blind console, ran 'passwd', set a new password and was able to reboot again.  After the reboot I am no longer able to telnet in and tried a SSH connection.  Same thing as telnet, SSH connects, but just a cursor on the screen, no 'could not connect' error message.  However, I was not prompted to accept any RSA keys.

As another test, from the Serial port console, I enabled HTTPD and tried browsing to 192.168.1.1 in my browser and the web page does not load.  Again, manually disabled the firewall before this test.

As yet another test, I created a wireless interface, and connected wirelessly and I am able to connect via SSH with no problems at all.

The big indicator to me is that telnet (prior to runnins passwd) over the ethernet port will accept my input/commands, but no output/results is shown in the screen.  Has anyone ran into something similar with the latest SVN for Atheros?

Any insight or advice is greatly appreaciated.  Thank you.
