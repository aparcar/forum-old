I have a network comprising 4 WRT54G running 0.9 WhiteRussian. I use WPA-PSK and WDS (not lazy).

The network is fairly stable. One WRT54G (call it M) serves two wireless Squeezeboxes. No real issues.

There are two intermittent clients: one laptop, one desktop. Router P serves the laptop, and router S serves the desktop.

I've discovered that these clients can sometimes claim to be connected to their respective routers, but attempts
to send traffic (eg ping) fail. The Windows ZeroConfiguration wireless manager shows the connections are up and "excellent".

I've finally discovered that re-running S41wpa will successfully correct this problem. I suspect the real problem is that
nas is no longer running!

I looked on the main gateway router (router F), and I see that although /var/run/nas.lan.pid is present, the corresponding
nas process is nowhere to be seen! Apparently nas was started, then died some time.

Is this a known problem with nas?

Are there known problems with nas?

Are there any known workarounds?
