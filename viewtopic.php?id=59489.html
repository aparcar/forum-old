I moved recently and I'm trying to get my ADSL connection working.

My DSL link is solid, and it works great if I use my ISP-provided DSL router.  However, it doesn't work very well at all if I try to use a DSL modem and OpenWRT's PPPoE.

I am running the latest and greatest trunk (as of yesterday), 3.18.21 kernel, and I experience dropped packets with pppd.

I've also tried half-bridge, where the DSL modem negotiates the DSL connection and then forwards the WAN IP to the internal interface.  That way my OpenWRT router gets the external IP without using pppd, and everything works.

Except that the two ADLS modems I have are flaky and unsupported in half-bridge mode.

So... Anyone familiar with half bridge mode that can recommend a solid, dependable ADSL modem?

I've found the Linksys AM300 which seems to be supported, but it's not available in Canada. 

I can get a TP-Link TD-8616 which may support half-bridge but it's unclear to me if it does and the documentation is contradictory.

Any recommendations?

I'm using DSL320B and had the exact same issue.
OpenWRT issues DHCP discover to the DSL320B and receives the external IP address.
That breaks all the routing since OpenWRT doesn't handle very well Half Bridge.

I've solved the issue placing a simple command in crontab.

* * * * * route add default eth1

Obviously eth1 is the WAN port of my OpenWRT based router. ;-)
