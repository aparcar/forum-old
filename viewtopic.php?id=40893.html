Little steps.
Try setting up a client bridge with luci and then see if you can access the net with your mac attached to the 703.
If that is OK, the next step would be to create an extroot on the 703 with a usb hub and flash card. Now you will have plenty of space to _install_ stuff.
Finally, if you are running low on RAM then there are to options: use a usb HDD and create swap partition, or upgrade the RAM chip.
I do not advocate the latter unless you are very handy with soldering!

Here is my writeup of using openWrt for a pan and tilt camera.  My devices were NSLU2 for openWrt and Picaxe for microcontroller, but the idea is the same with WL-520gU or WR703N and arduino:

http://www.picaxeforum.co.uk/showthread.php?13705

Grey day today here in Nova Scotia, you can see at http://www.lyzby.com/cam.html

Here is a thread on some of the many things you can do with openwrt acting as the operating system for a little computer:

https://forum.openwrt.org/viewtopic.php?id=23114

Among the enduser tasks an openWrt router can perform:

1.  Wireless bridge (it can do this with stock firmware if that is all you want). (As of 091222 this works with linux 2.4 & broadcom 2.4 wireless.) (As of 121204, works as client from stock for many devices, including WL-520gU and WR703n.)
2.  Webcam server--UVC webcam with mjpg-streamer and motion can be run
3.  Music player--with a hub, usb flash or usb hd can be attached along with a usb sound device.
4.  Internet radio--can stream internet radio
5.  Internet link for sensors--can talk through usb serial to picaxe or arduino or stamp microprocessors and can display a web page to monitor sensors or control devices.  The WL-520 has a lot of space inside the case, and is "solder-ready".
6.  Always-on internet data concentrator--enduser programs written in Lua or other language can constantly monitor the web and send alerts or present amalgomated web pages
7.  Remote computer for robot--wireless bridging allows this feature
8.  "Tweeter" or text messanger for emergency conditions
9.  In-car computer for simple tasks
10.  Cell phone controller for text-messaging of sensed events.
11.  Install to usb/sd flash for additional applications
12.  Add serial LCD for display
13.  Infrared control (http://www.trainelectronics.com/artcles/PulsedIR/index.htm)

Here's a thread on adding an LCD:
https://forum.openwrt.org/viewtopic.php?id=23172

Here is a client setup for WR703N--wifi and lan appear to need to be on separate networks.  I put wifi on 192.168.1.51 and wired on 192.168.0.51.  Here are the config files.

cat /etc/config/wireless

config wifi-device  radio0
        option type     mac80211
        option channel  auto
        option macaddr  38:83:45:d3:10:4c
        option hwmode   11ng
        option htmode   HT20
        list ht_capab   SHORT-GI-20
        list ht_capab   SHORT-GI-40
        list ht_capab   RX-STBC1
        list ht_capab   DSSS_CCK-40
        # REMOVE THIS LINE TO ENABLE WIFI:
#       option disabled 1

config wifi-iface
        option device   radio0
        option network  wireless
        option mode     sta
        option ssid     Omnibus1
        option encryption none

cat /etc/config/network

config 'interface' 'loopback'
        option 'ifname' 'lo'
        option 'proto' 'static'
        option 'ipaddr' '127.0.0.1'
        option 'netmask' '255.0.0.0'

config 'interface' 'lan'
        option 'ifname' 'eth0'
        option 'type' 'bridge'
        option 'proto' 'static'
        option 'ipaddr' '192.168.0.51'
        option 'netmask' '255.255.255.0'
        option 'dns' '8.8.8.8'
        option 'gateway' '192.168.1.1'

config 'interface' 'wireless'
       option 'ifname' 'radio0'
#       option 'proto' 'dhcp'
       option 'proto' 'static'
       option 'ipaddr' '192.168.1.51'
       option 'netmask' '255.255.255.0'
       option 'dns' '8.8.8.8'
       option 'gateway' '192.168.1.1'

With this setup, with no ethernet connected to the WR703N, I can ssh to 192.168.1.51 and can browse to the same address to view web pages.

Using mjpg-streamer and motion, I have run webcams on both the WR703N or WL-520gU.

This should give you some ideas.  I use the CLI--I find that luci takes up too much space on a small device--4mb flash for the WR703N or WL-520gU.

araner,

Since you have an address of 192.168.1.1 already on your network, you will need to change that on your openWrt device.  I would guess you can do that with luci, but I don't use it, so can't be sure.  Because of the conflict, you would have to disconnect from your network (wired and wireless) and connect a cable directly between your laptop and the device--either wl-520gU or WR703N.  Your laptop needs to have a fixed ip in the 192.168.1.n range, where n=2 to 254.  With that set up you should be able to connect via a browser to luci, or to connect via telnet (using putty or the like).

Once you connect through telnet, you need to use passwd to set a password--then you can sign in with ssh (and telnet will be disabled for security reasons).  Either while signed in with telnet or ssh, you can edit /etc/config/network with vi to change the fixed ip to something you prefer (for instance, not in the range which dhcp on your router provides).  Make sure you set up "option gateway 192.168.1.1" and "option dns 8.8.8.8" (that's google dns, likely to always be available).

When you reboot you should be able to connect to your network.  Check the wiki for wireless connection.

As far as serial goes, the serial on the wl-520gU and WR703N is the console, so without fiddling, it is not very useful for communication with a microcontroller.  I use these usb/serial dongles:  http://www.ebay.com/itm/New-USB-2-0-to- â€¦ 484bf4eb2f

Or similar--search ebay for "usb serial cp2102".  PL2303 and CH-340/Ch-341 dongles also work.  All tend to be cheaper than ftdi (which is otherwise fine).

For the WR703N you will need a hub if you are going to use wireless--there appears to be a bug which causes usb-serial to hang when wireless is used unless a hub is used.  You'll need that anyway if you want to add a webcam and usb storage.

I personally would not fiddle (again) with a serial mod on a WR703N.  The TPin and TPout pads are very small and very delicate--hard to solder and easy to pull up.  The usb dongle is much easier, and, if your time is worth anything, cheaper.

You would need modded serial if you mangle the WR703N, but with failsafe, that's fairly hard to do.  The WL-520gU is almost bulletproof, since you can always reflash it.  The serial console is useful for instances when you have done something to the network configuration--been there.  Don't know what your problem is--sure you've got a good ground connection--on both ends?

Re Casco Bay, I've taken the ferry from Portland to Yarmouth, Nova Scotia, many times.  I'm sorry that it's no longer running.
