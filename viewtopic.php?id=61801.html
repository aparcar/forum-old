You should be able to ssh to the device.
You should be able to tri-power cycle to the last image.
With DD you will need to ssh in and opkg update, opkg install luci, as luci is not there out of the box.

Edit: Things are fairly well documented on the wiki:
https://wiki.openwrt.org/toh/linksys/wrt1900ac
https://wiki.openwrt.org/toh/linksys/wrt1900acs

dhcp is from dnsmasq which if actually was not in the build would be very surprising.
You can always assign a static ip on the machine attached to the router i.e. 192.168.1.5
The last good image, i.e. the partition from which you flashed the none working image, should be there.

JohnnySL
29 Dec 2015, 01:45

no luci (and no wifi) is normal (in trunk builds). As anomeome said, ssh to 192.168.1.1, and configure from there.
ideally just connect your machine via the wan link to your old router (it will pick up a dhcp adress from the old router and use that for internet access, and conncet your laptop via the lan port of your new router.
then ssh in, and run opkg update and opkg install luci. after that you can log in via the webinterface, and finish the configuration.
