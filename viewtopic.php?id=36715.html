Hi,

recently I discovered by accident that OpenWRT drops ICMP packets with a wrong checksum in the ICMP header.
Other home routers don't drop these packets. iptables also doesn't filter these packets by default.

Here is what I did:
I have an OpenWRT router configured with standard settings, so it performs NAT (Network Address Translation).
When sending an outgoing ICMP port unreachable (type 3, code 3) packet from a host behind the NAT to the public Internet, I can see the translated ICMP packet on the WAN interface.
When sending the same ICMP port unreachable message, but with a bogus checksum in the ICMP header, the packet gets dropped by OpenWRT.
I tested other home routers as well as iptables on Debian-Linux, but none of them drops ICMP packets with a bogus ICMP checksum.


Why does OpenWRT act different? Does OpenWRT filter these bogus packets on purpose? What filter / rule is responsible for dropping the packets?

I know that's a very specific topic, but maybe there are OpenWRT developers around that can help me
