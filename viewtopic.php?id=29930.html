What I do is keeping two config files around (one for each target). Then I copy one I am using to the source tree and build. If it is after source tree was updated, I do make oldconfig first to catch all new changes. After the build I save the config for next time.

The reason why I do it this oldfashioned way is that I was never sure, what exactly changes when I change target and I found it easier to keep the configs up to day this way.
