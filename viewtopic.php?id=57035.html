Hello All,

I have openwrt running on DLINK 505. I am not able to ssh into the router connected over ethernet to my mac laptop. This is the network config settings on the router.

config interface 'loopback'
        option ifname 'lo'
        option proto 'static'
        option ipaddr '127.0.0.1'
        option netmask '255.0.0.0'

config globals 'globals'
        option ula_prefix 'fd97:8a69:3606::/48'

config interface 'lan'
        option ifname 'eth1'
        option type 'bridge'
        option _orig_ifname 'eth1 wlan0-1'
        option _orig_bridge 'true'
        option proto 'static'
        option ipaddr '192.168.1.2'

On my laptop I set the IPV4 to manually and assigned it 192.168.1.9 and subnet mask of 255.255.255.0 with router ip field left blank.

With these settings the telnet mode works fine in fail safe mode but ssh on 192.168.1.2 is timing out.

Added: May not have made myself clear above so adding this note. I telnet using 192.168.1.1in fail safe mode to get the router ip which is 192.168.1.2 . I then reboot the device and return to normal mode and try sshing using 192.168.1.2 and here is where I have issues. ssh command times out here.
Please note the ssh and passwd was already set on the device. I also set a master and monitor interface.

Please see firstlogin. By default, SSH will be disabled and telnet will be enabled. As soon as you set a password (command passwd or in web interface), telnet is disabled and SSH enabled.
