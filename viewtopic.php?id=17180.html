Hello,
I wanted to test how effective is a CompressionLevel on a ssh connection but it looks like that the dropbear ssh server doesn't offer any compression. I mande the following test

scp a 2MB pdf file to my openwrt box and I captured the traffic with tshark to see how much data are transfered on the wire:

No compression        : size of the captured file 2.1 MB
CompressionLevel 9 : size of the captured file 2.1 MB

The same test on a OpenBSD ssh server:
No compression        : size of the captured file 2.1 MB
CompressionLevel 9 : size of the captured file 812K

.. so it looks like the compression of a ssh is not offered by the dropbear server. Any ideas how to activate it ?

Thanks,
