MMCM
10 Feb 2010, 12:47

Exactly.

I usually let the adsl modem/router do the connection and forward all traffic to the openwrt router using a fixed ip (dmz function, or whatever, it depends on the adsl modem/router). Dyndns should be on the adsl modem/router too, if it does support it. Since speedtouch 510 does not support dyndns, I have a cron job running every minute on openwrt, which gets the public ip via snmp from the speedtouch and updates dyndns accordingly.
So we have a cascaded nat configuration, but I never had problems with it.
