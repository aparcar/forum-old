I am working on a wifi application. Our target router is  TPLink 1043 V1. I have run into a weird problem.

I modified the airodump-ng.c so that I could save the data to a file on the router  ( I know that -w serves the purpose , but it prints out too  much data, most of which I dont even need). I need to log the station mac and the power values and I save them to the file in this format

{ StationMac_1, StationPower_1},{StationMac_2,StationPower_2}, ..................

To do this I iterate over struct ST_INFO and do the  following stuff,

unsigned char  Temp_Station_Mac[7];  //  7 character used as NULL

unsigned int  Station_Power[2]; //Second Character used as NULL

While iterating I just  take abs(st_cur->power) and assign it to Station_Power  and then using multiple strcat calls I store them in a array which is written to the target file.
When I compile and run airodump-ng on my Laptop, I get  Mac and Power values in the desired format in the target file but if I compile for the target system using Openwrt Toolchain , power field is null in the target file for all entries.

The file contents looks like this on the target system

{Station_Mac1, }, {Station_Mac2, },....................

I cannot understand why is the same code unable to print the value of power in the target system and what can I do to make it work.
