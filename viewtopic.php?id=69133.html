I was inspired by the ars technica article on building your own router.

http://arstechnica.com/gadgets/2016/09/ … mpetition/

They didn't use openwrt, but I have been using openwrt from the begining, so it was an easy choice for me.  This router is replacing an old school asus wl500gp router (running openwrt) that just couldn't keep up anymore. 

So here's the x86 I bought.

QOTOM-Q190G4N-S08 Very cheap Barebone New computer hardware J1900 4 LAN Quad core Mini computer(NO RAM,NO SSD,NO WIFI,NO OS)
https://www.amazon.com/QOTOM-Q190G4N-S0 … ref=sr_1_1

It's honestly a very solidly built machine, I was pleasantly surprised when I received it.  It's rather heavy for its size.

I added a 32G kingston pcie ssd and a 2G ram chip I had from a prior upgrade.  This board would be perfect if it just had an SD card slot.  The pictures show a slot, it's for a SIM card, not an SD. 

No current build will work out of the box to get this machine up and running on openwrt.  I fired up Knoppix on a usb stick to see what the hardware is recognized as and used the dmesg and lsmod from knoppix as a guide. 

To get a working image (or enough so you can login and download the additional
packages yourself) download the x86/generic imagebuilder package.  I used
trunk, though if I have some time, I'll probably downgrade to Chaos Calmer. 

We need to add drivers for the ethernet so we add kmod-igb to the image.  The
wireless uses the other packages including kmod-usb3 to support the PCIe wifi
card. 

make image PROFILE=Generic PACKAGES="kmod-rt2800-lib kmod-rt2800-usb
kmod-rt2x00-lib kmod-rt2x00-usb kmod-usb3 luci kmod-igb"

Initially, I tried to get wifi working unsecured to test, but the wifi card was acting
strangely, so I gave up and installed hostapd to do wpa2 encryption and that
seems to be working, except I have experienced the problem detailed at:

https://dev.openwrt.org/ticket/14840

I tweaked the settings for the wifi and will report back if that solves the problem.

Good luck.
