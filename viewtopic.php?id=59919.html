Hi,

this is mostly a basic question and I was wondering, does it make sense to use QOS for LAN<->WLAN/LAN too, I always thought it wouldnt make sense and it is just for WAN<->LAN prioritizing, but if, does OpenWRT support this?

I noticed just recently, that there is QOS for LAN too and that it seems to be of useful relevance. For example in Steam In-Home streaming, there is an option called "Prioritize network traffic", and what it does is, it adds a "QOS flag" to UDP packets on specific ports I guess matching  source ports of 27031 or 27036.

The question now is, does this make sense for LAN<->WLAN/LAN, and if, is there a how-to to setup QOS on OpenWRT for LAN<->WLAN/LAN?

Thanks!

Well, with sqm-scripts (and luci-app-sqm) it is possible to set-up qos-instances per interface, so you can establish it on internal interfaces as well (just remember that for internal interfaces the download/ingress and upload/egress directions are reversed, as they are always configured fem the router's point of view). Now, per default openwrt bridges wlan and lan, so there might be no relevant interfaces to instantiate sqm on. You should always be able to change the setup to routed instead of bridged though, so that the multiple sqm instances method has a chance of working again.
Please also note that current sqm-scripts work okay for wlan/wifi, but are not perfect (as the make-wifi-fast project has not yet started for real partly due to lack of funding).
