Hello everybody I use an TL-MR3020 and installed openvpn

opkg update
opkg install openvpn

Then I edited the /etc/config/openvpn File so:

config openvpn client
        option enabled '1'
        option client '1'
        option float '1'
        option nobind '1'
        option dev 'tap'
        option remote 'XXX.XXX.XXX.XXX 49638'
        option tls-remote 'XXX'
        option up '/etc/openvpn/ipv6_con.up'
        option ca '/etc/openvpn/ca.crt'
        option cert '/etc/openvpn/tplink1.cert'
        option key '/etc/openvpn/tplink1.key'
        option verb '3'

My Ipv6_con.up Skript looks so:

#!/bin/sh
ifconfig $dev up
ifconfig $dev add fd25:a491:2:3a1::100/64 
route add -A inet6 fd23:a491:2:3a1::/64 dev $dev
route add -A inet6 fd00:feed:cafe:101::/64 gw fd23:a491:2:3a1::20 dev $dev

Now I go in the folder /etc/init.d/ to enable the openvpn with

./openvpn enable

But if I restart my Tunnel didn't open!
Also if I do the following command nothing happens

./openvpn start

Only If I start the Tunnel with the command I also use on Linux:

openvpn --config /etc/openvpn/client.conf

Then my tunnel open succesfully:

My client.conf looks so:

client
dev tap
float

remote XXX.XXX.XXX.XXX 49638
tls-remote XXX

#Skript laden
script-security 3
up /etc/openvpn/ipv6_con.up

ca /etc/openvpn/ca.crt
cert /etc/openvpn/tplink1.crt
key /etc/openvpn/tplink1.key

nobind
ping 30
verb 3
