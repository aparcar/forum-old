I would usually use a ttl converter and recovery via serial, but in reading the wiki (found here: http://wiki.openwrt.org/toh/tp-link/tl- … ut_serial) it seems you can recover it using the tftp method with out a serial connection


here 's is the factory bin https://downloads.openwrt.org/barrier_b … actory.bin

you need to rename it  "wr841nv9_tp_recovery.bin" as per step 2 below

from the wiki...

v9 (without serial)

As most other current TP-Link routers this device can be flashed and debricked without serial access. Basically, the procedure is as follows:

1.    Set your PC to use the static IP address 192.168.0.66 (the router will have 192.168.0.86)
2.    Put an OpenWrt factory image named wr841nv9_tp_recovery.bin in the root directory of an TFTP server and start it
3.    Power on the router while pressing the reset button until the lock LED is lighting up
4.   Wait for the router to reboot

For details, see reset_button_method_no_serial_cable_needed

good luck

again from the wiki ...

failsafe mode 

When the SYS led (under gear icon) starts to blink push the QSS/reset button for around 5-10 sec. Now the led should blink a lot faster than before.

it appears the sys led is the led beside the power led

basically once the led above comes on press and hold the QSS/reset button, the led should then start to blink twice as fast indicating the router has entered failsafe mode, once in failsafe mode your pc to static ip ...

ip: 192.168.1.11

subnet: 255.255.255.0

gateway: 192.168.1.1

login in via telnet using 192.168.1.1 and enter the following cmds 

mount_root

firstboot && reboot

if the router doesn't reboot right away wait a few seconds then cyle the power, set pc back to dhcp and login as normal via web browser
