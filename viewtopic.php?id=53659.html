Hi.
I was annoyed at the crippled Busybox VI, and installed 'vim (Tiny build)'.
To my surprise, whenever I run 'vi' or 'vim', I still got the simple 'vi', instead of 'vim'.

It appeared that the problem was because of a default alias in my /etc/profile of the format

     [ -x /usr/bin/vim ] && alias vi=vim || alias vim=vi

Meant to make either 'vim' or 'vi' commands to run the same executable - /usr/bin/vim if exists, or default Busybox VI, if not.
This would work of course in the next reboot, but sourcing /etc/profile now wouldn't solve the problem, as it leaves the old alias in place, and appends another alias.

The solution:
After vim installation, the post-install scripts should run 'unalias vi vim', and then 'alias vi=vim'
