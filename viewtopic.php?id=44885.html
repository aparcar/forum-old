I have a 8 input X 4 Output Speaker Selector that I want to control from a webpage hosted on my OpenWRT TP-Link 703 Router. I currently have a working HTML web page to control my Arduino UNO that has single buttons linked to CGI scripts. When a user clicks a button, the script runs, sending the command to the serial port, which in turn switches the Arduino outputs off and on. 

I have been working on a new PHP webpage with (8) Input and (4) Output check boxes and a submit button. I would like to write a string to the serial port of my TP-Link router based on which check boxes are selected. For example, if "Input 1" and "Output 3" check boxes are checked, when "Submit" is selected, I need to send "CL2I1O3 T" to /dev/ttyACM0. (The numbers after the "I" and "O" need to change based on the check boxes)

This is an my typical .cgi*file for the Arduino control-

#!/bin/sh -ax

#Send out the serial command

# Turns on Relay 1
echo -e "ON1/r" > /dev/ttyACM0
# Turns off Relay 7
echo -e "OF7/r" > /dev/ttyACM0

#Redirect the browser back to the index page

echo "Content-type: text/html"
echo ""
echo "<html><head><title>Arduino Audio Control</title>"
echo "</head><body>"
echo "setTimeout('Redirect()',0);"
echo " function Redirect(){* location.href = '../index.html';}"
echo " --></script></body></html>"

I am not sure how to make this work. I have had suggestions for a PHP page, or a CGI page, but I dont know where to go from here. Any ideas?
