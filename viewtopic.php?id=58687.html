I'm using a WRT1200AC as well, and only two of the LEDs work besides wifi activity and the hardware triggered lan port LEDs.  To be clear, which image did you attempt flash (the full filename)?  I'm assuming you flashed https://downloads.openwrt.org/snapshots … pgrade.tar.  Is that right?

Wifi is disabled by default, so it's always possible that the flash was okay and you need to plug in and ssh in via root@192.168.1.1.  At the very least, you can plug a computer and attempt to access 192.168.1.1 in a browser.  Sometimes, if you've not flashed a valid OpenWrt image on the WRT1200AC, you can still access it via http://linksyssmartwifi.com.

I'm assuming you've read the information about recovering to stock firmware here:
http://wiki.openwrt.org/toh/linksys/wrt1900ac.

If you're unable to get in via these methods, you can supposedly recover by entering into fail safe mode.  I'm not sure about how to get the WRT1200AC into failsafe mode.

Disclaimer: My knowledge of OpenWrt is pretty limited at this point, so if you're unable to get in via the methods listed above, wait for the cavalry before ordering the cable.

And success.

The issue I was having is that I had to open up the port (69) in the Adv Firewall settings.  Once I did that, TFTP worked like a charm.  I had to use the "run update_both_images" option and after it finished, "reset" and waited a couple of minutes for it to finish its thing.

Also had to switch from static to dynamic IP address on the NIC I was using to connect to it.  After that, "arp -a" to see what address the router was at and I'm looking at the LinkSys firmware now.

Do you have any advice or recommendations before i attempt this that might make it easier for me? Or perhaps maybe a list of steps you performed. I believe that i can manage based on the literature on the OpenWRT site but i figured you could point me in a more precise direction.

Yeah, make sure you can have the router VERY close to the computer, the USB cable isn't very long.  Also, you only need to connect the pos/neg/gnd and they are spaced apart (inside the router).  It won't be a physically firm connection, so it's important that there isn't any stress on the cable.  If you have the pos/neg reversed, it won't work (but it shouldn't cause any harm), just you won't get any response.  Simply reverse and try again.  If you have it set right and you see garbage characters appearing, readjust and try again.  Once it's clean, you're good to go.  Just make sure that you don't use backspace or the arrow keys, it might not work as expected.

From reading that, it might seem scary but really it's not, it's just a heads up in case it doesn't appear to be working or does appear to be working but with issues.  Would be better if there there was a way to insert the pins into a plug to put into the connector, but since it's only meant to be used for a few minutes to help you recover, "it is what it is."

Did have to RMA the router (Linksys sent me a new one and I sent them the one I started with), but that was because the uplink was about a quarter of what I should have been getting.  It was a hardware issue and the new one they sent me works wonderfully.

On other routers, I'm using dd-wrt.  Only reason I came to OpenWRT was that (at the time I tried searching) the only firmware option other than Linksys was OpenWRT.  For the 'master' router, I'm thinking it's a good one to use, with the 'slave' routers using dd-wrt.  Also help to distinguish when I'm on the main router or not.  That babbling said, one thing I really appreciate about OpenWRT is how easy it is to back up and even edit the config files directly.  I was able to use a few regex find/replace edits in Notepad++ and convert the data from the old 'main' router to the new one.  Made it quick to make the changes and only needed to upload the changes via a file.

Have fun.
