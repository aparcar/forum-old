Well... OpenWrt is a Linux system.
You code in C. You use uclibc (basically standard C library). There are plenty of other libraries for your disposal.

A typical download directory: https://downloads.openwrt.org/barrier_b … x/generic/
The Toolchain contains (cross) compiler for the particular platform/target (ar71xx) in this case.
The SDK contains the compiler and some more things.

Non-standard packages and libraries are available in packages directory and can easily be installed on your device.

I don't know how to access "MAC level frame" info... Best case you can access it by enabling logging and using some user space tool/library. Worst case you have to write a kernel module (I suppose).

Did that make anything clearer?

OpenWrt is based on Linux standard networking. Virtually any resource (book, web page, etc.) that describes how recent versions of  Linux handle the network stack will give you good information.

I do *not* recommend the Linux Advanced Routing & Traffic Control site. It once was the go-to guide, but it only documents Linux kernels 2.4 & 2.6, and has not been updated in the last three+ years.

Be sure that any material you review covers the capabilities of the 3.10 or newer kernel versions.
