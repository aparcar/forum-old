I recently installed a second router in my network to expand my wifi network. To have more options and control I have installed openwrt on this router. I already have another router in my network with openwrt wich does the dhcp and dns in the network.

The problem now is that this second router is also distributing ip addresses to the connected devices creating a second network. But I want all devices to be in the same network.

I have connected the new router with the old router via the wan port.

I have already tried disabling dnsmasq on the second router wich leaded to a brick.

The second router is a TPLink TL-WR841N with squashfs and luci as gui. Could someone tell me how I can set up this router to use the dhcp of the other router?

Hey there.

When areading the initial description: No, he's not trying to run OpenWRT in repeater mode. That's not his actual problem. He just connecte both (I guess via cable, but that doesn't matter to his problem) and now both are distributing DHCP announcing themselfs as primary gateway.

If you go GUI wise:

* Enter LuCI
* Go to "interfaces"
** delete the "wan" interface
* go to "switch"
** Assign the WAN port to switch, too. You don't need it as WAN interface
* Go back to "interaces"
** Edit "lan"
** In Section "DHCP Server" in tab "General Setup" tick the "Ignore Interace" checkbox
* Make sure to have both routers on different IP addresses, of course.

Save, done.

There's not a single step about repeater modes.

Regards,
Stephan.
