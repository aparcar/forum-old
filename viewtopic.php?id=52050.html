I am a (not-advanced) user of both Mikrotik RouterOS L5 (on their Routerboards, not on x86) and of OpenWrt (on TP-Link WDR-4300).
I love both OpenWrt and RouterOS but to be honest both feature and stability-wise Mikrotik stands out. Also the GUI is superior. Performance is not bad on either, but is hard to compare against dissimilar hardware platforms. 
The only clear advantage on the OpenWRT side is the flexibility offered by the huge number of optional packages (some of them being responsible for somewhat lower OpenWRT stability).
I noticed in the supported Hardware that a number of Mikrotik RouterBoards have OpenWRT developed for, I was wondering what would be the reasons to switch to OpenWrt for a board which comes bundled with RouterOS:
-is the flexibility of OpenWRT optional packages
-is the missing features of RouterOS <L3 set on some boards ?
-maybe the better compatibility when all routers in your environment are running the same OS (OpenWRT) ?

I think is a no-brainer to replace the vendor stock firmware of most routers with OpenWrt or DD-WRT but I'm still hesitating when is about RouterOS, specifically >=Level4 feature set.

What is your opinion on this ?

Actually it's more complicated.

Mikrotik does offer some advantages in specific usage scenarios, e.g. if you're a WISP and want to deploy a PtMP network over an area of tens of square Kms. Or if you're more comfortable with a Windows app for configuration (Winbox). But on the other hand, ROS is weak -even problematic- in several areas, including VPN support.

OpenWRT offers a more or less standard Linux system, using popular tools for DNS, DHCP, SSH etc. Most/all the Linux skills you've learned over the years are readily transferable to OpenWRT.
