If you don't want to use a VM and are tired of Cygwin error messages & problems, Andlinux (http://www.andlinux.org/) is another option which works as a very nice WinXP-friendly OpenWRT build environment with near-native-linux performance (at lest when I tried it over a year ago with the Andlinux pre-beta).  When you install it, just be sure to make a big enough 'base.drv' virtual drive (found in the Andlinux 'Drive' directory) to accommodate the entire OS and OpenWRT build-environment, create a non-root user, and to 'sudo apt-get install' all the standard 'build-essentialâ€™ and related build-environment utilities that you'd need if you were running in a native Linux environment.  If you run out of room in your base.drv virtual drive, you'll know because you start seeing a bunch of strange error messages (which weren't there a minute ago) when you run 'make.'  Andlinux has a utility to resize the virtual drive to create more space.

I had no problem running a development vm in vm ware player, using 128 MB ram and a 10G* image on a 1Ghz P3 with 384 MB Ram. As distro I used a current debian, without any gui (booted from the netboot image and installed only a minimal system, adding only the packages needed for compiling openwrt). It wasn't the fastest (a full compile took several hours), but it did its job. And I doubt it would have been much faster under cygwin.

To access it I just connected to it with a ssh client (putty), so I won't have to rely on the slow graphics emulation of vm ware.

KM

* 10G was enough for me because I only compiled the packages I needed, but a compile of all packages will need more than 10G. And you need to clean up the dl, build_dir/, and tmp/ directories sometimes from old source versions.
