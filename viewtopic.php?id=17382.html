Hello:
I am very excited about the developments with the BCM63xx port. Our deployment uses DSL on gateway nodes, ADSL+WiFi  is slick. I would like to help development of firmware for a particular piece of hardware. At the very least, we could provide hardware to developers... But what hardware should the focus be on?

Device recommendations?
Here is the wiki page devoted to BCM63xx http://wiki.openwrt.org/BroadcomBCM63xx … ht=(BCM63) I dont care if I use the BCM63xx platform, but there seems to be quite a few IADs with ADSL that use it.
D-Link DSL-2640B, Zhone 6228, and SMT-G3220 catches my attention Thomson has some good looking IADs, but they have tricky boot loader right?

Needed Features:
Support for failsafe and a compatible boot loader - do not what to mess around with problematic trickery.
Need vap / multi SSID in ad-hoc mod - this means stay away from broadcom wifi right? Atheros the only option?
Enough CPU and mem to support OLSR mesh.
Atheros (or dual radio).
>= 4 port vlan switch.

Desired Features:
OEM.
IPSec accelerator.
Dual radio.
Enough mem for small web proxy cache and embeded captive protal.
FXO and FSO ports.
In this application higher pwr and higher gain is better.
ADSL2+ and VDSL and G.SHDLS

Advice?:
If only that D-Link supported OLSR and not just WDS..... Does any one have insights and recommendations on the hardware selection? Any hang ups with the three that caught my attention?

VX-170WR http://www.versatek.com/products/vx170wr.htm
