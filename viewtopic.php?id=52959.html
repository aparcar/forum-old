Looking at the various recipes that have been posted for OpenVPN I'm pretty confident that I could get an OpenVPN client running on OpenWRT and make it the default gateway so that all traffic goes out over it.

However, I'd still like to forward connections coming in over the WAN interface to a host on my LAN. 

I suspect that if I don't set things up correctly that is going to result in reply packets from the LAN host going out over the VPN interface and not over the WAN interface, since that is the default route.

What is the best way to accomplish this on OpenWRT?  Would it be best to install one of the multi-wan modules (I found a forum post that suggests that mwan2 handles this automatically, but it wasn't in the context of OpenVPN - I'm not sure if they'll work together especially since the VPN interface won't exist at boot)?  I also found a recipe for marking incoming connections which might work to send the replies out the right interface, though that recipe didn't mention how to make the configuration persistent (I'm new to OpenWRT) - it just had iptables command lines which I imagine would get lost on poweroff and which would need to be run after the VPN interface is created.

I have to imagine that I'm not the only person who wants to use a VPN service but allow for port forwarding on the WAN interface, so I figured somebody might have a recipe they'd be willing to share...
