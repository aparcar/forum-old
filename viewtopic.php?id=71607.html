Looking at https://github.com/openwrt/openwrt/blob … mvsw61xx.c it seems there's no support for 88E6141 yet. Only 88E6171, 88E6172 and 88E6176 are listed as supported.

I'm not sure how far the 88E61xx driver source file can go when it comes to 88E6141. In the LEDE project, Imre Kaloz added support for the 88E6352 chipset in the commit https://git.lede-project.org/?p=source. … eca52eea05 using the 88E61xx driver as a baseline. Based on the commit, the changes were quite minimal.

Maybe you can do the same for your chipset? Or maybe give LEDE a spin, and see if your chipset is already supported by an upstream change?
