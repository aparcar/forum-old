The most common problem of not starting network services is absence of the IP address they bind to at the moment of process launch.
Sometimes they log useful information to syslog. Read the messages.
Do not bind to specific IP
Use your own script in /etc/hotplug.d/iface  to start the service when network is up.

Thank you bolvan.
I solve it now.
i ssh to router then creat a file name "30-udpxy" in directory " /etc/hotplug.d/iface" with permission 664, then i edit some word below:

#!/bin/sh

[ "$ACTION" = ifup ] || exit 0

/etc/init.d/udpxy enabled && /etc/init.d/udpxy start
/etc/init.d/igmpproxy enabled && /etc/init.d/igmpproxy restart

that all, whenI reboot andcheck: ps |grep udpxy and   ps |grep igmpproxy, it OK now.
Thank you so much
