I am also trying to get my device to use bluetooth.

I am having troubles getting it working but I thought I would respond to say there does not seem to be a lot BT response beyond cookbook stuff.  In other words, I have not found much in the way of troubleshooting.

I am using a Dockstar to connect a bluetooth dongle.  I infer from comments that some BT dongles require uploading firmware.  Whether that is still the case with recent Openwrt versions I am not sure.  This is an extra step or more.

There is other info on making a BT access point on the NSLU-2 forum.

I believe my problems are related to the latest version Backfire so I think using an earlier proven version of Openwrt should be pretty straight forward.  Just install the modules.

Almost any router with a usb port should suffice.  One of the common Asus devices has two ports already configured and I believe it works well on the 2.6 versions of the firmware so other features are built into the kernel. But 2.4 is pretty hardy so I am sure there are ways to get that going without much effort.

My goal is to use the dockstar in my car to pair my 3g phone connection to unit and use wireless dongle to provide connections to anyone in the car. Basically a portable hotspot.

The other dockstar application here describes using bluetooth to announce the presence of BT-equipped cellphone users when the come in proximity to the device for call routing.  It is part of the home pbx device.

I am not very capable with the openwrt tools, I use the images others have built.  I have not succeeded in comprehending everything necessary to do this myself although I am progressing toward that goal.  So I am not the most authoritative source to answer your questions.  But I hope this provides some guidance and perhaps prompts others to respond.

Paul
