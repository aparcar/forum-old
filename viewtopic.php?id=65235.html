MTK7620A 580M CPU, 16MB flash, 128MB RAM with internal micro sd card slot.  Also has WAN and LAN Ethernet.

Have you looked at the TOH?
https://wiki.openwrt.org/toh/views/toh_ … ll?dataflt[Device+Type*~]=travel

There are a few TP-links and HooToos that are fairly well documented with batteries and are easier to use in cafes etc.

You really want at least a LAN Ethernet or will need to have some function that addresses the AP/STAtion hang when you go from hotspot to hotspot.  The WWAN will hang the wireless when the device is booted and the currently enabled and configured STAtion\AP is not available.  Known "issue" and numerous posts on this.  An Ethernet connection will let you recover.  In addition, if you are in a Hotel and can use them, cables will improve your speed.

This issue is addressed in two manners.

  1. Immediately after flashing OpenWrt, set up extroot on the microSD card. This way, any configuration changes only get written to the microSD card. If you screw up any configuration, you can remove the microSD card and the router will boot into the the default AP Only mode. You can then re-insert the microSD card, connect to the router and correct the errors with vi.

  2. Set up a script that checks for the AP you are trying to connect to on power up. If the AP you are trying to connect to is not seen for 30 seconds, revert to AP Only mode. An example of such a script is here.

As for convenience, stability and performance, I still have to recommend the D-Link DIR-505. It's hard to beat for a complete package: Built-in power supply (multi-country plug, doubles as a convenient USB charger -- just don't set it to "Charge" mode or it will only be a charger), virtually unbrickable (emergency flash facility in Uboot), and a wired port (still preferrable to a Wifi if the hotel offers it, or only offers it) make it still my go-to device.

I own and run some of the alternatives, namely the WT1520 (a lesser WT3020) and the zsun. And while they are great devices and I never had any problems with them, they just don't offer the complete package the DIR-505 offers for traveling.

RangerZ wrote:

@vernonjvs, I have been following this ZSUN for a while now and concerned that this may be borderline technical for me.

If you don't have a Linux computer, it may be difficult. I have a Linux computer so I downloaded the firmware from here.

Flashed the firmware using the instructions from the Flashing from original firmware (mtd_write) section from here.

This basically involves opening a terminal window on my linux PC and pasted the following command

sudo dnsmasq -p0 --enable-tftp --tftp-root=`pwd` -d --user=`whoami`

Open another window on my linux PC and pasted the following commands

socat - TCP4:10.168.168.1:11880
(none) login: root
Password:]zsun1188

After getting a zsun login prompt, paste the additional commands in the zsun terminal window

cd /tmp
tftp -g 10.168.168.100 -r openwrt-ar71xx-generic-zsun-sdreader-kernel.bin
tftp -g 10.168.168.100 -r openwrt-ar71xx-generic-zsun-sdreader-rootfs-squashfs.bin

mtd_write write openwrt-ar71xx-generic-zsun-sdreader-kernel.bin /dev/mtd3
mtd_write write openwrt-ar71xx-generic-zsun-sdreader-rootfs-squashfs.bin /dev/mtd2

Note that the above assumes that your Linux PC is connected to the zsun AP
