I wanted to add to this thread as we have gotten the ME936 LTE module to work correctly (Thank you Bjorn!) using the NCM interface on Barrier Breaker.

There were 2 issues preventing it from working.

1) ARP is not supported on the card, so the interface must be brorught up with the '-arp' argument:  ifconfig wwan0 -arp up

2) DHCP is not supported on the card so the ncm.sh script had to be modified to support pulling the DHCP information from the serial interface and setting the IP/Mask/GW/DNS in the script as the dhcp service calls will never work.   The AT command "at^dhcp?" will return DHCP information:

<clientip>,<netmask>,<gate>,<dhcp>,<pDNS>,<sDNS>,<max_rx_data>,<max_tx_data>



Once these two items were addressed, the card works correctly.

I added this code to the ncm.sh script:

Before the connect command is executed via gcom:

# Make sure the interface is started before we connect.  Turn ARP off for ME936 card.
ifconfig "$ifname" -arp up

I added this code after the connection was made to retrieve DHCP from the AT commant, and removed the existing code that attempted to use udhcp to retrieve the DHCP information.  Please note this has only been tested with the ME936 and may not generically work with other adapters.

DHCP="$(gcom -d "$device" -s /etc/gcom/getdhcp.gcom)"

# Extract IP values from AT command
IPLOCAL=${DHCP:20:8}
IPMASK=${DHCP:31:8}
IPGW=${DHCP:42:8}
DNS1=${DHCP:64:8}
DNS2=${DHCP:75:8}

# Convert IP values to decimal form
IPLOCAL=$(printf "%d." $(echo $IPLOCAL | sed 's/../0x& /g' | tr ' ' '\n') | sed 's/\.$/\n/' | awk -F. '{print $4"."$3"."$2"."$1}')
IPMASK=$(printf "%d." $(echo $IPMASK | sed 's/../0x& /g' | tr ' ' '\n') | sed 's/\.$/\n/' | awk -F. '{print $4"."$3"."$2"."$1}')
IPGW=$(printf "%d." $(echo $IPGW | sed 's/../0x& /g' | tr ' ' '\n') | sed 's/\.$/\n/' | awk -F. '{print $4"."$3"."$2"."$1}')
DNS1=$(printf "%d." $(echo $DNS1 | sed 's/../0x& /g' | tr ' ' '\n') | sed 's/\.$/\n/' | awk -F. '{print $4"."$3"."$2"."$1}')
DNS2=$(printf "%d." $(echo $DNS2 | sed 's/../0x& /g' | tr ' ' '\n') | sed 's/\.$/\n/' | awk -F. '{print $4"."$3"."$2"."$1}')

echo "local IP address: $IPLOCAL"
echo "Netmask: $IPMASK"
echo "Gateway: $IPGW"
echo "Primary DNS: $DNS1"
echo "Secondary DNS: $DNS2"

# Override GW with IPLOCAL
IPGW=$IPLOCAL

proto_init_update "$ifname" 1

# IPMASK is invalid, just use 255.255.255.255 (32)
[ -n "$IPLOCAL" ] && proto_add_ipv4_address "$IPLOCAL" 32 "" ""
[ -n "$IPGW" ] && proto_add_ipv4_route 0.0.0.0 0 "$IPGW"
[ -n "$DNS1" ] && proto_add_dns_server "$DNS1"
[ -n "$DNS2" -a "$DNS1" != "$DNS2" ] && proto_add_dns_server "$DNS2"

proto_send_update "$interface"

You will also need to add a gcom script to execute the AT^DHCP command and retrieve the result.
