Hello,

I have to setup a "TP-WR1043N/ND v1" in our office. OpenWRT 12.09 has been installed.
I want to have 3 Zones:

1. LAN (internal, access to internet, manage router)
2. GUEST (external, access to internet, not be able to manage router).
3. WAN (PPPoE)

I now want to check if my ideas are fine:

1. ZONE LAN: Input=Accept, Output=Accept, Forwarding=Reject
2. ZONE GUEST: Input=REJECT, Output=Accept, Forwarding=Reject
3. ZONE LAN: Input=Reject, Output=Accept, Forwarding=Reject, Masquerading enabled, MSS clamping enabled

- To grant internet access for the GUEST and LAN zone I enabled the Inter Zone Forwarding "Allow to destination zones" to WAN zone. (Together with the Masquerading this should be the default SNAT to internet, right?)
- To enable DNS, DHCP and ICMP for guest zone, I created some custom Traffic rules (as by default the input to the router is rejected - see above)
- To allow and external Ping and DHCp Renew on the PPoE I also added some custom traffic rules - as be default this would be denied....
- To be able to reach Guest Clients from LAN I added a Source NAT from LAN to guest Subnet (with this rule it should only be possible to open session from LAN to GUEST, but not the way the other way arround)

Can anyone please tell me if I am on the right way? I use the LuCI 0.11.1 Webinterface to configure...
