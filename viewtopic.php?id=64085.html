Easy and to the point question.  The best summary I've found of the reasons + issues is over at DD-WRT. The tl;dr version: the router would need multiple BSSID capability to allocate different MACs (not just different SSIDs) to different virtual APs, and this is a hardware not OS capability so some devices or chips can do it and some can't. Also a bit about command line querying Broadcom's. Seems to match other comments online.

The OpenWRT device pages have a ton of info, but I can't find any hint in the specs/forum/wiki how to tell if different MAC per virtual AP is supported and achievable under OpenWRT for listed devices, when I buy one, or how to search for routers with that capability in the router platform databases. I'm guessing it's chipset/driver related but that doesn't help much either.

So - my current shortlist is something like Linksys WRT1900AC or WRT1200AC, or similar spec (Netgear R7500/8000? TP-Link Archer C2600?), if that helps. But I'd also like to know how to tell before buying for devices generally, in case something insanely decent spec'ed and cheap suddenly turns up online to be bought  Or is this just misinformation/outdated?
