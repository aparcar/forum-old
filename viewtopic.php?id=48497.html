The router's dhcp doesn't work - tried 192.168.1.* and 192.168.2.* .

That's nonsense. If the DHCP works, it just assigns an IP address. You can't 'try a range'.

You can try to use Wireshark to see if there is any announcement from the router, try another LAN port (in case the switch chip is acting up) try the WAN port, try a serial cable.

@lexey
19 Feb 2014, 20:15

The rt8366s switch doesn't work in 12.09 branch. You must open the case following instruction  https://forum.openwrt.org/viewtopic.php?id=21998, then solder console pins as described http://wiki.openwrt.org/toh/belkin/f5d8235-4. After that you should be able get boot loader prompt and flash last original firmware f5d8235v1_ww_1.01.28.bin via tftp, but before that you need to cut firmware header with hexeditor to 27 05 19. I hope this help you de brick this router.
