I'm quite the noob with embedded Linux and kernel modules, so I'm here to try to clear up a problem I've been having.

I've compiled some modified ath/80211 kernel modules for my PC Engine AP running Atitude Adjustment on a 3.3.8 kernel and I'd like to use modprobe to automatically resolve all the dependencies rather than relying on a script of my own making. The problem here is that modprobe is searching for modules.dep, which it cannot find. No big deal, you say, just use depmod. Well, for some reason depmod doesn't exist in this instance (nor modinfo for that matter).

So, my question is: why would modprobe be included in a system that doesn't have the necessary tools to support it? Is depmod supposed to exist, but there was a failure on my part to select it via make menuconfig? Or is this simply a matter of busybox being included in OpenWRT for the sake of space and ease-of-use and it's up to the user to provide dependencies for the more complicated tools?

Follow up question: how might I go about figuring out where modprobe is searching for modules.dep? I'm fine with the idea of rewriting this file by hand and placing it in this location, since I'll only be using modprobe for a handful of modules.
