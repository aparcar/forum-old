I have a TP-Link TL-WDR4300 v1 running Chaos Calmer 15.05 / LuCI (git-15.248.30277-3836b45). Over the last week or two I've started getting stalled connection issues on all the machines on my network. Browser connections just stall indefinitely until the page is refreshed. This setup was previously working great for almost a year, and before that for several years running an older OpenWrt on the same router.

Bandwidth and ping times consistently test excellent. Resetting the hardware has no effect, problem happens immediately. No changes in router config happened and no new machines were added to the network. Problem happens on wireless or wired the same. No strange log messages indicating anything on the router. I hooked a machine directly to the cable modem (Xfinity is provider) with no router and issue was definitely gone.

I ran some iperf tests on the local network and everything was fine. Also ran an iperf TCP test (ipv4) to a public server on the internet and seemed to work fine.

Maybe this is some ipv6 upstream configuration change that is causing issues with my router config?

Any other troubleshooting tips? This seems like it might be a hardware issue but I would think a silent fail there is unlikely. I guess my next step is to update OpenWrt and reset all the router settings.
