Finally, the OXNAS team has upgraded kernel to v4.4. However, I still face the same problem with this new kernel when I tried to boot OpenWRT off an external USB memory stick. The problem is Linux fails to recognize external rootfs and starts loading internal UBIFS rootfs partition (see the log below). At 0.000000, it boots with Linux version 4.4.0 (debian@debian) (gcc version 4.8.3 (OpenWrt/Linaro GCC 4.8-2014.04 r48363) ) #1 SMP Wed Jan 2016 and then loads rt3390sta: version magic '2.6.31.6_SMP_820 SMP mod_unload ARMv6 at 5.377139,  instead of '4.4.0 SMP mod_unload ARMv6 p2v8'. Note and AFAICT, this ONLY happens when the stock (original) firmware is still intact in the internal storage while trying to boot OpenWRT firmware from an external storage.

If I set CONFIG_UBIFS_FS=m in <OpenWRT root directory>/target/linux/oxnas/config-4.4, then Linux kernel has no problem to find the external rootfs to complete the boot sans the error messages.
