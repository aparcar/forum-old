hello, my provider use dual access pptp. on 10.03 backfire i used this config:

config 'interface' 'loopback'
    option 'ifname' 'lo'
    option 'proto' 'static'
    option 'ipaddr' '127.0.0.1'
    option 'netmask' '255.0.0.0'

config 'interface' 'lan'
    option 'ifname' 'eth0'
    option 'type' 'bridge'
    option 'proto' 'static'
    option 'ipaddr' '192.168.1.1'
    option 'netmask' '255.255.255.0'

config 'interface' 'wan'
    option 'ifname' 'eth1'
    option 'proto' 'dhcp'

config 'switch'
    option 'name' 'eth0'
    option 'reset' '1'
    option 'enable_vlan' '1'

config 'switch_vlan'
    option 'device' 'eth0'
    option 'vlan' '1'
    option 'ports' '0 1 2 3 4 5'

config 'interface' 'ppp0'
    option 'proto' 'pptp'
    option 'ipproto' 'dhcp'
    option 'server' 'vpn.internet.beeline.ru'
    option 'username' 'myuser'
    option 'password' 'mypass'
    option 'keepalive' '10'

and add to firewall config ppp0:

config 'zone'
    option 'name' 'wan'
    option 'input' 'REJECT'
    option 'output' 'ACCEPT'
    option 'forward' 'REJECT'
    option 'masq' '1'
    option 'mtu_fix' '1'
    option 'network' 'wan ppp0'

and it working perfectly, but i update to latest trunk r22871 and this config not work. pptp connected, wait one or two minutes and reconnecting, where fail? pls help.
