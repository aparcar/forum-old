So I couldn't help but notice that Attitude Adjustment is slated to have a 3.3 kernel, but the 3.2 and 3.4 kernels are the ones that are marked 'longterm' at kernel.org, meaning that 3.3 is no longer going to be patched. Also, distributions seem to be using 3.2, 3.4, or the 'current' release because they want to continue to get bug/security/performance fixes.

Considering that the OpenWRT images end up on devices that might not get touched frequently and have 5+ year lifetimes, wouldn't it be prudent to use only 'longterm' kernels?

Post #2
jow
11 Apr 2013, 17:55

What use is a longterm kernel (and its associated patches) if you don't touch (and thus update) the device for five years anyway?

Declaring a kernel LTS does not make it less buggy than other kernels in its initial revision.
