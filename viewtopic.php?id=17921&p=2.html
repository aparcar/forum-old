chris1982
31 Jan 2009, 18:37

Hello,
I have the same problem, because I have a Asus WL-500gpV1 with TL-WN560G mini-pci card.

The command improve my reception form -55 dBm to -39 dBm.
But I think "sysctl -w dev.wifi0.softled=2" is also possible.

sysctl -w dev.wifi0.softled=0    softled and LED is always off
sysctl -w dev.wifi0.softled=1    This is the default. LED keeps staying ON (no blinking at all!) when device athX is in status "up",
                                                and LED is off when device athX is     "down".
sysctl -w dev.wifi0.softled=2    LED blinks when device athX is "up" and transmitting or receiving data.
                                                LED is off when device athX is "down" or madwifi module is unloaded.

I modified my wirless-config-file (/etc/config/wireless), because the /lib/wifi/madwifi.sh skript change the value automatically . 

config 'wifi-device'     'wifi0'
    option 'type'     'atheros'
    option 'channel'     '5'
        ...
    option softled    '0'       <----- 

config 'wifi-iface'
    option 'device'     'wifi0'
    option 'network'     'lan'
        ...

Some Links:
http://madwifi-project.org/wiki/UserDocs/EnableLEDs
http://madwifi-project.org/ticket/1223
http://madwifi-project.org/ticket/1018
