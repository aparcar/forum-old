AFAIK there are no restrictions in the chips of the routers, which will block after a certain amount of users.

However the limitation could be the router's CPU, which in theory could have a limit before not being able to process its functions.

Thought you should consider speed an issue. Speed is already halfed being half-duplex, and if every user did generate traffic all the time, the speed would be halfed further per each user associated.
A little calculation I learned from somewhere else
300Mbit raw network speed = 150Mbit realistic optimal Wifi bandwidth
150Mbit divded the amount of users with traffic (users can be associated not doing any traffic load) = actual theoretic speed available per user. So in your case with 50 users worst case would equal to 3Mbit available per user.

So suggestion would be to divide the load over multiple routers (multiple radios), so everyone experience better throughput. But it depends on the activity.

NETGEAR routers can accommodate up to 32 clients per wireless band. If your router is dual band, the total wireless clients your router can handle is 64 (32 for the 2.4GHz and 32 for the 5GHz). If your in an area with many router use a Wi-Fi Inspector software if router don't have this feature to detect their wifi channel. Use the least used channel to prevent interference & avoid congestion.

As it's already been mentioned here, a router with a decent CPU and RAM would be the biggest factors with this many clients.

The use of the 5Ghz band would also be in your favor since it is much less likely to be congested, so a dual-band solution is a good idea.

It sounds like you are planning on an all-in-one approach, you should be able to get by with a 400Mhz CPU and 32MB of RAM. However, if you're planning to have multiple VLANs/subnets and many services (VPN,DNS,UPNP,DHCP...) I would strongly advise doubling down for something with at least 64MB of RAM.
