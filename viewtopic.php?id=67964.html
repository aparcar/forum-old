I'm referring to the configuration option in /etc/config/network: see wiki.openwrt.org/doc/uci/network

Can someone explain to me why the default setting for "force_link" is different depending on the "proto" option?

In my device, force_link was not specified, but proto was 'dhcp', so force_link defaulted to '0'.
This worked great.
But subsequently, if I changed proto to 'static', I was having issues. I eventually realised that these issues were due to the fact that force_link was now defaulting to '1', which meant that carrier sense events were not being handled.
I have now explicitly set force_link to '0', and all is good again.

But I'm concerned that I may be missing something. There must be a good reason why force_link defaults to '1' for proto 'static'. What is it? If I over-ride the default, are there side-effects of which I need to be aware?
