I resolved my problem about this RedBoot connection.

Now I can connect to DD-Wrt first I do:

1. sudo invoke-rc.d tftpd-hpa restart
Restarting HPA's tftpd: in.tftpd.
2. sudo invoke-rc.d tftpd-hpa status
in.tftpd is running.

then:

3. ifconfig eth0 192.168.1.2
4. telnet 192.168.1.10 9000

After that I can connect to DD-Wrt recovery console:

DD-WRT> version

Yes,

The documentation is for professionals its helpful but not always for people who flashing for the first time.

I resolved my problem  with tftp loading I couldn't remember the exact command but it should be enough hint for anybody who has the same problem. It was the IP problem, you should specify the IP address of your card and then it should work:

DD-WRT> load -h 192.168.1.2 ap61.ram

and it works!
