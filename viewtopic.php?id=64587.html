Well, what is written doesn't make sense to me.


pe0fko wrote:

The Gig bit switch on the device is not initialized and will switch on all ports (including the WAN)

That's the point I don't get. Either the switch isn't initialized (not working) or the switch transfers packages between the ports (working). It can't be both at the same time as long as it isn't schroedingers switch.


pe0fko wrote:

and loaded the latest OpenWRT (Chaos Calmer 15.05.1) on it. That works, only...

Regardless what's your real problem, please compile the latest development version of openwrt (How to download the OpenWrt source) and check whether the problem persist!


pe0fko wrote:

There seams a tool "switch_utility" and kernel module "ifx_ethsw" for this switch?

That is the kernel module and switch configuration utility from the original firmware. OpenWrt has a own (opensource) driver for the switch and the (opensource) utility swconfig to configure switches.


pe0fko wrote:

Also a other problem I mention on the device (only R02?) is that only one of the USB2 ports are detected. The other one is dead on both devices i converted.
