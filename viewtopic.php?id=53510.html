Hello
i discovered a method to tftp the file to router when you flashed the wrong firmware
first check if you have ethernet working
or keep pressing reset or wps button
i tested this on TL-MR3220 V2
requirements
pc (widows)
ethernet cable
ability to access ethernet between router/pc
wireshark
tftp server (http://tftpd32.jounin.net/tftpd32_download.html)

first give yourself static ip
(http://www.rfactory.org/OpenWRT/static.html)

then open wireshark then select your interface then press start
(http://www.rfactory.org/OpenWRT/unbrick.html)
go down to Using Wireshark

then power your router and get ethernet connection ( in my case keep presing wps button)

then look for "Who has x.x.x.x? Tell y.y.y.y"
but y.y.y.y is not 0.0.0.0

then realign your static ip to x.x.x.x
then restart seasion in wireshark
then power your router and get ethernet connection ( in my case keep presing wps button)
then look for y.y.y.y requisting tftp file
then rename your firmware to that file then power off the router

then install Tftpd32
then run as admin
then click on sittings
then goto tftp tab
then check "browse" in base directory
then select "c:\" then create new directory
name it "tftproot"

then ok then click on "none" on tftp security
then check "allow \ as virtual root"
then go to dhcp
then set diffrouter to y.y.y.y
then set mask to 255.255.255.0

then press ok then close it
goto my computer then goto c: then right click on "tftproot" then select properties
then select sharing then advanced sharing then tick on "share this folder"
then click ok then ok
then run tftp as admin
then select your interface then goto tftp server tab
then copy your renamed file to tftproot folder
then power your router and get ethernet connection ( in my case keep presing wps button)
then enjoy
