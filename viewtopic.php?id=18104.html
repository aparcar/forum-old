I have a a V2.2 WRT54G running 7.09 kamikaze. It supportw iwconfig and also the "wl" utility. I'm often in an area with multiple AP's with the same SSID but without any kind of dynamic roaming/routing. So when it drifts between the two AP's, it makes me have to kick udhcpc on the WRT, and of course all my connections reset. Not fun.

I've tried using "iwconfig wl0 ap xx:xx:xx:xx:xx:xx", and it doesn't give me any errors, but if I then do an "iwconfig wl0 essid myssid" afterwards it picks a different AP, usually the opposite one. 

If I issue the "iwconfig wl0 essid myssid" command first, and then the "iwconfig wl0 ap" command, the WRT *appears* to stay locked to that AP as long as I don't issue the SSID command afterwards, but I'm not sure if it'll stay that way for any length of time or if I'm doing it the right way.
