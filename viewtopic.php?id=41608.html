Hello,

testing 12.09-rc1 I had quite a problem to gain access to the internet. This came from my "old" 10.03 knowledge that only the "WAN masquerade" option is neccessary. The difference between the two versions is:

10.03 does not make use of notrack option
12.09 activates notrack option on every interfaces that has no masquerading flag

If you want to setup LAN->WAN routing via LUCI this means for newer builds that you have to enable MASQUERADE also on the LAN interface. With this option connection tracking is active and the important rule "MASQUERADE on WAN interface" is applied. That is no problem at all if you connect your local LAN with the internet. But what about two LANs? You cannot enable masquerading only in one direction it must always be for both directions. At least you can fix that with editing the config files option conntrack manually.

Is this desired by design or schould I open a ticket?

Markus

I can't verify this behaviour at all, I never had to enable masquerading on lan in order to have proper nat on wan.
You can also enable the conntrack flag in LuCI, no need to manually edit files.

Post #3
jow
12 Jan 2013, 19:04

What you describe can happen if ...
1) the default enabled drop_invalid option is disabled
... and ...
2) no "config forwarding" section exists between lan and wan
.. in this case the "need conntrack" property of "wan" is not propagated to "lan".

If this applies to your config you can cure it by setting the conntrack flag on the lan zone.
