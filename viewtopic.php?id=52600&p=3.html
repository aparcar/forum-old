You tried "smtp.uvtc.com"?

When you can ping an ip number but not a text address (i.e., www.google.com), it usually means that dns is not working. 

When connected via wifi, post the contents of /etc/resolv.conf

Mine is:
search lan
nameserver 127.0.0.1

You're getting into an area where my knowledge is minimal.  What I've posted is working for me, but I've never noticed the "search lan" option before, and don't know what it means in detail.  Sometimes I have temporarily fixed a dns problem by adding "nameserver 8.8.8.8" until I figured out why what I was doing didn't work.
