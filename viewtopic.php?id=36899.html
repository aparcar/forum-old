Thanks for this info, it was very exciting, but it worked without any problems. I give a feedback.

The TL-WR1043ND was connected on the _LAN_-Port to an AVM Fritzbox 7270, and the gateway was the Fritzbox, all devices in a 192.168.178.x net. There was _no_ connection on the WAN-Port. The TL-WR1043ND was flashed with "wr1043nv1_en_3_13_10_up120210.bin" from TP-Link before, which is the latest firmware. I flashed openwrt-ar71xx-tl-wr1043nd-v1-squashfs-factory.bin

I was especially nervous about this:
http://wiki.openwrt.org/toh/tp-link/tl-wr1043nd
It appears that in order to use the "Firmware upgrade" option in the original firmware for installing OpenWRT, the WR1043ND must be connected to internet, or at least to a dhcp server on its WAN port - otherwise it won't flash the OpenWRT image, but come up with original firmware again after reboot. UPDATE: not a problem with hardware v1.8 running stock firmware 20110429: leaving connected back to back the Windows client PC performing the upgrade was good enough.

The Fritzbox has a dhcp-server running, but as mentioned, the TL-WR1043ND was not connected via WAN to it, but LAN

After flashing I had to change the IP-address again to the "Frtizbox-net", which was no problem, using an ethernet-cable and my linux-notebook with Network-Manager (gnome)

Now I try to find out, if I can put openwrt on my Access-Points TP-Link TL-WA801ND and if it is so easy too.

Any hints about IPsec and Asterisk are welcome.
