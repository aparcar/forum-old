The StrongSwan and OpenSSL versions included in CC RC2 have known security vulnerabilities. Updates for both are already in place in respective branches, so the packages will be updated when the next RC or final is built. Just wait for the next build, or build your own image from the CC branch if you like.

StrongSwan currently doesn't build with musl-libc, which is the default libc in trunk. You can build your image with uClibc to get a working StrongSwan, in menuconfig you can change it under Advanced options, Toolchain options, C library. I can confirm StrongSwan builds in trunk using uClibc.

The CC branch still uses uClibc as default, so you can build from that instead: git://git.openwrt.org/15.05/openwrt.git. Some build hardening features were recently enabled in trunk, I don't know how well tested StrongSwan is with those features. These features are not enabled by default in CC. As I said StrongSwan builds, but that doesn't mean it won't have runtime problems.

Make sure to do make dirclean before doing any of the suggested changes, as you will need to rebuild the toolchain.
