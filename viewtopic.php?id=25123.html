It looks to me like the script is failing the telnet connection.  First, make sure you have enabled telnet access in the MI424WR control panel.  Try to telnet to your router and see if you can connect without the script (192.168.1.1 is the default I think).  Second, make sure you are supplying the script the correct password (use whatever worked above).  I think the default password for the MI424WR is "password".

Also, this info from the DD-WRT MI424WR wiki may be relevant:

Note: Windows 7 users must enable telnet under Control Panel -> Programs -> Turn Windows features on or off
Current secondary telnet port is 8083 for Rev C/D and you may have to edit the Jungo-image.py script to reflect port 8083.

http://dd-wrt.com/wiki/index.php/MI424WR
