have you tried this :
http://downloads.openwrt.org/snapshots/ … actory.bin

Bring the unit in tftp mode (press reset button for a few seconds after power up) and upload to file to 192.168.1.20.
from linux console: tftp -m binary -v 192.168.1.20 -c put openwrt-ar71xx-ubnt-bullet-m-squashfs-factory.bin
from dos console: tftp -i 192.168.1.20 PUT openwrt-ar71xx-ubnt-bullet-m-squashfs-factory.bin

The Nanostation and Bullet Ms appear to use same CPU and memory amounts judging from the data sheets so the bullet-m image might work.

The unit may come up after flashing asking for an ip address to be assigned per dhcp and until given that may be not reachable for a telnet session.
Depends on the default network settings and whether the PoE ethernet interface is initialised as lan (static 192.168.1.1) or wan (dhcp).
Can of course later be changed by changing //etc/config/network
