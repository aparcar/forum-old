We have 2 units of TL-WR1043nd v1.x
1 running Attitude Adjustment 12.09 (AA)
1 running Barrier Breaker 14.07 (BB)

Problem 1: (with default setup)
AA has no problem time synchronization when reboot or power failure/trip.
BB could not synchronise time when reboot or power failure/trip. Time deviation could be a few hours to 30 minutes behind actual time. Sometimes it is accurate.
AA is time sync is tip-top, we tested with power trip simulations.

Problem 2: (LuCi->System->Local Time->Sync with browser)
AA, click on the Sync with browser button restore time immediately. No network disconnection.
BB, click on the Sync with browser button takes 30sec and it seems pppoe-wan, lan interfaces got reset and connections restarted. This is annoying.

Problem 3:
Because of the above problem we have started to try out the actual ntpd package and disable the default busybox sysntpd service. The actual ntpd package did not help our situation. After remove actual ntpd package how can we restore the busybox ntpd command back?

From a user point of view BB time synchronization is horrible compare to AA which work (from default and no configuration needed). Are we missing anything here? Can someone kind to enlighten us. Many Thanks.

OK. Found what was the problem.

I have OpenVPN installed in the BB router. If I connect to the BB router remotely and (remotely) doing ntpd -N -q -p ntp-server-ip the OpenVPN connection gets broken.

Doing ntpdate ntp-server-ip do not have this problem.

Hope this able to help someone.
