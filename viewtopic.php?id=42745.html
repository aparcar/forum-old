My problem is that in E3131s-2 modems the memory card is accessible before _and_ after the modeswitch. As a consequence extroot on /dev/sda1 is mounted at the /overlay _before_ the modeswitch and during the modeswitch it disappears for a moment. So the fs at the /overlay has to crash.

Is there any way to prevent extroot fs on Huawei modem from being mounted at the /overlay _before_ the modeswitch??

It is impossible to permanently enable modem port in E3131s-2. It does not support the AT^U2DIAG command. You have to modeswitch every time. So the only solution is to delay mount.

Post #4
pld
9 Mar 2013, 21:26

I did some grepping through the filesystem and found /lib/preinit/00_extroot.conf. It contains one line: extroot_settle_time="20". This is delay before mounting the extroot. Unfortunately to change it you have to build the distro by yourself. Its because the file is processed before the overlay.
