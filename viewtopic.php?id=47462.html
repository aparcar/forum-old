Hi,

I just bought a TP-Link TL-WR2543ND (http://wiki.openwrt.org/toh/tp-link/tl-wr2543nd) and immediately installed OpenWRT.
First time I use Open-WRT, I'm totally impressed!! :-)


Anyway, this model has 8MB of ROM and 32MB of RAM.

I wanted to expand the storage with a USB stick, so I'm playing with an 8GB USB stick I had laying around doing nothing...

I want to create 3 partition:
1) extroot (for openWRT packages)
2) SWAP
3) Storage

I'm not particularly interested in how big "storage" is going to be, whatever is left after having created the other two is fine...

The question is, how big the extroot and swap partitions should be?

These are my guesses.

SWAP partition:
Normally I don't expect a Linux system to swap much, also because otherwise it would get extremely slow.
Even more in this case since it's using a USB stick.
So, having 32MB of RAM, I would say the SWAP should be something like 64/128MB?

EXTROOT partition.
My rootfs is 5.1MB with 3MB being free.
Now, I don't know how many more packages I'm going to install and nor how big openWRT packages can be but I want to keep the system as flexible as possible.
So, how big should this partition be?
1GB/2GB?

Any idea/suggestion is much appreciated.

Thanks in advance,
Andrea

Swap 64MB, Extroot 64MB (or 128MB if you know that you will be installing many additional packages) should  be sufficient.
