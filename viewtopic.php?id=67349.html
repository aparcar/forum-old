As an Archer C7 v2 owner I am considerng which firmware to use.

Current OpenWrt snapshots contain firmware-5.bin_10.2.4.97-1, which seems fairly new, looking at source.codeaurora.org/quic/qsdk/oss/fir … n/10.2.4/.

However, when I look at Kalle Valo's (apparently a QC employee) github at github.com/kvalo/ath10k-firmware/tree/m … /10.2.4.70, which is the recommended place to get up-to-date firmware for our radios, the most recent blob is firmware-5.bin_10.2.4.70.54.

Which one is actually more recent? The numbers indicate that .97 should be more recent, but it also looks like the .70.x line is still getting updates as well. Kvalo's .70.x line also comes with more features like raw-mode (for mesh networks) and mfp (frame protection) which the .97 firmware doesn't have.

This may seem like nitpicking from my side as both blobs work fine, but I am really curious about why the higher numbered blobs miss certain features and why openwrt uses them instead of kvalo's. Anybody has an idea?
