Hi, I'm the author of that package feed. Notice I said 'author' not 'maintainer' because I (unfortunately) don't have the time to maintain it ;-P

Please do not confuse that feed with (official) OpenWrt packages.

[...]

Most folks don't care much about CUPS on a router because it is very hard to make "normal" use of it. Routers normally don't have enough RAM/CPU, and even (most) high-end routers lack the storage space to the (full) Linux printing stack.

CUPS is an important piece of this printing stack but it is not the whole thing.

So when I made that comment, what I meant is: if I had a little time to improve that package feed, I would spend it on packages (of the printing stack) where I see a lot more need for improvements than at CUPS, namely ghostscript (a total PITA to cross-compile) and cups-printing-filters from the LinuxFoundation.

[...]

PS 1: If you want to understand the Linux printing stack, I strongly suggest you to read the Linux from Scratch documentation on it. http://www.linuxfromscratch.org/blfs/vi … nting.html

Notice that to actually make a system that prints like a Linux laptop and workstation do, you'll have to add a lot of the recommended packages as well.

PS 2: "Axet" has recently fixed a number of issues in that feed of mine, https://github.com/axet/openwrt-printing-packages

PS 3: the Linux foundation has loads of good pages on its OpenPrinting project. Take a read at http://www.linuxfoundation.org/collabor … job_format it explains a lot as well.
