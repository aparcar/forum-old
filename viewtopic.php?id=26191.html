The usb serial drivers probably don't recognize the device.

After you switch the device, post the output of:

cat /proc/bus/usb/devices

This will show the new USB interfaces and which, if any, drivers have claimed it.

If no drivers have claimed the interfaces, you can try tell the Option driver about the device:

echo -n "vendor_id product_id" > /sys/bus/usb-serial/drivers/option1/new_id

If the mode-switching worked, and the device is now exposing serial modem interfaces, the line above should result in Option claiming the interfaces and enumerating /dev/ttyUSBX devices.
