I've only recently installed Chaos Calmer (stable) and ran into some problems with dnsmasq not answering DNS requests over my VPN interface (tun0).

It seems dnsmasq's --local-service is the culprit: As soon as I add

option localservice '0'

to /etc/config/dhcp, dnsmasq answers over the tun0 interface.

Is there a more elegant option that I can use? I can't figure out how to use

option interface 'eth0.1 tun0'
or
option interface 'lan vpn'

Both don't lead to anything being added to /var/etc/dnsmasq.conf.

Any input is greaty appreciated!

Post #2
soopersonic
3 Nov 2016, 09:24

list except-interface 'pppoe-wan'

and removing

option localservice '1'

maks everything work as expected.
