Hi all,

I have a very annoying issue concerning WiFi interface on TP-Link WR703N running OpenWrt Attitude Adjustment 12.09.
Namely, if upon restart wireless 'myHome' network (from another router - cable modem) for some reason is not connected, my OpenWrt router interface radio0 (myRouter) is not accessible. When wired connected - LuCI gives me no signal for myRouter interface even after disabling/enabling interface. I need to manually disable invalid connection (myHome) and restart router in order to be able to connect to myRouter wirelessly (and to connect to LuCI or ssh).

My wireless and network settings are given below.

/etc/config/wireless:

config wifi-device 'radio0'
    option type 'mac80211'
    option macaddr '8c:21:0a:xx:xx:xx'
    list ht_capab 'SHORT-GI-20'
    list ht_capab 'SHORT-GI-40'
    list ht_capab 'RX-STBC1'
    list ht_capab 'DSSS_CCK-40'
    option country 'US'
    option txpower '20'
    option disabled '0'
    option channel '6'

config wifi-iface
    option device 'radio0'
    option network 'lan'
    option mode 'ap'
    option ssid 'myRouter'
    option encryption 'psk-mixed'
    option key 'myPasswordXXX'

config wifi-iface
    option network 'wwan'
    option ssid 'myHome'
    option encryption 'psk2'
    option device 'radio0'
    option mode 'sta'
    option bssid '00:1E:69:xx:xx:xx'
    option key 'myPasswordXXX'

/etc/config/network:

config interface 'loopback'
    option ifname 'lo'
    option proto 'static'
    option ipaddr '127.0.0.1'
    option netmask '255.0.0.0'

config interface 'lan'
    option ifname 'eth0'
    option type 'bridge'
    option proto 'static'
    option ipaddr '192.168.1.1'
    option netmask '255.255.255.0'

config interface 'wwan'
    option proto 'dhcp'

What might be the problem here?
Thanks ahead!
