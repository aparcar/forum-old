The error means the router and the PC don't agree on which cipher to use.  You need to reconfigure on the PC side to allow more ciphers.  By default, the Ciphers: config line is commented out so that the default behavior (use any implemented cipher) applies.  This is fine on a LAN where you are not concerned about the traffic being intercepted.

Also you can test that SSH is working (to some extent) on your PC by trying to ssh to yourself: ssh localhost.  This of course requires a ssh server to be installed as well as the client.

It doesn't matter from whence you ping if the server you're pinging is configured to not answer pings.
