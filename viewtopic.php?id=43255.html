I've just had exactly the same problem, changing browsers/clearing caches al yielded the same result (white screen after entering credentials). Let me describe my situation in a little bit more detail:

I'm running a "stock" (just downloaded the image of openwrt ( Backfire (10.03.1, r29592) on my wdr3700v2 ) and configured the device with ip-settings etc (through luci).

Then, I changed the password (whilst logged onto the system through ssh). After that I could no longer log onto the system with luci. After entering my credentials just a white page appeared. Running the binary on the cli just gave me the "regular" http output.

After powering off/on the system some times and starting reading a while on your forum I established that I hadn't installed additional software etc, just changed a lot of default settings (ssid's,passwords/passpharses,ips and the like).

Just before commencing a re-flash to the image I downloaded from the openwrt servers I decided to alter the root password "through" ssh to exactly the password I set to the Luci admin page. This solved my predicament instantly (no restart required or anything).

Anyway: hope this helps somebody out.
