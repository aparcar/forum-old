I've been searching around for a solution to this for a while and haven't found anything... which always makes me nervous that it's so obvious nobody else ever needed to ask... anyway...

   Context: Modifying the backfire ar23xx Madwifi driver on an ap51 board, frequently compiling new changes and testing them.

   What I'm looking for: A way to speed up the process.
 
   1) Is there a way to just compile the kernel (or even just compile the madwifi driver and install it in the kernel) rather than having to go through the whole make world process?

   2) And after that, is there a way to just load the new kernel to the board, rather than having to flash the whole thing? Currently I'm writing both the lzma and rootfs file to the board (via redboot) and it's time consuming, even with multiple boards...

   After perusing the forums/documentation I was under the impression that the lzma file had all of the kernel in it, but my changes don't propagate until I also re-flash the rootfs so I'm clearly missing something... I would greatly appreciate it if anyone could enlighten me or give any pointers on how to more efficiently carry out the steps mentioned above. :-)

1) Yes. Try "make package/madwifi/install".  This will compile a new package (.ipk) and install it to the image.

2) Yes. You can simply install the new package created by the compile (in the bin/<platform>/packages directory) to the router with "opkg install <package_name>.ipk".  You will have to get the .ipk to the router first.  You can copy it to the /tmp folder (either via wget from a ftp server or with a USB drive) and then install it.
