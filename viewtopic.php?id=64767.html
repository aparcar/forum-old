I have purchased LinkIt Smart 7688 & Breakout for LinkIt Smart 7688 from seeed studio. It is based on Mediatek 7688 soc & Openwrt Chaos Calmer.

My target project is mjpg-streamer along with Asterisk & Beresip running on same LinkIt Smart 7688 board.

But I am just Fed up with the USB support of LinkIt Smart.




I switch to USB audio. I have connect USB audio card through a USB hub as the USB audio card doesn't work directly with LinkIt Smart USB port. I have installled kmod-usb-audio and found amazing result. Audio playback & recording both working with mono channel and 8000 rate. I test sip call with baresip and found satisfactory result.

The issue I found as follows:

1) USB issue
===============
I turn to check mjpg-streamer. I have connected logitech USB cam (HD720p) to usb hub and started mjpg-streamer with desire configuration and found excellent result.

Now I tried to play audio (through USB audio card), vary bad.. no luck. The audio is not getting played. Error: set_params:1297: Unable to install hw params:

This USB issue I found in LinkIt Smart.

I stop mjpg-streamer and audio is getting played. Now I started mjpg-streamer while audio is continue .. no luck.. mjpg-streamer is not started.

So the conclusion is both USB device is not working simultaneously. Only one USB device can work at a time.

To check the performance of USB camera, USB audio card & USB hub I have replaced LinkIt Smart board with WrtNode board and run same thing and found amazing result. USB camera & USB sound card both are working simultaneously.

WRTnode, which is based on Wi-Fi AP-Soc, is an open source development board hardware. There is a Mediatek MT7620n chip in it.

http://wiki.wrtnode.com/index.php?title=Main_Page

Please note that I have test LinkIt Smart with both Firmware version v0.9.2 & v0.9.3.

Can anybody tell me uhat is the issue with USB of Openwrt Chaos Calmer????
If there is on issue with Openwrt CC the issue may only with LinkIt Smart Firmware version v0.9.2 & v0.9.3 then somebody help me how to build Openwrt CC/BB from from MT7688 SDK (available in openwrt site) which is fully compatable with LinkIt Smart board.

Please some one help me to fix the issue as I like to work with only LinkIt Smart as because of price and performance. LinkIt Smart is the only board having 32Mb flash & 128Mb RAM for just USD 13 a have found.
Please please please fix the issed and make LinkIt Smart highly popular.

Though I am late but finally I build linkit firmware from the source github.com/widora/openwrt_widora instead of openwrt official source (git clone git://git.openwrt.org/15.05/openwrt.git)
Now my linkit board detecting USB 2.0 high speed device and now mjpg-streamer & USB audio working simultaneously.

So I think there is some issue with Openwrt source of CC & trunk both related USB, this is why USB 2.0 device is getting detected as 1.1 full speed instead of High speed.

In this connection developers team are requested to loom in this matter if they really wants to fixed it.
