What's the best way to block module loading on an OpenWRT image?

I have a WRT1900AC, and with all of the wifi issues ongoing, I'd like to use it just as a wired router. As such, I want to disable the compat, cfg80211, mac80211, and mwlwifi kernel modules so they never get loaded.

Note that I'm not just looking to disable the wireless interface, but never load the kernel module at all (since it causes stability issues). The standard Linux modprobe mechanism isn't used, so it doesn't appear that I can prefix the module line with 'blacklist'. Is simply removing the /etc/modules.d/50-mwlwifi file sufficient and a supported path, or is there a better mechanism? I'd like to be able to use images as built instead of building my own, but entirely removing the kmod-{cfg80211,mac80211,mwlwifi} packages may be another option.

In openwrt kmodloader executable is responsible for loading kmods.
It's executed several times. For rom only, for overlay, for extroot.
/etc/modules.d will help only if module is not present in /rom/etc/modules.d
rom's modules.d are in squashfs and can be altered on compilation stage by editing some mk files
I havent tested that but it can also be possible to remove kmods in image builder.
Anyway to alter rom you need to customize image
Also its possible to rmmod in /etc/rc.local
