What I would like to do:
I want to set up a router to serve it's own DHCP for its own network, but also join an existing wireless network (no CAT5) to get internet. 

Router A is at 192.168.1.1, has internet, and assume I have no admin access (its just a simple dummy AP)

Router B is at 192.168.2.1, uses OpenWRT, Atheros chipset, and manages a local network that has no internet, yet.

Router B must operate standalone, AND have the added benefit of internet if/when Router A is accessible.

What I did:
With Router B on a clean OpenWRT with default settings (but LAN on 192.168.2.x, etc), I took it and (i think) joined the Router A AP as a client to get internet. This created its own new WAN interface, I did NOT attach this to the existing LAN.  I then created a new Wireless network in AP mode with DHCP enabled and attached it to the LAN (192.168.2.x).

The result is ALMOST exactly what I want.  Clients of Router B can join with a 2.x address and see each other AND internet.  HOWEVER, this is the problem:  if Router A is out of range when B is turned on, no SSID appears, and my clients cannot see or join Router B!  It's as if the whole thing falls apart when it can't join A.  Lame.  I want it to function fine but without internet when Router A is not available.

Questions:

  • Anyone know why my OpenWRT router B throws a fit and doesn't work, just because it can't find A?

  • What is the appropriate mode for the scenario I described?  Repeater?  Or is it a Routed Client Masqueraded?, or what?

  • Can I configure this through Luci?  (that is all I have used so far, I'm not a script guru)
