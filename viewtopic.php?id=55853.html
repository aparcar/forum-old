A general rule to deploy a router as an accesspoint (or switch), is to connect the uplink to a LAN port, not a WAN port, and switch off the DHCP server.

Of course in OpenWRT it should be possible to re-use the WAN port as a LAN port by bridging them, but maybe it's also possible to just reprogram the internal switch, to ditch the difference between the WAN and LAN ports. Which you should have to undo for step 2. So keep it easy and just use a LAN port.

You can't ping the 'outside router', because it's in the same subnet as your PC, and yet it isn't. Because it's in the same subnet your PC tries to connect directly, which fails, because there is a router in-between. When you try to connect to an internet host, the PC knows it's not in the same subnet, and just sends it to it's default gateway (192.168.1.1). That router also can't handle it directly, and sends it to his default gateway (192.168.1.254), and the package arrives.

BTW, in many cases this setup wouldn't work at all because the 2nd router can't decide which subnet to use to connect to it's default gateway. It has the same subnet at both sides, and so the gateway could be at both sides.
A router needs to have a different subnet on it's WAN port and it's LAN port(s).
