I figured out that connecting the server to WAN by a wireless interface isn't a good idea. First because missing certified drivers for the server, and if I need to test a WoL it is only possible through a wired connection. And the price of one wireless interface is greater that the price of one WR1043ND. Maybe the best option is to use two WR1043ND, one before the server doing the WDS connection and other after the server receiving the signal from the proxy and broadcasting it.
