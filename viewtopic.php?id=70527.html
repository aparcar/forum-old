Sorry, I did not remember about that option.

Now, according to the documentation (https://wiki.openwrt.org/doc/uci/firewall#zones), that option only logs "rejected and dropped traffic" (and that is my experience, also); as there are no reject / drop rules on the LAN interface (unless you added some), I think it is normal that only (rejected or dropped) traffic coming through the external interface gets logged.

Post #6
mk24
10 Apr 2017, 16:34

The default configuration will reject almost all traffic incoming on the WAN.

Normal usage such as reading web pages is all "outgoing" connections.  In other words an Internet server's response to an internal request is not considered "incoming."
