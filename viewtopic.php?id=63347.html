I'm trying to make an indoor location system based on the RSSI if the devices, the question is:

I can access the RSSI of the devices connected to the router with: w dev wlan0 station dump

But as we all know, indoor location needs way more data so I'm adding 4 AP to the network.

Is there any way to get the RSSI of the devices connected to the network relative to every AP? So I know all the RSSI to every AP and make some math to locate the device position.

It's a interesting idea.

The solution could be:
-Configure 1 router as AP, all devices will connect to this router.
-The others routers will be in monitor mode, reading the RSSI of every single packet with airodump-ng, identify the device using the source mac address and store its RSSI in order to get the position.
