Sure... On your Linux server install snmp ... Example apt-get install snmp OR yum install net-snmp-utils  depending on your distro.

For the router install snmpd...  opkg install snmpd

By default the router will run snmp with the community string of public and have all mibs in tact.

From the server, run the following command.. If the result doesn't understand -v2c just try -v2 instead.
snmpwalk -c public -v2c 192.168.1.1 system

If the above works then move on to this command.

snmpwalk -c public -v2c 192.168.1.1 IF-MIB

Copy the output.

What I'm going to show you is the down and dirty way of collecting using cron and outputing the result to a text file.
