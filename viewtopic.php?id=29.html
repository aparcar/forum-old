I just got a WL-500g the other day, and it seems to be working fine with the default trx with the exception of resetmon and failsafe. That is, if you boot "normally", it tries to go to failsafe but never actually assigns an IP to an interface that is pingable. If you hold down the reset button during boot, however (openwrt failsafe, not factory def), it boots normally. It appears that resetmon returns the opposite values on this box.

In any case, I couldn't get it to boot, so I started adding nvram variables and finally got it to boot in normal mode.

Here's the nvram settings I added to the default Asus ones:

~ # nvram set lan_ifname=vlan0
~ # nvram set lan_ifnames="vlan0 eth1 eth2 eth3"
~ # nvram set lan_proto=static
~ # nvram set lan_ipaddr=192.168.1.1
~ # nvram set lan_netmask=255.255.255.0
~ # nvram set lan_stp=0

~ # nvram set wan_ifname=vlan1
~ # nvram set wan_proto=dhcp
~ # nvram set wan_hostname=wl-500g

~ # nvram set vlan0hwname=et0
~ # nvram set vlan0ports="1 2 3 4 5*"
~ # nvram set vlan1hwname=et0
~ # nvram set vlan1ports="0 5"
~ # nvram set lan_hwaddr=<mac addr>
~ # nvram set wan_hwaddr=<mac addr +1>

(I was trying to avoid the bridge... yes, I should have changed lan_ifnames)

However, at some point these got changed (during firstboot?)

lan_ifnames=eth0 eth2
lan_hwaddr is unset
lan_ifname=br0
wan_ifname=eth1

So, I guess that you don't actually *need* the nvram vars to boot, but this box isn't booting in failsafe mode... I'll have to look into it more.

I have USB storage running fine from a usb->compact flash reader with a 170MB microdrive...
