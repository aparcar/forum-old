It's really weird in case of your old logs, that we can see:

[    5.970000] b53_common: found switch: BCM5325, rev 0

which means the first switch reset didn't cause a reboot. Than we can see

It all means that for some reason, it's the second switch reset that causes reboot. My guess is that the first (incorrect) switch reset changes GPIO value from (default) 0 to 0 and then to 1.
