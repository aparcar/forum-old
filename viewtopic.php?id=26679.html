Hi everyone,

I'm pretty new to openWrt. Even though I've assisted a conference about the openWrt project, I haven't really paid a deep attention to the project until today.
I have a bit of a "problem" and I think that openWrt could solve it. To be sure and to be guided a bit for the start, I've decided to post this message.
I hope the question hasn't been posted already.

So basically, I'm looking for a router that could suit my needs.

On one end, I have a PPP server that allow multiple connection since the MAC address is the same at each request.
On the other end, I have a couple of computers that would love to have an IP delivered by this PPP server.
Of course the MAC addresses of the different computers are different.

That's why I've been thinking about a router a bit special that I could put between my computers and the PPP server that would rewrite the source MAC address of my computers' frames by its own to communicate with the other routers.

Once the IP allocated, the router would receive frame from computers, record in a routing table the IP source and the associated MAC address, change the source MAC address by its own and send it to the other routers. When a frame is sent to one of my computers, my router would look for the source IP address and change the destination MAC address by the one that my targeted computer has. My router wouldn't have any IP address, which is not necessary.

Once the IP address is allocated to a computer, the process seems quite simple (I think), but before that, you can't make an entry in the routing table because the computer still has no IP.

That's why I would like a filter on the DHCP requests. I do not want the router to send it to the other routers.
Once an DHCP request is sent by my computers to the broadcast, my router should not have the default behavior to send it to it's WAN output and rewrite the source MAC address, but instead send a PPP request on this output to the PPP server and send the answer as a DHCP frame to the computer.

I currently have a WRT54G2 that doesn't seem to be supported. I'm willing to buy a new router, not very expensive (less than 60 bucks) that could support openWrt and the project that I'm having in mind.

That could be cool, if you have ideas of packages and some recommendations of a router.

Thanks in advance,

Richard.
