If you use -p1, patch will strip off the 1st path (trunk) before applying the patch. That said, you first need to change directory into the root directory of your local OpenWRT source tree in order to properly apply the patch.

Post #5
jow
18 Jan 2012, 16:06

The patch is indeed wrong and will not work for anyone but its author.
It patches files in build_dir which are not present on a fresh tree, in order to use it one has to manually redo the modifications done by the patch in the appropriate places. No amount of -p flags will make it work.
