How sure are you that your snapshot and the packages are from the same build ?

The trunk snapshot builds gets rebuilt automatically every  day or two   so even if you install an image today and then try to install packages tomorrow it may not work (i use the waffley "may" because i don't know if the signatures on the kernel package change with every build just because of timestamps and such or if it actually means something has changed).    That's an annoyance of working w/ trunk.    One way around it is to download the 300MB ImageBuilder archive  or just download a snapshot of the packages dir when you install an image you know you're going to upgrade to.  (or, if you are a linux developer/admin type and are so inclined to build your own stuff from source.  The ar71xx built quite cleanly for me (though you need to download a few hundred meg of source and  need about 6 or 7 GB of disk space) )

Rather than going in for weirder stuff... i'd say just try your sysupgrade again.

One thing that suggests that things aren't exactly how you think they are is that about 24hours ago you said that you used the latest snapshot, but r35673  was checked in a good 2 weeks ago on 2/19.

Checking out the build bot output shows that there have already been several snapshots that are newer:
http://buildbot.openwrt.org:8010/builders/ar71xx

2/25/2013  was already 35770

Anyway... there are other ways you can do things, some listed here http://wiki.openwrt.org/doc/howto/generic.sysupgrade

You CAN also download the ipk files and unpack them etc. but the kernel and modules also have version checking internally (in addition to what ipk does for all packages) so the insmod will likely fail.
