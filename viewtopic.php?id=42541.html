I am running OpenWrt Attitude Adjustment 12.09-rc1  on a Netgear WNDR3700 v 1.

I have:
kmod-ipt-nat
kmod-ipt-nathelper
kmod-ipt-nathelper-extra

all running on the box.
I am trying to set up a PBX (PBX In A Flash) behind the router and allow external (remote) extensions to register.

The PBX is supposed to run an iptables modifier which opens ports based on a dyndns check (on the PBX, not the router); if my uncle's phone in CA is sitting at X.Y.Z.A and is has a dyndns address of unclesphone.dyndns.org, the the PBX should open its firewall to requests from X.Y.Z.A.

On my cellphone phone (not attached to the local network/all wifi turned off), I can run a dyndns update and check with www.whatismyip.com and 'know' that the IP address of the phone is 166.137.X.Y; when I register the cellphone to the PBX and the ports are forwarded from the OpenWRT device the PBX sees the requests as coming from 166.137.Z.L (i.e. a different IP address).

The .X.Y address is current and can be checked via update.

So, why is the router passing/why would the incoming be seen as .Z.L?

Clearly if I unblock the dyn address and the incoming to the PBX is seen as another IP it gets dropped like a hot potato.

I can only see that the wrong IP address is registering by clearing the iptables and allowing anything.

Is there something I can set up and check on the router to see if .Z.L is incoming or if .X.Y?  In other words, is my network somehow mangling an IP address (yet allowing me to make a call) or is the router somehow mangling it all on its (incorrect) own?
