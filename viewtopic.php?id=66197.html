I have a pfSense proxy server, that I'm placing in my network.  It's create a blocking traffic to specific sites at specified times, but one thing that the box just can't do is serve wireless internet; this is largely because it's so hard to get a Wireless N card for FreeBSD when you're using hardware that's as old as mine is.

Since OpenWRT is great at serving wireless traffic I thought I would connect a wireless OpenWRT router on the ethernet LAN interface of the pfSense proxy and configure it so that pfSense would use DHCP to dole out ips to wireless clients though it.

I'm not entirely sure where to get started on this, but it seems to me that using relayd holds some promise in this arena. 

In the past I've used relayd to create a pseudo bridge between two wireless access points, for purposes of connecting computers with only ethernet NICs to the same network via a wireless link.  When I did this the DHCP Server on the local OpenWRT machine was disabled so that the one from the other network could be used.

In the pfSense situation I would like to hopefully use relayd to connect the pfSense LAN interface to the OpenWRT wireless, but I'm uncertain of where to begin. 

It seems to me that in the configuration file I should opt for a wlan instead of a wwan and make the gateway be pfSense.  Is that correct?
