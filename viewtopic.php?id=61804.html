I am a new OpenWrt user having used Tomato for a LONG time.

I've recently aquired a TP-LINK WDR3600V1 and straight out of the box, I loaded OpenWRT, and setup a USB WAN connection via usb_ether tethering to my wifi hotspot. It worked well. I was using bleeding edge Chaos Calmer.

In an effort to move back to the stable release, I reverted the unit back to stock TP-Link OEM firmware successfully.
I then upgraded (i think this was my mistake) to TP-Link wdr3600v1_en_us_3_14_3_up_boot(151104).bin.

My attempts to load the stable OpenWRT Chaos Calmer have been unsuccessful due to the following errors:

Error code: 18005

There appears to be a check in the TP-link firmware for the filesize and/or other criteria.

Has anyone experienced this?

Is there a workaround so I can put OpenWRT back?
All of TP-Link's images read a size of 8,001KB in windows, but not in OpenWRT...it's the only thing I've noticed.

If anyone can help, it would be much appreciated.

I've attempted upgrades via the factory web-gui as well as via the tftp recovery process.
The tftprecovery process downloads the image, but rejects the new image.

After a call to TP-Link support, they informed me that they have in fact implemented a firmware validation check that will not allow the installation of 3rd party firmware on their devices, at least using conventional methods.
This is to comply with new FCC regulations.
IMO, it was the easy way out for them to prevent 3rd party radio modifications.

I would recommend that users with older firmware on their TP-Link routers that wish to run 3rd party firmware like OpenWRT, DD-WRT or Tomato to not upgrade to the latest releases.

I am hopeful for a workaround.
