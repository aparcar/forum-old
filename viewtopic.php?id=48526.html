Hi all
I'm using TP-Link WR841N as simple WiFi Access point + ethernet switch. Since I need all the 5 ethernet ports, I've bridged WAN with LAN and WiFi in a single physical bridge in the firewall LAN zone.
Compared with the default "out of the box" setup, I've just disabled all the protocol on the WAN, bridged WAN with LAN, and disabled DHCP server, since my primary router does the job.
I've connected uplink (the ethernet link from the master router) to WAN, and connected some TVs to LAN ports and smartphones via WiFi.
The very strange issue is that my TVs can't find DLNA server running in the network (it's physically connected to the main router) but the smartphones, connected via WiFi to the same TP-Link can!!! And if I connect uplink to a LAN port instead of a WAN, the TVs can find the DLNA server!!!
It's like something blocks multicast (as far as I know DLNA use UDP multicast packets for discovery), but the interfaces (all of them) are bridged, so why (and why via WiFi works, via LAN not)?
