With OpenWrt firmware, you can flash via telnet with the "mtd" command or via the web interface.  Both methods use the TRX file.

The BIN file is just the TRX file with a modified header to make it look like the factory firmware (to make it happy and accept the image).

To go back to the factory firmware, I think you have to use the TFTP method, which will accept the BIN format (original firmware).  This can sometimes be tricky... search the forums for tips on flashing via TFTP.  But why would you want to go back to the factory firmware?
