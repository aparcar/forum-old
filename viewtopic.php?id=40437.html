Hi Swift,

LuCI is living in a separated "feed": it is not integrated in the OpenWrt base set of packages.

The available feeds are listed in the "feeds.conf" file in the root OpenWrt directory.

As it is alredy listed in it, you have first to download the LuCI feed's package descriptions using:

./scripts/feeds update luci

You can also download the optional OpenWrt non-base packages the same way:

./scripts/feeds update packages

Then install the LuCI feed, so you see it in the "make menuconfig" display in order to select it for inclusion:

./scripts/feeds install luci

You can also install the additional OpenWrt packages the same, way, it will add a bunch of things to select from!

./scripts/feeds install packages

All this is described in the Wikie here. The Wiki is always the best place to start searching
http://wiki.openwrt.org/doc/start?do=index

I strongly recommend to switch from the default W95-looking "OpenWrt" theme to the modern "Boostrap" theme. Provided you selected it in the menuconfig, once running on your device, go to "System > System > Language and Style > Style" and choose "Boostrap" from the drop-down list.

Hi Swift,

Well I don't understand: you asked first for an advice on how to get the LuCI packages for inclusion during compilation: what I described is the way to add the LuCI (and other) additional packages for compilation on your PC.

Alternatively, you can download on your PC the latest stable (non-trunk) firmware snapshot, that should also contain the LuCI packages by default.

But if you wan to compile your own firmware and you get an error when compiling, you should first make sure that you have a correct dev environment on your Linux PC. Under Ubuntu or any Debian-based distro, you can do that by issuing:

sudo apt-get update
sudo apt-get install build-essential

Once you have downloaded OpenWrt base distro (what you probably did already), type in the command I gave you previously, then "make menuconfig" to first choose your device and select the LuCI packages you want (probably "admin-full"). Then "make".

If you get an error, try re-running "make V=s" to get details on the error as suggested, and copy the end of the lengthy log near the end, when you start to see error messages and paste it here, so you can get help.

You should then obtain firmware images in the "bin" directory (for factory or sysupgrade), that you can install on your AP.

You then have to find a way to install this firmware into the device. If you don't have the LuCI web admin interface on it, you should still be able to connect to it using "telnet" if you still have not setup a password for "root", or using "ssh" afterwards.

You have then to install some kind of TFTP, FTP or HTTP server on your PC, so you can fetch the firmware file from the AP. Please refer to this thread for more information regarding the subject.

Once the new LuCI-enabled firmware is running on your AP, you can use it to configure your Internet access.

Hope it helps.
