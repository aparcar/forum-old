Hi All,

I have WRT54G and WAP54G, working as the nodes of the OLSR mesh network.
Additionally I wanted them to work as a tinc based router between two LAN segments.

The WRT54G is connected to the 172.19.1.0/24 segment (with IP 172.19.1.254)
The WAP54G is connected to the 172.19.4.0/24 segment (with IP 172.19.4.254)

The tun0 uses 172.19.1.254 on WRT and 172.19.4.254 on WAP with netmask 255.255.0.0

I can ping machines in the 172.19.4.0/24 segment from both WRT and WAP, however I can not
ping machines in the 172.19.1.0/24 segment from WAP (they are accessible from WRT though).
I can ping 172.19.1.254 (WRT) from WAP.

So it seems, that packets are lost between the tap0 and vlan0 in the WRT.
/proc/sys/net/ipv4/ip_forward is set to 1.

the "ifname", and "ifnames" variables are set as follows:
# nvram show | grep ifn
size: 1100 bytes (31668 left)
wl0_ifname=eth1
lan_ifnames=vlan0
lan_ifname=vlan0
wifi_ifname=eth1
wan_ifname=eth1

UPDATE - SOLVED
OK. I just had to add a single line to my tinc-up script:
iptables -A FORWARD -i tun0 -o vlan0 -j ACCEPT
