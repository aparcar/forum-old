Hello world,

I'm running into a problem connecting to an apple airport extreme that I don't experience with any other router - netgear, linksys, asus, etc. Per the logs below, supplicant successfully authenticates and associates with the extreme but fails with reason 2: PREV_AUTH_NOT_VALID in the dhcp state, each time, and only against the airport extreme. I'm not sure if this is related to dhcpc or it just happens to be a timing issue with how/when syslogd handles the logs. This is becoming a nuisance. Assuming for a second that this is not an Apple router issue, how is wpa_supplicant supposed to respond/react upon receipt of reason2? I'd appreciate any help and direction. Thank a lot!
