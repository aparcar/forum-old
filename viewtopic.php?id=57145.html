Even though I have been using openwrt for years, I suddenly can't seem to figure out how to do some thing as simple as open and forward a port.. Sorry for my stupidity..

I want to forward a port (say 8080) from WAN to LAN to a specific computer to SSH in to..
I can SSH in to this computer fine from with in the LAN..
But port-checking websites still say port 8080 is closed from the outside (WAN)..

So far I have tried this:
In the firewall page, allow receiving, sending, and forwarding to/from the WAN zone..
In the port-forwarding page, I made a rule saying to forward any thing from WAN that is trying to access port 8080 to forward to LAN (to the computer running SSH server) on port 8080..
In the traffic rules I have allowed port 8080 from any host and any zone..
In desperation I made an other traffic rule to allow any traffic from WAN to access any place on LAN, and I put this rule at the very top so it would be checked first..

My settings make it seem like all computers from the internet can access my entire LAN, and literally I can see nothing that is rejecting any thing any where, yet every check-port site I try still says port 8080 is closed, even though I can SSH in to that same port and computer from with in the LAN..

What am I missing here?..

5 steps.

1. /etc/init.d/firewall disable
2. opkg update
3. opkg install rinetd
4. vim /etc/config/rinetd
5. enjoy
