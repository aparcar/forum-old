Hi,

TLDR: I have 2 routers in my network, one from my internet services and one openwrt device. The problem I am facing is that if the openwrt one is turned on, I have quite some trouble with DNS resolving.

The setup:
Telephone comes in, into the standard router I got from my internet provider.
A long network cable is connected to that, it goes to the top of my house into a openwrt router.
The openwrt router has clients connected by cable and by wifi. The SSID of the openwrt router and the main one are different.

Exact problem: When the openwrt router is turned on like this, I sometimes have very bad dns resolving. It doesn't matter on which router I am connected. It looks like it gets worse once a machine is connected once to the openwrt router, but it stays bad once it connects back to the main one. Linux and Android seem to do better than Windows.
I also have a third router which is also running openwrt. When I connect this one to the network, using internet becomes almost impossible sometimes.

Note: When I go on VPN with my pc all problems seem to be solved. But I don't want to be on vpn all the thime

Is openwrt doing something weird in my network with dns?
