router is a GS with rc5 (but problems are also on a rc6 router)

1st issue is nsslookup ignores the 2ns statement in cmd line (server)
so nsslookup host otherdns  always "asks" the 1st dns in /etc/resolv.conf
a bug or a feature ? :-)

2nd is that it seems my update script sometime stuck (depending on when it's called in /etc/hotplug.d/iface/
and nc semms to eat all cpu time
303 root     R        348   287 97.8  1.1 nc
here is the script....
(i inserted sleep 15 then log entries are after log entries of ppp when its connected, otherwise the update message is in logfile/logread before
ppp shows that it's connected...

#!/bin/sh
sleep 15
IPADDR=`ifconfig -a | grep -A1 ppp0 | grep inet | awk '{ print $2 }'`
LASTIP=`nslookup hoerst.dtdns.net ns1.darktech.org | grep -A1 hoerst |
grep Address | awk '{
print $2 }'`

echo -n "`date` - "

if [ "$IPADDR" != "$LASTIP" ]; then
     echo -e "GET /api/autodns.cfm?id=hoerst&pw=amiga&client=BSD
HTTP/1.1\nHost: www.dtdns.com\nUser-Agent: bash\n\n" | \
            nc www.dtdns.com 80 > /dev/null 2>&1
  echo "Posted IP change from $LASTIP to $IPADDR"
else
  echo "IP still $IPADDR"
fi
