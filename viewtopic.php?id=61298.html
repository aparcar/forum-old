You mean the "waiting time" that the system waits for you to trigger the failsafe mode via a button press, right?
It is 2 seconds by default, which is a ridiculously short time.

That can be changed in two ways, both of which require you to compile a personal firmware and flash it. (The changes need to be in the main original flash, not in the modified jffs2 "overlay".)

Two ways:
* Use "make menuconfig" in Openwrt buildroot and change options for .config
    ( Image configuration --> Preinit configuration options --> Failsafe wait timeout )
* currently in my own firmware, I patch the preinit script code directly with the following patch:

--- a/package/base-files/files/etc/preinit
+++ b/package/base-files/files/etc/preinit
@@ -16,7 +16,7 @@
 fs_failsafe_broadcast=192.168.1.255
 fs_failsafe_netmask=255.255.255.0
 
-fs_failsafe_wait_timeout=2
+fs_failsafe_wait_timeout=4
 
 pi_suppress_stderr="y"
 pi_init_suppress_stderr="y"
