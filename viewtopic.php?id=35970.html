If no local ipv4 address is given, 6in4 attempts to use the local ipv4 address of the interface carrying the default route.
If there is no default route and no local ipv4 address specified, it will give up.

You can try to set "option ipaddr 192.168.5.51" in your 6in4 interface declaration.
Note that 6in4 is not able to perform nat traversal, you have to either forward protocol 41 from the router to your openwrt device or ensure that the openwrt device actually gets a public ip.
