I see your problem and I don't think we have the same issue, though we both have difficulty downloading the source files.

Mine was that the filename in the download path was not complete because that I didn't update the configuration file after an SVN update. I tried to download http://downloads.openwrt.org/sources/gc … o-.tar.bz2 from http://downloads.openwrt.org. I could successfully visit http://downloads.openwrt.org/sources, but could not download gcc-linaro-.tar.bz2 because there is no such file on the server.

Your problem is that the download site is totally down. If you go to the site kernel.org, you will see the maintenance notice. You should be able to download the source file from another mirror at http://downloads.openwrt.org since the Make process tried this address, but for some reason the source code linux-3.0.3.tar.bz2 has not currently been included on the OpenWrt source server.

Probably you can manually search for this file, download it and copy it to the dl directory under the trunk directory.
