Well. Transparent proxy mode only works for HTTP but not for HTTPS unless you manage to fake foreign host certificates. Which of course is possible depending on your network infrastructure, but I refuse to suggest such a thing just because of security reasons.
Besides: in case of apps using certificate pinning (like I hope every online banking app on Android and iOS does) you will create at least some confusion since those apps will tell the user about a security breach.

To do it properly:
* Only enable transparent mode for HTTP but not for HTTPS.
* Make your gateway route HTTP to the proxy automatically for every host that is not your proxy.
* Block outgoing traffic to HTTPS entirely for every host that is not your proxy.
* Announce the proxy config to your clients. By wpad, by dhcp or by yellow post its on every coffee mug you find.
