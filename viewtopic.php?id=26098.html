Hi,

I have tried to build myself custom image for TP-Link WR841ND as described in wiki.

While building I didn't do much, used the default configuration, just with ppp and ppp-mod-pppoe packages removed:

make image PROFILE=TLWR841NDV5 PACKAGES="-ppp -ppp-mod-pppoe"

After building was done, I picked up image file 

bin/ar71xx/openwrt-ar71xx-tl-wr841nd-v5-squashfs-factory.bin

and flashed the router via web interface. Everything went fine, but after the reboot - router is not responding anymore... I can clearly see that it boots up by blinking leds. Also when I plug the cable between PC and router - LAN1 (or any other too) LED gives light as well - but that is all... I cannot even ping or telnet to the router - always getting "Request timeout"...

Tried accessing by old IP address and by default - 192.168.1.1  Also tried hard reset as described in documentation (holding reset plug for 15 seconds) - to be sure that router has default 192.168.1.1 IP address - but still with no luck...

Can it be that by building custom image I've missed some important package/lib, which was needed by router for network to run properly?

Is there a chance to flash the router again with "stock" OpenWRT image?

Thanks in advance!
