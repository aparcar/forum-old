Theese routers gets mac address 00:0c:43:30:50:77, all routers gets the same address making a mess on my LAN! It looks like a Ralink MAC. Using original firmware it gets mac 54:04:A6:xx:xx:xx. Thats an Asus MAC, and it is also printed on the router. As a workaround I set the mac address in /etc/config/network.

Currently I have no time to investigate this further, just want to inform the community of this bug.

Wrong offset was used when reading MAC in target/linux/ramips/base-files/lib/preinit/06_set_iface_mac, offset should be 4, not 40.

Fixed in trunc r32877 and AA r 32879.
