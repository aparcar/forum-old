Hello,

I found this while I was searching for board pictures http://www.dd-wrt.com/phpBB2/viewtopic.php?p=539053 and I agree with them

If you want to try to run OpenWRT on this small machine, know that OpenWRT currently does not have ADSL drivers.

So this device would be used for a wired router with VLAN support eventually.

Please post pictures of the board where we can read the labels of the chips. We also have to see if it is possible to connect a serialTTL cable to the board, JTAG,...

Also if you can get into the current firmware with telnet/SSH and issue and "sh" command, do you get a # prompt? If yes - post the boot log or output of dmesg command

Post the boot log if you have access to it from the Web interface.

Thank you.


P.S. from the firmware files on TP-Link website I see that the v2 (haven't checked v4) board_id most probably is 96338L-2M-8M this means 2MB flash and 8MB ram if we believe the tag.

Maye it is possible to fit OpenWRT on 2MB flash if you compile it yourself as a very stripped version. The image that could work would be the Broadcom generic one, if the CFE accepts it at all.

Check this to see what you're signing up for if you choose to try it: https://forum.openwrt.org/viewtopic.php?pid=128156
