I think you can understand target as an architecture with a vendor specific solutions.

For example bcm47xx is a group of devices based on MIPS, but you can't just use some MIPS-generic build. Target bcm47xx is Broadcom specific, so it contains drivers for Broadcom specific bus, for NVRAM, for Broadcom flashes, etc.
There are many different MIPS based targets, for example ar7. It's Atheros specific (not Broadcom), so it won't work with the same init, drivers, etc.

You can take a short analyze of any target to understand it a litter better. For example take a look at target/linux/brcm47xx/config-3.8. This is config for bcm47xx target which enables Broadcom specific drivers. It's completely different from ar7 (and mostly all others).
Apart from "config" target also contains patches for kernel, that have to be applied to the mainline kernel. Not all drivers have been upstreamed, so we keep some patches just in OpenWRT.
