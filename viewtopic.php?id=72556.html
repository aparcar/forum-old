Dear All,

I'm new here. I have an problem with an D-Link DGS-1248T switch. I know this site is mostly for routers but i will give it an try. I think there is a lot of simularity with routers.

First of all i will give all info i could discover:

Brand: D-Link
Device: DGS-1248T
Hardware Rev: B1
Initial FW: 3.00.07

CPU: Marvell - 88E6218-LG01
RAM: Mira - P2V64S40ETP
ROM: Intel - JS28F160C3BD70

There are 2 places for headers i populated:
1) 6 Pin - UART Connection to CPU
2) 20 Pin - JTAG Connection to CPU

The issue is that the switch doesn't run, the CPU LED on the front stays off. I connected the UART to my PC and started the terminal. The CPU runs ARMboot. The "Hit any key for boot setup" appears.

If i do nothing the CPU will boot and after the message "wss-ii driver initialization done" the CPU hangs and the terminal stops working.

If i hit any key during boot i get into the ARMboot Menu.

I think the ROM flash is corrupt or broken. D-Link only supplies .hex files for the firmware. Should I load this file in the CPU or in ROM Flash memory? And how?

Would great if somebody could help me get this thing running again.
