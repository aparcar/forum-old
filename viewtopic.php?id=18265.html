After that I need some help.
I try to access my automounted hdd (on my router asus wl500g P V1 with openwrt kamikaze 8.09 RC1) via FTP.
Due to small place on the router memory and dropbear is already installed I followed the idea from (http://wiki.openwrt.org/FTPDHowTo) where they say that dropbear can be used with openssh (server) to allow a secure ftp access. Then I followed this (http://wiki.openwrt.org/SFTPWithDropbearHowTo) but when I tried to connect to the router in ftp with konqueror (ftp://root@192.168.1.1:21) or with krusader I get an eror message that tell me : unable to connect 192.168.1.1.

I presume it comes from the fact the firewall don't get automatically access to the port 21.
Then I added those lines as explained in :
iptables -t nat -A prerouting_rule -i $LAN -p tcp --dport 21 -j ACCEPT
iptables        -A input_rule      -i $LAN -p tcp --dport 21 -j ACCEPT
iptables -t nat -A prerouting_rule -i $WAN -p tcp --dport 21 -j ACCEPT
iptables        -A input_rule      -i $WAN -p tcp --dport 21 -j ACCEPT

I putted the firewall.user under /etc and also tried under /etc/config but nothing changed.
I can not get ftp access to my router.

To get that working with 8.09 RC1 all I had to do is:

root@OpenWrt:~# opkg update

root@OpenWrt:~# opkg install vsftpd openssh-sftp-server

root@OpenWrt:~# /etc/init.d/vsftpd enable

root@OpenWrt:~# /etc/init.d/vsftpd start
