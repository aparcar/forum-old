Hello

I have been reading all of the posts about DG834 on here and all over the web in regards to flashing OpenWrt on it, though I have now reached a road block.

This is the steps I have taken in an attempt to flash OpenWrt:

1. Patch bootloader and load it dd if=mtd2-patched.bin of=/dev/mtdblock/2

2. echo "my_ipaddress 192.168.0.1" > /proc/sys/dev/adam2/environment

3. Pulled the power out and it rebooted fine

4. svn co svn://svn.openwrt.org/openwrt/tags/8.09.2

5. make menuconfig - loading either of this and this as .config

6. Installed a ethernet patch for AR7 (which is for a DG834 v3)

7. make V=99

8. dd if=openwrt-ar7-squashfs.bin of=mtd1.bin bs=1 count=720896 and dd if=openwrt-ar7-squashfs.bin of=mtd0.bin bs=1 skip=72089

9. On the router cd /tmp/ then wget the mtd0.bin mtd1.bin from a PC running httpd

10. Then dd if=/tmp/mtd1.bin of=/dev/mtdblock/1 and dd if=/tmp/mtd0.bin of=/dev/mtdblock/0

11. Pull the power out and now it is not responding to PING

The lights are not flashing in an alternative state and when pinging the router the port light on the router which the pc is connected to that's issuing the ping command is flashing so maybe some packets are getting sent there but not replying back?

I have made sure that the IP is the same as the one in .config because if I try and ping any other IP address it gives a "no route" or something different.

Could it be that I applied that ethernet patch or that I used a config file which was initially used on a older build of Kamikaze?

Any ideas would be appreciated it. Thanks!
