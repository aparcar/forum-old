I use both version alternatively and there is not much difference. The QoS change to codel was included in AA as patch to kernel 3.3.8, while it is built-in in kernel 3.5 onwards. That is perhaps the most significant kernel change between those versions.

If you are happy with AA 12.09, there is no major reason to change yet.

One factor against an update is that as trunk builds are sanpshots, you need to add all packages at the same time, as the packages compiled a week later might not be compatible. AA has a stable package download library, so it is much easier for a "casual user".

If you compile Openwrt by yourself, then it is much easier with trunk. (But like Nilfred already said, a spare router can be handy, or at least a router with easy recovery from a bad firmware flash.)

For some reasons the  Wikipedia Article on OpenWrt contains more information then the OpenWrt wiki.
For the current Kernel version, quickest way would be to have a look in the ftp downloads directory (for your platform!): ar71xx Downloads. Search for "kernel". It the meta package for the Linux kernel included, e.g. "kernel_3.10.4-1-675..." Sometimes Kernel versions are different between platforms!

But I cannot tell you where to get more information about existing differences between OpenWrt current stable and OpenWrt next.

Only the developers know that, and I doubt anybody has the time and the will to answer such questions... you could try on IRC, but I would not.

You could look at the changes since the stable fork, but this misses the changes in the kernel. You would also have to look for changes in the packages upstream.
  * E.g. OpenWrt says: "update busybox package from version 1.19.1 to version 1.20.0". This tells you nothing about the changes inside. So then you have to go to busybox page, and see there, what has changed!

Sometimes there are changes, you would profit from big time. But that depends on what you are doing with your device.
