You have to set connbytes over luci with -bytes / -1024 without you get crazy high pings over 1000ms 6000ms & more

Sad is openwrt qos don´t support marking for incoming traffic by default for this you need custom build and scripts.
But with connbytes set you don't have problems with freaky  high pings UP/DOWN the same time

examble:

config classify
        option target 'Normal'
        option proto 'tcp'
        option ports '20,21,25,80,110,443,993,995'
        option connbytes '-1024'

l7filters mostly useless with crypt traffic any torrent or game downloader with peer to peer  app enable crypt by default today,
set prios for port range for this apps
