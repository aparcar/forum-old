Hi There,

I am currently using the 8.09/brcm47xx/openwrt-wrt54g-squashfs.bin firmware on my WRT54G-V2.0 and am experiencing some wifi stability problems. I have seen other people with similar issues running kernel 2.6 firmware on broadcom chips, but haven't been able to find a definitive answer.

The problem is that under conditions of moderate to heavy load (for example streaming video over wireless), my wireless connection drops and the router needs to be rebooted before anyone can connect to wifi again. This was happening much more frequently using WPA-PSK/TKIP but still happens at least every 10-15 minutes (under load) with WPA2-PSK/AES.

When I go to the system page in luci I notice that although I am using a brcm47xx firmware, my router actually is using the Broadcom BCM3302 V0.7 chip. I thought the V2 used a brcm47xx chip... Maybe I am mistaken? Could this be the source of my problems?

Also, does anyone know if using one of the kernel 2.4 builds of kamikaze could alleviate this issue?

Thanks for the knowledge,
Stephen
