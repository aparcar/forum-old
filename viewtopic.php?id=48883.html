It would be nice if this feature was included in LuCi, but as of this post it is not.

For those who would like to use their WNDR3700v2 in place of the Actiontec router, you will have to give a VID 35 (for Bell/Aliant) on the WAN Port. Currently this feature is not available in the web interface, however it is still possible by modifying the configuration file in the router itself.

Using PuTTy, connect to the router. (Default 192.168.1.1)

login using root and use the password you set in the web interface.
issue the command: vi /etc/config/network
Scroll down with the cursor keys until you see the following:

config interface 'wan'

you will want to add the line, option ifname 'eth1.35' and option macaddr 'xx:xx:xx:xx:xx:xx' where xx:xx:xx:xx:xx:xx is the mac address of the actiontec router which can be obtained by looking on the underside of it under WAN MAC.

as an example, my configuration looks like this:

config interface 'wan'
             option proto 'dhcp'
             option _orig_ifname 'eth1'
             option _orig_bridge 'false'
             option ifname 'eth1.35'
             option macaddr '11:22:33:44:55:66'
             option hostname 'R1000H-Router'

remove the actiontec from the ONT and replace with the wndr3700v2. Reboot the router if not already done so, and you should have internet access now.
