I'm using an Academic campus network and we have a proxy server that connects us to the outside world. I use an Android phone and right now it's impossible for me to set proxy settings (with authentication) on my phone without rooting it. I was thinking, I can setup some kind of forwarding on my wireless router to send the packets to the proxy server.

Many online articles give solution to this problem with the use of iptables, but all their setups consist of connecting to a custom proxy server which requires special Transparent proxy capabilities turned on. I don't have admin access to the campus proxy server, so I need a solution that only needs modifications on my router side.

My basic understanding is that my router needs to mark all the packets coming from my phone and forward it to the proxy server. It would also need to add or modify certain proxy headers in the packets. I know how to achieve the first step with the help of iptables. What I can't figure out is the second step.

Note: I am connecting to an authenticated proxy server that uses basic password based authentication.
