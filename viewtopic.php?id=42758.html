A short answer to the floatingpoint topic:
most CPUs used in routers don't have hardware-floatingpointunits, and for network stuff you don't need it anyway. So if you run code with floatingpoint operations (and most audio/videos-codecs do) the system have to emulate a  FPU (=softfloatingpoint) with really bad performance. So the best way is to run codecs which only use fixedpoint-operations.
For ogg you have the codec tremor and for mp3 you got libmad as fixedpoint-implementation.
For aac (faad2) seems to be some build option to compile it as fixedpoint-codec.
In AA faad2 is compiled with floatingpoint support, but there is a changeset in trunk, so faad2 should be fixedpoint-code there.
So try trunk, or apply the changeset to AA, and make sure you compile mpd-mini (with tremor, libmad and faad2). mpd-full uses vorbis-codec for ogg (floatingpoint-implementation)
