compat-wireless is the core mac80211 wifi driver package. You most likely need it.

And what do you mean by "kernel version is wrong"? ar71xx is set to use 3.14 kernel at the moment.
https://dev.openwrt.org/browser/trunk/t … x/Makefile

3.14 kernel version was bumped from 3.14.18 to .25 yesterday. See https://dev.openwrt.org/browser/trunk/i … version.mk

After a kernel version bump you need "make clean" and also delete build_dir/toolchain.../linux* files.  (or just do "make dirclean" to clean the whole toolchain.

Openwrt's kernel version support is complicated. See the two files I linked earlier...
* include/kernel-version.mk defines the  kernel revisions that are currently supported for some target platform
* ar71xx/Makefile defines that ar71xx uses currently 3.14
* there are separately config & patches for each major kernel revision for each target platform.

You may attempt to change that 3.14 to 3.18 in ar71xx/Makefile, but then you will also need to create the necessary 3.18 config & patches for ar71xx. The "generic" linux contains 3.18 config & patches (see directories in https://dev.openwrt.org/browser/trunk/t … ux/generic ), but the config and patches specific for ar71xx platform have not yet been updated for 3.18. There are only 3.14 stuff in https://dev.openwrt.org/browser/trunk/t … nux/ar71xx . If you want to use 3.18 with ar71xx, you need to create the needed config parameters.

Most likely it will be easier for you to backport your own code to 3.14.
