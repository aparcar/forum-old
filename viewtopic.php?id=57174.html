Hello OpenWRT Community,

I've got quite a weird problem. I have a TP Link WR841N v.9.2 Router which can connect to the Internet with its original Firmware without any problems. I just put in the MAC Adress I gave to my provider and it immeadiatlyy works perfectly.
So far so good: But I want to use a VPN Service and I dont want to always start the Software on my Desktop and I also want my Smarthphones and my other Devices to benefit from the VPN Connection, so I decided to Upgrade to OpenWRT.
Problem here is not with the VPN Integration but much weirder.
I installed the newest supported OpenWRT for my router and configured the MAC Override for the WAN interface but: No Internet.

Well actually thats not true: I have Internet for like 2 seconds after connecting the interface.
Enough time to load up google but then the connection dies. But the weirdest thing is this: My PC still says it has Internet Connection on the little symbol right next to the clock in Windows 8. But I can guarantee that I don't get any connection at my PC.

This is so confusing to me, why does the original Firmware work and deliver Internet with no problem, and OpenWRT has such an unusual bug?

I hope you can help me,
Sincerly
Nyard

SSH in to the router and do 'ifconfig' to see the MAC that is actually used.  It may also be using the factory MAC at first then switching and the provider doesn't like that.

If this is the router that you will be using (either stock or OpenWRT firmware shouldn't matter) you should tell the provider to allow its factory MAC.
