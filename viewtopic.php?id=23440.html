Post solution I found for myself. (only works for mimiPci type of wireless card)
Change Regdomain info in EEPROM by a laptop running Linux, take the wireless card if you have miniPci type.
Insert it into the linux machine. Depends on the driver check country code and regdomain.
ath5k: iw set get
madwifi: sysctl -a | grep country
madwifi: sysctl -a | grep regdomain

if both country code and regdomain are 0x0, it's default to US (only ch1 to ch11)

use ath_info tool to write 0x36 regdomain in EEPROM.
#ath_info -w 0x(hardware memory address, you can get it by using lspci -v) regdomain 0x36

After this reboot, and check by iwlist channel, you are about to get channel 1-13, tested on AR5413 chip with ath5k and madwifi

WARNING: You may break local law by legally using unauthorized frequency band.

Have Fun!
