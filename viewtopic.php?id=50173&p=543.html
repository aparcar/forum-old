If you don't have a USB-TTL cable, it's highly recommended to invest in one, as it not only allows one to debrick their router from a bad flash, it also helps troubleshoot boot, as well as other, issues.

If flashing w/o selecting keep settings selected results in the same issue, it's likely one of the following applies:

  • Corrupted download, which can be verified via checksums

  • Unsure what revision completely removed telnet, allowing for SSH without a password set, as I haven't used CC for around a year.  (I know I flagged the message in Outlook with the revision and will look when I have time tomorrow)

  • It's also possible kaloz's images do not have the NAND patch applied, resulting in a kernel panic ~10s into the boot process.
