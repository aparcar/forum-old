Attitude Adjustment will "work" on a WRT54g, but don't expect it to be usable. You can give it a try if you want some pain. I couldn't make it through configuration.

https://forum.openwrt.org/viewtopic.php?id=39117

blogic wrote:

The OpenWrt Team is happy to announce the beta release of Attitude Adjustment (12.09)

[...]

EOL Notice for brcm-2.4:
Kernel 2.4 support has been dropped from Attitude Adjustment. For most devices using brcm47xx images is fine, but older models with only 16 MiB RAM or slower CPUs (200 Mhz) will not run properly with it. So for those devices, sticking with backfire is recommended.

You can always download the toolchain and build your own packages.

Edit -- It appears that at least the Makefile for each package in question would need to be updated for newer source versions. For example, Backfire busybox builds v1.15.3, yet AA builds v1.19.4
