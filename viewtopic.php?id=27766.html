fatbob

trumee wrote:

Hi all,

I am running backfire rc4 on a Buffalo wzr-hp-300NH with ssh running on the wan side. I have the following rules in /etc/firewall.user

iptables -A input_wan -p tcp --dport 22 -m state --state NEW -m recent --name SSH --update --seconds 30 --hitcount 10 --rsource -j DROP
iptables -A input_wan -p tcp --dport 22 -m state --state NEW -m recent --name SSH --set --rsource -j ACCEPT

Any idea what might be wrong?

The above lines should work. Make sure you have iptables-mod-conntrack-extra + kmod-ipt-conntrack-extra installed. Also make sure that you do _not_ let port 22 through in your /etc/config/firewall. That will bypass the rules in firewall.user.

I also found that the above did not work on 10.03.1-rc4. The following does seem to work, it's a variation on a method described in another thread:

iptables -t filter -A input_wan -p TCP --dport 22 -m recent --name SSH --rcheck --hitcount 3 --seconds 120 -j LOG --log-prefix "SSH_BRUTE "
iptables -t filter -A input_wan -p TCP --dport 22 -m recent --name SSH --update --hitcount 3 --seconds 120 -j DROP
iptables -t filter -A input_wan -p TCP --dport 22 -m recent --name SSH --set -j ACCEPT

Of course running ssh on a different port and turning off password authentication are methods also worth considering.
