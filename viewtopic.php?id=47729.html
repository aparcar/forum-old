I have the following situation:

My internet provider gave me a modem/wlan/router device which I placed in my living room. I connected that device via ethernet cable with my TP-Link router with openWrt on it which is located in another room.

The LAN-Zone of the provider router is 192.168.0.* and the LAN zone of the openwrt router is 192.168.1.*.

Everything is working fine, but my TV (192.168.0.10) which is in the living room connected via lan-cable with the provider-router  can't speak with my PC in the other room (192.168.1.100).

How can I configure my openwrt router to get a working connection between my TV and my PC? But I want to use the openwrt router as a DHCP server and not only as access point. Any ideas`?

Here is example http://wiki.openwrt.org/doc/recipes/bridgedap Use this with 192.168.0.* network

vertigo wrote:

But I want to use the openwrt router as a DHCP server and not only as access point. Any ideas`?

If you have access to ISP router, you can add route 192.168.1.0 is behind yours OpenWrt router and on OpenWrt disable masquerade.

Second way configure on OpenWrt firewall redirect traffic to your PC. for example all traffic to 192.168.0.44 will redirect to 192.168.1.100.
