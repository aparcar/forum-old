Router Model    Buffalo WZR-HP-G300NH
Firmware Version    OpenWrt Attitude Adjustment 12.09 / LuCI 0.11.1 Release (0.11.1)

Under v10.03 the WAN status would show the WAN IP correctly, and would only revert to the LAN IP if the WAN connection dropped; it was an easy way to check if the WAN was down.

Under v12.09 the WAN status seems stuck on the LAN details, even when the WAN connection is working.

Is there a setting somewhere which controls what the IPv4 WAN Status section displays? I've been through all the LuCI tabs and can't see one (or I've missed it).

Hi,

I had the same issue:

OpenWRT version:

Model    : TP-Link TL-WDR4900 v1
Firmware Version: OpenWrt Barrier Breaker r39770 / LuCI Trunk (svn-r9960)
Kernel Version: 3.10.32

Solution :

Go to "Interfaces" -> "LAN" -> and leave the "IPv4 gateway" blank!

Now the WAN is showing correctly in the overview page.

I got the hint from here:

https://dev.openwrt.org/ticket/12381

See ya....
