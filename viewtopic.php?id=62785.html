sok wrote:

An easy solution would be to copy those files you want to be protected to a separate (encrypted) partition and symlink them via a script. The script can be called in rc.local (on boot).

That is a good idea. I have put together this process:

1) Create two partitions on the SD card (second one will be encrypted later)
2) Place openwrt on the first one and install all OUR necessary packages, dependencies, scripts, etc. as per normal. Make sure that nothing autostarts with the OS boot.
3) Cut all necessary packages, dependencies, scripts, etc. to second partition and create symlinks from old locations to point to the new locations.
4) Encrypt the second partition and store the private key on a cloud server.
5) Put a script in the boot process to authenticate with that specific RPi's userID, pull the private key from our server via an encrypted channel (SSH is fine) and to decrypt the second partition.
6) Put another script that runs after decryption to start all services, scripts, etc. that are on the second partition.

If I didn't do it this way, i presume that I could just hide the decryption key on the non-encrypted partition but that would blow a hole in the security. 

Is this the best way to do it?
