Hi. It seems that only the NEW packets go through the iptables rules. When I insert a rule to OUTPUT or FORWARD chain with -j LOG, only the first packet of each connection is logged. "-j DROP" does not cut existing connections.

Compared with a desktop Linux. There all works as expected.

Seen this:

root@OpenWrt:~# iptables-save | grep ESTAB
-A delegate_forward -m conntrack --ctstate RELATED,ESTABLISHED -j ACCEPT
-A delegate_input -m conntrack --ctstate RELATED,ESTABLISHED -j ACCEPT
-A delegate_output -m conntrack --ctstate RELATED,ESTABLISHED -j ACCEPT

Tried to insert rules to those chains BEFORE the ACCEPT rule, but it had no effect.

Perhaps some performance optimization is turned on in OpenWRT, that prevents established packets from going through the rules? Some flag that can be controlled via echo 1>/proc/... ?
