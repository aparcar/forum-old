Well it is not the original firmware anymore after you put openwrt on it. (it is much better).
Openwrt is highly configurable firmware, that has a lot of packages that you can install on it.

Back to the iphone if you connect to the Iphone over wifi you don't need aditional drivers at all. Because it is part of the standard installation.

Here is an example config for the device, I made the ethernet port part of the lan (sorry forgot if that was default with orig firmware anyway).

Here you go, an example on how it could work with all wifi, the ethernet port is in this case part of the lan

/etc/config/wireless

config wifi-device 'radio0'
        option disabled '0'

config wifi-iface
        option ssid 'IphoneSsid'
        option encryption 'psk2'
        option device 'radio0'
        option mode 'sta'
        option key 'Iphonekeyhere'
        option network 'wan'

config wifi-iface
        option device 'radio0'
        option mode 'ap'
        option ssid 'OpenWrt'
        option encryption 'psk2'
        option key 'iamverysecure'
        option network 'lan'

/etc/config/network

config interface 'loopback'
        option ifname 'lo'
        option proto 'static'
        option ipaddr '127.0.0.1'
        option netmask '255.0.0.0'

config interface 'lan'
        option ifname 'eth0'
        option type 'bridge'
        option proto 'static'
        option ipaddr '192.168.1.1'
        option netmask '255.255.255.0'

config interface 'wan'
        option proto 'dhcp'

You can edit those files with vi on the router trough ssh, or via winscp: http://wiki.openwrt.org/doc/howto/basic.config
Or you can use the gui, and use this as a guideline.
