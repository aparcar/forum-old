Hi, first you have to explain what kind of settings are looking for. For example, I do not need the relayd to make a bridge between WLAN and LAN (WAN access is other, no wireless). In this case, how to is:
1. Create a wireless access point, with a new interface called wlan.
2. Go to LAN interface, physical settings, add check bridge with wlan.
(3. check that wlan is together lan on firewall settings)

I think that I try to do same but check "lan" interface on wireless initial settings, but I think remember that it give me some mistake (also, no try to do bridge from WLAN interface, or you have a broken wired lan). With this simple bridge, you extend your local wired network to wireless interface. That interface say "unsupported protocol" is safe, because WLAN is managed by LAN directly (clients get ip over local network from same dhcp server) and WLAN clients have same access to WAN network.

Regards.

Hi, this is why I asked you to describe what you want: I explain a simple bridge, but maybe you need a realyclient, if you get wan from another wifi router. My plan (share wifi and lan on same router):

WAN <- MR3220 -> LAN + WLAN

But I think that you talk about this other case (share wan from another main router):

WAN <- WIFI-ROUTER <-> WLAN <- MR3220 -> LAN
                     ^

In this second case, sure my comments do not help: check relayclient (although still a little confusing for a newbie). Hi, If you don't have difficulties with the failsafe (saving settings before testing is easier), you can try to check "Disable DHCP for this interface." on LAN interface. Target on a relay client is get dhcp ip from main router.
You can see:
"Run ifconfig and take note of the IP address assigned to wlan0. Then tell the relayd that this is the routers IP address with the following config."
It's set on "Local IPv4 address" on relay protocol page (not in the tutorial with an old version of luci).
Regards.
