Can you please elaborate on the topology of the devices?  I ask because if you NAT at the ISP router and again at the OpenWRT router, it is likely that no amount of tinkering at the OpenWRT side will get it to work.

UPnP is the correct tool to use for this, but it will only work if there is one "layer" of NAT'ing.

First prize would be to NAT only at the OpenWRT router.  If you have config-level access to your ISP router, you should investigate whether it uses PPPoE (it likely does) and whether you can set it to bridge mode and initiate the PPPoE connection from the OpenWRT router instead.  That would require the username/password for the PPPoE session; some ISPs that supply pre-configured routers don't make that information available.
