Pfft, sure, take the easy way out.

Works for me as recent as r21639.

/etc/config/ddns

       option enabled 1
       option interface wan
       option program ez-ipupdate
       option service dyndns
       option username myname
       option password mypassword
       option hostname mydomain.dyndns.org

KillaB
14 Mar 2011, 02:37
hanz wrote:

i modified my ddns-config as killab stated.

/etc/config/ddns is the only file you need to edit. Don't touch the init scripts.
Also, leave the interface option as wan. You shouldn't need to specify ppp0.

config ddns
       option enabled 1
       option interface wan
       option program ez-ipupdate
       option service dyndns
       option username myname
       option password mypassword
       option hostname mydomain.com
hanz wrote:

but how do i know if ez-ipupdate is updating my ip? the old startup-script won't work anymore, i guess:

Use dig or nslookup to perform a dns query on your DDNS hostname. You can also cat /tmp/ez-ipupdate-dyndns-<wan interface>.cache

written_direcon wrote:

I'd simply try and use the ddns-scripts package.

And how is this being helpful? You still have to configure it!
ez-ipupdate works perfectly fine. The *only* reason to use ddns-scripts is if you're running on a system with extremely limited flash space.

EDIT: I guess another reason would be if you're using LuCI?
