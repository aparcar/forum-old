I have a WDR3500 that I flashed with the firmware I found in this thread; https://forum.openwrt.org/viewtopic.php?id=42292

Everything works great, except LAN clients can't talk to eachother.  Wifi clients can talk to LAN clients, but when I try to send packets from one wired desktop to another, they are mysteriously dropped.  Even running tcpdump on my LAN bridge or straight on the eth0 interface doesn't show anything on the router.  (Although on the desktops, I can see the packets leaving the computer)

Here is my /etc/config/network file:

config interface 'loopback'
        option ifname 'lo'
        option proto 'static'
        option ipaddr '127.0.0.1'
        option netmask '255.0.0.0'

config interface 'lan'
        option ifname 'eth0'
        option type 'bridge'
        option proto 'static'
        option netmask '255.255.255.0'
        option ipaddr '10.10.11.1'

config interface 'wan'
        option ifname 'eth1'
        option proto 'dhcp'

I have tried messing around with vlans and switches and such, but I can't seem to get anything to work; I can't seem to figure out how to get OpenWRT to allow me to talk about single ethernet ports on the back; I don't seem to have that kind of control.
