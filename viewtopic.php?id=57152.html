So tell me, whether uhttpd is be good for my page ? And whether all technology they will be handled ?

This question cannot be answered because of several (!) reasons:

- Responsive and "all technology" is very vague
- strictly speaking: responsive "guarantees": means some sort of Realtime
- performance varies WIDELY between Platforms/Chipsets/Compile settings that are possible

    * Compiling with -O3 might behave differently from normal -Os
- There are no serious Benchmarks for OpenWrt available - feel free to create one/one test protocol
- see usb or openssl benchmark pages in wiki
- Embedded systems are often design with specific goals - there is no "all round" System because of Hardware design limits and trade-offs: USB performance, Wireless performance, power consumptions, attachable GPIOs, I2C performance, other bus that are attached (I2C, Multimedia), NAT perfomance, DSP integration ... so this has to be taken into account when developing on embedded systems

So before I create my system I whant to ask what to expect from serwer http installed into OpenWRT

This totally depends - How many users, "hostile environment" , USB or Wifi enabled / active /how many clients there etc. - see above

Notice that there are several projects like Piratebox and possibly others that do offer web services from OpenWrt nodes, run nginx etc. but from very different HW / usability povs.


PS: Example: If I enable "security" on OpenWrt - performance of USB copies etc. gets lowered to low single digit Mbyte/s. I consider this "not responsive" compared to 20Mb/s that are max. possible.

Yes, uHTTPd can execute PHP, although in CGI mode which might or might not provide sufficient performance, see http://wiki.openwrt.org/doc/uci/uhttpd#using_php5

PHP5 is a bad choice for embeded systems imho due to its rather high ram requirements so maybe evaluate smaller alternatives for the backend.

HTML, Javascript and CSS are static assets, the choice of the webserver on the backend does not matter for that.

You could serve the static stuff from the small MT7620, and create the dynamic part (PHP) on another, larger server.
OR put everyhing onto a bigger server in the background, and use squid with careful config on the MT7620, to cache your static data/part of dynamic data. Performance  should be similar as served from apache on the MT7620.
Some of your "tasks" are done within the browser, anyway, like "Responsive Design", which usually depends upon JS and CSS.
