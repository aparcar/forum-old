Hello!

I use VLANs on my TPlink WR1043 to separate several machines on my network. From the router's point of view the modem of my cable TV provider represents the WAN side. I need to split my net there, because that modem is controlled by the provider and thus it is not trustworthy to me..

However, my TV is also placed inside this WAN side, because it needs the TV signal from that modem. Basically, the network looks like this:

WAN (modem, TV) <-> OpenWRT <-> VLANs (my pc, servers, printers, and others)

Now I want to run a DLNA server inside VLAN3 and I need to route its traffic from there to the WAN zone. Following these instructions, I set up igmpproxy and I simply replaced WAN with VLAN3 and LAN with WAN in that instructions:

config igmpproxy
        option quickleave 1

config phyint
        option network dlna
        option direction upstream
        list altnet 0.0.0.0/0

config phyint
        option network wan
        option direction downstream

The server is running in zone "dlna".

Of course I applied the firewall entries shown on that page, and of course I switched the zone names there as well. But that does not work, my TV does not find the DNLA server.

Before I do more experiments without any success, I'd like to ask if it is possible at all to use a DLNA-Server *inside* LAN and feed a client in WAN with it. Thanks in advance for any help.
