The Nanostation loco M2 (one Ethernet port) is different hardware than the regular Nanostation M2 (two Ethernet ports).  Be sure you're using the appropriate firmware.

Use TFTP if necessary to flash stock firmware (5.6 version), then use that firmware's web interface to downgrade to 5.5.  (Using the web interface changes the bootloader, using TFTP does not.) Then install OpenWrt.

OpenWrt safe mode is different from Ubiquiti TFTP mode.  To get TFTP mode, hold down the reset button before connecting the power, and keep holding it the whole time through the boot until the signal LEDs start flashing in a back and forth pattern.

Then set your PC to a static address such as 192.168.1.10 and use a TFTP client to PUT (in binary mode) a firmware to the router at 192.168.1.20.  The LEDs should start flashing in a different way while the firmware flashes, then it will reboot.

Once you flash Openwrt with the newer Uboot the only way Ive found to recover is to use a serial connection to tftp flash the device.
1. Open up the device and locate the serial jumper, labeled J1.



2. Connect the USB-TTL Cable to the jumper, which has the following pinout:
---1. 3.3V+ (Red: Do not connect)
---2. S-in (GREEN)
---3. S-out (White)
---4. GND(Black)



3. Open up a terminal program, such as tera-term, and configure it to the following settings:
---Bits per second: 115200
---Data Bits : 8
---Stop Bits : 1
---Parity : None
---Flow control : None



4. Plug the POE adapter into the wall.



5. Plug the LAN side of the device into your PC.



6. Configure your PC IP settings to the following:
---IP Address: 192.168.1.123
---Netmask : 255.255.0.0

press any key when prompted to stop boot

urescue -f -e
tftp v 5.5.11 to 192.168.1.20
reboot
urescue -f -e
tftp v 5.6.xxx to 192.168.1.20
reboot
gui flash back to 5.5.11
Then you can flash with Openwrt
