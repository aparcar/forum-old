Orca
5 Jun 2010, 01:35

I had the same problem. It drove me crazy.  Check if there is /tmp/fstab and if it is populated as intended in your /etc/config/fstab. IF yes, and if you have a /etc/fstab, too, THEN we both have been antagonize by the same bug:

Try vim /etc/init.d/fstab  and put a # in front of the line echo '# WARNING: this is an auto generated file, please use uci to set defined filesystems' > /etc/fstab.

This line creates the file /etc/fstab and thus prevents creating a symlink to /tmp/fstab. In order for uci to work, there need to be symlink instead of the file!

I already put this info into the wiki:
http://wiki.openwrt.org/doc/uci/fstab

It is a real pain in the ass to find something in this forum...
