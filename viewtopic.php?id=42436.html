electric_hammer
21 Feb 2013, 09:41

ok.

see this link for more         http://luci.subsignal.org/trac/wiki/Installation

I think your source should be
http://downloads.openwrt.org/snapshots/ … x/packages (without /packages.gz)

in which case you will probably get "package_not_found" or similar

to pull the luci package add it to the source
from terminal#  vi /etc/opkg.conf
# i (edit mode)
add line "src luci http://downloads.openwrt.org/attitude_adjustment/12.09-rc1/ar71xx/generic/packages"
press Esc key
#:wq

(# above represents terminal prompt)

worked for me. good luck!

Snapshot download directory normally includes Luci. But yesterday's build was erroneous, and Luci was temporarily missing. The next build is now ready, so Luci is again there now:

The advice to add some luci feed to opkg.conf is wrong.

I can download http://downloads.openwrt.org/snapshots/ … ackages.gz quite normally, so that "bad address" error looks like a DNS problem at your router. If it is something like that, then no Openwrt version will work before you are able to sort DNS out.

For novice users, the safest bet is to install the most recent "stable version", which is currently Attitude Adjustment 12.09 RC1. Downloads from http://downloads.openwrt.org/attitude_a … 12.09-rc1/
