Hello,

i've set up OpenVPN with a static key and a bridged interface tap0 which is part of my lan-Zone.

I have the following problem:
To try the VPN i connect from my local PC (which is directly connected to the router). If i configure my PC to connect to the local ip (192.168.1.1) OpenVPN does connect.
If i try to connect "over" my WAN-IP it does not connect (last line in the status windows is: "Wed May 02 14:06:05 2012 UDPv4 link remote: ...:1194").

I think! the necessary ports are not open.
In Network->Firewall->Traffic Rules i created two rules

config rule
    option target 'ACCEPT'
    option name 'VPN Rule 1'
    option family 'ipv4'
    option dest_port '1194'
    option src '*'
    option proto 'udp'

config rule
    option target 'ACCEPT'
    option dest_port '22'
    option _name 'ssh-wan'
    option family 'ipv4'
    option src '*'
    option proto 'tcp udp'

BTW: Do i need the ssh rule?

If i try an online Portscanner both ports are closed.

Can anybody help me out? Not shure what i am looking for.

crushi
