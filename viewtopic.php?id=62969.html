Through SSH and iwconfig I can see that the client interface correctly associates with the correct access point (I get ESSID and Access Point MAC on iwconfig) , but just briefly. Backfire is really old, so I don't know how much current documentation still applies. In current OpenWrt, you set the bssid option of the client (STA) interface to the MAC address of the access point you want to connect to. See doc/uci/wireless.

lucas.hartmann wrote:

The router I have at hand is really old himself. A Netgear WGT634U. I could not find an image for it in the latest version, and the old-versions server seems to be refusing connections.

Disclaimer, I've never used a WGT634U myself nor have investigated this in depth - but...

From the specifications it looks like the "Netgear WGT634U" would be rather comparable with the "Asus WL-500g Premium v1", slightly less CPU performance, but also 8 MB flash/ 32 MB RAM and a ath5k compatible wlan card straight from the factory. Given these baseline specifications, I don't see a problem with running current OpenWrt versions (both the CC release branch or trunk) on that particular router. While the webinterface might not be a runner, it should work and still fullfill routing/ AP needs just as well as it did a decade ago with older OpenWrt versions. Just that today's requirements tend to be slightly more involved, making a hardware upgrade certainly desirable.

I still use an Asus WL-500gP (with retrofitted ath5k wlan) occassionally for testing (trunk snapshots and-) larger configuration changes, before deploying them to (newer) production routers and it's still quite fine.
