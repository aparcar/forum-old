How to access OpenWRT router if there is no network connectivity? For example if I make a mistake in /etc/config/network file and instead of "option proto    dhcp" I type in "option proto    dhpc", save the file and reload the router:

config switch eth0
        option enable   1

config switch_vlan eth0_0
        option device   "eth0"
        option vlan     0
        option ports    "0 1 2 4 5"


#### Loopback configuration
config interface loopback
        option ifname   "lo"
        option proto    static
        option ipaddr   127.0.0.1
        option netmask  255.0.0.0

#### LAN configuration
config interface lan
        option type     bridge
        option ifname   "eth0.0"
        option proto    dhpc

For example Cisco routers allow one to configure the device over console(RS232) port. Or is the only option to upload new image to bootloader TFTP server and flash the router? Or is it smart to set up a MAC-telnet option?

m4rtin wrote:

in order to access the bootloader, you need a serial access anyway. Or what do you mean by "may also be something in the bootloader"?

Some routers have recovery functionality (mainly for reflashing) embedded in the firmware.

E.g. my WNDR3700 has a recovery mode with TFTP server and automatic reflashing that can be accessed during the boot process via the reset button. A new firmware can be uploaded with it and the firmware will then get automatically flashed. That enables de-bricking without console access. That is not full access to the config files on the router, but can be used to recover from cofig errors like yours.
