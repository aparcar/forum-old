Being honest I don't like CPE510 factory firwmare mainly because don't let you "improve" things and support forum for this product is very poor in terms of users and questions/answers. You only have the WebGUI to perform changes (Yes, you have PharOS Control but it's basically the same thing!). I want to be able to create scripts, using SSH commands but I don't want to them be deleted every time device is rebooted. Also the hierarchy of configuration files inside the device (using WinSCP) is very messy. I used Ubiquiti once and you got every configuration parameters in one file (Root SSID, MAC, IP, etc...), in this one only you have is files with iptables everywhere.

The savior: Digging a little I found out that CPE510 can use another firmware (an open one) named OpenWrt. Never have used this product (once used DD-WRT in a router). I guess with OpenWrt things are more "changeables" lets say. I can't post links!! OMG

And when I uploaded the firmware in order to perform the change I got "Hardware version is not supported". After getting this error message I started to "googling" and found several things related such as:
- "Firmware is not supported in v1.1 hardware"
- "put in tplink-safeloader.c the line CPE510(TP-LINK|YOURZONE or FABRICATION COUNTRY (CN)|N300-5):1.1\r\n"
- "People still have this unsolved"

I'm not a developer but I guess this can be solve easily (if only is required to add a line and compile the .bin again). I really need to get rid of CPE510 factory firmware but I'm stuck with this issue. Tried even changing zone after set the device to factory defaults. I could even change the tplink-safeloader.c and compile it again so instead of getting this job done from other person I could solve this with a "recompiling-tutorial".
I'll google this after post but how can I get this trunk (not familiar with this term): svn.openwrt.org-openwrt-trunk@47212 3c298f89-4303-0410-b956-a3cf2f4a3e73. I think the answer is in here.
