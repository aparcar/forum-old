The short of it: I'm trying to get hold of the original (v4.12-UK) firmware for the above linksys 'modem' OR advice on my problem:

(UK based btw)

Was using Zen Internet ADSL fine using the "Bridged Only Mode" option on the modem, which means it takes PPPoE packets and converts/tunnels via PPPoA (which all UK ADSL suppliers use)

This was all working fine untill I decided to move to freedom2surf's 8meg LLU (via easyNet iirc) - the migration happened on friday. When I got back from work the migration appeared to have happened, and once I'd found the login details i tried it - no joy!

The modem had a DSL sync fine, however it would just not get a PPP link, setting the modem into bridge mode and using pppd with debug mode enabled, it appeared that as soon as the ppp link was actualy established, it got disconnected from the other end.

Anyway, after hours of googling I decided to try flashing the firmware - except... (wait for it...) there aren't any offical firmware versions on any of the linksys' sites that I could see. so anyway I found a version that claimed to be 4.19UK, and it had two files: upgrade.this.first.360c.bin and LKS...419.img

Anyway, I applied both patches and lo and behold - I can get a PPP link to stay up! Sadly only when the modem was doing the PPPoA link itself, when it was in bridged mode (i.e. the mode where the modem behaves as an actual modem, rather than a router with no port forwards) it seems to just ignore the pppoe packets coming from my linux box.

So I see my options as this:

- Get a new modem (prolly not another one of same model)
- Install OpenWrt/AR7Port (whats the status of this?)
- Nasty hack with iptables on the router (possibly also using dhcp-fwd)
- Hope someone (joseangel?) has a copy of the original 4.12 firmware
- something else i've missed?


Any help would be greatly appreciated.
