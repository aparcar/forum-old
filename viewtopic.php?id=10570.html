if your planning to do vpn, and the device you have in mind can't do it with it's cpu, i'd recommend getting a more powerful device rather then comromizing on features by using the stock firmware.  I think via makes fanless 1-2ghz boards x86 now for example, which would obviously handle anything you wanted to throw at it (and seem to have supported hardware crypto, actually, not that you need to push vpn data at 20gbps). or one of the higher end embedded boards (routerboard, soekris, wrap, etc) designed for this sort of thing, which would be cheaper.

as to whether these consumer devices can do it without hardware accelleration, i've never tried, but I don't suspect you'd have much problem with a small number of clients, on a normal internet connection.  someone else will no doubt chime in with their experiences on this

I've been running OpenVPN on Linksys WRT54GS v.1.1 for quite some time now.  It doesn't have any problem saturating my 768 kbit uplink at either end of the VPN tunnel.   While doing so the cpu usage for OpenVPN is steady at 17.8%  This is on WhiteRussian RC5. 

Depending on you network speed that the VPN is on just about any hardware that OpenVPN runs on should handle your VPN just fine.  Number of clients is not much a factor, the bandwidth that must be encrypted is the biggest factor of CPU usage.
