I was able to get the iwlmvm module working, so the card works now. I can see AP scan results to verify that it's working. The problem is, OpenWRT doesn't seem to actually want to do anything with the card. If I manually run hostapd with the following config, the card will work in AP mode:

interface=wlan0
driver=nl80211

hw_mode=g
channel=1
ssid=mac80211 test
ieee80211n=1
ht_capab=[HT40+][SHORT-GI-40][DSSS_CCK-40]

The problem is, if I try to create the same config with /etc/config/wireless as follows, it doesn't work. 

config wifi-device 'radio0'
    option type 'mac80211'
    option path 'pci0000:00/0000:00:01.0/0000:01:00.0'
    option hwmode '11g'
    option country 'US'
    option channel '1'
    option htmode 'HT20'

config wifi-iface
    option device 'radio0'
    option mode 'ap'
    option ssid 'mac80211 test'
    option encryption 'none'

When I scan from another machine, no AP shows up. Also, on the wifi page in LuCI, it says "Wireless is disabled or not associated" (clicking the Enable button doesn't change this) and says that it's in client mode, despite it clearly being set to AP mode.

Also it works perfectly fine in client mode, I can scan, connect, and do DHCP successfully.

Edit: Upon further investigation, OpenWRT isn't even running hostapd, nor creating a config file for it in /var/run like it does on a working both with an ath9k card.
