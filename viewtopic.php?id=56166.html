download sanpshots from here:
http://downloads.openwrt.org/snapshots/
find your version.



1 update packages

opkg update
2 install luci web admin packages
opkg install luci    luci-mod-admin-full
3 install language, for example zh-cn
opkg install luci-i18n-base-zh-cn
4 make uhttpd start at boot

/etc/init.d/uhttpd start
/etc/init.d/uhttpd enable
5 install other packages (my setting for a nas purpose)

opkg install block-mount mount-utils mountd ntfs-3g-low ntfs-3g
opkg install kmod-fs-ext4 kmod-fs-vfat kmod-fs-ntfs kmod-fuse kmod-ide-core kmod-ide-generic fuse-utils
opkg install luci-app-hd-idle luci-app-minidlna luci-app-samba luci-app-shairplay luci-app-upnp 
opkg install blkid e2fsprogs dosfsck fuse-utils
opkg install luci-app-samba luci-app-shairplay luci-app-upnp luci-app-commands

if you have any opkg update link problem, please edit your /etc/config/network:


vi  /etc/config/network

my openwrt device located at a subdomain 192.168.3.*, my ip address is 192.168.3.11.
since it is not a primary router, so I have to set gateway and dns server, or your opkg update may fail. you can use

/etc/init.d/network reload

to make your new configuration into effect. ping www.google.com for testting.

parts of my network configuration:
...
config interface 'lan'
        option ifname 'eth0'
        option force_link '1'
        option type 'bridge'
        option proto 'static'

        option ipaddr '192.168.3.11'
        option netmask '255.255.255.0'
        option ip6assign '60'

        option gateway '192.168.3.1'
        option broadcast '192.168.3.255'
        option dns '8.8.8.8 8.8.4.4'
...

we can refer from:
1 luci http://wiki.openwrt.org/doc/howto/luci.essentials
2 network http://wiki.openwrt.org/doc/uci/network
