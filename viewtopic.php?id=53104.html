OpenWrt always defaults to disabled wireless, only way to enable by default is if you include your own /etc/config/wireless with enabled wireless in your image, or otherwise include scripts that enable it on boot.

You can't delete the drivers from a device with OpenWrt installed, because SquashFS is a read-only filesystem. If you delete a file there will only be a marker in the overlay indicating that this file is inaccessible. However, if the device is booted in failsafe or otherwise booted without mounting the overlay, the drivers will be there.

If you really want a device with no wireless drivers available, you will have to use the Imagebuilder or compile from source, and build an image without the kernel modules needed for your wireless chip. You will probably also need to modify your platform and/or target profile, because wireless drivers are included by default, as you can see in e.g. target/linux/ar71xx/Makefile (kmod-ath9k is used by many ar71xx devices).
