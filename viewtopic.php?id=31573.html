I determined whats wrong...
In LAN section dont touch anything!

Configure only WAN.
In that section you can do what you want.

Nano Station M2 have two rj45 ports primary and secondary.

With original firmware on primary there is poe, and in secondary often unusable.
So openwrt made that secondary port - WAN.


I think this is common users mistake

Post #10
LinkZ
5 Oct 2011, 18:42

As previously mentioned, the DHCP server settings on the LAN interface page defaults to disabled in LuCI.  I believe this is a bug.  So whenever you make a change to the LAN, you inadvertently turn off the DHCP server.  Just make sure the DHCP server is checked before saving changes.

To recover, you will need to set your PC to a static IP address, then connect to LuCI and enable the DHCP server.

I don't know if there is a bug report for this yet, I will check.
