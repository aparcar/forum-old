Hi Slammer and other

Slammer wrote:

I would like to see a Crisos and Openwrt merge!
What the reason to have separate development efford?

I'm the Crisos maintainer and I start this project months ago with the two main goals:
1. Fix the etrax on Openwrt
2. Provide a wide range of package as binary repository for cris arch

During my work I always release fix on openwrt trac system (see list on mywebsite) and the platform can be consider stable after the patch proposed on this very long thread of openwrt mailing list http://www.mail-archive.com/openwrt-dev … 01939.html
All diff between openwrt and crisos are published on http://download.tuxfamily.org/crisos/da … 8.09.patch (important diffs are target/linux/etrax/patches/100-compile_fixes.patch and package/iptables/patches/1.4.0/008-table_alignment_fix.patch) and are also been reported to cris openwrt mainteiner (john), so we should only wait and hope

I like also invite all openwrt developer to commit this patches because this really fix etrax, many users are using it now on several etrax platform.

Regards
