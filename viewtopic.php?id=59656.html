you can add users(to use with various servers).. but they can read config files.. you can deny them to use system commands by not giving them a shell environment (/bin/false); keep in mind that owrt is not design to restrict users.. as its based only on root user access
add them inside /etc/passwd file

regarding firewall config http://wiki.openwrt.org/doc/uci/firewall

- look inside /etc/passwd
- copy one of those users and rename it.. give him a home folder and set shell to /bin/false (its all in passwd file)
- you can assign a passwor for you user by using

passwd <user>

then you can use this user for login with servers.. but it cant access shell(/bin/false restriction)
