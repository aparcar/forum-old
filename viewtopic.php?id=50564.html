Most probably they used a Linux environment to edit the *.img file. Also i don't know (yet) how to edit those.
Or (as mentioned in R38063) you have to make a Linux environment to 'Make' your own *.img file. This is the regular way for OpenWRT.

nikki81 wrote:

It is in the supported devises list.

Furthermore there is no information on the openWRT Wiki site about running openWRT on this device, other then the mentioned R38061.

Model    Version    Status    Target(s)    Platform    CPU Speed (MHz)    Flash (MB)    RAM (MB)
WNR2200    v3       r38061    ar71xx       Atheros AR7241    360             8             64

You probably should build your own image, or try to flash your device by TTL serial connection (to 'break' the default boot process) and then tftp the new image to your device.
As i did with a ZyXEL (other base type) router.

Regards, DG.
