What is your basic setup? ISP gives a native ipv6 address through router advertising? dhcpv6? (I get a bit confused as you talk about an "6in4 address", but do not mention how the address is supposed to get assigned.)

And 6in4 and 6to4 are two separate things, so make sure that you are not mixing them.

Divide the problem to two steps: making sure that the router itself gets and uses ipv6 address, and providing ipv6 connectivity to the clients.

Does your router get full ipv6 functionality? Can you ping ipv6.google.com from router console?

root@OpenWrt:~# ping ipv6.google.com
PING ipv6.google.com (2a00:1450:4010:c00::63): 56 data bytes
64 bytes from 2a00:1450:4010:c00::63: seq=0 ttl=52 time=29.507 ms
64 bytes from 2a00:1450:4010:c00::63: seq=1 ttl=52 time=32.213 ms
64 bytes from 2a00:1450:4010:c00::63: seq=2 ttl=52 time=40.273 ms
^C
--- ipv6.google.com ping statistics ---
3 packets transmitted, 3 packets received, 0% packet loss
round-trip min/avg/max = 29.507/33.997/40.273 ms
root@OpenWrt:~# wget http://ipv6.google.com
Connecting to ipv6.google.com ([2a00:1450:4010:c00::63]:80)
index.html           100% |*******************************| 11293   0:00:00 ETA
root@OpenWrt:~#

If not, make sure that it is set to listen/accept router advertisements (or use dhcpv6, if that is your ISP's technology).

After you have got the router to work with full ipv6, then it is time to config the LAN side.

EDIT: you might also read this: https://forum.openwrt.org/viewtopic.php?id=27541
It is about 6in4 tunnels (to SixXS), but if you actually have 6in4, then it might be useful.
