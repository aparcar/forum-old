I'm wanting to set up a few routers as access points only, allowing other clients on the switch (including Wi-Fi clients) to be on the main network and receive their IP addresses from the main network's DHCP server. I also know that this would normally require setting an IP address statically on the AP.

Is there a good way of still allowing the AP to receive its IP address via DHCP whilst keeping all the conditions I stated before? I'm wanting to place the APs, leave them and forget about them, but if I need to re-structure the network, I'd rather do it from the main router rather than having to go to each and change the static IP.

radicalbiscuit wrote:

Is there a good way of still allowing the AP to receive its IP address via DHCP whilst keeping all the conditions I stated before?

Yes, with "alias".

config 'alias'
        option 'interface' 'lan'
        option 'proto' 'static'
        option 'ipaddr' '10.0.0.1'
        option 'netmask' '255.255.255.0'
