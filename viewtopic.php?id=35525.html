Hello,

I have a simple script i've written on my WNDR3700 to start or stop an OpenVPN VPN tunnel that looks like this:

#--> /etc/scripts/myscript.sh
cat /etc/scripts/config.on > /etc/config/openvpn
/etc/init.d/openvpn restart

I'm able to invoke this script without having to login to the router by using ssh keys and running this command:

ssh root@myrouter '/etc/scripts/myscript.sh'

But when it's run, the second command fails (to restart openvpn). I get this error message:

/etc/rc.common: eval: line 1: start-stop-daemon: not found
/etc/rc.common: eval: line 1: start-stop-daemon: not found
/etc/rc.common: eval: line 1: start-stop-daemon: not found
/etc/rc.common: eval: line 1: start-stop-daemon: not found
/etc/rc.common: eval: line 1: start-stop-daemon: not found

I found in another post somewhere that the start-stop daemon must be installed but it was a default busybox setting. I didn't disable this (at least not knowingly...).

Can anyone offer a suggestion on how to get the script running? It's very basic but I seem to be missing something.

Thank you for your time,
Mike
