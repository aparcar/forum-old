Did you try the method of copying the openwrt image to the /tmp directory of the dd-wrt firmware-ed router and then Telnet or SSH into the router command line interface to use the command "mtd -r write openwrt.bin linux" where openwrt.bin is the name of the firmware?   It takes a while for it to load.

If you need a way to copy and are a windows user, you should go into dd-wrt and enable SSH.   This will allow you to use the Windows app  WinSCP.  With it you can define a session to the IP of your router, using the SCP option (say YES to allow the certificate)   It opens up a Windows-like directory structure of your router where you can view file contents there or copy files back and forth between windows and  your router via Windows drag and drop.   With this method you would download the OpenWRT firmware to windows and use WinSCP to open the router directory /tmp and drag and drop the firmware to that folder (directory) on the router.   

From there you would use either Telnet (if not found, go to Control Panel, Programs and Features, turn Windows features on and off, enable Telnet) or via SSH (install Windows app Putty to define a SSH session) to get a command prompt accessed on the router and then you would enter:

cd /tmp
mtd -r write openwrt.bin linux

Noting that openwrt.bin is the name of the file you copied to /tmp

Your lights will start to flash on the router and eventually you should be able to get to Luci.   If nothing has happened, wait at least 20 minutes before you begin working with the router if you don't want to brick it.

Note: The web interface of dd-wrt is where you turn on and off the ability to use Windows apps like PuTTY for SSH or Windows telnet/PuTTY telnet to access a router command line interface.

Warning:  If you edit files that will be placed on a linux system like your router, be sure to use an editor like Windows free app ConText or the native editor that comes with WinSCP.   Never use Windows Notepad to edit a file and will be copied to a Linux system.

******* UPDATE: The method is device specific; this method I have used is suggested for some Buffalo models.   Use at your own risk.
