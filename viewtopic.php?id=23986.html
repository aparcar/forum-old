It seems to me then that your unit does not have access to the internet. If you have access to the web interface, change the 192.168.1.1 port to a DHCP setting and connect it to your router that is on the Internet. It should obtain the correct Gateway settings and an IP address from the existing router/modem. Once it's done that it will have access to the net and be able to 'opkg update' and install the packages you require.

Wirelessuser,

Not sure if you are still looking for an answer, the reason you are not able to ping outsides is because the default gateway in your lan config isn't set.

config interface lan
        option ifname   eth0
        option type     bridge
        option proto    static
        option ipaddr   192.168.1.1
        option netmask  255.255.255.0
        option gateway '???' <-----  add this line with the correct gateways address and restart your network (/etc/init.d/network reload)
