Issue is resolved. I enabled VXLAN support into the kernel template of my build environment and successfully built an image having ip-full package integrated in it, but now with vxlan-enabled kernel.

Now the vxlan virtual interfaces can be manipulated through the iproute2 CLI.

I think that the TT can also be closed.
