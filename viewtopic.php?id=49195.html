Problem is solved. Need to link the start script in /etc/rc.d.


Hi,

Please help with this question: How can I start the "dynamic_dns_updater.sh" automaically after a reboot?

I can start is manually as described in: http://wiki.openwrt.org/doc/howto/ddns.client after which it runs.
    sh
    . /usr/lib/ddns/dynamic_dns_functions.sh # note the leading period
    start_daemon_for_all_ddns_sections "wan"
    exit

But after a reboot it is not running anymore.

I read that it starts after an interface up trigger but I have "URL" configured via LuCI. So there is no interface "linked" to it, I guess.

The configuration file is:
config 'service' 'myddns'
        option 'force_unit' 'hours'
        option 'enabled' '1'
        option 'service_name' 'no-ip.com'
        option 'force_interval' '24'
        option 'domain' 'HOSTNAME.no-ip.org'
        option 'ip_source' 'web'
        option 'ip_url' 'http://checkip.dyndns.org/'
        option 'username' 'USERNAME'
        option 'password' 'PASSWORD'
        option 'check_interval' '15'
        option 'check_unit' 'seconds'

As I said, it runs happily after a manual start but I need it to run automatically after a reboot.

I have already created a file: /etc/init.d/ddns
With content (I found this in https://dev.openwrt.org/browser/package â€¦ -scripts):
    #!/bin/sh /etc/rc.common
    START=95
   
    start() {
            . /usr/lib/ddns/dynamic_dns_functions.sh
            start_daemon_for_all_ddns_sections
    }
   
    stop() {
            killall -9 dynamic_dns_updater.sh
    }
... and did a reboot. It did not help.

I have a WRT54G v2.2 with kamikaze/8.09.2/brcm-2.4.

Can anyone help me with this please.
Thanks in advance,
Johnny.
