Hi all

I am a long-time OpenWRT user and would like to bring up the subject of security of this distribution. It seems somewhat neglected.

First off, I don't mean to be offensive. I use and like OpenWRT, but security seems to be an area that isn't given much attention. I wonder about this subject and would like comments from others.

So, here are some questions and comments I have on this subject...

Does OpenWRT not have enough manpower to do security announcements/updates?

There is a security-announce mailing list here, but it is never been used:
https://lists.openwrt.org/cgi-bin/mailm â€¦ y-announce

It seems like there are no security announcements for OpenWRT. I don't think there is any mechanism for users to know when a stable-distro package gets updated for security reasons. Is that true?

How often are packages in the stable releases updated for security reasons?

Personally, I stopped using the stable releases of OpenWRT some years ago because there was a period of years with no stable releases and some of my hardware platforms were unsupported in stable. As a result, I started building my own from sources and I still do that today, dealing with all the occasional bugs and brokenness which that brings.

I use StrongSWAN in some of my deployments, and there was a security advisory for racoon and similar IPSec/VPN packages recently. I am going to have to update my builds here soon for that. I got a security advisory for every other distro I use, but I've never seen any kind of security notification from OpenWRT. That got me thinking.

If there are any security professionals out there, do you have any comments on OpenWRT in general?

OpenWrt seems to me to be very on top of security issues in packages, as updates are quickly committed to source. Git repos have RSS feeds, you could subscribe to those (or use some RSS-to-email gateway). It doesn't differ between security and other fixes, but the stable branches aren't noisy in that regard.

Does OpenWRT not have enough manpower to do security announcements/updates?

OpenWrt does not have enough manpower to do regular release schedules.
While security updates sort of work for the important basic packages other aspects
like announcements, tracking, stable versions etc. require more manpower and management
"overhead" that is not available at this point in time.

sdwalker has some CVE tracking that package maintainers can use
(see https://sdwalker.github.io/uscan/index.html )

PS: Feel free to participate in the wiki/mailing list by reviewing and improving security & procedures
