I found that when I connected to the guest network, the 4g connection would drop almost instantly. I'm running mwan3 for failover, the regular network is allowed access to the 4g through firewall zone config but the guest traffic is not. My best guess is that some internal packets from the guest network got forwarded to the 4g connection and Verizon dropped the connection just like the link said.

The resolution was to add the following to the custom firewall rules... The connection has been up over 2 hours now, and I was lucky to get 5 min before..

iptables -I FORWARD -i br-guest -o 3g-wan2 -j DROP
