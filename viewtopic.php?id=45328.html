katananja wrote:
iptables -P FORWARD DROP

You don't really need to do that manually. You can make that default via Luci or UCI like this:

config zone
    option name 'lan'
    option network 'lan'
    option input 'ACCEPT'
    option output 'ACCEPT'
    option forward 'DROP'


katananja wrote:
iptables -N web_access
iptables -A FORWARD -i br-lan -o pppoe-wan -p tcp --dport 80 -m comment --comment "WEB Access" -j web_access
iptables -A FORWARD -i br-lan -o pppoe-wan -p tcp --dport 443 -m comment --comment "Secure WEB Access" -j web_access
iptables -A web_access -s 192.168.1.0/24 -j ACCEPT

You also don't really need to create an additional chain just to insert lan to wan forward rules. The default OpenWRT firewall has already created chains for custom user rules. You can modify the example above and add them into "forward_lan_rule" chain with a single line:

iptables -A forward_lan_rule -p tcp -s 192.168.1.0/24 -m multiport --dports 80,443 -j ACCEPT
