After figuring out what I'd broken trying to get any kind of IPSec VPN up and running, I've encountered a new problem - IPSec seems to authenticate nicely, but I don't get any data going through. If I'm on the router itself, I can ping the client device, so there is some kind of connectivity, suggesting a routing issue. Sadly, I haven't been able to resolve it yet.

The configuration is as such:
The WAN side is a /29 subnet, with the router at x.y.z.70. Connectivity to the outside is provided over PPPoE.
StrongSwan is bound to x.y.z.68, and an alias "ppoe-wan:0" is created with this IP address - otherwise this IP isn't available for StrongSwan. Firewall rules are in place to permit connectivity for IKE/ESP to this address.

StrongSwan configuration looks like:

config setup

conn %default
 keyexchange=ikev2
 auto=add

conn roadwarrior
 left=vpn.changed.me
 leftauth=pubkey
 leftcert=gatewayCert.pem
 leftid=vpn.changed.me
 leftsubnet=0.0.0.0/0
 leftfirewall=yes
 right=%any
 rightsourceip=192.168.249.0/24
 rightauth=pubkey
 rightcert=certname.pem
 esp=aes-aes256-sha-modp1024,aes256-sha512-modp4096
 ike=aes-aes256-sha-modp1024,aes256-sha512-modp4096

I want to be able to set up OpenWRT so an external device can VPN and have access to both internal network resources and the Internet at the same time. This is due to the other end of the link being on potentially hostile networks.

What routing magic am I missing with this?
