Following setup with two routers, first is a Fritz!Box-Derivative, second is a OpenWrt-powered Router:

Everything works.

Now when I disable NATing on the OpenWrt-Router, the hosts behind it, cannot connect to the FritzBox nor to the Internet. The OpenWrt-Box can connect to the FritzBox and to the Internet. This is due to the NATing done in the FritzBox-Derivative.

Does anybody have an idea, howto solve this issue?

As described here: http://freetz.org/wiki/packages/iptables.en NAT is done by dsld and test with tcpdump done on the OpenWrt-Box, suggest, the FritzBox refuses to NAT unknown IP-addresses. I.e. even if 192.168.2.1 sends packets to openwrt.org, the response-packets are NATed to 192.168.1.50 and not to 192.168.2.1.

Out of curiosity: in case Router-No1 was an OpenWrt-device as well, following configuration should work, right?:
iptables -t nat -A POSTROUTING -o $dsl-interface -j MASQUERADE

The fritzbox needs static routes to 192.168.2.0/255.255.255.0 .. 192.168.5.0/255.255.255.0 each via gateway 192.168.1.50

Also consider setting the wan input- and forward-policies of OpenWrt to accept.
