BusyBox 1.4.2 -> doesnt use the Nvram command to make configurations  it uses uci source = http://kamikaze.openwrt.org/7.06/release.txt

after i found the commands list i managed to fix the subnet and the Wan port
i configured the Wan port via DHCP easly with  revert to DHCP:
uci set network.wan.proto=dhcp
uci commit network
ifup wan

From then on i could connect to the internet....Finally !!!
a very usefull book with some commands i needed 
( https://we.riseup.net/assets/48812/Link … acking.pdf )

From then on i had to Update the firmware in order to get the web GUI, coz the original intention was to get the device back to original firmware, that guide didnt work for me : http://www.makeuseof.com/tag/unbrick-tr … ys-router/

Knowing what my device is and what firmware i got i was looking for updates...i found some usefull info here : http://wiki.openwrt.org/toh/linksys/wrt … ng.openwrt

i used those commands to get the update and it worked!
cd /tmp/
wget http://downloads.openwrt.org/backfire/1 … uashfs.trx # The file must be the trx file.
mtd write /tmp/openwrt-brcm47xx-squashfs.trx linux && reboot


After the reboot in my putty telnet console after auth i saw BusyBox v1.15.3, Backfire (10.03.1, r29592)
OMG from 1.4.2 > 1.15.3 :@
moment of truth typed in the browser 192.168.1.1 and there it was OpenWrt web GUI Success!!!
After a fast look around i i realized that that firmware is 1000 times better than the original and way more fun

Any way that post is for any one out with the same problem like mine, u never know
