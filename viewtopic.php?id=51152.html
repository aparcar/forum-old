From my own notes:

Question: What's the clean way of adding custom configuration (running custom scripts) on first boot.

Answer: /etc/uci-defaults/foo.sh, a plain shell-script which can contain arbitrary commands. It is deleted by the system after it has been invoked once and if it exited with code 0. In case the script did not exit with code 0, it is not deleted and will be re-executed during the next boot until it successfully exits with exit code 0.

The script will be called in /etc/init.d/boot, after the rootfs has been mounted, but before modules and hotplug are loaded.
