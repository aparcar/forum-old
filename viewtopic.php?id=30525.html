I'm not sure what you mean with your question. Firmware or RTOS?
Openwrt uses the linux kernel and the linux kernel is not optimized for realtime operations. There are some ways to optimize it for (at least soft) realtime operations, but I think openwrt is optimized for performance. So, openwrt is not a RTOS.
And if openwrt is a fimware?? For my unterstanding/definition of firmware, I would say it depends on where the systemfiles are stored. If you have a device where you flash openwrt into the internal flash, you could say openwrt is a firmware. If you run it from the harddrive of an i386-pc-hardware I would say it's an operatingsystem.  ... but in my opinion it doesn't really matter how you call it, just enjoy that you can install it almost everywhere

Mostly the definition of firmware is "software stored on hardware." such as on a ROM chip. I'd consider OpenWRT a firmware replacement for most devices, sure. Since if you're having issues with your Linksys you'd download a "Firmware upgrade" from the manufacturer. thats true and thats was my point. it doesn`t need to be embedded to be a RTOS, and it doesn`t need to be a RTOS to be embedded or vice versa
I was irritated by the question if openwrt is a real time operating system OR a firmware. this question makes no sense or you have another understanding of whats  a firmware and whats a RTOS.
I thought a RTOS is a OS with special scheduling so that it can garantee a specific response time for specific operations.
And a firmware is the programmcode of some sort of embedded hardware (normally stored in flash, or (e)eprom).

with this in mind, I would say openwrt is a linux distribution optimized for performance and not for realtime operation, so it is not a RTOS.
and if it is considered a firmware depends on your hardware. If it is an embedded device you could say openwrt is a firmware, and if it is a pc you will say it is an OS. But that are just names, and this defininitions were more destinctiv 20 years ago, when you were not able to run a full featured operation system on a  so called "embedded device".
And especially with openwrt it is hard to draw this line. Most users consider their wifi-router as some embedded device on which some "firmware" is running, and they have limited control over the hardware.  If you put openwrt on it you get a full featured operating system and you have full control over your hardware, but it is still an embedded device and the programmcode running on it can still be called its firmware. so in this case you can say openwrt is a firmware. but where is the point?

"Optimized for performance" = written to be fast as possible. But slowdowns can happen.
RTOS = written to react at  events in very strict time constraints. If slowdowns happen = catastrophe.

Well, this OpenWrt *fork* is an RTOS:   https://github.com/bifferos/openwrt

The above incorporates kernel patches for RTAI, lxrt kernel modules, and a demo application to show how you might use it.  Perhaps one day someone will incorporate similar into OpenWrt, but for the moment it is most definitely *not* an RTOS.
