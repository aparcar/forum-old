nuentes wrote:

So I bought my USB to serial adapter.

To get the obvious out of the way, make sure you are using the proper kind of an adapter. The nomenclature can be a bit misleading, since the word "serial" is often mentally associated with the concept of COM-ports found on x86-compliant motherboards. Unlike the UART port that is found in microcontrollers, a COM port commonly uses the RS-232 protocol, which is incompatible with the TTL (or CMOS) UART port on the microcontroller.

What you need is a USB-to-TTL-UART (or CMOS-UART) adapter. Your run-the-mill "USB to serial" adapter will fry the microcontroller, if you try to send data through it.

Make sure you have flow control set to "none". 

To test adapter, do not connect to the router, connect a jumper from TX to RX on the adapter.  When you open the terminal and type in some text, you should see what you type.  When you break the connection, you should no longer see what you type.  This shows the adapter and PC software are working.

Next connect ground TX and RX to the router (never connect the red +5 wire, leave it open) and boot it up, you should see boot log text.  If nothing happens in the terminal, unplug the power to the router and switch the RX and TX pins and try again.
