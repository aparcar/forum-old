Post #29

So it looks like the default make targets do not include hostapd, without which the wireless interface can't be set in to master mode. The card was running fine in client mode. Doh!

For future reference: If you want your wireless to run as an ap it helps to have hostapd running.

The latest 8.0.6 stable Kamikaze image: openwrt-rdc-squashfs-ar525w-web.img seems to work very well. 

I have been running it for a few hours and was very pleased with the smooth installation process and the simple luci interface. 

I just loaded the image using the firmware loader available in the Airlink web gui, it rebooted as designed. 
Openwrt did not enable the wireless adapter by default, so i plugged my laptop using an ethernet cable, got a 192.168.1.x ip address
logged in, and started playing around.   

Getting WPA seemed a little harder than it seemed at first since I was getting an error about a required package,  but playing around in the shell I did an opkg update and an opkg upgrade which gave me no feedback,  but when I went back the WPA option were available. 

Im very excited as the airlink firmware had some issues i'm hoping will be resolved, eventually I intend on using the unit as an AP client so I can bridge some other devices. 
I do want to update to x-wrt and perhaps play around with some of the VPN modules/packages and perhaps some more advanced firewalling features to replace my aging sonicwall unit.
