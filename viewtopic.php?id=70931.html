Hi there, I have WRT1900AC router with firmware OpenWrt Chaos Calmer 15.05.1

I started using this router yesterday (due to suspecting my previous router of dropping connection) and everything worked, even connecting via VNC viewer to another PC on LAN (the PC has broken display but otherwise works).

Today when booted up the router and the PCs, I found the VNC viewer will not connect to the VNC server on the LAN PC (with that broken display), after a while it pops up a message "Timed out waiting for a response from the host computer".
I can connect in via TeamViewer just fine and check on the VNC server on the PC but it is all fine there.

I suppose it must be the router, that is OpenWRT firewall which is blocking VNC server from responding to VNC viewer request for connection?

I tried making a firewall rule:

VNC Server
Any traffic
From any host in lan
To any host in lan 

But that didn't help. What bugs me is why would it work on the first day and then stop.

Maybe related thing, when I booted the router the first day and made it to pickup IP from my ISP, I found not long after that DHCP Leases were showed as 'expired' (Status-Overview), I checked and the lease time was set some short time, I think maybe 12minutes, I changed that to 24h and after some reboots it caught on and everything was OK, till today when I can't connect via VNC to that PC on LAN

LAN to LAN traffic goes through the internal switch in the device, it does not even reach the CPU, and it is very unlikely that OpenWRT could be interfering with it.
