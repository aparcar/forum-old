I am attempting to clone a router using "Generate archive" on the "Backup / Flash Firmware" page of the first router and then using "Restore backup" with the archive on a second router.

This does NOT create an exact copy however.  On the "Wifi" page of the second router there is a second entry "Generic 802.11 Wireless Controller (radio1)" in addition to the "Atheros AR9285 802.11bgn (radio0) entry configured on the first router.  I have attempted to delete it from the /etc/config/wireless file, but it returns following reboot.

Questions:

Is there a better way to clone an exact copy of a configured router?

Failing that, is there a way to eliminate the second wireless entry?

2 May 2013, 19:31

The wireless config usually embeds the device specific mac address to identify the radio, so if you upload this config to another model, OpenWrt will not find the radio and create a new default section for it. If you want to share the wifi config across multiple devices you have to replace the "option macaddr ..." lines with "option phy phyX" where "phy0" is the 1st radio, "phy1" the 2nd and so on.
