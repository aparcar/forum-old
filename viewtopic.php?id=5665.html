Hi Folks,

We are trying to breach (modification ) the existing synchronization system of Wireless Access
Point (AP) to make AP synchronize (time synchronization) with TDP router for our research purpose  (http://ip-flow.dit.unitn.it/projectwireless.html   ).

Working with OpenWrt by developing new package.

We believe that, there should be some modification in beacon generation process
and also timestamp, beacon interval field in beacon frame. We may need to change beacon interval field daynamically. Precisely this is like AP will send beacon frame not in a static interval all the time. It can change the interval but before sending in new interval, it should put the new value of interval in the last beacon frame.

The key question here: How power saving mechanism will be affected, if beacon interval is changed dynamically?
If a STA is in “Doze”,  power saving mode,  Will it be out of synchronization ?
