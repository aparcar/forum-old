Hi,

the problem with the u-boot stuff is, you can't flash the u-boot partition with a stock openwrt, because of security reasons, the u-boot partition is write protected and you cannot update it.

The original NEXX firmware wasn't on the security alley (at all...) and there you could easily flash a new U-Boot.

But there is the problem: The u-boot mentioned in the wiki is.... lets say : not good....
After flashing this one I could only run exactly THAT openwrt version which came with the archive of the new U-Boot.
Any onther openwrt downloaded, or compiled directly would not start at all.

Luckily, the u-boot web-frontend worked flawlessly and you could reflash another firmware and another u-boot.

Look in the forum, there is some u-boot-hack (dirty Hack, to be precise, not only violating any terms of the gpl, no its webinterface is also in chiniese or some other language I don't even recognize the characters....) But at least; This "U-Boot breed" seems to work - it boots ChaosCalmer RC2 right out of the box.

Hope the guy which named himself "hackpascal" or so will release the sourcecodes, so that some normal characters can be used and the GPL violation ends....

Cheers,

att15
