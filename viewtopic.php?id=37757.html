If BOARD is set in the command line (Which I'd imagine would come from using a specific-to-the-board build), then you can access it with the $BOARD environment variable (echo $BOARD)

Otherwise, /proc/cpuinfo is probably the best place to get it from, if it can be detected. (cat /proc/cpuinfo | grep Hardware), or even dmesg (dmesg | grep Machine)

Barring that, perhaps an identifier/board id or data stored in an EEPROM somewhere?
