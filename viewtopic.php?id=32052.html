Hello,

I've just started dabbling with collectd / rrdtool in LuCI and have hit a roadblock when trying to customise it.

I've got the 'ping' plugin enabled and am able to see rrdtool graphs working in LuCI. However, I note that the ping page only shows round trip times and does not show failure rates. The RRD for failures rates is present and is being populated (/tmp/OpenWrt/ping/ping_droprate-*.rrd), but it isn't graphed.

Based upon the other LuCI graphing modules I assumed the following would do the job in /usr/lib/lua/luci/statistics/rrdtool/definitions/ping/ping_droprate.lua:

module("luci.statistics.rrdtool.definitions.ping.ping_droprate", package.seeall)

function rrdargs( graph, plugin, plugin_instance, dtype )

        return {
                data = {
                        sources = {
                                ping_droprate = { "value" }
                        }
                }
        }
end

After putting this in place I simply get the ping round trip times graph *twice*. If I look at the rrdtool command generated I can see that it is calling the same "rrdtool graph" command twice.

If I then remove the original ping.lua then I get the failure rate graph alone. So, I suspect the problem lays somewhere with LuCI matching "ping" as a substring of "ping_droprate" and subsequently short-cutting the reading of the droprate RRD.

Anyone have any suggestions?

Thanks

Sam
