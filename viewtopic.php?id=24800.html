OK, I did some (more) searching and found this helpful post - I'm going to be able to sort this out tonight, let you know the results later.

sithsasquatch wrote:

I was able to tftp this router model using ubuntu and ran into a similar situation.

What finally made it work was to add an arp entry to map the router's MAC to the IP.

You can do this before plugging in the router and running the tftp.

//disable the automatic network config
sudo service networking stop
//set up your own ip
sudo ifconfig eth0 192.168.11.2
sudo ifconfig eth0 netmask 255.255.255.0
//add the arp so your machine can find the router immediately
sudo arp -s 192.168.11.1 {router mac}

The router's MAC is listed on the label of the router as the SSID, just separate it into clumps of two like so:

SSID: 123456789ABC
MAC: 12:34:56:78:9A:BC

after that, run the TFTP commands and boot the router.

Once you are done, you can reset your network config with:
sudo service networking start
