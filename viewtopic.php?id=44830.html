Hi,

I try to use on my openWRT box the DHCP/DNS server from my clearos server, because I need the DNS capabilities of this server (there are DNS configs for my webserver etc.). Actually, this works fine, if I mask an external server (like a google server) with another name. But the local mask do not work - e.g. my webserver should be available with the name wpad -> that is needed, so the browser can auto-detect the proxy.

But the webserver and the wifi are on a different subnet, because openWRT lan uses its own dhcp server (webserver/clearos: 192.168.0.0/24, openWRT-LAN: 192.168.1.0/24)  When I disable the dhcp function on the lan, I have no dhcp at all on the openWRT box.

Is it possible to configure an external dhcp server and where can I do that?

Ok, I solved it.

I found a tutorial for what I was trying to accomplish: http://wiki.openwrt.org/doc/recipes/bridgedap

Because I have to wifi routers (that are both in switch mode) I also activated the Spanning Tree Protocol. The main problem was an IP filter I setted up on another switch.

Now, both routers are served by my DHCP server from the clearOS box, with the DNS server right up.
