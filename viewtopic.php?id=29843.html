If you find LuCI hard to use you may give Gargoyle a shot, it's based on OpenWrt but aims to provide an interface similar to that of commercial routers. Visit the Gargoyle site, select AR71XX and scroll until you find the proper WR1043ND image. You can flash the firmware using LuCI (choose the one with -squashfs-sysupgrade in it)

To edit files you would have to login using ssh, to do it using WinSCP you'd have to install the openssh-ftp-server package (this can be done from LuCI as well). If you don't have a basic familiarity with Linux I wouldn't recommend doing that as you can easily break things and then you might end up in over your head.

You're welcome. Transmission is not so hard to install, but you most likely will find necessary to add virtual memory in the form of a swap partition on an USB device (a hard disk preferably, but a pendrive will work if you don't care too much about shortening its life)

You need to do that if you notice the router rebooting. I'm not sure this is just a consequence of the limited amount of memory and not a kernel bug instead, but this is how things are today.

Hmm, I successfully operated transmission on my WR1043ND. Worked fine for me for about 3 months. 24/7.

The only problem I saw and still see, is the disk usage. You have merely 32MiB of RAM and can use a certain amount of it for disk buffer. So the disk is constantly being written to or read from. My harddisc is still ok (old Samsung 160GiB with 7200rpm I put into a USB-Disk-Thingy) but I didn't want to break it and also, it consumes quite some current. The harddisc gets very warm as well.

As long as electrical current isn't an issue, a lot of things are possible and fun.

If you try to work aroung this issue, you option become limited very fast. Whatch the movie Apolo 13 for inspiration.
