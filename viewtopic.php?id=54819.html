I have a PPTP connection and I'm source routing a specific IP address through it.

I need to execute the following commands every time the device reboots:
         ip rule add from 192.168.1.10 table tunnel
         ip route add default via 182.168.68.31 dev pptp-tunnel table tunnel
         ip route flush cache

I added those 3 lines to rc.local (before exit 0) and it doesnt seems to work.

Any ideas?

rc.local:

         # the system init finished. By default this file does nothing.

Try putting sleep 15 before your ip commands, and see if it helps. If it works, you can try to lower the number to reduce the wait.

Even though rc.local is the last startup script to run, it may be started before all other started tasks are completed. The tunnel may not be ready by the time rc.local is run.
