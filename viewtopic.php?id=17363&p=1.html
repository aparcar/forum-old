Wipster

I have an ar7 I get my net up using Network ---> ppp ---> ppp-mod-pppoa and Kernel modules ---> Network Devices ---> annex A when compiling, then this in config/network

config interface wan
        option ifname   ppp0
        option unit     0
        option proto    pppoa
        option encaps   vc
        option vpi      0
        option vci      38
        option username (your username here)
        option password (your password here)

And then edit /etc/init.d/network in the boot() section replace /sbin/wifi up with start and I'm away.

KillaB

symbiansucks wrote:

What I want to do is to use the builtin modem in the netgear DSL router. Is it possible to use the builtin modem?

Yes, of course.  Do you have the modulation variable set in ADAM2?
What Annex is included in the image, and is it correct for your location in the world?

If your modulation mode is set correctly and you're using the proper VCI/VPI settings for your ISP, you should end up with the word SHOWTIME using the following command:

# cat /proc/avalanche/avsar_modem_training

Once your modem is trained, DSL stats can be found using the following command:

# cat /proc/avalanche/avsar_modem_stats

symbiansucks wrote:

I didnot set any modulation variable in ADAM2. How can I do it?

There may be another way to do it, but I like to hard code it in ADAM2 the same as setting the my_ipaddress variable.
From the ADAM2 FTP prompt, use:

quote "SETENV modulation,GDMT"

Other common modes:
MMODE
T1413
GDMT
GLITE
AD2MOD (ADSL2)
A2PMOD (ADSL2+, used by my ISP)

Not sure how to help beyond this point.  Sure you can run pppd manually, but I wouldn't bother.

The only things I can stress are proper VPI/VCI settings and that all other options in "/etc/config/network" are correct.  You may also want to try changing the payload to routed.

Working PPPoE config on my fathers DG834:

config atm-bridge
    option unit     0
    option encaps   llc
    option vpi      0
    option vci      35
    option payload  bridged # some ISPs need this set to 'routed'

config interface wan
    option ifname   nas0
    option proto    pppoe
    option username "my_username"
    option password "my_password"
