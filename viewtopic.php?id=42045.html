I'm no longer selling Bifferboards, and there isn't much activity on the Bifferboard group so I thought I should post here as well.

There are a few developments that may be of interest to owners of other RDC-based devices, possibly Sitecom routers based on RDC3210 or NAS dongles.

1) I've released a version of the Python JTAG software for the Bifferboard.  This was reverse-engineered from the RDC Loader tool, and allows you to flash RDC devices, and to send x86 instructions to the CPU and execute them.  It allows you to dump flash as well.  You need a parallel port JTAG cable to use it.  See: https://github.com/bifferos/bb/tree/master/tools/jtag

2) The Bootloader from the Bifferboard (Biffboot) is still closed source but now available under a fairly permissive license, which includes provision for commercial use.  The latest version runs under Qemu as well as all versions of the Bifferboard - 1, 4 and 8MB.  I can look at porting it to other RDC devices if there is sufficient interest.  It should be fairly easy.  The bootloader can be found here: https://sites.google.com/site/bifferboa … y-editions

3) There is now a debrick utility for Bifferboards, which allows you to recover a bricked board without a parallel port JTAG cable.  Unfortunately you will need a second Bifferboard to use it, instructions are here:  https://sites.google.com/site/bifferboa … er/debrick  There is also work on an Arduino-based version of the JTAG software, but it's only partially working: http://groups.google.com/group/bifferbo … bf53a4a50.
