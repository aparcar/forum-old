Tallgirl,

The wiki page for the Alix provides a lot of information and talks about how to flash the device. I know it's still up to date because myself and another user rewrote it only last month! See http://wiki.openwrt.org/toh/pcengines/a … ilt.images

In short: You'll just want to download one of the images from http://downloads.openwrt.org/backfire/1 … 6_generic/ (the ext2 one is fine), flash it to the CF card as per the instructions, and then install it in the device. Again, as the wiki page says, you will need to install some additional packages to use USB or WiFi. There are too many different models of Alix to provide a universal pre-built package that covers all of them.

If you find something on your travels that is missing from the wiki page then please do go ahead and add it.

Usually one just has to "dd" the combined image onto the CF card.

wget http://downloads.openwrt.org/backfire/10.03.1-rc5/x86_generic/openwrt-x86-generic-combined-ext2.img.gz
gunzip openwrt-x86-generic-combined-ext2.img.gz
dd if=openwrt-x86-generic-combined-ext2.img of=/dev/cfcard

Afterwards you can use the usual utilities like fdisk, tune2fs etc. to resize the system partition (its only around 40MB by default) and maybe upgrade the filesystem.
Sometimes you need to change the root= parameter in the grub settings if the booting kernel does not find the root device. You'll find the menu.list within the first partition.

Post #7
samcrawford
25 Oct 2011, 18:47

You'll want to select the "x86" platform, and then the "Alix" profile (it may be named Alix2, that'll be fine, they're all basically the same hardware).

That Compex card you have is supported by the ath5k or madwifi drivers (see http://www.wikidevi.com/wiki/Compex_WLM54GP23), so you'll need to find one of them in the make menuconfig menus and enable it (under Kernel Modules > Wireless Drivers if I remember rightly).

If your Alix board also has USB (and I think yours does), then you'll want to enable the USB kernel modules (kmod-usb-*).

Hope this helps,

Sam
