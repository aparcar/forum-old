I was trying to install openwrt on my TP-link WDR4300 router.

Steps:
1. flashed squashfs-factory.bin image and restarted router
2. used putty to access router at 192.168.1.1 (telnet) and created password
3. used putty to access router at 192.168.1.1 (ssh) with root and new password - successful
4. tried to download packages using opkg update - unsuccessful
5. unable to download packages with error returned as (bad address, wget =1) - no internet connectivity, I figured
6. realized router and modem ip address is clashing
7. changed modem ip to 192.168.1.2 and set dhcp on modem with address allocation starting from 192.168.1.3
8. still cannot download the packages (router still accessible via 192.168.1.1)
9. found this : https://forum.openwrt.org/viewtopic.php?id=38122 and followed (most of) the instructions to change ip of router, i.e.:
uci set network.lan.proto=dhcp
uci commit network

uci set network.fallback=interface
uci set network.fallback.ifname=eth0
uci set network.fallback.proto=static
uci set network.fallback.ipaddr=192.168.1.1
uci set network.fallback.netmask=255.255.255.0
uci commit network

10. restarted the router and modem
11. from modem, new router ip is 192.168.1.4
12. problem starts: no longer able to ssh into router on 192.168.1.4 (connection timed out), also unable to ping router on 192.168.1.4 (request timed out)

I realized the instructions above for changing the router ip was for WR703N and had thought it was reasonable enough that it should work for my WDR4300 too. Apparently it doesn't or I have made a mistake somewhere.

Can anyone out there help me out?

Cheers
