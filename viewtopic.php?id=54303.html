Flashing a tplink firmware with boot via openwrt= bricked for sure.
Try tftp recovery method (press and hold reset button and turn on the routet.) But first, set your PC IP to 192.168.0.66 wih tftp server running.

This link might help: https://polkaspots.zendesk.com/hc/en-us … FTP-method

P.S. you'll have to rename tplink file according to your router. Try packet siniffer tool such as wireshack to see what file name it looks for.

If it doesn't work then youll have to go with usual serial debrick method.

Post #7
makro
30 Nov 2014, 19:32

The important detail is the environment you are flashing from. When flashing a stock firmware from OpenWrt, you need to cut out the bootloader. When flashing a stock firmware from a stock firmware, you can flash the file as is (unless TP-Link tells you something different, which in that case is their responsibility). So if you want to update the bootloader, first return from OpenWrt to stock with a cut firmware, then flash the firmware with bootloader from the stock interface.

OpenWrt generally does not recommend writing to the bootloader, this is why you can't (easily) flash a bootloader update. What exactly happens when you flash a boot-firmware with mtd I don't know, but it probably writes the bootloader part of the firmware file to the firmware partition, and thus pushes the kernel and data out of position, and on next boot the bootloader can't find a kernel to load.
