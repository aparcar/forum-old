Hello all,

I am hoping someone here can help me with an issue I have been trying to solve, but been unable to.

I have a Pi B+ installed with the latest RC3 version of OpenWRT.  I have a Trendnet TU2-ET100 USB to Ethernet Adapter (ASIX chipset) which is plugged into one of the USB ports, and is happily detected and running.  I have configured the USB-eth adaptor as eth1 (WAN), and the built in Ethernet as eth0 (LAN).  In this configuration, everything works fine.  Config is as per below:

[MODEM] <-cable-> [eth1 | Pi_OpenWRT | eth0 ] <-cable -> [LAN network switch]

The issue I am trying to solve is that of network bridging.  When I bridge the networks and reboot the Pi the WebUI can no longer be accessed, nor can I SSH to the unit – it effectively disappears from the network.  If I connect to the Pi directly with HDMI + keyboard, I can see the Pi dmesg log stops at:
“port 1(eth0) entering forwarding state”

If I manually remove the bridge info from /etc/config/network and  reboot the Pi, then I can access everything again without issue. 

Any help or guidance would be much appreciated.

Thanks,
Ross

OK - so I finally got this sorted. 

After MUCH playing around and pulling my hair out, I thought I'd try the latest Barrier Breaker stable release as opposed to the Chaos Calmer RC3.

Bridging worked as intended immediately with the Barrier Breaker img.  Chaos Calmer RC3 bridging must still be a bit buggy...
