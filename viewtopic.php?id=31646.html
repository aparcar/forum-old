I am running Backfire on a D-Link DIR-615 rev E3.  I am trying to use OpenWrt as a client to an Access Point (AP) that is sometimes near me but not always.  I also want the router to be an AP for a wired network it shares with others in the area.

Everything works fine when the client mode is connected, but when the client disconnects, the router stops sharing its AP.  Is this related to powersave (ie.  the router thinks that since the client is down that the AP should be down)?

I noticed that power save was enabled on the AP's interface:

# iw wlan0 get power_save
Power save: on

But I could not turn it off:
# iw wlan0 set power_save off
command failed: Operation not supported (-122)

I am able to alter the power_save mode of the client, but it doesn't fix the problem or alter the reading wlan0's power_save value.

So, two questions:
1.  Is this really a power save issue?  If not, please give me some ideas what is wrong.
2.  Is there some way to disable power save?
