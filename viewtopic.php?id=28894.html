which topology should your VPN have?

the standard configuration would be to have one openvpn-server and many clients which would connect to it. But that's a star-topology. so all traffic would be routed through his node. This can be usefull if you create the mesh for internet-connectivity with one uplink, than you could run the openvpnserver there.

If the topology of the VPN should be a mesh too, openwrt isn't the right choice. you should look at n2n http://www.ntop.org/n2n/ which is this nearest thing to a p2p-VPN I know, and it should work on top of a mesh very well, and there are openwrt packages. But it's still in development, and it's not very secure compared to openvpn. only preshared key,...
