You can use this
http://downloads.openwrt.org/backfire/1 … uashfs.bin
or this
http://downloads.openwrt.org/backfire/1 … uashfs.bin

if you need fast and stable wifi I would suggest the brcm-2.4 version, with kernel 2.4 and the proprietary broadcom wifi-driver.
if wifi isn't that important you should try the brcm47xx-version (with kernel 2.6) you will have wifi too, just not that fast, and not that stable.

if you need a autoreconnect-script (vpnc doesn't come with one) check my post here:
https://forum.openwrt.org/viewtopic.php?id=31853

and don't expect more than 2-3Mbit/s, your CPU is quiet slow.

vpnc is designed as client for temporarly connecting to a vpn-concentrator (on a workstation with a user in front of it). And the user could just reconnect if it happens that the connection is lost.
So yes it does't have autoreconnect and other features you would expect from a vpn-service.

But with the script I posted it runs just fine as a service. I used it in a productionenvironment for more than a year without any problems.

But I'm afraid I can't help you with finding the right image to flash
