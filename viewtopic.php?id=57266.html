I've posted my solution in another thread. Here's what I did.

I'm using DSL320B and had the exact same issue.
OpenWRT issues DHCP discover to the DSL320B and receives the external IP address.
That breaks all the routing since OpenWRT doesn't handle very well Half Bridge.
I've solved the issue placing a simple command in crontab.
* * * * * route add default eth1
Obviously eth1 is the WAN port of my OpenWRT based router. ;-)
