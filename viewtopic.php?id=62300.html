Hello community,

I have an OpenWRT router in my internal home network interconnecting 10 subnets/VLANs. I have configured a local DNS domain which looks similar like "homenet.XX" where XX is a nonexistent (that is, not yet in use on the Internet) top level domain. My OpenWRT router is configured to resolve local DNS names directly and query my Internet router for all other top level domains. Hosts accross all my VLANs can resolve names like "host1.homenet.XX" unless my OpenWRT router loses connectivity with my Internet router. Is it possible to configure OpenWRT in such a way that it continues resolving local DNS names when my Internet router goes down?

Thx in advance,

Steijn van Essen

By default, Chaos Calmer sets up a local domain called lan which is assigned by OpenWrt dnsmasq as per /etc/config/dhcp. For example, if you have a PC named happy which is connected to OpenWrt, its fully qualified host name becomes happy.lan.

Chaos Calmer also sets option rebind_protection '1' /etc/config/dhcp. This will prevent OpenWrt from using the DNS servers in /tmp/resolv.conf.auto for lookups. To correct this, set option rebind_protection '0' in /etc/config/dhcp.
