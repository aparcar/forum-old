I managed to solve my problem of getting serial console. My mistake was basically with soldering and cabling, quite dumm .

My board, which says REV 1.2 on it, didn't have serial pins. I didn't solder the ones in the middle good enough at first, so I corrected that.

Then I noticed another dumm mistake with my cable. I had connected RX to RX and TX to TX between MR3020 and FTDI PCB, which should have been TX to RX on both pins

Also, I noticed on a blog, that it could cause problem to connect Vcc pin between two boards, so I took that cable out.

I did all these three changes at once, then tried, and boom!. So I'm not sure which one(s) helped, but I now have serial output.

Well I'm no expert on soldering, but this one didn't seem hard. Just make sure all the pins you solder gets actually connected on the board, and not touch each other.

Also, have you put a 10k pullup resistor between tx and vcc?

Make sure of these two, and don't connect vcc to anything else. Everything should be ok then.

A guy has posted detailed explanations on his blog, but I can't add any links to this message. Just search for "3020 serial pin vcc" in google. One of the results should be a blog page from 9 june 2013. Domain is ediy dot com dot my.

Post #588
tomi_engel
24 May 2017, 15:14

Just tried to flash a TL-MR3020 v1.9 via the origial TP_link "Flash Upgrade" web UI … and when clicking the "Upgrade" button it always complained:

        "Please choose a file to upgrade"

Turns out that the latest 15.05.1 image … has a name which is too long.
So renaming

     openwrt-15.05.1-ar71xx-generic-tl-mr3020-v1-squashfs-factory.bin

into

     openwrt-ar71xx-generic-tl-mr3020-v1-squashfs-factory.bin

and then trying the upgrade with that renamed file resolved the issue.

So there actually seems to be a file name length limit in the TP-Link web UI code.
Maybe this information is helpful to others as well.

Honestly, don't. As much as I love the ones I already have, there's really no point in buying this router anymore. It has never been particularly powerful, even when it came out more than half a decade ago, and by today's standards it is only barely usable. For anything beyond basic OpenWrt router performance you'll (figuratively) have to bend over backwards, and even then it won't run great (i.e., forget about transmission).
