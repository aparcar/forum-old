My TP-Link TL-WR841-N worked mostly OK with the original firmware, but needed to be rebooted daily to prevent connection speed from dropping to near zero.  I finally decided to switch to OpenWRT.  I really do think it would be an improvement, but there's a problem.

I flashed the most recent image for the WR841-N, v9.  Everything seemed to work fine.  However, each time I turn off or disconnect the power, everything gets reset back to the original flashed image.  In case this matters, I have not enabled its internet connection yet, because the default IP address is not compatible with my existing network configuration.  Is this an >essential< step in completing the initial configuration, or just a good idea?

At first I was configuring from the web interface, but switched to SSH in order to see what was really happening.  I created a file in /root then rebooted and it disappeared along with every setting change I made while to attempting to configure it.

I can't find anything in the forums that relates to this - in fact, a lot of people seem to use and enjoy this device.  I would very much like to join them.  Any ideas?  Am I doing something wrong, or should I have returned this to its maker during the warranty period?

I'm about ready to start looking in /rom to see where all the raw, defautl configuration files come from, and "configure" that stuff so that when it resets I get what I want anyway...
