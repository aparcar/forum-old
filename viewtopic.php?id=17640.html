After loading Kamikaze 8.09rc1 2.6 kernel on my venerable WRT54GS v1
I tried to set hostname of system using the typical:

uci set system.cfg1.hostname=spike
uci commit
reboot

This did not work!

When I view with
uci show system
I see strange numbers where I would expect cfg1 it shows:

system.cfg02f02f.hostname=OpenWRT

If I set using these numbers I get the right results.

I have noticed this on a few other files as well.

Bug or just something wrong with my platform or install?

Post #2
SnapShot
11 Nov 2008, 03:27

The system configuration uses unamed sections, that's why you see the strange numbers instead of section name. Try with:

uci set system.@system[0].hostname=OpenWrt

This command means set the hostname variable in system config file, in the first system section to "OpenWrt".
