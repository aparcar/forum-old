Hi all, i'm managed to sucessfully run OpenWRT on ASUS DSL-N12U, but there is some issue i can't find way to solve for days.
It works greatly when flashed with Serial command, everything works (except DSL). I want to release it for people without Serial hardware.
Seems like it has patched U-Boot that don't flash entire memory when used with "RESET" key. But it works correctly if i choose this method on boot using Serial:
   2: Load system code then write to Flash via TFTP.
In "reset" way it can't flash standart OpenWRT bin, it needs special header, i'm added it by replacing OpenWRT mkimage utility with asus patched one from GPL source release. After doing it I managed to flash it using reset button, but seems like it writes only first 1MB of firmware and drops rest of it. It flashes faster and data partition is not flashed correctly, so system can't mount root.
I don't has much experience in porting firmwares, maybe i'm just going the wrong way?
