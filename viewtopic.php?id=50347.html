Hello all,

I've been talking to an ed-tech nonprofit in Rwanda (http://kepler.org/), and they have an interesting problem:

Net access in Rwanda is... weird. There's actually a pretty big fiber infrastructure, but no one can afford to use it, so most home networking is done with 3G routers instead. I was talking to them about helping them set up some kind of proxy solution to minimize bandwidth needs, but the problem is that many of these 3G plans have bandwidth caps so they plan to have multiple routers, and just assign each student to a router, which could make giving everyone access to the proxy difficult.

I then had an idea, but I'm very much not a router/network guy so I'd like to post it here for sanity checking and advice. What if they...

1. Disabled the WAP features of the 3G routers, so they basically just do wired<->3G

2. Set up an OpenWRT WAP with all of their routers (and the proxy server) plugged into it

3. Somehow configured OpenWRT to spread outbound traffic between the 3G routers and/or stop sending traffic to a router when its bandwidth cap is hit (ie when it stops reliably sending traffic)

The idea would be for all of the students to connect to the same WAP, and thus be on the same LAN, and have the WAP deal with figuring out how to juggle 3G routers.

To me this seems like a bizarre but plausible setup, but again I don't do this sort of networking stuff very much, so maybe it's either completely unworkable or quite common.

If anyone can provide guidance about this, I would appreciate it!
