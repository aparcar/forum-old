Ugly vs. non ugly is a matter of taste but I cannot understand the more complicated argument. Its the very same forms as in 0.9, just tabbed to bring the more esotheric stuff out of sight. If you mean the "mini" admin, well yes its is certainly easier to use but contains a lot of duplicated code and I stopped maintaining it for the time being to get more important framework stuff finished in order to be able to start working on applications again.

However I don't want to dismiss your arguments but I am always faced with "ugly", "complicated", "its crap" arguments without any specific pointers or suggestions. I know that some parts are still crude and could be made look better / work smoother but this is hard to right without good user feedback.

As for being slower: the framework is now compiled in full source mode by default, this means there's a higher startup overhead since Lua has to byte-compile the files first but on the other hand the resulting source code compresses up to 50% better compared to precompiled files.
