eleon216
1 Mar 2015, 20:34

maybe the new firmwareimage was a little bigger than the previous one, so now the new overlay-partition is smaller, maybe luci or it's dependencies did get a little bigger, or you used sysupgrade with "-c", so now also non-modified configfiles need space on the jffs2-partition, or something else, but I'm quite sure the reason isn't some bug. 
just look in /overlay to see what occuies the space.

4MB is the minimum for running openwrt, and over the last years the minimal set of packages got bigger with every openwrt-version. On kamikaze you could add some additional packages, on barrier breaker  hardly any.  You had only 768kBytes on your overlay partition, and the filessystem need one eraseblock so you have only 704kBytes left, that's not much for your configfiles and additional packages.
So if you need luci, get a firmwareimage with luci build in, or create such image yourself.  Or you could try a fresh install (do not save configuration over reflash), maybe it will fit, but your best change is to put all packages you need, in the firmware!

What image did you use ? AA should work just fine on the v3's. To recovery using failsafe method you will need to set your network up static to reach the router in failsafe mode. Set your computer ip to 192.168.1.11, subnet 255.255.255.0, default 192.168.1.1. Put the router into failsafe mode(led blinking fast) then you can reach the router via telenet. nope I don't think none the less of you, I have been working with openwrt for years and from experience i can tell you that each recovery can be quite unique.

lstart with the basics..

1. make sure the ip has not been changed ?
- open up cmd prompt and type ipconfig /all and check for default gateway:
* this should not matter when using safe mode anyway..

2. make sure the router is the only device connected to the computer, turn off wireless

3. make sure cable is hooked from computer to the a lan port (1-4) on the router

4. set computer up for static ip:192.168.1.11, subnet:255.255.255.0, gateway:192.168.1.1 as before

5. reboot router in safemode and use telenet in putty with address 192.168.1.1


if you still can not access the router then you may need to recover with via serial port which will require a ttl converter and some soldering skills

hnyman

In that video you are trying to connect to 192.168.1.11, which is your PC's address, not the router's...

Router will have 192.168.1.1 as its address in failsafe mode.
And like said above, ssh mode works in router's normal operations, but with failsafe mode you need to use plain "telnet" mode.

And like hostle19 said, you may need to set your PC's own IP address to a fixed IP address, as it probably normally gets the IP address via DHCP which is not working now. The address might now be 169.254.x.x link-local address instead of something from the 192.168.1.X block. check from control panel, or from CMD console with "ipconfig".
If necessary, set PC's network settings to a fixed IP address something 192.168.1.X, e.g. 192.168.1.22. (I am not saying .11 as you may mix router's .1 and PC's address.)
