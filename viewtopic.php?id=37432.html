loyukfai wrote:

Is there any way to prevent clients from the guest network from accessing the router's administrative interface (port 80 and 22)...?

Create a reject rule for INPUT chain, where destination port is 80 and source address is from the guest network address space. And a second rule for 22.

You might insert the rule to be the first rule in the INPUT chain (so that no other rule will give ok first).
