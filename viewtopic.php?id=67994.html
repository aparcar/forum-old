Hey Guys,

I am getting a lot of the following messages in my system log:

Sun Oct  9 18:32:42 2016 daemon.warn odhcpd[885]: DHCPV6 SOLICIT IA_NA from 0003000180ea235a58d2 on br-lan: ok fd37:3552:7e0::792/128

It looks these are related to clients asking the DHCP server for an IPv6 address, and since there is no gateway to route this traffic to the internet it gives this warning. The thing is, my router is configured as a dump access point. This means I have deleted the WAN interface, and for the LAN interface I have the "Ignore interface - Disable DHCP for this interface" box checked. I also have the "Router Advertisement-Service" and "DHCPv6-Service" set to "disabled" on the "IPv6 settings" tab.

For IPv4 this seems to be working just fine. The DHCP is successfully disabled and is not giving out IPv4 addresses. But for IPv6 the DHCP is still enabled it seems. Clients are able to get IPv6 leases from the DHCP server even though I presumably disabled it.

My question: Am I missing something? How do I disable DHCP completely, for both IPv4 AND IPv6?

Thank you very much in advance!

For access point you dont need odhcpd.
/etc/init.d/odhcpd disable
/etc/init.d/odhcpd stop
But dont forget what you have done for future
