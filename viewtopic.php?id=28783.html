Post #3
eleon216
1 Mar 2011, 12:57

WPA2-encryption is specified in 802.11s but isn't implemented in open80211s (now).
WPA isn't possible in ad-hoc modus.
WPA2 with CCMP should be possible in ad-hoc mode but I'm not sure if it works in openwrt.

Also, a caveat from the olsr-users listerv about using WPA2 encryption (should it become supported in ath9k + adhoc/mesh) with the higher bitrates:

"You need to listen to 40000-80000 encrypted IP packets before you can calculate the 128 Bit key. Thats only a few minutes on a 802.11g full speed download."

The poster likened this to erecting a 50cm tall fence to keep out intruders.  So, there does appear to be a point of diminishing returns.

Encryption is implemented in open80211s!
So maybe you should give it a try and set up an 11s-meshnetwork with encryption.
You should be able to run 11n, but I'm not sure if the encryption is done by the wifi-chip or if it's a softwareimplementation.

Check out http://o11s.org/trac/wiki/HOWTO#secmesh for the securemesh implementation
and http://o11s.org/trac/wiki/OpenWrtBuilding for the openwrt-specific part.
