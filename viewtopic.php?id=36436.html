From what I have seen on here for a while now it is unlikely someone will post to help you with such an issue as it's usually down to you to figure out the problem or possibly such an issue has been previously discussed enough that a search should find the answer.

Of course with OpenWRT having such a steep learning curve it can be difficult to diagnose problems or search for relevant posts.

I have learned that sometimes it is necessary to try cleaning up when problems arise. I know very little about your problem so I don't know if it is necessary. You must decide if it is necessary. You can read about cleaning up:

http://wiki.openwrt.org/doc/howto/build#cleaning.up

Also keeping a copy of source downloads is a time saver when you feel it is necessary to try a distclean so before trying it you would move the dl folder to the parent folder for example:

~/openwrt/trunk$ mv dl ..
~/openwrt/trunk$ make distclean
~/openwrt/trunk$ ln -s ../dl

Sometimes you will decide a clean up isn't necessary and you can choose to revert back to a known working version in case the latest trunk is currently broken. You can choose to wait until the next day and update to the latest again with the hope it is working again.

Also you can browse the latest changeset by going here:

https://dev.openwrt.org/changeset

You can work backwards by clicking "? Previous Changeset" and hopefully you will get an idea of what may have caused a problem for you.
