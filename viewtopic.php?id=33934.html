Deroute wrote:

I have similar problems with "packet drop" and "standard deviation" graphs in "ping".

What is similar?
The realtime charts (that this thread is about) are drawn in a completely different way from the Luci statistics charts (that you are talking about)...

What you are probably seeing is two empty charts for two different reasons:

The deviation should be 0 as the default sample size is just 1 ping, so that there is no deviation inside the sample. If you change the sample size and interval values, you will get deviation.

The packet drop is almost always 0 as the drop chart is scaled wrongly  in AA Luci statistics (1/100 of the correct value, so that 100% drop is shown as 1%). I wrote a patch for that and trunk is already patched along the collectd update to 4.10.8.
See: http://luci.subsignal.org/trac/ticket/462 and https://dev.openwrt.org/browser/package â€¦ ?rev=34487
