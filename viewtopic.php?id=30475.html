Hi guys!

Using ar71xx r27153 (trunk) for my D-Link DIR-825 and installed the newest luci-build, later on updated the packages and downloaded luci-app-qos. (did everything just a few hours ago)

I can adjust the qos-setting in the webinterface without any problems, but unfortunately, they don't seem to have an effect.
My connection has 66,74MBit/s downstream and 5,21MBit/s upstream. I've tried it with download-speeds from 45000-66000 and upload-speeds 4000-5000 (webinterface says that it wants kBIT/s), enabled/disabled overhead calculation and of course I've enabled the profile.

Also tried numerous classification rules:
Target | Source | Destination | Service | Protocol | Ports | Number of bytes

priority | 192.168.1.193 | all | all | all | 27000-27050 |
priority | 192.168.1.193 | 95.223.72.1 | all | all | 27000-27050 |
priority | 95.223.72.1 | 192.168.1.193 | all | all | 27000-27050 |
(and the using the last two rules at once)

(each of those with target "express")

(each combination with the standardrules set to low-target)

There is a gameserver running at 192.168.1.193 using 80kbyte/s downstream and 110kbyte/s upstream in the worst case.
95.223.72.1 is the gateway of my isp to which I'm connected to.

However, running the speedtest from another computer at the network and taking a look at the players ping, they all increase while the speedtest is running, regardless of which of the above settings I'm using.

Any help would be appreciated.

SOLVED!

Although "enabled" was checked in the qos-menu in luci, the startscript "qos" in System--->Startup was disabled by default.
Maybe it's a bug (never accessed that menu-tab before), I don't know, but it now works as intended.
