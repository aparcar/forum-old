Post #9
FriedZombie
30 Apr 2013, 16:21

Well here is an example

config switch
    option name 'switch0'
    option reset '1'
    option enable_vlan '1'

config switch_vlan
    option device 'switch0'
    option vlan '1'
    option ports '0t 2 3 4 5'

config switch_vlan
    option device 'switch0'
    option vlan '2'
    option ports '0t 1'

config interface 'loopback'
        option ifname 'lo'
        option proto 'static'
        option ipaddr '127.0.0.1'
        option netmask '255.0.0.0'

config interface 'lan'
        option ifname 'eth0.1'
        option proto 'static'
        option netmask '255.255.255.0'
        option ipaddr '192.168.222.1'


config interface 'wan'
        option ifname 'eth0.2'
        option proto 'dhcp'

the wan port is eth0.2 in this setup, all other ports are put under eth0.1

edit: I just extended your 'default' settings

You probably don't want Masquerading   in the vpn zone in this case.

And the log does complain about tls handshake failure and the certificate, so that is what is going wrong.
see: http://wiki.openwrt.org/doc/howto/vpn.openvpn#build.your.certificates
for more detailed info see http://openvpn.net/index.php/open-source/documentation/howto.html#pki

My guess is that you created the client cert, without using the private key/cert of the vpn server
