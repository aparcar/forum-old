Possible reason for low interest is limited hardware, performance capabilities of this generation devices.

Last I checked, OpenWRT itself was not prepared for proper user experience, easily wasting users' time if they tried to use it with the nice OpenWRT features such as the web interface, firewall, packages.

If features are enabled, allowed - it should be made safe - more testing and fixes towards user experience would be needed.

I do not see a reason for such projects to be deliberately geeky (unless we keep evil and want to limit the user base or waste people's time when they come across a technical difficulty built in the Open Source software by means of non-software-management.)

In the WiKi article I posted, I show that there is not enough space on the flash, and thus removing of packages.
http://wiki.openwrt.org/toh/netcomm/nb6plus4w




People are interested in switch functionality to use such devices for VLAN tagging switch next to a router on a stick as well as next to a Wi-Fi AP L2 only devices. As well as L2 separation.

Another use is - unmanaged switch with a pingable IP for monitoring.
