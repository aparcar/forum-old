I'm running openwrt on top of raspberry pi 2. When I try to configure wireless config via UCI it throws me "invalid argument" error. I searched little  bit and I found "wifi-iface" in my wireless config file cause this problem because it contain hyphen in between two words which is not allowed by the openwrt.

according to some users  removing the hyphen will solve their problem but to me, it still exists. I look through UCI documentation but there is no helps to fixing this issue

I tried following two UCI commands

uci set wireless.wifi-iface.ssid=test
>> invalid argument

uci set wireless.wifiiface.ssid=test
>>  invalid argument


Then I tried to rename  "wifi-iface" to "wifi_iface" in actual configuration file but then after reboot my wifi is not working. then I revert.

I'm building simple front end to openwrt using PHP and I want to use this command. How can I solve this? is there any fix for this or should I go with different approach other than UCI

thanks
