Take baby steps. Instead of trying to configure everything at once, follow the process I have done. https://forum.openwrt.org/viewtopic.php … 14#p332914

You config has extra bridge config in the network interface configs.

If you do it step by step then you can see where it works/fails.

Test it with encryption 'none' (and no password as there was a bug in at least one version of the code that automatically enabled encryption if a password was set, ignoring the encryption setting) to begin with. And make sure the static IP on the mesh interface is different for each device.
Test by ping between the devices. And by
You don't need to configure anything else via the cli ie

iw dev wlan0 interface add mesh type mp mesh_id smesh
ifconfig mesh 10.0.0.1

isn't required. Basically https://wiki.openwrt.org/doc/howto/mesh.80211s up till the section of 'known issues'.


iw dev $MESH_IFACE mpath dump

Gives a list of actual connections so you should see the IP of the connected mesh device.


In summary,
1/ simple, one step at a time.
2/ try no encryption to start with and get working
3/ add encryption and get working.
4/ add on bridge/route/firewall etc.

wrtboy wrote:

4a.  add a bridge.
4b.  route
4c.  firewall

At his point you have 'some sort of network' on each router (I don't know what your setup looks like) as well as a mesh link between them.
You have pinged between the two routers via the mesh connection. So you have connectivity via the mesh link. Now you just need to get the appropriate traffic passing through the link.

What you do now depends on what you are trying to achieve. The mesh link is just another layer 2 link. (ie same as a piece of cable between two switches.

If you just want to bridge the two routers via the mesh, just bridge the 'mesh' network with the required network at each router.

As an example for using the mesh as an extension cable for two routers.
ie
on router A
bridge wifi sta, mesh and ethernet
on router B
bridge wifi sta, mesh and ethernet
Any firewall is already in place and no routing as it's layer 2.

If you have different networks on each router (ie layer 3) then you'll need to add the appropriate routes on each router pointing to the mesh link as a gateway to the network.
ie
on router A
bridge wifi sta and ethernet, add route with mesh router B IP as gateway for router B network
on router B
bridge wifi sta and ethernet, add route with mesh router A IP as gateway for router A network

Firewall is also dependent on what you want to do.
