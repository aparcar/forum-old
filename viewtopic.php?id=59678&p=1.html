Use Case Summary

Micro technology (SOC, etc) has allowed us to shrink the router footprint substantially, but few of the manufacturers have begun to address the use case paradigm.

Home routers are expected to always have power and a wired connection to the ISP. 

A portable router will require its own power and be expected to connect to what ever AP is available under a variety of wireless and wired configurations. 

To be truly portable, these devices all require the following minimum set of features:
1 – Battery power for the device and its real time clock.
2 – Dual radios so the WWAN and WLAN can be configured independently.
3 – Connection management tool set.
        A - Easy to access connection management tools.
        B - Function on the smallest of mobile devices (i.e. cell phone)
        C - Support the retention and maintenance of a connection list.

The goal of this post is to highlight the use factors in hopes that manufacturers and developers will consider these as important criteria in new designs.  Using my portable router should be as easy as using my cell phone.  I should be able to press a button to turn it on, and if configured, connect automatically to my saved connections, or be prompted to connect. 

For those interested in the details, read on....

We have been using the terms "portable device" and "mobile device" for years now.  Travel router is relatively new.  Would you buy a laptop, tablet or phone that only ran off an external battery?  I think not.  The expectation of devices in these classes SHOULD be they run on their own internal power, otherwise I will suggest these are just "micro-routers".  Some devices, do have batteries, however it does not appear that any of the powered device can maintain the real time clock.

There is nothing wrong with micro-routers, and there are a lot of good applications for these devices.  I am suggesting they are not "portable" to the standards that we are now accustom to, and fail to meet the suggested use case.

In addition to the power issue, there is a radio issue.  The single radio supports a single channel and configuration for both WAN and LAN. While one may be able to work around this in the home environment for a static configuration, it does not support the dynamic client side configuration requirement that users are faced with connecting to hotspots, unless they also reconfigure the LAN side client and AP.  There is no device, in any class that I am aware of, that offers 2 radios in the same frequency band. Bridging on home units is substantially the same use case as what these travel routers need to be able to do well all the time.  It's well documented that this function is not without problems and constraints. http://wiki.openwrt.org/doc/howto/clien … modeissues   Unfortunately the current OpenWRT design will frequently result in an inaccessible wireless network if the WAN side connection is not found.

Some vendors, like HooToo and TP-Link and offer internal batteries and software features, including mobile apps, in their products with stock firmware.  Unfortunately the software features disappear after installing OpenWRT.  Gl.inet has no battery powered offerings, but their routers include OpenWRT BB 14.07 and an additional proprietary user friendly GUI included with the device, though this is not clear from the website. http://www.gl-inet.com/ar150/

No one that I am aware of is doing both. 

There are no OpenWRT mobile APPs.(IOS/Android)  There is no mobile device compatible Web GUI for OpenWRT.  There are no connection manager specific tools.  There is no capability to store previously used connections.  The lack of these features makes the devices inconvenient to use in the real world, and can be addressed by the community.

The "Road Warrior" market is substantially unaddressed.  This group of generally technologically savvy users fill the cafes, airports, and hotels daily and lack the ability to connect securely with any of the off the shelf current products.  None include VPN tools.  The current functionality is basically a small workgroup.

If one looks at the current crop of product broadly, with stock firmware, one will find that these products APPs are targeted at multimedia and file sharing.  They support USB data devices, some support cameras, most have some level of DLNA and can stream media.

OpenWRT can meet the need of the "Road Warrior", technically, with VPN, Samba, USB support, miniDLNA and other packages.  The usability, unfortunately, is not so good on a small device, and indeed most of this relates to the connection management tool set.  The OpenWRT GUI, as noted here and in some other of my posts, is not mobile compatible.  It is a few too many clicks away from being considered "easy to use" and does not allow one to maintain a list of previous connections.  For Hot Spot use the process must be quick and ideally transparent.  The user will be there a short time and spending any significant time connecting reduces the time “using”.

There are numerous other router features that could be discussed.  The number of USB ports, GPIOs, battery size, CPU speed, etc are differentiating product features that will be of different levels of interest to different consumers.  The importance of these is secondary today to meeting, what I will suggest are common core features of the product class (battery, dual radios and connection tools).

I am not affiliated in any with any product offered for sale by any manufacturer or site, nor have I received any incentive from any to promote the same.  I own a HooToo TM02, and have been working to establish an easy to use OpenVPN connection to access files from my home server for about a year.  It's been extremely frustrating in today's world that I am unable to do this.

The community can not fix the hardware issues, however we can lobby the vendors to do so.  We do have some ability to address the soft features in the product.  Hopefully some of the developers will come to our aid, but if not maybe there are some creative souls that can improve the tool set.

@alzhao, Thank you for your reply.

The core value of a portable router is to allow multiple devices to connect to a single device that can then connect to a single AP. 
*In many hotels a user is provided with only a single connection, and a need to connect more than one device.
* In many hot spots one does not have easy access to a power connection, and carrying a separate battery is just not convenient. 
* I want to run all devices through VPN and may wish to allow a device with no VPN to attach.

Regarding redundant features, personally I would forgo the mobile WEB UI for a iphone/android APP.  We are in an omni-channel world and the user should have a choice as to how to connect.

As for the radio, I think this may not be as big an issue as I thought just yesterday.  My usage has been in OpenWRT software, and I am learning that the process vendors are using in their firmware solutions may address some of the problems OpenWRT has with connection management.  See posts 295 and 298 here.  https://forum.openwrt.org/viewtopic.php … 76#p292276

The hardware features I am suggesting are items 1-3.  While I am writing on the OpenWRT forum, and looking to use this software (4), I am not opposed to a vendor adding VPN to their existing firmware as a solution.  Indeed, some vendors are close to a solution and only need to add VPN.  I am not clear if most devices with factory firmware have a working RTC or not, but see 1 & 2 as the same (1 & 1a).

The ease of use features (5.6) are available in most firmware and LOST with OpenWRT.  They should be supplied by the community, not the hardware vendor, though the vendor may be motivated to offer a complete unified solution and the first will probably gain my business.  I would prefer an app that I can use across all vendors, but either works (Google Play or Apple Store have no OpenWRT apps, but there is a DD-WRT APP).  Item 9 again is offered by most vendors today, and available in OpenWRT.

I have no problem with a $50 device.  Again, I differentiate micro router from portable router, and battery is the single biggest factor.  I am not suggesting this as a hobbyists device.  The price of most of the battery units are closer to this number.  Small does not mean cheap in my book.  I am, as I think many road warriors are, willing to pay for features and convenience.  As for the size (and weight) I expect it to be about the size of a cell phone and up to twice as think.  A unit with NO wires does not even need to come out of my bag or pocket.  All I need to do is press the button and work.

In summary, to use this at a mobile hot spot, for hardware I need a battery and for software VPN and easy to use connection tools.  I am now of the opinion the radios can be made workable.

Regarding the battery, my HooToo seems to consume about 500 ma/hr based on some real time testing (your mileage may vary).  3000 I would expect be a minimum, 5000 a rough upper end.  That should be enough to get through the day.  I do not expect it will be on all the time.  The battery technology in most device is in the form factor of an AA or AAA if you busted them open (done it), even in your laptops.

RTC is important for a few reasons.  I would like the time to be correct in the logs and some programs like DNScrypt require accurate time to run http://wiki.openwrt.org/inbox/dnscrypt., NTP does appear to update about every 10 minutes.  I am not smart enough about the hardware to know if this can be done with out the device being "fully" on.  I know most devices have a CR2032 or similar to keep the time right.  Exactly what needs power I do not know.  I have assumed the main battery can also do this even when powered off.  Maybe I'm wrong.

Regarding 2 radios, see post 4 in the thread.  I am becoming more convinced that a better software solution can at least handle a Router config, if not an AP and relayd.  Never used WPS, and do not see it as essential.  It's just a front end in my mind (software solution).  It's a design issue.

OpenVPN and TOR are different tools and have different purposes.  They are complimentary,.  Tor hides the connection points, but not the data.  VPN hides the data, but not the connections, and to be complete DNScrypt and DNSSEC makes sure the DNS requests are real, and not spoofed.

I am not really too concerned about the world knowing the end points, but I do want my content secure.  And if I route everything through a VPN Service, they really can infer nothing from the end points.

As for the boxes, I'm a manufacturing guy.  I appreciate the time and effort it takes to design, build and support a product.  They are not the box OEM, so there is a layer of cost above say GL.inet or NEXX.  We also need to face the fact that there is an entire audience of people who have no interest or ability to flash a device, but still want a solution.  Actually, I am willing to pay $50-80 to get what I want in a box; this has been a major time suck.  Other than GL.iNet, I am not aware of any other vendor including the firmware as stock, and lets not fool ourselves, it was a cost savings to them.

In terms of software, we have something close to what you are looking for. The travel router software we are offering has the following ready to use features;

VPN Ready

Works with all latest smartphones, tablets and computers alike.

Can connect wifi to wifi (on single radio, but we manage the dirty details in our software like managing and scanning connections, disabling station mode if we can't find any hotspot etc.)

Can connect over RJ45 cable

Full usb flash drive control using web interface that works on all platforms. You can do all the common file operations using Iphone, Android, and Windows Phone >=8.1 (no apps required) . This includes uploading files from and to your cell phone. File uploads offer auto retries (useful for large files e-g > 1GB) retries can utilize  http file upload continue mode (much like ftp continue).

Copy Files from one USB flash to another just with your cell phone and the travel router.

Map android phone as  a virtual drive (app required for this feature). This will allow you to manipulate android internal and external sd card just like USB flash.

Ez File sharing lets you share selected files on your router's USB flash drive and or Android phone virtual drive with others in 3 modes
a)Using hotspot's wifi
b)Using router's wifi
c)Over the internet (Possible even when your travel router does not have public IP address, but you will need our home/office router to get this feature working or our VPN service)
Ez File share lets others upload files to selected folder on the USB using any browser capable of uploading files (with your permission).

Home/Office VPN server router
========================
Home VPN server router is ready to work with ikev2, ipsecure, openvpn, l2tp, l2tp/ipsec, pptp. That should cover pretty much any cellphone. Just add a remote user account, and you are ready to go with xauthipsec, l2tp/ipsec, pptp. For openvpn and ikev2 you need to download cert and .ovpn file from the router to your cell phone as an extra step.

We have dual mode support (in select models). i.e the router can act both as the vpn server and client with any combination of protocols. For e-g it may be connected to the vpn server using l2tp/ipsec and to your iphone using Xauth/IPSecure or your Windows phone using IKEv2 simultaneously.
