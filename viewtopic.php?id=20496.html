My OpenWrt system needs to run two instances of OpenVPN as it connects to two OpenVPN servers out on the internet.  I have used UCI to configure to configure the following:

openvpn.custom_config=openvpn
openvpn.custom_config.enable=1
openvpn.custom_config.config=/etc/openvpn/client1.conf

This works great and OpenVPN starts up  using client1.conf with no problem.  However I cannot figure out how to add another client.  In regular Linux like Ubuntu, OpenVPN will load any *.conf file in it's /etc/openvpn, however this doesn't seem to be the case with OpenWRT.  I have created a custom init.d script to start the other client, but I think there is probably a better way to add additional custom configs using uci commands.  I want the cleanest way to do this and I want to avoid making all kinds of custom changes to the stock Kamikaze scripts. 

Thanks,
Johnny

just edit /etc/config/openvpn

you can have many sections like this one :

config openvpn config1
        option enable 1
        option config /etc/openvpn/client1.conf

config openvpn config2
        option enable 1
        option config /etc/openvpn/client2.conf
