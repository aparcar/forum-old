I have Openwrt on a WRT54g with my wired lan behind it. There is also a WAP11 connected to the WRT54G which I used for wireless access. The reason for the WAP11 is that the WRT54G is in the far corment of the house and the WAP11 is in the attic and provided house-wide coverage.

The WRT54G is downstream from another single purpose router which performs similar firewall functions to OpenWRT and each machine on the lan has a full compliment of security software.

What I would like to do is:

1) Eliminate the wireless support. To this end is setting wl0_radio enough or should I do more?

2)Turn off the OpenWRT firewall without impeding access.

The configuration without these changes is running fine but to optimize a little I might as well turn this stuff off.

Can someone comment on how to do this?

obrienj wrote:

1) Eliminate the wireless support. To this end is setting wl0_radio enough or should I do more?

Well, the easiest way is to set wl0_radio="0" in NVRAM.

Than if you like to get rid of the proprietary software parts and get free space for other things you can build your own firmware image using OpenWrt's Image Builder. There you can build images without the iwlib, kmod-brcm-wl, kmod-wl-compat, wificonf, wireless-tools packages. These packages are only required if you need wireless functionality. Makes room for lots of other packages.

obrienj wrote:

2)Turn off the OpenWRT firewall without impeding access.

I would like to prefer of changing the permissions on the init script instead of removing it.

chmod -x /etc/init.d/S45firewall
