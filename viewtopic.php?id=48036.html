Hi,

First time OpenWrt user here. I'm very happy with Attitude Adjustment (12.09, r36088) on my TL-WR1043ND.

I installed the wifitoggle package through the GUI. All is well with the core functionality. I press the button, the radio toggles on or off. And the WLAN LED toggled on and off correspondingly, in the beginning. It has stopped doing that now. It is always on.

Here is my /etc/config/wifitoggle.

config wifitoggle
        option button   'wps'

        option persistent       '1'
        option timer            '0'

        option led_sysfs                'tp-link:green:wlan'
        option led_enable_trigger       'none'
        option led_enable_delayon       '500'
        option led_enable_delayoff      '500'
        option led_disable_default      '1'

I am a little wary of the item for which I gave a 'none' value. I can't find any documentation for what it does. These are supposed to be its possible values:

# cat /sys/class/leds/tp-link\:green\:wlan/trigger
[none] switch0 default-on timer phy0rx phy0tx phy0assoc phy0radio phy0tpt netdev usbdev

The documentation says it's actually the led_disable_default value that determines whether disabling wifi turns off the LED. In any case, I did not change my config file between the last time I saw the LED work and the first time I didn't.

Thoughts?
