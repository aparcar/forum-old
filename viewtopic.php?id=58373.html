Hi guys,

in the most current trunk debug logging from hostapd has been disabled. ( See https://dev.openwrt.org/ticket/15658, http://wiki.openwrt.org/doc/devel/debugging).
Apparently even authentication messages for radius authentication are influenced by that change.

Usually my APs had messages like these:

Jul  7 17:11:00 ap-05 hostapd: wlan0: STA xx:xx:xx:xx:xx:xx IEEE 802.11: disassociated
Jul  7 17:11:01 ap-05 hostapd: wlan0: STA xx:xx:xx:xx:xx:xx IEEE 802.11: deauthenticated due to inactivity (timer DEAUTH/REMOVE)
Jul  7 17:11:07 ap-05 hostapd: wlan0: STA xx:xx:xx:xx:xx:xx IEEE 802.11: authenticated

with

logger_syslog_level=2
logger_stdout_level=2

according to /var/run/hostapd-phy0.conf

With the most current change this is not the case anymore and the AP is mostly silent. ( on loglevel 2)
logread does not reveal anything as well.

Changing

#CONFIG_NO_STDOUT_DEBUG=y
to
CONFIG_NO_STDOUT_DEBUG=n

and rebuilding did not help either.

Does anyone know how to get the old behaviour back? Right now I'm trying to find the corresponding commit and working it out from there on. I hope this is not a duplicate topic as the only forum resource related that I've found is this thread from 2013: https://forum.openwrt.org/viewtopic.php?id=44942

Edit:
This issue has been resolved by removing syslog-ng in favor of the new ubox logging system.


Regards
