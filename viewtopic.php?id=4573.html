don't really understand why but you can get the route to the modem added to the vlan1 interface, by temporarily interrupting operating through ppp0, by
(assuming you start from dynamically assigned IP)
/usr/bin/killall -9 ifup.pppoe
/usr/bin/killall -9 pppd
/sbin/ifdown wan
/etc/init.d/<< your script that sets nvram parameters for static wan IP>>
/bin/sleep 1
/sbin/ifup wan
/sbin/ifdown wan
/etc/init.d/<< your script that restores nvram parameters to previous dynamic pppoe state>>
/bin/sleep 1
/sbin/ifup wan   (this will restart ifup.pppoe and pppd)
/etc/init.d/<<your regular firewall script >>
/etc/init.d/<<your additional route definitions script>>

Not sure whether the sleep seconds are really necessary. I found the above effect by accident.
The nvram parameters to change and change back:
                           dynamic        static
wan_ifname          ppp0             vlan1
wan_proto            pppoe            static
wan_ipaddr           ""                 "192.168.254.1' in your case
wan_gateway        ""                 unset

No need to commit these values to permanent nvram.
If you now type 'route' the 192.168.254.0 subnet should show as reachable via vlan1 while also the ppp0 with dynamic IP is active.

Would not consider this as elegant at all. Perhaps somebody knows how to add a net to the vlan1 interface while operating through ppp0.
