7621+7612+7603_WE3526KY-2升级固件.bin

it's a ZBT WE3526 (MT7621A+MT7603+MT7612)
4G modem feature implemented by MiniPCIE modem card (only USB pins and special pins for sim slot used actually)
try LEDE and use a WE3526 profile to compile/build image

Firmware Version    OpenWrt Barrier Breaker 14.07 / LuCI Trunk (unknown)
Kernel Version      3.10.14

Your device is running Mediatek APSoC SDK OpenWRT firmware, which is based on Barrier Breaker, but contains _proprietary_ kernel and drivers. Compiling OpenWrt/LEDE will decrease your wifi speeds, NAT speed (if it compiled with HWNAT support). Probably you will lose SD card reader feature. Probably you will have troubles with your 4G modem drivers.
But you will get all OpenWrt/LEDE features without SDK build restrictions (like wifi/switch manage etc)

mkf.vip wrote:

OK, Why they still use "Barrier Breaker 14.07" rather than newer Barrier Breaker Chaos Calmer 15.05.1???

Cuz they (Mediatek devs) started before barrier breaker release (it's based on pre-release BB snapshot). It was an experiment, both shiity, old and lot of bugs 3.10 kernel (mainline kernel for Mediatek SDK is 2.6.36) and openwrt build with this kernel. But this experimental openwrt build became popular suddenly (say thanks to small vendors who don't wanna make their own customized firmwares and use this crap for their products as a stock, yes, im talking about ZBT and other "nonames")

mkf.vip wrote:

and actually need just repeater option why it not supported??? do you have any suggestion

cuz it use _proprietary_ driver as i said before
in a few words - it's faster, much better optimized and have more features than MT76, just cuz it written by guys who work in the same company as chip engineers

but

It completely have NO support of mac80211. So you can't configure it by usual openwrt way (by uci/luci) without special "adapters" as it made for proprietary broadcom drivers (wl) for example.

To "adapt" drivers it requires some special scripts which will translate uci settings to special config files used by driver and make some movements to up/down radios, additional apps (like sdk's "apcli" app to create a client connection) and special iwinfo driver to translate some driver data for openwrt enviroment.

Mediatek devs implemented AP mode only and very limited iwinfo support (no support, it works just as a generic WEXT and very few generic ioctls work actually), just cuz it was an experiment as i said before and it was enuff for devs to be proud they made sdk based "openwrt"
