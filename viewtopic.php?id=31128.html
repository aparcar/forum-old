I respond to my self. After investigation I discovered a config error (my fault).
After correcting for the config error I was able to receive email but not to transmit. I was in the same situation as in discussion http://old.nabble.com/DNS-problems%3A-c … 37089.html
I was actually getting SMTP-Error = "Bad format for DNS name record" .
As conjectured in the link above the problem was that the package (from backfire 10.03.1RC5 repository) was build with the wrong endiannes (at least for the AR71xx platform of my TP-Link WR1043 ND router).
After modifying the sysmachine.h patch for using BIG ENDIAN  configuration and rebuilding all problems disappeared and the server is correctly working.
PS: I am not expert enough with openwrt /linux to submit a proper patch or opening a ticket for correcting the error.
