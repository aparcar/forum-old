Depends. Not in the way of actual port forwarding since IP doesn't know about domains.
The only thing you can do is set up a distinct (reverse) proxy server which takes every request on port 80, understands HTTP and does a new HTTP request to one of your backend servers depending on the request domain.

Some examples:
https://httpd.apache.org/docs/current/m … l#examples
https://www.nginx.com/resources/admin-g … rse-proxy/
