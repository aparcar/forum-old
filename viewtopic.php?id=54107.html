probably there is post on this subject, but I can't find solution for my case.

So, here is problem:
I have openwrt routers Attitude Adjustment 12.09 and Barrier Breaker 14.07,

wan ports connected to local domain network, and routers is configured to use a domain dhcp server (win 2003), not static ip for wan interface, because it will change ip subnet quite often.

My question is, how to configure openwrt router to send its own hostname to domain dhcp server (win), in order to update dns record,  the goal is to access openwrt routers over its hostname?

For example on Ubuntu is

send host-name "<hostname>" ;
send fqdn.encoded off ;
send fqdn.server-update on ;

When you create a DHCP-Client interface using LuCi, there is a "Hostname to send when requesting DHCP" entry; that value goes to the "config hostname" entry of the corresponding interface in /etc/config/network.
