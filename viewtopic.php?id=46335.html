monty wrote:

I just don't understand why Auto DHCP doesn't work like on any other device.

The default dhcp on Openwrt's WAN port should work when connected to a LAN port on the other router (acting as DHCP server).

First guess, why it does not work in the basic stacked NAT setup: you have 192.168.1.x as the LAN address space on both routers. I have not used dd-wrt, but I think that also it uses 192.168.1.x as the default LAN address space...

The Openwrt router uses it too, so the inner router has thus 192.168.1.x both on the WAN side (dd-wrt's LAN) and its own LAN side. Routing logic gets screwed.

You might make sure that the routers use different LAN address space, so that the inner router get routing logic in order. Setup a fixed LAN address of 192.168.2.1 for the other router and thus 192.168.2.x as the LAN address space.
