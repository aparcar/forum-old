Hi,

this all depends highly which version of openwrt you had installed in the first place. If  the versions are too far away from each other,  the packages could have  changed quiet (especially IPv6 related ones, but also many others).

So, I you would like to restore your openwrt, you should use the same release as you had before, else, many bad things could happen.

I hope I translated your spanish correctly, with the help of translate.google.com.... this should be an english spoken sub-forum, so please try to write in english.

The problem of your opkg does not work probably is because you used an (now very old) trunk version... Those have no long lasting package repository.

If you didn't save them for yourself, they are probably lost forever:-(

problem solved.

steps:

1.- telnet 192.168.1.1

2.- passwd root

3.- cd /tm

4.- wget http://downloads.openwrt.org/backfire/1 … pgrade.bin

5.- mv openwrt-ar71xx-tl-wr741nd-v1-squashfs-sysupgrade.bin tplink.bin

6.- scp openwrt-ar71xx-tl-wr741nd-v1-squashfs-sysupgrade.bin root@192.168.1.1: /tmp

7.- mv openwrt-ar71xx-tl-wr741nd-v1-squashfs-sysupgrade.bin tplink.bin

8.- mtd -r write /tmp/tplink.bin firmware

waiting y ready

thanks
