Openwrt has strict kernel module checksum checks.
In practice you need to install kernel modules (kmod-*) from the same build run that built the main firmware.

Either
- use an official release so that you can install kernel modules from the official repo
- use a third-party firmware and use kernel modules from him.
- compile everything by yourself.

AFAIK, packages for trunk images are rebuilt every time the trunk image is rebuilt which could result in some packages not being installable due to kernel dependencies if your installed trunk image is behind one currently posted.

The tips would be to either:
1. Go back to stable (CC) version so you wouldn't experience kernel dependencies issues when installing packages in the future.
2. Upgrade to the current trunk and then immediately install the packages you need.
3. Make a custom image (hint: OpenWrt Image Builder) of trunk snapshot with packages you need built-in.
