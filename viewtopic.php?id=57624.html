See if 'empty' can be a replacement for 'expect' in your case:

empty     0.6.18b-1     empty is an utility that provides an interface to execute and/or interact with processes under pseudo-terminal sessions (PTYs). This tool is definitely useful in programming of shell scripts designed to communicate with interactive programs like telnet, ssh, ftp, etc. In some cases, empty can be the simplest replacement for TCL/expect or other similar programming tools.

http://empty.sourceforge.net/
http://www.linuxcertif.com/man/1/empty/ -> links to 'expect' in the "see also" section

Since you write LAN, perhaps you don't worry about security:

You can use netcat (nc):

server # nc -l -p 9999 > file.txt
(waiting for connection)

client # nc server 9999 < file.txt
(connecting and sending file)

The server listen thing you can put in a little loop:

#!/bin/sh

echo > /tmp/keeplistinforfile

while [Â -f /tmp/keeplistenforfile ]
do
  nc -l -p 9999 > file.txt
done

You may need to install "real" netcat (nc) on OpenWrt. The nc that comes with busybox is limited.
