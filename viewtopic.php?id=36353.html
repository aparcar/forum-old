The WAN interface showing an IP and the status showing not connected suggests something isn't configured properly.

You could reset to defaults and test but it may be unnecessary if you are confident your changes aren't causing the problem:

Reset router to defaults.
Connect PC to a LAN port and using DHCP it will receive a 192.168.1.x IP from the router.
Connect internet connection to the WAN port and using DHCP the router will receive an IP, DNS and gateway IP from whatever provides the internet connection.
Use putty to telnet 192.168.1.1 and run ping www.google.com.

If the problem still exists after the default configuration test you should switch your attention to the configuration of the internet connection device for example perhaps both devices are configured to use the same broadcast domain? You can check using the route command when you ssh into the OpenWRT router. If that is the problem you could configure the OpenWRT router to use 192.168.2.x instead of 192.168.1.x.

You have not mentioned the purpose of the OpenWRT router but if it is to work as a wired/wireless switch you could configure OpenWRT appropriately and connect the internet connection to a LAN port.
