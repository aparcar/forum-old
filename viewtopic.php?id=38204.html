Regarding permanent ipv6 addresses:
You can just fine assign permanent ipv6 addresses for the interfaces in Openwrt's netowrk interface config sections. But I would recommend you to use new trunk instead of Backfire 10.03.1, as the ipv6 support has improved quite a lot since original 10.03 was published over 2 years ago. By default, Backfire has ipv6 routing support disabled. (See http://wiki.openwrt.org/doc/howto/ipv6#routing and edit sysctl.conf)

That fc01::3135 looks a bit strange. It is from the local address space fc00:: , which addresses are not routable ipv6 addresses. They only work inside the local network. http://en.wikipedia.org/wiki/Unique_local_address

If your goal is only to provide internal traffic with no outside connectivity, using that address might be ok. But if you want to surt internet using ipv6, that address will not work. You will need a proper routable public ipv6 address.

All currently used normal global routable ipv6 addresses start with 20xx::  (e.g. 2001::). Have you been assigned a /64 address space by your ISP?
