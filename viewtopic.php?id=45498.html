Hello jenom;

The xmodem and ymodem transfer protocol date back to the days of CPM and prior (I'm dating myself).

I think you have to check and see if your boot loader supports xmodem or ymodem transfer.
There may be other issues such as memory mapping etc. that has to be considered also.

My version 4.66 of Tera Term supports xmodem and ymodem transfer protocols.
File-->Transfer-->x/y-modem-->send/receive.

I use this on a 64 bit windows 7 system and with the new IE10 I have had issues using it so be aware.
I have no problems with XP 32 bit and various flavours of Linux.

I have included some links that may help:
http://wiki.openwrt.org/doc/howto/generic.flashing
http://en.wikipedia.org/wiki/XMODEM
http://www.dd-wrt.ca/wiki/index.php/La_Fonera_Flashing
"NOTE: There are other methods of transfering the files over to the fonera, including directly over a serial connection (if you are using one) or using HTTP (the standard web protocol). For serial users, add "-m xmodem" to the end of your load commands, find the send file option in your terminal, select the right file, and it should to the rest."
http://wiki.openwrt.org/toh/arcadyan/arv752dpw?s[]=xmodem
http://wiki.openwrt.org/toh/astoria/arv752dpw22?s[]=xmodem

~john
