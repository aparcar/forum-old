[English]
I have a router with OpenWRT equipped to run a web server on which webcams can display and further information for an event. There is no internet at the venue. Thus the visitors can log in to the WLAN via their mobile phone but the time in which they are logged into the WLAN can not access the Internet. So my question.

Can one set the OpenWRT so that one can access the web server in the network but still use its own Internet from the mobile phone?

A further question would be whether there is a setting with which I am after one connected with the WLAN directly the side of Webserver gets opened. This would be easier for the visitors than then to enter the IP.

[German]
Ich habe einen Router mit OpenWRT ausgestattet um einen Webserver darauf laufen zu lassen welcher Webcams anzeigen kann und weitere Informationen für eine Veranstaltung. Am Veranstaltungsstandort ist aber kein Internet. Somit können die Besucher sich in das WLAN  über ihr Handy einloggen aber die Zeit in der sie im WLAN eingeloggt sind können sie nicht ins Internet. Daher meine Frage.

Kann man den OpenWRT so einstellen das man auf den Webserver im Netzwerk zugreifen kann aber trotzdem sein eigenes Internet vom Handy benutzt?

Eine weitere Frage wäre ob es eine Einstellung gibt womit ich nachdem man mit dem WLAN verbunden wurde direkt die Seite von Webserver geöffnet bekommt. Das wäre für die Besucher einfacher als dann die IP eintragen zu müssen.

1) This is device dependent. As client device has WiFi connected, but no internet, then it _MIGHT_ connect to web using mobile connection. So, not an openwrt issue.
However, device might be set to _NOT_ use mobile conn, when WiFi available ...
2) Device dependent, too. Certain Androids do _NOT_ allow redirect from 'minibrowser' for login to full browser functionality.
IoS does it, though.
But instead of typing IP in browser, there might be some shortcut defined, like "cameras.show/".

Hmm. I don't get the problem. What if there's a network that just doesn't provide DNS and Gateway through DHCP? Shouldn't that be enough?

Just a wild guess:

config 'dhcp'
        list 'dhcp_option' '3'
        list 'dhcp_option' '6'

http://lists.thekelleys.org.uk/pipermai … 00257.html

As for accessing the web server by host name: Just create an A record in a public zone pointing to the internal IP address of the web server should do the trick.

You could try DHCP option 160 to provide your web server as captive portal login. But I'm not sure about any client supporting this.
https://tools.ietf.org/html/rfc7710
