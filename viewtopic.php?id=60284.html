I have a TP-LINK WR841N V9 Wireless-N router, and accounts with the TunnelBear and PrivateTunnel public VPN services. I'm interested in setting up the WR841N to act as a client for either of these services, so that all devices on my home network — or better yet, only specific local devices that I designate — or better still, only certain local devices communicating with certain remote hosts — will have their traffic routed thru the public VPN service rather than the open Internet. What is the best way to achieve this functionality with the OpenWRT firmware installed?

Although my use-case is for public VPN services, this same configuration would be useful for those who wish to connect their home router as a client gateway for a private VPN, such as that provided by an employer or an educational institution.

You have some homework to do
This article is to set up a server, but is one of the best written.
http://wiki.openwrt.org/doc/howto/openv … rver-setup
Go to the upper right and type OpenVPN into the search box.

A quick look at your providers indicates that neither are set to use OpenWRT.  PrivateTunnel is OpenVPN, so if you install their config on a PC, you can grab the CONFIG.OPVN file and have a good start for converting this to the file to use with OpenWRT as a client.  It's unclear what VPN TunnelBear supports.

You need to install 2 files, OpenVPN-openSSL and luci-app-openvpn to have the gui.  Some dependencies will also be installed.
If you are on Windows, you will want WinSCP to make editing the Openvpn config file easier.

Separate the which user goes where from the equation until you have both systems working for all clients.  Note, you will probably want to turn one off while you are using the other.  It's a separate exercise that I have never done.
