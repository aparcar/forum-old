I see:

openwrt-15.05.1-ar71xx-generic-tl-wr741nd-v4-squashfs-factory.bin - used to upgrade from stock (stock = default tp-link firmware)
openwrt-15.05.1-ar71xx-generic-tl-wr741nd-v4-squashfs-sysupgrade.bin - used to upgrade ONLY if you already have OpenWRT installed

both of them have Luci.

Wiki say nothing if there is a difference or not between tl-wr741n and tl-wr741nd, usualy the only diference between n and nd is that nd has detachable antennas, but I have no clue what TP-Link is actually doing. So if you use any of those images you do it on your own risk, if you break your router I can only say that you have been warned.

-------

Before flashing OpenWRT you should understand that things can go wrong, like:
1) you flashed the wrong image and you bricked your router
2) if the pc and router are not connected to an ups and the electrical power is cut (totaly out of your control) you will probably brick the router

Also you should kinda know why you want OpenWRT. Some reasons can be:
1) security reasons mostly cause tp-link hasn't released a firmware upgrade for years for your router
2) your isp offers ipv6 but the stock firmware doesn't offer support for ipv6

The nice part regarding OpenWRT is that you can add/remove things based on your needs. In the 4MB flash routers case if you want luci + other things (that are not in the images you can download) the bad part is that you will kinda have to compile OpenWRT yourself to remove/add new things (when you remove stuff from it you don't actually free the space cause what you removed it's not deleted so even in remove case if you want/need the free space you have to compile it from sources).

If you download OpenWRT from the above link, and flash it, compared to stock firmware you will miss: wps, upnp and qos.

vico wrote:

I do not want to run an old, unmaintained release... So my options are run Barrier Breaker or no gui? Hmm...

Main question is what you understand by a maintained release.
If you use the image you can download from the above link you are stuck with an release that is missing some fixes (I'm talking about 15.05.1 here). Your router has 4MB flash and I doubt you can install any package on it (you'll have ~400kB left after flashing 15.05.1 and saving the router configurations).

If you want a relative newer version either you compile 15.05.1 from git and add luci (luci is not enabled by default) (you get a minor kernel version bump, several fixes compared to what you can download from the link above; not sure what they added in 741 case but in the case of the routers I use I have to enable the force use of reg domain (no clue why it's not enabled by default..., so i can use channels 12,13), remove the usb support (because without a mod there is no usb port, no clue why the usb part end up enabled by default in 15.05.1) and add iputils-traceroute6 (not really needed but well it doesn't hurt to be able to do ipv6 traceroute from the router) or you compile trunk or LEDE add Luci so you have a web interface (keep in mind that trunk and LEDE are not considered stable).

CVE-2016-5696 for example is not fixed even if you compile 15.05.1 from git, I know it's fixed in LEDE, no clue about OpenWRT trunk. They backported the fixes for CVE-2016-5696 in 3.18.41 (https://cdn.kernel.org/pub/linux/kernel … og-3.18.41 at the bottom) and if someone will bump the OpenWRT kernel from 3.18.36 to 3.18.41 it will be fixed (alternative is to make the patch yourself or see if the patch for kernel 3.18 from LEDE works (mostly check if the removed lines in the patch exist in the file that is beeing changed , you might or might not see some not so nice messages (some complains saying that someone hasn't done something ) while compiling if you just copy the patch from LEDE but the patch is applied )).
