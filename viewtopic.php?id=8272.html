It seems that the Wii does not play well with OpenWRT.  Testing the connection in the Wii dash results in an 'Error code: 51330'  It is able to detect the broadcasted SSID, but does not connect.

I have moved the Wii to the same room as the wrt, adjusted the tx speed to 1.5mbps (wl0_rate=3,) as recommended on the Wii support pages, disabled all macfiltering, disabled all security (WEP,WPA, etc.)  Still no dice.

I tried to run a tcpdump as suggested in #openwrt, but I see no traffic related to the wii at all when it is 'testing' the connection.

I have also tried using dhcp, and manually setting connection info.

Relevant Wii Techsupport pages:

http://www.nintendowifi.com/customersup … =0&y=0
http://www.nintendowifi.com/customersup … 54Gv5.html

This is on a WRT54G v2, running RC5.

Any help would be apperciated!

Looks like I solved my own problem, for others with the Wii and having troubles, try setting your wl0_gmode to 1.  This worked for me.
