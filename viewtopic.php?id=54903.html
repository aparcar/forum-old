Hi all,

I need to connect a laundry room to the internet and to be able to give it a public static IP. (Some of you might ask why, and the reason is that its required from the laundry billing software. It needs direct access using a public static IP + Port 5000 to 5010). The problem is, that I have no connection to the laundry room today and it will take a least 6 months before I get a wired connection there, so my best choice would be to put 4G/LTE capable router there. Unfortunately its not possible to get a mobile-subscription with public static IP, so for that I would like to make the laundry room network a part of my own network. I got a public static IP so I will share the 10 ports in the range from my own network. The traffic will be close to none.

1) I got an Asus RT-AC66U with VPN Server enabled
2) I buy a 4g LTE dongle, a flatrate mobile subscription and an OpenWRT compatible router for the laundry room.
3) I connect the OpenWRT router to my Asus RT-AC66U using the VPN client in OpenWRT as a permanent solution.
4) I wire the OpenWRT routers LAN side to the laundry system, and disable WIFI.
5) I give the laundry system at internal static IP address (from my local range)
6) I make a WAN- Virtual Server / Port Forwarding rule on my Asus RT-AC66U to the laundry systems IP address.

QUESTION:
1) Will it work: Hitting my public IP with fx port 5000 will give direct access to the laundry system?
2) Will OpenVPN be able to keep a connection open, even if the router in the laundry room is rebooted?
3) Any better ideas?
4) What 4g/LTE dongle and router will work well with OpenWRT?

LTE dongles are hit and miss, make sure you know which model number works with OpenWRT. You don't really need public static address if you use dyndns services.

1.When you hit your public IP port, you need to tunnel into your VPN server first. Then you access your 5000 port using your internal IP address.
2.OpenVPN connection will be reset if your laundry room router has been rebooted.
5.You might want to experiment with OpenVPN first ideally using easier to setup system like a plain Linux laptop, it seems your networking concepts are not that clear.

Hi dear,

1- OpenVPN server need Public IP
2- OpenVPN client don't need Public IP

With OpenVPN client you are able to redirect all port you want to the OpenVPN server, then the server need to redirect this port out of your WAN with Public IP :

OVPN Client ------------>>OVPN Server ------------->> WAN Public IP

In a router with OpenVPN support ( OPENWRT ) is the same concept, but you need to make firewall rules which redirect port or some port to an internal IP out through OpenVPN tunnel .

I don't tested OpenWRT with any mobile dongle and I can't give you an answer, but if it will not work, you can install two machine :
First for the dongle support and setup it as bridge...
Second with openwrt as router mode and openvpn client...
