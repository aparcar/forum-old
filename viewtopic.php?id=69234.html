I have a trouble with port on router. I setup Web server on ip 192.168.1.xx
DDNS: abc.xyz,port forwarding port 443 to 1234 is setup on my router (192.168.1.1) to access router's lucipage from Internet.

My router redirects to 192.168.1.xx when enter abc.xyz and to router's page abc.xyz:1234  Everything is OK when access from Internet.
but I can't access to my router page by entering abc.xyz:1234 from LAN.

How I can use abc.xyz:1234 from LAN?

Thanks for advice!

Sorry, I've realized, that abc.xyz is domain name, not WAN IP?
For the case of WAN IP it looks like https://wiki.openwrt.org/doc/uci/firewa â€¦ _same_host:

config redirect
    option src              lan
    option proto            tcp
    option src_dport        80
    option dest_port        3128
    option dest_ip          192.168.1.1

You should change numerical parameters and add option src_dip. For the case of domain name it should work also, if DNS resolves IP. If not, you should use proxy like squid etc. https://wiki.openwrt.org/doc/howto/proxy.overview

You can also avoid port forwarding and configure uhttpd: https://wiki.openwrt.org/doc/uci/uhttpd. By default it listens on all interfaces, so you should only open ports in firewall. For security you should also enable https and disable http.
