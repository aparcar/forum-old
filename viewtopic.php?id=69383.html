I fixed the issue of not getting IP when setting "option  input    'REJECT'"


Had the following entry for DNS queries:
config 'rule'
        option 'name' 'Allow DNS Queries'
        option 'src' 'work lan'
        option 'dest_port' '53'
        option 'proto' 'tcp udp'
        option 'target' 'ACCEPT'

option 'src' 'work lan'        // this line was the culprit, it was considering work and lan as one one zone.
Removed the whole entry as it was not needed.

I would still like to know when the   "option  input    'REJECT'"    is  used?

cheers
yudi
