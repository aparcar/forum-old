It took me a long time to find out that it is actually very easy to connect to a WPA Enterprise network wireless, for example eduroam, so I wanted to share it. I was put on the wrong foot by all the guides & howto's on configuring wpa_supplicant files. You dont need to do all that. You can do everything through your browser. (The only exception to that seems to be the case where you have to put in an anonymous identity. Then it is more complicated).

Here is what you do if you have a router that is able to run on openwrt (not sure about additional requirements regarding technical feasibility). I used a TP Link WR841ND myself, which is very cheap. For the method I describe hereunder I used a cable to connect my computer with router to do the set up. I connected the cable to a LAN port.

1. Install openwrt, see Installing Openwrt.
Method 1 is the most easy. With this method you just have to find out which file you need based on your router, see hardware list, and then log in to your router using a browser (using the standard ip adres for that router). YOu still make use of the original software on the router. Now select update firmware, where you select the appropriate openwrt file. After the update openwrt is installed and ready for use.


2. Remove wpad-mini and install wpad
Option A Use internet connection on your router. The most easy method is to do this through the web interface (Luci) that comes with the standard Openwrt package, but you need an internet connection to your router for this first. I used my mobile phone to create a wifi, and connect the router to that by setting up a wifi network.

Go to: Network -> wifi. Here you can set up the wifi connect to your phone. I ticked the "wwan" option, but I think this is router specific.

Now you go to System -> Software and update package list. Remove wpad-mini and install wpad from Available packages.

In case it doesnt work: I have noticed the web address with the packages is sometimes off line. Under System -> Software, Configuration you can find the address of the website. Copy past in your browser to see if that caused the problem.

Option B  Use Putty (&WinSCP). Download the wpad package on your computer and put in on router using SCP (use Putty, or WinSCP, which is very easy to use: I like the visual aspect). You just need to log in through "root@192.168.1.1" for example. Put the .ipk file in the folder /tmp. Then use Putty to do:
cd /tmp
opkg remove wpad-mini
opkg install wpad-xxxx.ipk

If you have problems with the exact commands, see this very old guide


Set up wifi client connection with the enterprise network
See under 2.A how to set up wifi. Just you have more options now for the enterprise wifi (if the extra options dont show up immediately, then you probably have to reboot first to make wpad active). The provider should be able to give you the required information about the protocols.

Add Access Point
Assuming you want to have wifi access from your home devices you just add a "Master" wifi, also under Network -> Wifi. Tick the "lan" connection, and set security setting.

Now you should be connected to wifi and have your router function as an Access Point. I have no clue about the standard security settings, so I have no clue how safe you are now... but at least you should be connected.
