Hi,

I'm trying to turn my WRT54G into a (modest) VPN server for a Windows XP box to connect to it. Every connection attempt fails with the "error 737 - loopback detected" message.

This was an issue well known by l2tpd developers (see for example [1] and [2]). In theory, this had been fixed by a patch coded by Damion de Soto, which is integrated in l2tpd-0.70pre-4. However, I keep running into this error when running it on my WRT54G. I have also tried using the Debian Project's sources and patch to build my own l2tpd package with the White Russian SDK with no more luck.

I have also tried the connect-delay workaround described in [3]. This barely changed things, making maybe one out of ten connection attempts successful, then requiring a reboot to get a new one again.

What makes me so persistent is that I can connect the same client, in the same network, using the exact same version and configuration of l2tpd, on a Pentium II box.

Has anybody managed to turn their router into a VPN server for NT clients that way, or instead run into this problem ?
