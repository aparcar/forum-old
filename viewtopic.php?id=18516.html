I'm running 8.09 RC1 on a WRT54GL (or a v4 WRT54G, can't remember which one I have where... same hardware anyway) with vnstat on it, and I want to be able to maintain a persistent database of bandwidth usage.  It currently stores the vnstat database in /var/lib/vnstat, which is on the ram.  I'm not sure moving the file to a position in flash is a good idea: with updates every 5 minutes, I'd be somewhat concerned about flash wear.

One thing I'm considering is a cron job on my desktop to pull the file down via scp and store it locally.  It seems the vnstat init script has the capability to download the file from a webserver, so I could probably just put it somewhere accessible by apache.  When it comes back up, the data would be restored.

Can anyone think of a simpler way to address this problem?
