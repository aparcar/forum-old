Hello OpenWRT community! I've been a DD-WRT user for sometime and I'm in general happy with the performance I've had on that router (TP-Link WR841N). What attracts me to OpenWRT is the fact that it allows a lot more customization.

In selecting a good router to flash with OpenWRT, I've narrowed down my choice to 2 routers
TP-Link WR1043N (64/8 RAM/flash in MB, v2 and above)
Asus RT-N14U (64/16 RAM/flash in MB)
(feel free to drop more suggestions that have a good quality/price ratio, I haven't bought any router yet, still doing research)

The available flash on each router, plus the fact the Asus box is roughly 20€ cheaper than the TP-Link leave me inclined to the Asus box. In spite of the documentation for the TP-Link WR1043N being one of the most complete hardware-specific pages I've found in this forum.

As it seems, the Asus RT-N14U supports the chaos calmer (15.05) image. Even-though this router's page (still) points to the 14.07 image:
https://downloads.openwrt.org/barrier_b … pgrade.bin

The SoC on this router is the Ralink MT7620N. I would like to confirm that the 15.05 image for this router is located here:
https://downloads.openwrt.org/chaos_cal … pgrade.bin
Note that the SoC in the 15.05 URL is referred to as mt7620, without the "n".

A related question is about the naming of these images. For an upgrade from OEM firmware, I should be using a "...-factory.bin", but I cannot find such an image for this router. Is it ok to upgrade from an OEM image with a "...-sysupgrade.bin" image?

Now for an unrelated question...
Many routers have a USB port, including the two I've mentioned. Is it possible to expand the space available for OpenWRT packages by using USB storage?
