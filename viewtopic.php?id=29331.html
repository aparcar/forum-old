Have no fear. I am sitting right next to a WRT54G v3.1 that I flashed with the latest 10.03 Backfire release with no problems. On second thought, some fear is good because it will make sure you are paying close attention.

Following the instructions here can be a challenge, I have found. The OpenWrt project has grown to support many platforms over time (although I've only recently registered on the forum here I've been using OpenWrt since the transition from WhiteRussian to Kamikaze) and keeping the wiki up to date is daunting. I suspect most people (myself included) get their routers up and running and then don't come back to share their experience. I'm trying to fix that lapse right now.

Here is what I did to get my router flashed:

(1) Visit the Linksys table of hardware to confirm what model you have (some WRT models will not work). The current table lives at http://wiki.openwrt.org/toh/start#linksys , but the links on that page for the devices are broken. The old wiki still has good information for this model: http://wiki.openwrt.org/oldwiki/openwrt … sys/wrt54g (Sounds like maybe you did this already since you know you have a v3 WRT54G.)

(2) The key to knowing what image you use is in the table of hardware (TOH). Note, in particular, that your model can use either of two "targets": brcm-2.4 or brcm47xx (from the current TOH). Now you can navigate the downloads directory:

  (2a) start here for the backfire 10.03 release: http://downloads.openwrt.org/backfire/10.03/
  (2b) you can use either brcm-2.4 or brcm47xx; I use brcm-2.4, so I know it works. Try that first.
  (2c) you need a .bin file for a first flash (see here for why, and why you only need .trx after the first flash: http://wiki.openwrt.org/doc/faq/linksys … e.with.mtd ). For your model, download the "openwrt-wrt54g-squashfs.bin". All of the other .bin files are for other models.

(3) check the md5 signature of the file you've got with the md5sums listed in the downloads directory to be sure nothing got corrupted during download (this also makes doubly sure you're using the right image before you flash)

(4) if you haven't already done so, connect your computer *directly* to the router using an *ethernet cable* (no hubs or routers or other points of failure in between, thank you)

(5) visit the router's firmware upgrade page and load in the .bin image you downloaded and double-checked

(6) submit that form and do not interrupt the flashing process (keep your pets and toddlers away from those power strips and ethernet cables and cross your fingers for no power outages)

(7) welcome to OpenWrt! I suggest continuing from here: http://wiki.openwrt.org/doc/howto/basic.config

Other helpful links:
http://wiki.openwrt.org/doc/start#uci.configuration (skim all of the sections and go in depth on configurations relevant to you)
http://wiki.openwrt.org/doc/start
http://wiki.openwrt.org/doc/faq

-Matt

P.S. I wrote this out from (recent) memory. Please let me know if you find any missing details and I'll update.

Great to hear it worked for you.  There are instructions for configuring it as a bridge on the page listing documentation above (under "Recipes"):

  http://wiki.openwrt.org/doc/recipes/bridgedap

This "recipe" appears to be for a specific hardware configuration, so you'll want to read the documentation on the network and wireless config files and be sure you understand what all of the lines in the file are doing before you apply them to your router. You will likely need to modify the settings to fit your network. Although none of these steps have the potential to actually brick the router, it could become difficult to get back into the shell if you don't remember the IP address.  Be sure you keep track of what you use as the static IP address in the lan interface (and your root password!) and you should always be able to get back in.

Post again if you have any questions!
