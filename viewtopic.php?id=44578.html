I compiled Altitude Adjustment (r36842) recently, and found that some of my firewall rules was missing from the generated rules. Log of iptable (gotten by executing "FW_TRACE=1 fw3 reload 2>/tmp/iptables.log") shows something like this for both ipv4 and ipv6:
fw3_ipt_rule_append(): Can't find match 'state'
The needed kernel module is there, but seems the corresponding iptable module was missing. I checked the menuconfig and no module for 'state' match was listed (for trunk version, it was built-in.) So my question is how to add it? I need it for AA since recent trunk version has wireless stability issue for my WNDR3700 v1. Thanks.

Regards

Post #2
CyrusFF
6 Jun 2013, 10:47

The state match is deprecated for a while now, you should try to replace "-m state --state ..." with "-m conntrack --ctstate ..."
