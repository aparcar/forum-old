I saw there is a monitor mode that the wifi can be put in so it does not transmit. Maybe that can be used to scan for a special package.

Anybody knows if something like this is already created?

Post #2
nozombian
28 May 2014, 09:55

I don't know about special packet, but quite easy solution came to my mind. If you have smart phone, you can turn it into ap (tethering). Then you can scan on openwrt for nearby wireless networks in a loop using

iwinfo wlan0 scanning | grep "yourphonessid"

If openwrt will see your network, it can enable wifi. Scanning should work in monitor mode too.

Given most Wifi devices don't support or use dynamic transmission power, a client "looking" for a hidden SSID does not have any effect on any other stations in range. Not only that, a client "looking" for a hidden SSID doesn't behave any differently from any other client "looking" for a non-hidden SSID (unless it's a mobile phone with various non-standard power saving hacks). There would still be no reason whatsoever for any other clients to "bump up their transmission power", ignoring the fact the vast majority always transmit at maximum power all the time anyway.

No, the point is the only "wifi radiation" the AP is generating when no clients are connected is the beacon packets that broadcast the SSID. Turn off said beacons and it therefore has nothing to transmit. A wifi device that has no data to transmit generates no RF radiation.

Dunno about what document you read but in practice most clients will probe for *any* known SSID hidden or not. Most clients don't even have an attribute to store whether an SSID in their list is "hidden" or not. Access points will only respond to probe requests for *their* SSID, and as I say clients will probe for *all* known/saved SSIDs so they would have responded anyway.

Although perhaps there's an alternative scenario, where the known SSID is in range and broadcasting, where a client won't probe for it if it already sees it in a passive scan. In that case what you say might be true, but the client will still broadcast probes for every other SSID it's ever connected to that is not in range.

And I totally agree about the radiation thing, I'm just entertaining the OP who obviously has a different opinion.
