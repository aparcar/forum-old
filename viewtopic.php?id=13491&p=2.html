Update: Same error when I try vmlinux.gz instead of vmlinux.lzma.  I had a look in the make output and couldn't see that an decompressor code is added after vmlinux is compressed with gzip or lzma, so I assumed that the bootloader natively supports gzipped images, but not lzma.

I'm still puzzled by the base address of 0x80080000, which is not contained in the fis entry for vmlinux*.

Update: It seems like the Redboot unzipper breaks due to a missing timestamp and filename in vmlinux.gz. This was introduced with svn revision 46361 (aka git commit b5bb9272ee773d30c1fde2da17c087618264ab6f) in July 2015.
I've removed the -n option for gzip and now vmlinux.gz from HEAD boots successfully:
