Yesterday I finally successfully compiled and installed OpenWrt on my new router (MikroTik RouterBOARD 951G 2HnD) with help of wiki documentation (http://wiki.openwrt.org/toh/mikrotik/rb951g_2hnd), however it looks like it contains small errors. An image for first boot, openwrt-ar71xx-nand-vmlinux-initramfs.elf, is referred, but the doc does not describe, how/where to get it. After some research, my solution (which is working) is:

  • When configuring the build, I added "ramfs" to target images in configuration (make menuconfig) (the documentation states to select "tar.gz" only).

  • Used built "openwrt-ar71xx-mikrotik-vmlinux-initramfs.elf" instead of referred unknown "openwrt-ar71xx-nand-vmlinux-initramfs.elf" for the first boot. (By "used" I mean copy to /tftp/ and use it as argument of --dhcp-boot.)

My questions are:

  1. Is this correct way? I'm just green beginner with OpenWrt... Could someone with more experience review these steps?

  2. Who/where to ask to update the wiki?

Edit:
I forgot one more thing - my router (Mikrotik RB951G 2HnD) is not listed in hardware list (http://wiki.openwrt.org/toh/start#mikrotik), but it's page exists (http://wiki.openwrt.org/toh/mikrotik/rb951g_2hnd). Is there reason for that? Should it be updated?
