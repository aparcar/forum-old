Hi,
the attack on WPA/WPA2 is done offline after the attacker captures the handshake packets exchanged between the router and the station.
The process of capturing the handshake can be absolutely passive.For more info try google.
About the secound question Ii believe you can check system log for failed authentication attempts.
If you want to be secure enough use long keys with alfa-numeric,lower and uppercase combined with some special characters like +-!@... etc.

+1 A patient hacker listen (undetectable) all day long for a single handshake packet from a genuine connection. An impatient one may inject a (detectable) deauthentication packet to force the handshake in a genuine connection.

Then offline check (undetectable) the packet against a publicity available dictionary. You may offline check your key too against same dictionary. Please DON'T use a online service!

Below this point you have a weak key, can stop reading now.

Next step is to bypass the MAC filter: Use the MAC of the router (can't filter itself but is detectable) or use a captured one at times (noticeable) when not in use. At times when in use may inject a (detectable) deauthentication packet to (noticeable) disconnect a genuine MAC owner.
If you have a strict connection schedule can use crontab to follow it turning on and off the radio.

Next step is to obtain a (detectable) DHCP lease. Really? Use a captured IP/MAC instead without ask for a lease, but can't avoid the (detectable) ARP registration.

If you read so far then you are already owned.
