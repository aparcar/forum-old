well, there's nothing wrong with using tinyproxy, it blocks websites just as well as a regular proxy server, I just mean that if you intend to block a LARGE amount of websites (1000+ would be a very rough estimate depending on what else is running on your router/how busy your network is), you may run out of cpu cycles and take a preformance hit.

The only thing that limits the wrt is it's processor speed and it's memory.

*hits himself on head for not remembering* lol... duh... you could also block the ip of the websites with iptables by using the builtin firewall

edit /etc/config/firewall
drop:src=<ip> (not too sure of you can use a range of IPs. you could probably do IP/netmask but not sure)
reboot

Please note that I had been looking into using my router to act like peer guardian, but i discovered that there were too many ips to effectivley block using iptables (or a proxy server) due to the CPU. If you have less than the number of IPs peerguardian blocks, you should be fine =)

So if (for whatever reason) you wanted to block all udp traffic and reply with an icmp unreachable message, you could do that with iptables, or if you wanted to block all packets coming from or going to a certain ip, you could do that using iptables.
