Luci interface reflect technical details as they appear in configuration files. It can be not too friendly for users but it precisely reflect what happens on low level. When you understand how things work you understand why web interface looks as it looks. That's what I personally love because it does not try to simplify anything and give it as it is. Its for professionals, not for newbies. I guess if you try to configure a professional appliance (for example cisco router) you also will be confused because it requires some learning. Home-grade systems have simple and easy to use interface but they also stuck to pre-designed scenario or two and do not allow deep customization because it would make everything too hard for user. Also if someone try to make professional appliance user-friendly it will probably break its initial purpose and make professionals hate it because of its stupidity.
I have nothing against alternative simplified web UI for ordinary users but it must be optional and it must not break  internal flexibility. But for the moment i'd suggest move to another firmware such as dd-wrt because its closer to your demands.

Hi eival,

while I am not sure the displayed attitude will help your cause, let me address your sqm-scripts gripe. Sqm does not care what interface you instantiate it on, putting it onto an internally LAN-facing port is just as valid as putting it onto an externally WAN-facing port, so there literally is no default value that will work for everybody. Add to that the fact that different router models end up using different ethernet interfaces with or without VLANs (the .2 in your eth0.2 example sqm has no easy way of detecting the actual WAN interface. BUT you need to manually configure SQM anyways with meaningful and well researched numbers (for bandwidths and overhead). So if selecting the desired interface seems to burdensome, you might simply want to deinstall sqm-scripts and luci-app-sqm as selecting the correct interface is most likely the easiest part in setting it up. (But since SQM-scripts is not enabled after installation and has no real run-time or memory cost if inactive you can simply leave things as they are...)
