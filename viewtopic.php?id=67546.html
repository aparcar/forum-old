Hello to everyone!

I have purchased a RavPower RV-WD02 and started to use OpenWRT on it for a while now. And I've been using it as a wireless client travel router. So, where I go, I connect my router to any wifi near and connect my devices to my wifi. The config was made using LuCI.

Even if I want to use it as a wireless client, I need to have, everytime, a network cable (a RJ45 cable) to configure it to use the Wifi I want the router to connect to. To explain this, I need to use a example.

Let's say that I use my WD02 to connect to my HomeWIFI, it will then be configured to every time the router is turned on, it will try to connect to my HomeWIFI. It works flawlessly and I am able no navigate using it. After that I go to my work. There, I need to reconfigure my travel router to connect to a new WiFi. It is at this point that the problem arrives.
Because the travel router is configured to use my HomeWIFI, when it boots, it tries to connect to my HomeWIFI and  fails. If the router is not able to connect, it won't turn on it's wifi. So I am not able no reconfigure it, only If I use a cable, connect to it's RJ45 and access LuCI again (or CLI).

So, here's my question: is there any way to force the WD02 to turn ON the wireless radio even if the router can't connect to a WiFi?

I hope I was able to explain it.

Thank's for reading and, as we Brazilians always say: sorry for my bad English.

Yes, you can have a script running on your router which disables the STA interface if it can't connect (and then you can connect to AP). There're multitudes of those in this forum.

Not sure if the router you have has a slider (like D-Link DIR 505 does) for different modes, but if it did, you could program each setting of the slider, like position 1 would be home, position 2 work and position 3 would boot into just AP with STA disabled so you could configure it thru Luci.
