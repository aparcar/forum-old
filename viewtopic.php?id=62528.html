I have a GL-AR150 mini router that I have installed OpenWRT on, as well as OpenVPN to use as a client to connect to my provider. I was hoping to use this device to:

* Connect to wifi in a public location, especially in places like hotels that only allow one connection
* Re-share that connection with my devices
* Be able to connect the router as an OpenVPN client.

I've done all of this and it wasn't terribly hard. I am using the Luci web interface. (FWIW I could not make any sense of the luci openvpn app, I couldn't get anything to start or find any logs so I bailed on that and just use the CLI)

OpenVPN is running and connects to my provider on startup.

Everything is working as intended thus far I am just trying to understand how some of it works!


When OpenVPN is connected all of my traffic is directed through that connection. When I am not connected, traffic routes through whatever the router WAN is connected to.

What is happening to make the traffic re-direct through the OpenVPN tunnel? I'm trying to understand because ideally I'd like to modify this so that if the OpenVPN connection drops, I cannot access the internet.

Your VPN client changes the routing table after establishing a new connection. You can have a look to the routes on the web interface, and compare before / after.

If you want to block all internet when the VPN is down, I guess you should delete your default route on the WAN / WWAN interface, and create one specific to your VPN provider.
