Hi,

I'm running arokh's build of OpenWRT on my Netgear WNDR3700.
I've got regular WAN + a he.net tunnel running.

I can't find the best solution to force route certain sites to ipv4. For example, Youtube currently runs through ipv6, which results in slow buffering. Is there a solution for that? How to route Youtube through ipv4, so it wouldn't use the he.net tunnel.

I am sure there are many ways one can do this, but one is simply to reject the IPv6 traffic, and it should fall back to IPv4.

Example /etc/config/firewall

config rule
        option name 'Google-Ireland-IPv6'
        option proto 'all'
        option target 'REJECT'
        option dest_ip '2a00:1450::/32'
        option src '*'
        option dest '*'
        option family 'ipv6'
