I'm trying to set up a batman-adv mesh network on Bullet M2 hardware.  I've been able to build OpenWRT, load it on one unit, and configure it, but I'm running into a strange problem trying to replicate the setup on additional units... after loading the configuration file saved from the original unit (either onto a new unit or the same one), OpenWRT thinks that there are two radios instead of the one physically present in a Bullet, neither one of which works.  For example, 

# wifi up
PHY for wifi device radio0 not found
'radio1' is disabled
PHY for wifi device radio0 not found
'radio1' is disabled

I can't find anything in the configuration file relating to radio1, so I have no idea where that device is coming from.

Post #2
jow
14 Jan 2012, 00:04

The mac adresses do not match. Replace "option macaddr ..." with "option phy phy0" in your predeployed configs and it will work.
