Yes, for example using a TP LINK MR 3220 V2 with OpenWRT.

Make a c program using tun/tap driver, create 2 virtual network interfaces with that driver.
poll two virtual interfaces, intercept ip datagrams you want and resend data to another interface.
use brctl to bridge eth0 with tun0, and eth1 with tun1 (eth0 and eth1 connected to the bacnet/ip network).
use wlan0 to access tcp/ip network via wifi and post data.
