I have installed Kamikazi 7.09 with webif on an NSLU2, accessed from a Vista PC through Putty.  I want a limited installation and have everything installed on the flash--PHP5, dropbear, httpd, gmediaserver, and a number of php scripts.  The php scripts build web pages which I want to be semi-private.  I have their output in the /www directory. 

I want to access the files cross the local lan using another php program running on a Windows PC.  I can access them using the PHP command file_get_contents($Geturl), where $Geturl is in the form "http://id:pw@10.0.0.210/filename.xml"--successfully when id="root" and pw is my root password.  I don't want to have this combination in plain text on my PC, so I have tried to create a user and use that password.  This is not working for me so far.

On the NSLU2 I created a user, say "joe".  The command, adduser, didn't work, so I edited /etc/passwd, put in "joe::8000:8000:Joe:/home/joe:/bin/ash".  I ran "passwd joe" and supplied a password.  I modified /etc/httpd.conf and put in a line "/filename.xml:joe:joe's_pw".  Now when I run my PC php program with the id and password changed for joe, I get a "401 Unauthorized" message.  With "root" and "rootpw" it works.

So I went back to the NSLU2 and tried "login joe"--this returns "login failed".  When I just try "login", it immediately says "login failed" without prompting for the user name.  My experience with linux is modest.  I may be missing something simple, but how do I log into a non-root user?
