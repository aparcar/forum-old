Hi,

I've just setup a WNDR3700v2 with a recent trunk snapshot: BARRIER BREAKER (Bleeding Edge, r41679) to replace my Buffalo, which is still running Backfire I think.

However, i'm having a really odd problem with vlan's.

Unless, I have an untagged port, the vlan doesn't seem to appear on the tagged ports...

eg:

If I have:

config switch_vlan
        option device 'switch0'
        option vlan 4
        option ports '0t 1t 5t'

Then vlan4 doesn't work with the switch connected to port 0, or the server connected to port 1.

If I change it to this:

config switch_vlan
        option device 'switch0'
        option vlan 4
        option ports '0t 1t 2 5t'

Then vlan4 works with the switch connected to port 0, and the server connected to port 1 - as well as untagged on port 2.

This is my full switch config:

config switch
    option name 'switch0'
    option reset 1
    option enable_vlan '1'
    option blinkrate '2'

config switch_vlan
    option device 'switch0'
    option vlan 2
    option ports '0t 1t 5t'

config switch_vlan
        option device 'switch0'
        option vlan 3
        option ports '0t 1t 5t'

config switch_vlan
        option device 'switch0'
        option vlan 4
        option ports '0t 1t 2 5t'

config switch_vlan
        option device 'switch0'
        option vlan 5
        option ports '0t 5t'

config switch_vlan
        option device 'switch0'
        option vlan 6
        option ports '0t 5t'

config switch_vlan
        option device 'switch0'
        option vlan 7
        option ports '0t 1t 3 5t'

config switch_vlan
        option device 'switch0'
        option vlan 8
        option ports '0t 1t 3 5t'

config switch_vlan
        option device 'switch0'
        option vlan 9
        option ports '0t 5t'

Interestingly, vlan 2 seems to work fine, but 4, 7 & 8 show the symptom above - so i've had to add port 3 to them both (there is nothing plugged into port 3).

(i've not yet had chance to try vlans 3, 5 or 9)

Any ideas?

Thanks,

Ian
