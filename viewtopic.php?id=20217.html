Hi, I read on the wiki that we can upgrade openwrt with this command :

mtd -r write firmware.trx linux

root@OpenWrt:/etc# cat /proc/mtd
dev:    size   erasesize  name
mtd0: 00040000 00010000 "cfe"
mtd1: 007b0000 00010000 "linux"
mtd2: 00355c00 00010000 "rootfs"
mtd3: 00010000 00010000 "nvram"
mtd4: 003e0000 00010000 "rootfs_data"



Doesn't this write the firmware.trx file to the linux "partition"? I want to flash a 4.4 MiB firmware, which is larger than the size of this partition, won't this be a problem? Or is it that mtd only need the adress of the beginning of the memory space and will flash over the other partitions too, in my case : linux, rootfs, nvram, rootfs_data

Hi.

Exactly. Linux is the start address, following partitions will be overwritten. Partition map is recalculated on next boot.
