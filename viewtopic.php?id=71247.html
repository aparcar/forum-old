I've had this happen a few times now, where my Spectrum cable modem doesn't seem to be renewing the DHCP lease.  I'm forced to restart the cable modem.

I'm going to open a case with Spectrum to check the logs on their side, but just wonder if anyone else has seen a similar issue and resolved it already?

config interface 'wan'
        option ifname 'eth0.2'
        option proto 'dhcp'
        option hostname 'router'
        option broadcast '1'
        option delegate '0'

FYI, I've turned off all IPV6 support.

If you are running OpenWRT on your Spectrum cable modem, then ensure that the firewall configuration allows incoming DHCP Renew responses from the DHCP server.

You'd find these configs from /etc/config/firewall.

If you are running OpenWRT somewhere else, and the Spectrum cable modem is a DHCP client to this server, then it's possible that the Spectrum cable modem's DHCP client is malfunctioning, in which case you'd need to contact their support to fix it.
