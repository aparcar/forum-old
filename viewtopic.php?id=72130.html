They're currently am running David's snapshot 2017-08-20 kernel 4.9.44  WiFi driver 10.3.4.0-20170810.

Sadly the diminished wireless performance is observed just same with the 2 latest Linksys firmwares I've tested as well.  I hadn't tried DDWRT for the very reason you mention...that I'm hearing that DDWRT has been slow to adopt the much improved wifi driver that LEDE has already implemented.

In every scenario each WRT1900 5GHz WLAN is 80MHz on separated channels 36, 161 and function in  VHT-MCS 8-9.

▬▬ Configurations Tested ▬▬
    • ACv2 as router & ACSv1 as AP
    • ACSv1 as router & ACv2 as AP
    • Out of box OEM FW on both

    • Updated OEM FW on both
    • ACv2 as router w/ OEM FW & David's LEDE on the ACSv1 as AP (Kernel version  4.9.44  WiFi driver 10.3.4.0-20170810)
    • David's LEDE on both

▬▬ Observed Throughputs ▬▬
WLAN to WAN on either router or AP via speedtest.net = 240mbps ~ 30MB/s
Router LAN to AP LAN = 100+ MB/s
Router LAN to Router attached storage = 90+ MB/s
Router WLAN to Router attached storage = 40MB/s
Router WLAN to Router WLAN = 14-15 MB/s
AP WLAN to AP WLAN = 14-15 MB/s
Stats -> Graphs -> CPU hasn't exceeded 6% for either WRT unit

I've tried numerous client PCs and Macs w/ Intel AC and Broadcom AC adapters.
