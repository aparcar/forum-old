Does anyone know how to recover from the following scenario without re-flashing? Is this just a known issue with yaffs2?

After getting errors about running out of (flash) space, I (expectedly) see output like the following from 'df':

# df -h
Filesystem                Size      Used Available Use% Mounted on
rootfs                   60.0M     58.7M      1.3M  98% /
/dev/root                60.0M     58.7M      1.3M  98% /
tmpfs                    14.3M    388.0K     14.0M   3% /tmp
tmpfs                   512.0K         0    512.0K   0% /dev

However, despite removing some files that I had stored on the router, I still see "Use" at "98%", even though 'du' suggests otherwise:

# du -sh /
11.1M    /

This has occurred to me several times, but I'm not sure what the precise repro steps are. Most recently this occurred when I tried installing a package with opkg, and ran out of space during the install. I removed a large file I had on the router, and could see that "/" was back down to "21%" usage, but after rebooting the router, it was back up to "98%" even though 'du -sh /' showed only 11.1M.

The only way I've been able to recover from this in the past is by re-flashing, and I'd rather not have to do that if I can avoid it.

This post looks related, althout it's quite old: https://forum.openwrt.org/viewtopic.php?id=13171

OpenWrt version: ATTITUDE ADJUSTMENT (Attitude Adjustment, r36088)
File system type: yaffs2

Hardware: RouterBoard 750UP
