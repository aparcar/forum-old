Hi,

i've attached a few scripts to the reset button on my router, which should get executed on the basis of how long i press the button. But this doesn't work. Only the first script, which is configured to be executed when i press the button between 0 and 3 seconds, gets executed all the time. The other scripts doesn't get executed.
So i inserted the line "logger seen is "$SEEN"" in the /etc/hotplug.d/button/00-button script which gets executed when a button press oder release event is triggered to see what's the value of $SEEN. The value of $SEEN is all the time 0, so thats the cause why all the time only script 1 is executed.

Has anyone an idea why $SEEN gets set incorrect or maybe doens't get set? Or where $SEEN gets set or rather from which program or script?

My button config in /etc/config/system looks like this:

config 'button'
        option 'button' 'reset'
        option 'action' 'released'
        option 'handler' '/usr/local/script1''
        option 'min' '0'
        option 'max' '2'

config 'button'
        option 'button' 'reset'
        option 'action' 'released'
        option 'handler' '/usr/local/script2'
        option 'min' '4'
        option 'max' '7'

config 'button'
        option 'button' 'reset'
        option 'action' 'released'
        option 'handler' '/usr/local/script3'
        option 'min' '12'
        option 'max' '30'

The /etc/hotplug.d/button/00-button file/script is the standard file/script, which was there from installation on. I've nothing changed within the file or any other hotplug stuff.

I'm running openwrt backfire branch r31074 with kernel 2.6.32.33 on a fonera+ (aka fon2201), which is AR231 based.
I hope this helps.
