Hello,

I have a TL-WR1043N, with OpenWRT 12.09.
I am trying to have access my ftp in safe way, using TLS.
Already I inserted in the archive etc/vsftpd.conf the following lines.

##### SECURITY
ssl_enable=YES
allow_anon_ssl=NO
force_local_data_ssl=NO
force_local_logins_ssl=NO
ssl_tlsv1=YES
ssl_sslv2=NO
ssl_sslv3=NO
ssl_ciphers=HIGH
#nopriv_user=ftpsecure
rsa_cert_file=/etc/ssl/private/vsftpd_cert.pem
rsa_private_key_file=/etc/ssl/private/vsftpd_privkey.pem
debug_ssl=YES

However, when I give to the command in Putty “/etc/init.d/vsftpd restart”, I receive the following message:

ssl_enable is set but SSL support not compiled in

I read in forums that I would have to install package “vsftpd-tls”. But it is not available….

Somebody knows as must proceed? It would like to keep the serving ftp for some few users, jailed, making possible access out of LAN, in safe way…

michelmetran wrote:

I read in forums that I would have to install package “vsftpd-tls”. But it is not available….

It is available for Barrier Breaker/14.07 and later.  Do you want to spend a lot of time messing around to learn how to build an isolated package for an old release like AA/12.09?

If not, please consider installing a newer stable release like the latest, CC/15.05.1.  Then this package is installable.  HTH.
