Hello,

for range-maximization on my home-network I am running one WiFi-Router (A) with openWRT and two other routers (B, C) as simple AP for wireless roaming, connected via LAN-cable with the router A.
Maybe I did something really wrong, but I don't know what.
The problem is that throughput via the APs (B) and (C) is really poor, and e.g. my tablet I have to reconnect every day to work. Without doing so after a day there's absolutely no throughput. The wireless channels are wide away (1, 6 and 11), the ssid and the wifi-password are all the same. I never got error-messages, only one time with connectbot (android-app) I got "no route to host".
I try to describe my network:
On all devices latest trunk ar71xx r38303 is running, but the problem occurs a long time. On router (A) a "normal" configuration is running. It is working as router after a DSL-modem and is handling the connection. dnsmasq is running, most wireless-devices are installed as devices with known MAC and therefore statip IP-address. On the other routers (B) and (C) I've stopped and disabled dnsmasq and firewall, so th√§t they are working as simple access-points. Most devices, especially my Androids, are getting a static IP and connected with it.

/etc/config/network on (A), a Buffalo Airstation WZR-HP-AG300H:

cat /etc/network
config 'interface' 'lan'
        option 'type' 'bridge'
        option 'proto' 'static'
        option 'netmask' '255.255.255.0'
        option 'broadcast' '192.168.1.255'
        option 'ipaddr' '192.168.1.1'
        option '_orig_ifname' 'eth0 wlan0 wlan1'
        option '_orig_bridge' 'true'
        option 'ifname' 'eth0'

/etc/config/network on (A) and (B), two D-Link DIR-825:

cat /etc/network
config interface 'lan'
        option ifname 'eth0.1'
        option type 'bridge'
        option proto 'static'
        option netmask '255.255.255.0'
        option ip6assign '60'
        option ipaddr '192.168.1.17' # (and .18 on the second unit)
        option 'broadcast' '192.168.1.255'
        option gateway 192.168.1.1
        option dns 192.168.1.1

Any hints what I've done wrong? Any other information needed?
