Hi,
I'm trying to configure a login server with mgetty and ppp in a VM.
First I downloaded the Openwrt VM-image from http://backfire.openwrt.org/10.03.1/x86 â€¦ -ext2.vmdk
After starting the VM I install the package "mgetty"
and edit in /etc/inittab the row ttyS0 to 

ttyS0::respawn:/usr/sbin/mgetty -x 0 -a -r /dev/ttyS0

In /etc/mgetty+sendfax/mgetty.config I add:

port ttyS0
    speed 115200
    direct y
    issue-file /etc/mgetty+sendfax/mgissue

In /etc/mgetty+sendfax/login.config I add 

/AutoPPP/ -     a_ppp   /usr/sbin/pppd file /etc/ppp/options

And in /etc/ppp/options.ttyS0 I add

debug
-detach
asyncmap 0
crtscts
lock
proxyarp
noauth
192.168.50.1:192.168.50.2
netmask 255.255.255.0

If I try to connect to the OpenWRT-Server I get the following error in Syslog (logread):

May 10 08:38:15 OpenWrt daemon.notice pppd[1558]: pppd 2.4.4 started by root, uid 0
May 10 08:38:43 OpenWrt daemon.err pppd[1558]: Couldn't get channel number: Invalid argument
May 10 08:38:43 OpenWrt daemon.info pppd[1558]: Exit.
May 10 08:38:43 OpenWrt daemon.info init: process '/usr/sbin/mgetty -x 0 -a -r /dev/ttyS0' (pid 1558) exited. Scheduling for restart.
May 10 08:38:43 OpenWrt daemon.info init: starting pid 1562, tty '/dev/ttyS0': '/usr/sbin/mgetty -x 0 -a -r /dev/ttyS0'

What make I wrong?
The error "Couldn't get channel number: Invalid argument" seems like there were a incompatibility between kernel and the PPP-daemon.
