As 741 and 740 are nearly identical models, I'd rather use firmware directly for your router, so not for 741n. BEWARE, that tp link firmwares, which do not contain boot in the filename might still include boot, which will lead into bricking your router. In fact, most of tp link firmwares do contain the boot, even if they have no word boot in filename! The only sure way is to look inside using hex editor such as bless...

Flash this file:

69db411f88af2d1d79c9d5b4e4bc77a6  wr740nv4_en_3_16_5_noboot-130329-openwrt-back-to-original.bin
http://bayfiles.net/file/RF5x/VgSusn/wr740nv4_en_3_16_5_noboot-130329-openwrt-back-to-original.bin

Be sure to download using orange button "premium download", or you will download some spyware or adware of wtf, don't worry, it is free download, but they need to click their ads badly

It is basicaly this firmware: http://www.tp-link.com/resources/softwa … 130329.zip, but with uboot removed. Made it myself for you, should work, but if you will brick your router, don't blame me. I don't have this model, so I was not able to test myself. Be sure to check md5 before flashing!

You have syntax errors, because you need to include file name into qutoes "", when there are special characters present, such as parenthesis. I'd not flash the firmware provided by Boompje123, if you don't want to brick your router, it is for different model

TP-link touters have 5dbi antenna, which is above average and I have no problems with coverage. They are cheap, but perform very good. I have about 20 tp-links (all with openwrt) between family, friends and company I work in. Thanks to openwrt they are easily manageable from remote and quite stable. There are some problems in openwrt, but nothing that serious, openwrt is much better than stock firmware. If you have problems, you must have bad hardware peiece, but keep in mind, that most problems are between keyboard and chair So I recommend tp-link, cheap and solid, never needed to replace defective piece (exept a few I have bricked myself, when I was new to openwrt).

nozombian
24 Jun 2013, 17:18

First, download the bin file I provided, then check md5, if it is correct, open your web browser and into adress bar type your router IP, which will take you to the luci. Here you can flash router with the bin.

If you don't have luci (you have used trunk image), you will have to transfer the bin file using scp, or any way you like into /tmp, see wiki how to do it. Then flash using mtd.

This should work, but I have not tested myself:

1. run cmd from windows
2.type PSCP.EXE -p c:\wher\ever\it\is\wr740nv4_en_3_16_5_noboot-130329-openwrt-back-to-original.bin root@192.168.1.1:/tmp
3. open putty to 192.168.1.1 (or ip of your router)
4. from putty type mtd -r write /tmp/wr740nv4_en_3_16_5_noboot-130329-openwrt-back-to-original.bin firmware

But it would be much better for you if you could learn openwrt, because it is really better than the stock firmware It may look uneasy on the first look, but if you will give it a try, you will like it. Returning to stock firmware is a good idea only if you are going to return the router
