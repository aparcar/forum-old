OK, so the NEXX lan port is connected by cable to your client PC and shows the LAN address of 192.168.1.1.  That means the NEXX is running DHCP on the LAN side correctly.  As frietpan also says that this is the LAN port, I think this is correct and working.

So now your wifi screen shows two connections, the master is wireless to the PC.  The other, based upon the IP info at the bottom of the above post is the WAN connection to your home router using the second connection on radio0 and it should say client.  The IP info indicates that you have the WWAN connection to the home router working.  You should be able to go to the internet.

I think we have validated what connection is what.  the only other thing I can suggest is to go to the WAN interface and make sure that the current wireless and the eth0.2 are bridged (check box at top on and the wireless and ethernet both checked).  If not please do so.

If you now DISABLE the wireless connection for the WAN side and connect a cable, if it works great, if not I would say the port is dead.
