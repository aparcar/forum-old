I had the R7500 beta builds that MyOpenRouter put out, and those were not very easy to get back to official NETGEAR. They support a mix of OpenWrt and NETGEAR boot features, so you don't really know what you are supposed to do.

The solution was:

  1. Connect router port 1 directly to computer with Ethernet cable

  2. Open router and connect serial cable to header

  3. Router: power on, Enter (IPQ) by hitting key within 3 s

  4. Use command BOOTM listening tftp on port 69

  5. Computer: ifconfig en2 192.168.1.2 netmask 255.255.255.0 up

  6. tftp; connect 192.168.1.1; mode binary; put R7500-V1.0.0.82.img

  7. Router echos bunch of dots, then flashes to nand, then reboots

  8. Browse to 192.168.1.1, you should see NETGEAR Genie wizard

Problems:

  • OpenWrt only flashes .bin

  • NETGEAR only provides .img

  • I have no idea if those are compatible.

  • NETGEAR suggests flashing openwrt-ipq806x-sysupgrade.bin

  • openwrt-ipq806x-sysupgrade.bin is no longer available

  • OpenWrt does not supply openwrt-ipq806x-sysupgrade.bin b/c ipq806x not supported platform

The franken-rom only supports:

  1. FACTORY Reset mode by power-on with reset for 6 s

  2. Funky preboot mode (IPQ) by hitting a key on serial cable interface within 3 s of power-on

  3. Netgear 3 s nmrp some proprietary image-flashing bootup. Regular tftp/pxe no work

  4. OpenWrt failsafe mode by hitting f, Enter late in boot process.

OpenWRT doesn't change the bootloader.  The process involving the serial cable and tftp is entirely through the bootloader-- it will work no matter what OS firmware is presently installed, as that firmware never actually boots.
