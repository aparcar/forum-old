See first how it works with 15.05.1. If your problem is fixed then you can just happy use 15.05.1: "If it's not broke, don't fix it". If the problem is not fixed you can read below but you have to understand that there are some risks.



You can find most of the informations regarding how to compile OpenWRT from git here https://wiki.openwrt.org/doc/howto/buildroot.exigence and here https://wiki.openwrt.org/doc/howto/build

The Package Installations part depends on the distro you are using.

There is always a chance to break your router. If you can't recover from a bad flash/can't afford to break your router you might want to stop.
There is also a chance to face the same problem even after you flash your router with the lastest 15.05.1 from git. You might want to think about it first.

******

I can NOT be made responsible for any damage caused by the informations provided below.

It works for me but there are an army of reasons why it can just not work for someone else. I just want to make sure you understand the risks.

I use Mint Mate 17.3 x64, if you are using another Linux distro this line will probably not be like mine (this is the The Package Installation part):

sudo apt-get install subversion build-essential libncurses5-dev zlib1g-dev gawk git ccache gettext libssl-dev xsltproc zip

Rest should be the same on most Linux distro. Anyway this is what I do:

git clone git://git.openwrt.org/15.05/openwrt.git

cd ~/openwrt

./scripts/feeds update -a
./scripts/feeds install -a

make prereq

make menuconfig

In that menu:

  • target: Atheros AR 7xxx/AR 9xxx (if I remember right it's already set like this)

  • target profile: TP-Link TL-wr841n/nd (you want the images for 841, anyway you will get the images for v3,v5,v7,v8,v10 also)

  • luci -> collections -> luci : press y to have it selected (-> = navigate in the menu) (you probably want luci, else you will have to manualy edit the config files; if you don't want luci ignore this step)

  • Exit -> exit -> exit -> save when asked if you want to save

make

And you wait until it's done. In my case it takes 2-3 hours but I use an old AMD dual core.
If you see any error during this last step, it's not good at all, just stop if you got errors.

The images are in: ~/openwrt/bin/ar71xx . Be carefull when you pick the image check to have v9 in the name (you get the factory and sysupgrade images for v3,v5,v7,v8,v9,v10 in this folder and a couple of other things). Flashing with the wrong image will break your router.

I prefer to delete the openwrt folder and git clone it again when I build a new image. I can do git pull to update it I know.
