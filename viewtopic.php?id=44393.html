there is no point in doing this on the AP-side.
Initiating roaming to another AP only makes sense if you know that there is a better signal to/from the another AP.
Disconnecting a client just because it has a "weak" signal doesn't make sense, what if this weak signal is the strongest signal the client gets?
So the logic and decision should be in the client, and it should change the station because the other signal is better/stronger, not because the current signal is weak.  The only thing that the APs should do, is make the roaming, fast and without or only little interruption.
