Hello,

I cam across with exact same error message, "has no valid architecture, ignoring."

Hardware: ravpower RP-WD02 with CC 15.0.1 openwrt-15.05.1-ramips-rt305x-ht-tm02-squashfs-sysupgrade.bin

Before this error I was playing to revert original firmware using tftp and various "Kernal.bin" file : no luck

I extract "firmware.bin" from sysupgrade file, rename it "Kernal.bin" and update with tftp,
openwrt came back to life after that but with strange error like the one valid architecture
(and others: like list upgradeble modules list to update, even luci-theme-openwrt was not working)

Solution: since luci was working, partially but working, I update with the full sysupgrade file:
now all is back to normal.

I don't know if my problem is the same than you, but for me uploading firmware.bin from sysupgrade file using tftp and flashing with luci (apart from uboot, it's strictly the same file !!) don't give same result.


EDIT: I think my mistake was to use firmware.bin from sysupgrade witch is not a complete firmware,
to upload with tftp a need a complete file.
