heureka
it was pretty simple in the end.
the problem was my old GS108Tv1. the config is rather awkward. i now ordered a cisco 300-10.

1) config sophos utm 9.2
i configured a free 1g/bit port with two new vlans 100,200 inkl dhcp, dns, fwrules,...
and connected the openwrt tp link wr841n v9 directly with a lan port, not the wan port.

2) config openwrt
then i configured 2 new dhcp interfaces, one on eth0.100 and another on eth0.200,
and moved both to the internal firewall zone.
in addition i uncheckt 'this is the only dhcp' in systems dialog

3) config openwrt
now i added vlan 100 and 200 as new vlans to the switch
and tagged, the cpu and the connected lan port, with both vlans
(now i had to connect to one of the default gateways ip of the new dhcp, of course)
suddenly both interfaces got a correct ip from the dhcp's.

4) config openwrt
now, i configured two new wlan's, one for each new dhcp interface.
now, both wlan's are working correctly.
all wifi clients getting ip's from sophos dhcp.
because of the fwrules on the sophos, one wlan has no connection to a internal devices, just the connection to the internet works.
i even can throttle the throughput for the wlan's separately by configure it on the sophos utm

all config for openwrt are done with LuCI web user interface.
