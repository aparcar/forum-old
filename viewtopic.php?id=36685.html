Hi!

The flash layout for D-Link DIR-825 are defined in target/linux/ar71xx/image/Makefile:

dir825b1_mtdlayout=mtdparts=spi0.0:256k(uboot)ro,64k(config)ro,1024k(kernel),5184k(rootfs),64k(caldata)ro,1600k(unknown)ro,6208k@0x50000(firmware)

vargalex

Hi!

I think, you haven't understood me. You must only change the DIR-825 MTD layout in the target/linux/ar71xx/image/Makefile to:

dir825b1_mtdlayout=mtdparts=spi0.0:256k(uboot),64k(config)ro,1024k(kernel),5184k(rootfs),64k(caldata)ro,1600k(unknown)ro,6208k@0x50000(firmware)

When you check, than can you see, that I have removed the ro attribute from uboot partition. The DIR-825 haven't art partition, but you can set with the same method the config and caldata partition to writeable.

The flash layout for DIR-825 is completely different defined than for 1043nd (or I can say, that the layout definition for TP-Link routers differs from other devices).

vargalex

Hi!

I have now readed your full post! :-)

When you want to write the full flash, than the simplest way is, when you change the mtd layout to:

dir825b1_mtdlayout=mtdparts=spi0.0:256k(uboot)ro,64k(config)ro,1024k(kernel),5184k(rootfs),64k(caldata)ro,1600k(unknown)ro,6208k@0x50000(firmware),8192k@0(fullflash)

Than you have a fullflash named mtd device, which represent the whole flash content.
