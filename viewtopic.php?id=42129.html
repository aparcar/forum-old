hi XTF,

there are tons of "what is the best router"-threads in this forum, and in almost every thread you will see, that you won't get a recommendation if you don't have specific requirements.

Nozombian already answered that any (recent) router will fullfill your requirements.

So, if you really have no specific requirements, there won't be a recommendation.

Flash-memory is important if you want to install a lot of packages.
RAM is important if you want to run memory-hungry applications.
CPU is important if you run CPU-intensiv applications (e.g. VPN)

25Mbit/s and 16 users needs none of that.
You could run this on a router with 400MHz CPU, 32MB RAM and 4MB flash (but I would recommend at least 8MB flash)

qasdfdsaq

XTF wrote:

Ah, is processor performance no longer an issue at all?

Processor performance for what?

Modern MIPS processors do about 15-30Mbps per 100Mhz for WAN throughput and about double that for WLAN. Throughput is CPU limited, connection number for practical intents and purposes is not.

If connection number is the main issue then I would first try limiting the connections per user and/or increasing the connection limit on the router. With DD-WRT I've personally managed to increase the limit to as high as 12000-16000 on a router with 32MB RAM, Openwrt ships with 16384 as the default but can struggle with that on 32MB devices.

I have no idea why you have 82% sirq. And it's really hard to figure it out without any info what happend at the moment  you decided to take this top-output.
wlan-drivers tend to produce high sirq-values, especially older wlan-driver.
But you didn't post any detail so, it's no use to speculate. So lets just say it's DD-WRTs fault.

So some test of mine:

1. 120Mbit/s (netto) from WAN to wifi-client (NAT, firewall, QOS) on a 680MHz MIPS (Netgear WNDR-3700 v1)

CPU:   0% usr   1% sys   0% nic  49% idle   0% io   0% irq  47% sirq
Load average: 0.02 0.02 0.05 1/50 17524

2. 120Mbit/s from WAN to LAN-client (NAT, firewall, QOS) on a 680MHz MIPS: Netgear WNDR-3700 v1
CPU:   0% usr   0% sys   0% nic  69% idle   0% io   0% irq  30% sirq
Load average: 0.12 0.04 0.05 1/49 17552

3. 140Mbit/s LAN to wifi-client  (bridged)
CPU:   0% usr   1% sys   0% nic  72% idle   0% io   0% irq  26% sirq
Load average: 0.03 0.04 0.05 1/49 17582

So yes, I really think a 400MHz MIPS-CPU should be able to handle 25Mbit/s.

if you want gigabit you can get a tl-wr1043nd (or any other supported 400MHz atheros-router with gigabit!)
