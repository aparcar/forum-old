I have implemented a new routing protocol SSR (Scalable Source Routing) for the OpenWrt linux and standard linux.
The purpose to develop SSR protocol is to enable routing in large random networks with memory and message efficiency. The routers can find the right path to the destination in arbitrary network topologies, even with many churn, i.e. the start or crash of some routers inside the network.

www.linyphi.net is the website for this project. You are welcome to have a test. (Of course, you have to have at least 2 machines.)
So build your network as you like. Once they are connected, the route can be found.

After installing Linyphi, the machine gets one or more IPv6 address, which should be used for the communication with other machine. In other words, the destination address should be IPv6, for example, type the IPv6 address of the 'web server' in firefox, which should also have IPv6 addresses generated by linyphi program.  If you have internet connection, you can also create a gateway to translate the Ipv4 packet from the internet to your linyphi net.

Any test and suggestion are appreciated.
