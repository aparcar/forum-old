my problem is
I want to login via ssh to a git repo
but my git binaries are installen via opkg -d usb
so I have to add the /opt/bin PATH in the /etc/profile

the problem is, when git logs in via ssh, the /etc/profile is not read and therefore PATH has no /opt/bin in it

I tried to add a /etc/bashrc and /root/.bashrc and even /bashrc and /.bashrc with the PATH update, but that does not have any effect
i can also type
ssh router echo \$PATH
to view the PATH when i log in with the non-interactive console

so how can i run a script bashrc-like for ssh access
that changes my PATH var?

ps
perhaps bashrc does not work because openwrt uses ash?!

problem solved:
had to change the following on my router:
/etc/ssh/sshd_config: PermitUserEnvironment=yes
~/.ssh/envirnonment:
PATH=<PATH>
LD_LIBRARY_PATH=<LIBS>
