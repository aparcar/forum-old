Hi,

i'm a new OpenWrt-User and searching for some way to use My TL-WR1043ND as a Router between two LAN-Ports.

I have this Setup:

DSL-Router (with multiple external IPs, i can't configure this router) <==> TL-WR1043ND <==> Internal Network

Now I need a routing setup between the external and the internal ip for every device.

Like:

92.x.x.1 <==> 10.0.0.10
92.x.x.2 <==> 10.0.0.11

...

I already did this on a real computer with rules like:

iptables -t nat -A PREROUTING -d 92.x.x.1 -j DNAT --to 10.0.0.10
iptables -t nat -A POSTROUTING -s 10.0.0.10 -j SNAT --to 92.x.x.1

iptables -t nat -A PREROUTING -d 92.x.x.2 -j DNAT --to 10.0.0.11
iptables -t nat -A POSTROUTING -s 10.0.0.11 -j SNAT --to 92.x.x.2

But i have no idea how i could to this with OpenWRT.

Not sure if a empty system would be the best for me. I just need iptables, dhcpd and a small dns and some way to use 2 LAN ports for routing. Then i would be happy.

Should i check the Image Generator? If yes:
Is it enough to build the kernel with "make image PROFILE=TLWR1043NDV1" or is it necessary to define some packages? Not sure if packages like a ssh/telnet-daemon are available with that minimal image.



Thanks
