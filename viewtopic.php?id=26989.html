This is the question I already asked at the DD-WRT forums. I since migrated from DD-WRT to OpenWRT which is more stable on this router. I recieved no answer there so I will ask here also. The same problem exists with OpenWRT and I don't believe the the hardware is not capable of near to gigabit speed. Ignore the DD-WRT and optware parts.

----------------------------------------------------------

I have recently bought a Buffalo WZR-HP-G300NH. Until now I was really pleased with the device. I installed "DD-WRT v24-sp2 (08/07/10) std" on it and it worked ok. Upon that I installed some of the Optware packages following this blog:

http://g300nh.blogspot.com/2010/06/soft â€¦ art-1.html

I configured samba and connected a 2TB ext3 disk to the USB on the router. Still everything was ok.

But then I decided to switch some parts of my LAN to gigabit. Namely my main desktop and my NAS storage. To my suprise the speed after upgrade was still the same. So I investigated. I used "iperf" to meassure the TCP throughput. First I meassured it between my desktop and NAS. The route was:

Desktop -> Gigabit switch -> Router -> Gigabit Switch -> NAS.

The speed was 10MB/s as it was before the upgrade. Then I meassured speed between NAS and some other server in the same room separated only by the switch. So:

NAS -> Gigabit switch -> Server.

The speed was 75MB/s whis is fine by me. Ok then I meassured the speed between my desktop and the router and between the server and the router. The connections were:

Desktop -> Gigabit Switch -> Router : 10MB/s
Server -> Gigabit Switch -> Router : 18MB/s

Ok so I thought the cables (CAT 5) were the problem. The last test should confirm that. I unpluged the server and brought it up to the router with the same CAT5 cable that achieved 75MB/s in the test before.

I pluged the server to the router directly (even no switch was present). And to my surprise the speed was the same: 18MB/s. The router is supposed to be gigabit and has a decent processor and a lot of RAM.

So what is wrong. Is DD-WRT causing problems or is the problem the hardware (hard to believe)?
