of course there is a way, don't be afraid. First you need a working firmware and then follow this steps:

1- You need to change the IP options (IP 192.168.1.10 netmask 255.255.255.0  gateway 192.168.1.1) and the pc directly by RJ45

2- Enter cmd (if using windows) and type PING -t 192.168.1.1 and wait

3 in another cmd navigate where you have the firmware and run  tftp -i 192.168.1.1 PUT [firm_name] and press enter just when you restart the WRT160NL.

After that check if the first cmd start to receive and wait untill the router reboot. Good luck!

I've been in the same boat. The solution is weird, but it works

You'll have to download this file: http://dd-wrt.com/routerdb/de/download/ â€¦ e.bin/3806 (This image is specially crafted so that the WRT will accept it. Stock Linksys or OpenWRT images won't work).
Now:
1: Set your NICs IP to 192.168.1.2.
2: Fire up a TFTP client (I used tftpd32 (http://tftpd32.jounin.net).
3: Set TFTP host to 192.168.1.1.
4: Select the downloaded linksys-to-ddwrt-firmware.bin
5: Power off from your WRT.
6: Power on your WRT.
7: Press "Put" in tftpd32.

Upload will commence after a few seconds, after which the WRT will reboot after ~50 seconds. You will now have a micro build of DD-WRT on your device, which can be used to flash stock Linksys firmware (direct DD-WRT to OpenWRT will fail and you'll have to repeat the above process). Once on stock Linksys firmware you can flash OpenWRT
