From the command line, you can set up a "cron" job.  For details, google "man crontab linux".

You edit the crontab file with "crontab -e".  Then you enter a line like this:

0 * * * * /sbin/reboot

This says on the zeroth minute of every hour of every day of every month, reboot.  It is safest to fully qualify any command (include the full path).

In openWrt, crontab uses by default the "vi" editor, so if you're not familiar with that, google it too.

You may or may not have to start the "crond" daemon.  If you don't see it when you do "ps | grep cron" then try "/etc/init.d/cron enable" and "/etc/init.d/cron start".

(But ... why do you need to reboot every hour?)
