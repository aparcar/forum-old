ha! self-help it is!

for those looking for a solution to a similar problem:
the problem seems to lie with the DFS mechanism. As far as I can tell this is broken cfr the error posted above.

so the workaround is to select a band without DFS requirement (and set a correct country code)
You can check with freqs are fine with: iw reg get

802.11ac is now working fine.
a quick check with a file download on windows from a NAS wired to the xiaomi runs at 11MB/s.
(while on a 2.4 run a 2MB/s - i'm swamped by other people's wifi)
