I have been running OpenWRT flavors since day 1 on a WRT54G.  I finally updated my very old white russian build to OpenWRT backfire.  So far it has been very smooth and after a dozen plus reboots of the router everything is still running perfectly.

The major problem I've hit head first is the firewall setup.  I have a webserver running at 192.168.1.106.  Port 80 is forwarded.  External users can access it via my world ip address and domain name.  Internal users (so computers behind the router) cannot.  My solutions to this problem on the old build do not work.

nmap on my world ip from a box external to me shows port 80 is open.

nmap on my world ip from a box internal to me shows port 80 is closed.

I've tried to solve this on my own reading all the documentation I can and googling.  I have come up empty.

Any advice?  What changes do I have to make to /etc/config/firewall?  An example forwarding port 80 in such a way that outside can access it and inside can access it both by the world ip.

Apparently I just screwed things up during the initial configuration.

Placing redirect rules after the custom firewall.user script fixed everything.


config 'redirect'
        option src              wan
        option src_dport        80
        option proto            tcp
        option dest_ip          192.168.1.106



Works from outside and inside.
