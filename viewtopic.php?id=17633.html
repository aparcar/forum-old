I have a ddwrt54gl v1.1 with a 1gb SDA card add on, running dd-wrt, in a fan cooled box up on a 15 meter tower and attached to a 22 dbi gain parabolic antenna attached to a mechanical rotor that I can control from the ground.  Actually there are two antennas and routers on the tower - the top one on the rotor, and the pole can be mechanically twisted.  I wanted two lan feeds as I plan to mix bandwidth with two LAN cards in a box running RouterOS load balancing software.  Who knows, maybe I'll get greedy and make a Christmas tree out of that pole with branches of antennas and router ornaments.

Here is my question: I want to install Wicrawl http://midnightresearch.com/projects/wicrawl/ in the router, but wicrawl doesnt run inside a shell command, and it isn't in the Opt packages, so I don't know how to get it to work with a router, as opposed to with a wifi card.

I'm assuming to run it I'd need to do this:
1) install a swap partition on the SDA card and configure the router to use it (the card is soldered directly to wires, and the router is already up the tower - can I format it from here?)
2) install a graphical front end such as gnome or kde desktop into the router
3) install a vnc server into it
4) install wicrawl - possibly by creating an Opt package for it.
5) run wicrawl through vnc viewer on my local computer.

The problem is I have no idea if #2 and #3 and even #4 are possible.

On a different note, I was able to install an older optware ported version of aircrack-ng in the router, and plan to figure out how to get the newer version working in there.  Can I just copy files from my linux box to the SDA card into the same locations the older version files are? 

But wicrawl seems much faster and easier and more intuitive, and it would be too perfectly geeky cool to get that hacked into a wrt54gl. 

Anyone want to help?  I can pay a few paypal dollars for progress.

A few notes: I have Optware installed, and access the files on the SDA card using WinSCP.  It took me many hours last troubling with learning how to transfer files, unsuccessfully trying to use sftp in shell to move files, and trying to set up an FTP server.  WinSCP in Ubuntu under Wine works a treat.  I struggled to get a samba share working under CIFS and using Ubuntu, but gave up and opted for the SDA card hack.  Cost $6 for the card, and I paid a repair shop guy $5.00 for the soldering.  I'd still like to get Samba working, but from what I understand many people have spent many sleepless nights and given up.  I wish there were some hardware hack to install a usb hard drive through an sda interface, as I expect the SDA card to wear out after a while, and it transmits data at a slow 180kbps.  I don't need the hard drive in the router for file sharing over a lan, the functionality I need is to run wardriving applications through an antenna that has very high gain and transmission power.  If there were some wireless access point that was hackable to alter at will transmission signal strength, I wouldn't need to mess around with installing gnome desktop and a vpn server into a glorified radio.  I don't think an imported $100 1 watt signal amplifier would make my C-band dish antenna neighbours happy, nor the other wifi consumers in the area, and I'm too lazy to twist it amps in and of the wire as needed, and in any case, I'd lose a lot of reception strength down a long (and expensive) wire going up the tower.  I've got to use cat 5 going up to an antenna that can vary signal transmission, and I've got to either run a full OS on that router/antenna, or have a pure radio recognized as wifi service and not a lan.
