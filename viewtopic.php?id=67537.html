Dear all,


i tried to establish sftp/ssh from my ubuntu 16.04 machine, getting connection error every time, if i do ssh from ubuntu terminal it works fine as always.

but me want to access file and folders like WinSCP via ubuntu for easy operations.

somewhere i read openwrt wiki--> Some clients, e.g. GNOME Files or Cyberduck on the Mac, cannot connect to the SCP server provided by OpenWrt's standard ssh server, dropbear, unless an additional binary is installed in /usr/libexec/sftp-server - this can be done with:


# Enable sftp, work around https://trac.cyberduck.io/ticket/4161
opkg update
opkg install openssh-sftp-server

sftp/ssh to pfsense from same ubuntu machine it works like charm without any issue.

# Enable sftp, work around https://trac.cyberduck.io/ticket/4161
