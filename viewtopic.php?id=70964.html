"adblock" and "simpleadblock" are not on any custom feed, but in the main packages feed that is common to both Openwrt and LEDE. But as those packages have been introduced long after the Chaos Calmer 15.05 was branched, the packages have not been compiled for the CC15.05.1 release.

Note that "adblock" exists in the CC15.05 packages feed, but you need to compile it yourself. (The version there is not quite the newest, but should do the basic job.)

As those apps are plain scripts without binaries, there is no major reason why you couldn't install trunk/master/DD versions of those packages to your CC router. Just download the package files and use opkg to install them. ("adblock" used to work in CC, but I am not sure if the trunk version currently uses features that are not available in CC. So using the trunk version may fail.)

Adblock trunk buildbot snapshot build download links (main app and LuCI app):

There are talks about LEDE and Openwrt uniting again. The basic scenario is that the Openwrt codebase would be abandoned and the current LEDE renamed as the new Openwrt. (longer explanation: https://lists.openwrt.org/pipermail/ope … 42976.html )

Practically all active Openwrt core developers left for LEDE in May 2016, so there has been almost no development on the core Openwrt for the last year. (packages and LuCI feeds are common, so those have continued to evolve).

The LEDE 17.01.1 can been seen as the Openwrt DD release that will most likely never materialise. So, if you want adblock easily, just install the current stable release 17.01.1. It is not branded as "Openwrt", but in practise it is Openwrt DD trunk of May 2016 plus one year of further development.
