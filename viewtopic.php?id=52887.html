Using TP-Link TL-WR1043N/ND v2 on Barrier Breaker 14.07-rc3 / LuCI Trunk (svn-r10467), Kernel Version 3.10.49.

Previously, was using DD-WRT on Buffalo WHR-HP-G300N and stock firmware on D-Link 615, 1 AP each, attached over wire to a single router (from the ISP) and DLNA was working fine using Android wifi clients running BubbleUPNP accessing media on my 2 NAS (a DLink DNS-320 and a Thecus) and a Win7 laptop. The android clients were able to watch videos on the 2 NAS over DLNA, as well as file-browse the 2 NAS using ES File Explorer on Android, and of course detect the Win7 laptop over DLNA.

Then I bought 2 newer TP-Links and installed OpenWRT on them. I duplicated the network settings replacing the Buffalo and D-Link with the 2 TP-Links. Now my android clients cannot detect/find my DLNA servers, so need advice/help on what I can do.

I've disabled multicast snooping by doing:

echo "0" > /sys/devices/virtual/net/br-lan/bridge/multicast_snooping

as well as install avahi-daemon and enabled/started it, but I suspect I didn't need to do that. I've also removed firewall profiles from the br-lan interface.

After disabling multicast_snooping I am now able to see the Win7 laptop, but not the 2 NAS.

I received a miracast dongle together with my TP-Link routers, and my android client is able to detect the miracast dongle and communicate with it over the network, and BubbleUPNP can see the miracast device to "cast" media over. Problem now is my android device cannot detect my DLNA servers to cast media over to the miracast device, nor watch video via DLNA on my android devices. I can still use ES File Explorer to browse into the file server and open up the video files to watch though, just not via DLNA/BubbleUPNP.

Simplest solution I used:
1. Login to your router
2. Go to System -> Software
3. Go to available packages
4. (optional) Search for UPNP
5. Install luci-app-upnp, this will also install miniupnpd

Then DLNA/UPNP traffic should work over the network. I hope this answers your question.
