All,

I wanted to share the success of running 10.03 trunk on WRV54G with unmodified hardware (Prism PCI) with all but some LED functionality working.

Thanks to 'markusl' for his post https://forum.openwrt.org/viewtopic.php?id=24650 and the Redboot info on the WIKI.

1. Get a serial connection - takes all the guessing work out
- get TFTP server ready
- get HTTP server ready
2. Install redboot as indicated in the wiki (http://wiki.openwrt.org/toh/linksys/wrv54g)
3. Get the source code for openwrt (I use debian linux squeeze to compile)
- apt-get install quilt  gcc-multilib bison autoconf screen gcc g++ binutils patch bzip2 flex make gettext unzip libc6 git-core libncurses5-dev gawk zlib1g-dev  python sudo
- git clone git://nbd.name/openwrt.git
4. make menuconfig and set thing up for ixp4xxx
- I disabled all driver  -> net -> wireless but p54 (Prism 54g)
5. make kernel_menuconfig and:
- System Type -> Intel IXP4xx Implementation Options ->  Gemtek WX5715 (Linksys WRV54G)
! Note I disabled all platforms that where obviously not relevant
6. make
7. Install (Follow wiki) basically:
- From openwrt compiled directory: bin/ixp4xx/openwrt-ixp4xx-generic-zImage and openwrt-ixp4xx-generic-squashfs.img
(put them on the TFTP server :-) this is 192.1681.2 in this example)
- fis init
- load -r -b %{FREEMEMLO} -h 192.168.1.2 openwrt-ixp4xx-generic-zImage
- fis create linux
- load -r -b %{FREEMEMLO} -h 192.168.1.2 openwrt-ixp4xx-generic-squashfs.img
- fis create rootfs
8. Configure redboot (init script) as mentioned on the wiki)

Reboot, configure your internet access and install whatever you need via opkg.

Voila!

If you do not all steps in the notes, consider not doing it - or allocate a lot more time (>5 hours) AWAIK.

M.
