Yes, client and AP, (STA via AP mode).
Wifi to Ethernet adapter, make it as wifi receiver (sta mode), no problem at all,
much more better than any USB-wifi receiver, at least no additional driver and software is needed.
(routed or non-route just up to you.)

However if you reproduce a AP, make it has Repeater or AP Bridge capability.
you will get some stability issue.
Sometimes even just scan the network, 703n will hang or reboot.
But in older trunk doesn't experience such problems.

I didn't test WDS, so I don't know.

So you have changed password ?

After login with putty or telnet or whatever, that is Linux shell environment.......  openwrt is Linux base.
Something like DOS prompt or Windows Command prompt(cmd)    (actually DOS is copying from Linux/unix)
If you know nothing about Linux then it will be difficult for you,
because the command sets are totally different from DOS.

Your 703n won't have internet connection unless you properly config the network of it.

LuCI is a Web-interface, it let you use your web browser(IE / firefox) to connect to your router rather than login and typing commands.
It is not pre-installed in openwrt.

If you really want to learn something new, welcome to openwrt.
Since you are noob, I would recommend you use Sam's firmware.
It is custom build of openwrt for 703n, it also build-in many useful functions, such as USB storage / audio support, NAS support, and of coz come WebUI.
https://forum.openwrt.org/viewtopic.php?pid=150596

Or, maybe it is easier if using dd-wrt.

Now, you are inside the shell, it is no turning back.
Go back to basic.config, take a look to "uci" command, it is easier for you than the text editor.
