Hi guys,

I'm posting here after trying to setup a VPN client with OpenWrt.

I have the following setup:
- 1 router (A) TP-Link WR741N with original firmware, connected to internet via PPoE
- 1 router (B) TP-Link WR710N with original firmware, connected to the previous router (A) via WiFi in client mode
- 1 router (C) TP-Link WR703N with OpenWrt, connected to the previous router (B) via Ethernet, on which I would like to setup a PPTP client

I tried to do a simple configuration of the WR703N, bridging the LAN and WLAN cards, and un-ticking the case "this is the only DHCP server on the network" (as I guess the router A should be the main DHCP?)/

I faced several issues:
- I have internet on my laptop, connected via WiFi to router C, however, when I SSH to router C, it doesn't have internet
- It seems it is a DNS issue. When I ping a host, it says "ping: bad address 'xxx'". I checked /etc/resolv.conf, and it was set as "domain lan / nameserver 127.0.0.1"
- It seems I had a dnsmasq service running on the router, although I didn't activate it. Is this normal? I tried to deactivate, then force the DNS to Google's DNS, and succeed to get internet on the router
- I tried to set the DNS server as 192.168.1.1, but then the network was down again
- Then I tried to reach the main router (192.168.1.1), but it didn't work, either from the router or my computer
- Now, I can't get internet on the router anymore, even when forcing the DNS to 8.8.8.8
- I used to try to connect to PPTP connection using dd-wrt, but it didn't work (though everything else working). I decided to switch to OpenWrt to see if it could work better (and I guess the performances are better for these small devices on OpenWrt). I still haven't tried to connect the VPN using OpenWrt as I don't have any internet on the router anyway

Any advice/troubleshot procedure for this setup?

Cheers!
