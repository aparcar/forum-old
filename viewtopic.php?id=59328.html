Device can handle more than one SSID (I tried it). Do you have some suggestion to configure wifi wan + wired wan? Any special consideration or is as simple as creating the network/interface, putting it on WAN firewall zone and associating it to a existing wireless network?

i cant tell you exactly since i cant test it.. but more or less thats exactly how it goes

there is a phisical layer derived from the switch (eth.. system and phy) and the logical layer (interfaces)..
an eth/phy can have more interfaces

you can have as many interfaces as you want(more then 1 less then 65k on newer hw) the only link between them beying the bridge system(selecting which lans to bridge or wifi) and the firewall(selecting which is zone wan or lan)

so you can have as many wan zones as you want but dont bridge them(donow if OWRT support the function.. but considering it supports lan bridging...)

frietpan
2 Sep 2015, 00:51

@petiso
good question about having WAN+WWAN. I also experienced difficulties with it myself. And have not been able to figure it out.  It looks like OpenWRT expects only ONE situation and that you stick with that.
I was trying to set up a captive portal with nodogsplash, i had it working with WAN as connection to the outside
But when I switched to WWAN the next day then the whole config went tits up. Even when switching back to WAN nodogsplash didn't work any longer. 

And yes, make sure you have a USB to TTL at hand like makarel already suggested. I managed to get locked out several times... Would be very welcome to have a safety net for that...  You don't want to get locked out when simply changing a network setting on a travel router when you are fare away from home and don't have a USB TTL plug in your survival kit

I prefer to stay with the BB install on the GLiNet. I actually like the GLiNet basic front end.

Looks like there isn't really a simple way to do what you have in mind.  Unless that issue is caused by some kind of bug.  I'm using CCrc3 on the NEXX in that case... (Mediatek 7620)
