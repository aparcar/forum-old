When I mount my USB drive from /dev/sda1, it mounts it as ext2 although the filesystem is ext3.

In the log I see this message:
EXT2-fs warning (device sda1): ext2_fill_super: mounting ext3 filesystem as ext2

If I mount the filesystem with '-t ext3' it is mounted correctly without problems. I have already deleted /etc/fstab without success.
Does someone else have this problem - I'm using Kamikaze r12302 on a wgt634u.

Just for the record: in case your media isn't HDD, you most likely don't _want_ this to be ext3, since ext3 has a lot higher write rates due to journalling (see media durability discussions in SSD circles, too). In case non-ext3 mount is impossible, then parameter commit=XXX might help a bit.
