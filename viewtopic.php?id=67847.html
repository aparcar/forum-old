Using a Linksys WRT1900ACS with OpenWRT CC 15.05.1 under the following configuration:

ISP Router <- [ 802.11ac WiFi] -> WRT1900ACS <-> LAN

I am unable to get a ethernet link from the ISP's router to my network edge due to proximity and ownership issues (I rent a room where the landlord owns the ISP router).

ISP provides both IPv4 and IPv6 and I'm trying to run native dual stack on my network edge, having both DHCP and DHCPv6 interfaces configured on the WiFi radio. Whilst IPv4 connectivity works just fine, IPv6 connectivity to my LAN is not present, with the network edge spitting out:

daemon.warn odhcpd[1230]: A default route is present but there is no public prefix on br-lan thus we don't announce a default route!

In the system log every 3 or 4 seconds.

Running ping and ping6 on the network edge seems to work just fine (i.e. the OpenWRT router has IPv6 access). As per documentation, I've also tried to toggle RA between server, hybrid and relay without any effect. i.e.

dhcp.lan.ra=server

In case it's relevant, here's my network configuration:

config interface 'loopback'
    option ifname 'lo'
    option proto 'static'
    option ipaddr '127.0.0.1'
    option netmask '255.0.0.0'

config globals 'globals'
    option ula_prefix 'fd6b:9daf:8937::/48'

config interface 'lan'
    option force_link '1'
    option type 'bridge'
    option proto 'static'
    option ipaddr '192.168.1.1'
    option netmask '255.255.255.0'
    option _orig_ifname 'eth1 wlan0-1 radio1.network1'
    option _orig_bridge 'true'
    option ifname 'eth1'
    option ip6assign '64'

config interface 'wan'
    option ifname 'eth0'
    option proto 'dhcp'

config interface 'upstream_iface'
    option proto 'dhcp'

config interface 'upstream_ifacev6'
    option proto 'dhcpv6'
    option reqprefix 'auto'
    option reqaddress 'try'

Where 'upstream_iface' defines the connection to the upstream IPS router via WiFi.

Does anyone have any ideas regarding this issue?
