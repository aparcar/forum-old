Hi, I recently bought the book 'Linksys WRT54G Ultimate Hacking', a WRT54GL, & goodies for the SD card & serial port hacks. I know these are ageing products but having minimal experience I hoped to establish a modified device quickly using the guide.

Following the book I upgraded the firmware, made some configuration changes including X-WRT which swallowed the remaining memory. Some rash commands lead me to find & figure a parallel port, make a 'Hairy Dairy Maid' de-brick cable, re-flash cfe & nvram, then tftp a new kernel.

I telnetted into the router and set the ssh password but can't log back in?

I can login with fail-safe mode but I get 'Read-only file system' warnings?

I have reloaded the Linksys firmware which works fine.

I have used the Linksys firmware upgrade facility to re-re-install OpenWrt White Russian 0.9 but still cannot write to the file system?

root@(none):/# mount
/dev/root on / type squashfs (ro)
none on /dev type devfs (rw)
none on /proc type proc (rw)
none on /tmp type tmpfs (rw)

root@(none):/etc# echo Try to change something > banner
/bin/ash: cannot create banner: Read-only file system

I can make & save changes in the web browser GUI!

I know this is old stuff but I hope the knowledgeable may sympathize, thanks - Alex.

Once you set passwd via telnet, telnet is disabled and you will have to connect with ssh. Read only file system in failsafe mode is normal, don't worry it. Just set the password, reboot and "telnet" to ssh port.
