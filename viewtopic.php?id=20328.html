Hi,

I'd like to compile support for XFRM into my OpenWRT kernel but I'm not sure how to go about it properly.If I dump the output of make using v=99 I can see that the XFRM options are being checked and they are set to n:

Transformation user configuration interface (XFRM_USER) [N/m/y/?] n
Transformation sub policy support (EXPERIMENTAL) (XFRM_SUB_POLICY) [N/y/?] n
Transformation migrate database (EXPERIMENTAL) (XFRM_MIGRATE) [N/y/?] n
Transformation statistics (EXPERIMENTAL) (XFRM_STATISTICS) [N/y/?] n 

I have tried simply adding the appropriate lines (CONFIG_XFRM=y, CONFIG_XFRM_USER=y, etc) to the .config in the Networking Utilities section, but whilst the kernel still compiles successfully, the options appear to be ignored. Can anybody recommend the best way to switch these options on?

Thanks in advance.

Ok, I managed to find the solution for this and it's pretty straightforward. Doing make kernel_menuconfig before doing make menuconfig allows you to alter the appropriate kernel settings for XFRM under Networking-->Networking Options.
