speedyb0y.rev0ltz wrote:

> mtd: partition "rootfs" must either start or end on erase block boundary or be smaller than an erase block -- forcing read-only
i thing it doesn't happen when I use the trunk firmware (not compiled from me)!

that part is 100%  normal. About 2 years ago firmware was changed to leave less empty space between partitions. That message has been there since then.

I would suggest that you start from scratch. Remove .config and all other options. Build the basic vanilla firmware for your router with no extra packages, just the defaults for everything.

EDIT:
Additional comments:
It is hard to go through the full .config, as it mostly contains non-essential lines. Post the output of "./scripts/diffconfig.sh". That should give you the difference to defaults.

EDIT2:
Also this is rather normal as it is also in my own bootlog:
    2.910000 m25p80 spi0.0: found mx25l12805d, expected m25p80
    2.920000 m25p80 spi0.0: mx25l12805d (16384 Kbytes)

It merely reflects that there is a general defintion for the chip type in this devices of this architecture, and the actual memory chip varies by device.

Sometimes there are big enough changes in trunk that you need to run a "make distclean" on your source. You can also remove .config from the source folder, which will reset all config options to default.

I just recently had to do this on my source tree, as compiling stopped producing the final .bin files. A distclean fixed it right up.

For even worse problems, it's occasionally necessary to delete the entire source tree and redownload from git.
