Hello everyone,

I'm running Attitude Adjustment (the last one, RC2 IIRC) on a TP-Link WR1043ND. My cable modem is a Thomson DWG874. I've been using OpenWrt for years now, but only with DSL/PPPoE, so please blame me if I'm asking/doing something stupid.

My goal is:

1) Leave the Thomson box as dumb/transparent as possible
2) Leave all the network tasks to the OpenWrt/TP-Link router.
3) Avoid double-nat
4) Redirect a few ports to some computers behind, in the LAN.

Physically, everything is like this:

Internet -> Thomson (192.168.1.254/24) -> WR1043ND (192.168.0.254/24) -> computers (192.168.0.x/24)

The only configuration I could use that allows all of this is OpenWrt as DHCP, and with its IP address as a DMZ host in the Thomson configuration. However, not always the Thomson router will assign the same IP to OpenWrt (since it's DHCP), so the IP I put in DMZ may change, breaking the port redirection.

Ideally, I could put OpenWrt WAN as an static IP, and this static IP as DMZ in the Thomson, but no matter what I do, I can't get Internet access. Even when I configure the static IP as the same IP the router was just using with DHCP.

So, what do I need to achieve port forwarding without a double NAT in this case? How to configure a WAN with static IP in this case? Is DHCP more than just "automatic IP assignment"? I thought it was strange that I lost Internet access even though static IP had the same IP given by DHCP.

Thank you in advance.
