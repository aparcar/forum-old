Hi everyone,

I was wondering about how a packages travels through my router (TP-Link WR1043ND)? My setup is:

[pc1 = 192.168.1.2] -- [router = 192.168.1.153] -- [internet]
[pc2 = 192.168.1.4] --/

pc1 and pc2 are connected to ethernet ports, which are one vlan port (eth0.1) and bridged with the wlan interface to br-lan.

When I sent a package from pc1 to pc2 I thought the following would happen: packages enters router, router makes routing decision (and maybe other stuff), package leaves router.

Somehow that seems to be wrong because with tcpdump on the router I can't see any pc1-pc2 traffic, and iptables doesn't work on any pc1-pc2 traffic.

Can anyone explain to me where I am wrong or at least have some pointers?

Cheers,
Simon

Post #2
got_milk
22 Apr 2012, 20:35

The router only performs routing functions on WAN<->LAN traffic. Traffic between connected machines doesn't need routing and is handled by the switch.

Thanks! That means that openwrt doesn't "see" the traffic because it travels through my tp-link via the switch and openwrt can only work on routed traffic?
