I would give the 12.09 beta a try before 10.03.1 as your device may ship with a newer bootloader (which may not compatible with 10.03.x). It's code base is also quite old.
I've been running 10.03.1 since it was released and it's been very stable, but I just recently upgraded to a recent trunk build and it's been solid as well, so might as well stick to the newer code base.

Personally, I prefer editing the various configs via the CLI (vi or UCI), but if you require a GUI, I believe LuCI is still the preferred one to install.

Is there a reason why the MAC needs to be changed via a script? The UCI framework can handle this for you with a simple line addition (see http://wiki.openwrt.org/doc/uci/network):

 
config interface wan
    option ifname   eth0.2
    option proto    dhcp
    option macaddr  'xx:xx:xx:xx:xx:xx'
