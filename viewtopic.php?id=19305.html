Hi,

I don't want to teach you how to suck eggs or anything, but if you are completely new maybe you didn't spot that ./Scripts/Feeds is not the same in the Linux world as ./scripts/feeds

Also, the significance of the . is that it will execute in the current directory, so if you are in /home/test it will execute /home/test/scripts/feeds

I hope it helps a little.
