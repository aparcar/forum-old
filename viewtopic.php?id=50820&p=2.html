No, don't change it to that.

You need to change the IP to something outside the 192.168.1.x range now, e.g.:

uci set network.lan.proto=static
uci set network.lan.ipaddr=192.168.10.1
uci set network.lan.netmask=255.255.255.0

Gateway and DNS should be unset on the LAN interface. Also don't plug it into the LAN, it is not supposed to work that way (yet).

Good news is the wireless is now correctly connected from the repeater to your host AP.

Set your laptop to DHCP and connect your laptop to the LAN port of the repeater and see if you get internet access.

Post #36
qasdfdsaq
29 May 2014, 16:04

OK, so now everything upstream of the repeater is working fine, as is the routing and DHCP.

The final step is to add a repeater network to the wireless configuration so you can connect to it wirelessly. Just add the following to the bottom of your /etc/config/wireless:

config wifi-iface
    option device 'radio0'
    option mode 'ap'
    option ssid 'RepeaterWirelessNetwork'
    option encryption 'psk'
    option key 'testing12345'
    option network 'lan'

Obviously replacing "RepeaterWirelessNetwork" "psk" and "testing12345" as appropriate, making sure the SSID remains different from the host network.

And yes, feel free to install Luci.

Just remember this setup creates a separate network for your repeater network so you will need to switch between it and the host network manually when moving between the two. Also remember wireless repeating will reduce bandwidth somewhat so don't expect amazing performance.
