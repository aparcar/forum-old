how to protect one network with fake ssid's with madwifi

This is current working in Kamikaze 8.09 RC2 in fonera 2200.

edit the file /etc/config/wireless and copy the config 'wifi-iface' heads, do not copy the network option. ex:


config 'wifi-iface'
        option 'device' 'wifi0'
        option 'network' 'lan'
        option 'mode' 'ap'   
        option 'ssid' 'openwrt'
        option 'encryption' 'psk2'
        option 'key' 'very secret key'
                                 
config 'wifi-iface'
        option 'device' 'wifi0'
        option 'mode' 'ap'     
        option 'ssid' 'openwrt1'
        option 'encryption' 'psk2'
        option 'key' 'very secret key1'

config 'wifi-iface'
        option 'device' 'wifi0'
        option 'mode' 'ap'     
        option 'ssid' 'openwrt2'
        option 'encryption' 'psk2'
        option 'key' 'very secret key2'

this example will create two fake ssid's (openwrt1 and openwrt2) and one real ssid (openwrt)

this only works for 4 ssid's (ath0-3).

if you want to create more ssid's you need to add to the file:

/etc/modules.d/50-madwifi
in the line: ath_ahb  to ath_ahb autocreate=none maxvaps=32

the max maxvaps is the limit of the ssid's. actually the limit is 32.

then reboot the device. Warning, you device now will boot but takes more 10 seconds or more to boot, needed to create the interfaces, more ssid, more time to boot, and more memory need, and more probability to have a kernel panic.

I hope this post helps some one

Your method just decreases the chance a little bit, a novice attacker will try the right network, out of hundreds of other networks around.
A little more talented one will listen to the network traffic and find out the right one very quickly.
If you really want to be on the safe side, only allow access to your internal network from your wireless network via VPN solutions, like openvpn.
