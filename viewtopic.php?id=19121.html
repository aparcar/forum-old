Hi.

The error level of the reported messages is a "bug" in busybox' handling of syslog-input, it always uses error as message level. However this does not mean that's something wrong with it, just a glitch in the syslog api.

The second highlighted error is explained easily: after cron was started, the ntp client is fired and sychronizes the current system time, cron detects this time warp and complains about it in the system log. This is uncritical too as cron will resync with the correct time.

HTH, JoW
