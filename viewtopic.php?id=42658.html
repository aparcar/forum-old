So you have: Internet <> a router <> openwrt router <> PC + WIFI?

Two sorts of bridge. One is when a modem/router is turned into just a modem with another router behind it that does authentication, DNS, DHCP etc. and the second is what happens inside the router when you bolt logical networks together to look like one. Classic example is WIFI - LAN which shows up as "br-lan". Which router was bridged?

Second they all need to be on different addresses. If you are that new to CLI and networks it just might be simpler to take just the router cabled to a PC and nothing else and re-install OpenWRT.
on it and start fresh.

HTH

What you  could do is forget about the OpenWRT router being a router. Instead of connecting a LAN port of the first to the WAN port on the second, which I assume is what you are doing,  connect to a LAN port on the second. Is is now just a switch with WIFI. Turn off DHCP  etc. on it so everything uses the first router - it will still need it's own address for configuring.
