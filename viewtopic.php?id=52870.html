Hi,

I'm trying to understand how the Inter-Zone Forwarding and zones output policy work. If Inter-Zone Forwarding is configured between zone Lan and zone Wan, why doesn't the output policy of the Lan zone have any effect? Considering this quote from luci "The input and output options set the default policies for traffic entering and leaving this zone [...]", shouldn't the output policy be involved once data leaves the Lan zone to Wan? I thought that the output policy would set the default action for communication between zones that can later be override by rules, eg. whitelisting.

Post #2
hnyman
22 Sep 2014, 20:42

The default policy sets the default, but you have to understand the context correctly.

input = incoming traffic, targeting the router itself, from Zone X to router
output = outgoing traffic from router, originating in the router, from router to zone X
forward = traffic passing through the router, from zone X to zone Y

http://wiki.openwrt.org/doc/uci/firewall#zones
