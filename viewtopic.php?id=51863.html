So I installed Barrier Breaker on a TP-Link TL-WR1043ND v2.1.. I installed the trunk from about a week or 2 ago.. 

So I get it all set up, install Luci, etc.. and set up a wifi network and leave it at the home of the person I was helping.  First time I've used OpenWrt and there was a learning curve but I got it to do what I want and it's really slick.

A week or two later and they say things aren't working anymore.  I go there.  I look for wireless networks with my iPhone.  Nothing found!  So I figure I will log into Luci with their PC and see what is up.  There is no response as if there is no webserver operating.  So I try to ping the router.  Pings fine.  I try to ssh.  I get a login prompt.  I try "root" and the password that I had set.  Invalid password.  And I know I didn't forget it because I wrote it down.  I did power-cycle several times.  Same result.

So... I have to hold the reset button in and reset the configuration. Then telnet in, set password, ssh in, setup wan, install luci, etc, etc...  I get it all set up again and it now seems to be working fine.

Has anyone else seen this kind of configuration loss?  I'm wondering if it's just a bug or if someone hacked in or if a virus on the LAN got to it (there was a computer that seems to have a virus as it keeps setting the browser homepage to safesearch.ch and it is seemingly impossible to remove).

Any suggestiong appreciated!

first you should format C and reinstall windows on your pc, and then change all your passwords,
i really doubt openwrt would change the root password by itself...

after installing luci on the trunks, you need to make sure the webserver starts up automatically,
aka "/etc/init.d/uhttpd enable", if you don't do that and reboot the router, luci will not be working...

I can confirm this issue on the TP-LINK TL-WDR3600. This happened to me last night. I've had OpenWRT installed on this model for months and months now with the works. Print server, DLNA, DNSMasq Etc. on ExtRoot. Everything stopped working LAN, WAN & WIFI. All the extroot data was still intact, and no problems with the usb drive.

I has to telnet in, and reset root pw, but no opkg installs or configs would stick. I had to mtd -r erase rootfs_data. After I wiped everything and reinstalled configs (I had backed up), all has worked fine.
