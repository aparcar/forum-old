Stop for a moment and think of the implications of the name of the release changing from "experimental" to "development" or "unstable" etc

This would mean all docs referring to experimental will have to be updated, as experimental moves to development etc etc

eg from http://openwrt.org/OpenWrtDocs/Hardware â€¦ WrtDocs%29

1.3. WRT54GS v2.0

    The WRT54GS v2.0 uses a BCM5325EKQM switch and a BCM4712LKFB processor. These units are supported in the EXPERIMENTAL branch of OpenWrt.

The situation gets worse, as with each subsequent release, the docs will again have to be updated. Maintaining the docs as a Wiki with this approach rapidly becomes unworkable.

OpenWRT is now mature enough to warrant a version numbering scheme for each release, with full release numbers and point releases. Plus it would be easier to refer to OpenWRT1.2 rather than the "non-experimental 24-5-2005 release".
