        option ifname 'nas0'              
        option ipv6 '0'                   
        option proto 'pppoa'              
        option username 'XXXXXXXX'
        option password 'yyyyyyyy'             
        option vpi '0'                         
        option vci '38'                        
        option encaps 'vc'

#config interface 'wan6'

Hi Omiez,

you probably know this, but you can only use PPPoA if your router has a direct ATM/ADSL link (the oA part of the akronym means "over ATM"). So if you have an external DSL-modem PPPoA on the router will not work for you.

Hi Omiez,

some modems/dsl-routers offer to translate a PPTP connection from a router/client via ethernet into PPPoA over the ATM link, (see for example http://de.wikipedia.org/wiki/PPPoA ). If your modem does support that you need to set-up your router to use PPTM to talk to the modem. But again this really depends on your modem supporting this mode, which as far as I know is rare. According to wikipedia this seems to be used in Austria, so maybe getting an austrian adsl-modem might do the trick (I would first research which austrian ISP's still use this method and then would try to get a used dasl-modem from that ISP (potentially branded?) vie e-bay... Also I would check whether Austrian modems support the required Annexes for my country)
     Hope that helps...

Best Regards
        M.
