Last Friday the server changed to 78.24.191.177 but several people have seen problems with the propagation of the new IP address
through dns world-wide for all prefixes before .openwrt.org.
I had a problem with www. until yesterday, others have reported problems with svn. and you with downloads

You can overrule the dns server of your provider by defining 78.24.191.177 downloads.openwrt.org in the 'hosts' file.
When the time to live of the dns setting expires it will probably get updated in your provider? dns server with the correct address.

Proper procedure before changing an IP address would be to first shorten the validity period for the old address forcing more frequent refreshment of cached values,
then change it, and then extend the validity period again for the new address. But apparently it was not or could not be done that way. The dns records contain a validity
period of an entire week so if the last refresh was done just before the change it may take until coming Friday before your provider'? dns server decides not to use the cached
value but instead do a look-up from the authoritative dns server.
