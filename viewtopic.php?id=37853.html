Hello!

I'am using TP-Link 741ND (HW rev. 1.4) and last trunk firmware. I have next configuration on my router for VLAN switching:

config interface 'loopback'
        option ifname 'lo'
        option proto 'static'
        option ipaddr '127.0.0.1'
        option netmask '255.0.0.0'
config interface 'lan'
        option ifname 'eth0.2'
        option type 'bridge'
        option proto 'static'
        option ipaddr '192.168.10.1'
        option netmask '255.255.255.0'
config interface 'wan'
        option ifname 'eth0'
        option proto  'pppoe'
        option username 'usern'
        option password 'userp'
config interface 'iptv'
        option ifname 'eth0.3'
        option proto 'none'
config switch
        option name 'eth0'
        option reset '1'
        option enable_vlan '1'
config switch_vlan
        option device 'eth0'
        option vlan '3'
        option vid '777'
        option ports '0t 1t 2'
config switch_vlan
        option device 'eth0'
        option vlan '1'
        option ports '0t'
config switch_vlan
        option device 'eth0'
        option vlan '2'
        option ports '0t 3 4'

port ?1 - WAN port
port ?2 - for VLAN 777
port ?3,4 - for Internet via NAT

With this configuration from ports 3 and 4 I have ethernet frames with VLAN ID=0. Why? Windows 7 and Vista works fine, but in Windows XP I have problem. Windows XP can not properly handle such frames. Please, help me.
