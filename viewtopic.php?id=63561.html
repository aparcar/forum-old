The patch has been applied to the mainline linux kernel in the meantime. But it has not been backported to the stable kernels 3.18 and 4.1 which OpenWrt Chaos Calmer and trunk use. The official stable kernel trees only accept patches/backports for real issue. A harmless log message won't be fixed in stable kernels.

Now, you could still ask the OpenWrt devs to include a patch in their source trees. But it seems like a lot of effort for something that is only a cosmetic issue.
