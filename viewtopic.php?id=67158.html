First guess: the kernel version for ar71xx has been bumped from 4.1 to 4.4 since your previous build. And somehow you have now a version mismatch between kernel modules available and the running kernel. The error messages "insmod: failed to insert /lib/modules/4.1.23/lockd.ko" and "insmod: failed to insert /lib/modules/4.1.23/nfs.ko" hints to that direction. If you have an extroot setup, you might need to ensure that modules for the correct kernel version are in all places.

blackdragon169 wrote:

I still get the same error after flashing my new firmware i have build with linux kernel 4.4 you suggest.

[    8.377956] insmod: failed to insert /lib/modules/4.4.7/lockd.ko

I think the problem not come from mismatching between kernel modules available and the running kernel.

You might be right that it is not the reason, but you are apparently still using the old abandoned git.openwrt.org sources for your build. There 4.4 is still at 4.4.7 (like your error shows), while it should be either 4.4.19 (LEDE) or 4.4.14 (github.com/openwrt/openwrt). I suggest that you git clone totally new sources and rebuild everything. There are lots of other fixes (also for compatibility with 4.4) that are not present in git.openwrt.org, which has been abandoned during the spring months.

Another thing that you should do, is to rebuild your .config from scratch. Defaults may have changed while your personal .config still uses old values for something.

I don't quite get your comment on squashfs. Having read-only squashfs and writable jffs2 overlay (on top of the read-only part) is the normal thing. But preinit stuff is done before any overlay functionality is applied, so the files used for preinit come from the read-only squashfs partition. Thus you need to compile them into the firmware.

Ps. Has it ever worked for you without errors? Or was this changed in the last build? If it worked without errors with the previous build, what was the exact revision? I scanned your posts and it seems to me that you are trying to do rather special config (e.g. https://forum.openwrt.org/viewtopic.php?id=66685 )
