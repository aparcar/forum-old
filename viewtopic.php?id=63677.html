Hi,

I want to wake up my nas while not on my LAN. I had a previous set up that worked (a stock NBG6716 which had an easy "enable wake on wan" feature.) I've recently ugraded to CC15.05.1 and am trying to replicate that feauture.

plenty of recipes can be found on forum/web. mostly boiling down to:

1.set up a portforward (7/9 udp) from wan to lan to a pseudo-broadcast ip (....254)
2. map pseudo-broadcast to real broadcast with magic incantation:
    ip neigh add 10.0.0.254 lladr ff:ff:ff:ff:ff:ff nud permanent dev br-lan

this last step can be a bit fidgety, as the arp table can get flushed, and the command will fail when ...254 is already in the arp table. (can't seem to flush that addr from the table with ip neigh del ...)
moreover, mapping any incoming udp on port 9 to a broadcast is plain dirty.

so, is there a better way to achieve the same functionality.
if not, what's the best to map the ip to the bcast addr
rc.local - but only run when the interface changes
init.d script - .same problem i think
procd - seems to have this feature... but AFAIK it's undocumented (the relevant page is work-in-progress at best ;)

thanks.
