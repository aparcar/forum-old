wvdakker wrote:

How to proceed? Can I fix it if I solder a serial port? Or do I have to get a JTAG
interface?

If bootloader (CFE) is still in it's place, having a serial console should be enough. It's unlikely that "sysupgrade" killed CFE, writing to CFE partition is forbidden by MTD layer by default and I don't think flash driver could fail too.

So serial console is mostly likely to be enough to fix your device. All you have to do is use CTRL+C to stop booting process and then type "tftpd" in CFE to make it wait for a new image.

With a little of luck, maybe you can put "CFE" is "tftpd" mode without serial console too. Not sure about that, but maybe booting with reset button pressed can help?

I can connect to the router again.

After a CTRL+C during the first boot I got the CFE command prompt and
was able to ping the router on one of the ethernetports (WAN port did not work).

I first downloaded the openwrt-e3200_v1-squashfs.bin, renamed it to code.bin and
started a tftp upload to the router.

After that a:
flash -noheader : flash1.trx

loading and rebooting I got an invalid boot file message.

Downloaded the openwrt-brcm47xx-squashfs.trx file, renamed it to code.trx and
flashed it again.
I ended with a working router (without wifi).

So my problem is solved.
