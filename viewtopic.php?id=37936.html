Hi,

I am using OpenWrt Backfire 10.03.1  on a TP-Link TL-WR1043ND Gigabit Router with an USB disc that I want to export using NFS.
I installed and configured the kernel-nfs packages as described here http://wiki.openwrt.org/doc/howto/nfs.server

However something with the portmap/rpcbind is wrong, executing rpcinfo -p my_routers_ip from the client results in a timeout
as does nfsmount without mount-options.

The strange thing is that I can mount the nfs shares succesfully using udp protocol:

uruk:/home/paco # mount -t nfs 192.168.0.100:/mnt/share2 /mnt -o udp,vers=3
Using NFS over UDP can cause data corruption.
Please refer to the WARNINGS section of the nfs(5) manual page.

Well, as you can see^^ this method is not recommended but it works. It also causes issues with KDE nfs mount icons because they can not handle the udp warning message.

Any ideas how to activate nfs over tcp on Openwrt?

RPC and portmap seems to be running on the router:

root@sin:~# ps aux | grep rpc
  634 root         0 SW   [rpciod/0]
 1148 root       956 S    /usr/sbin/rpc.statd -p 32778 -o 32779 
 1157 root      1256 S    /usr/sbin/rpc.mountd -p 32780 
 1411 root      1372 S    grep rpc 
root@sin:~# ps aux | grep portmap
  715 nobody     928 S    /usr/sbin/portmap 
 1413 root      1372 S    grep portmap
