OpenWRT is a mini linux system... so you really need to deal with basic linux remote access to get to those files you're talking about. If you're on Windows then Putty and telnet are your friends. I'm sure in the getting started documentation there is a note about after you first flash you'll need to Telnet to the router, set yourself a root password, and reboot. After you do that, SSH should be enabled.

Putty is an SSH client, so you use it to connect from Windows to the SSH server (called Dropbear on OpenWRT). Once you connect, you login there and can then access files, make changes, etc from the linux command line.

I agree.  In general this product EXPECTS that you have a GOOD working knowledge of Linux, and the documentation is not very helpful to most Windows users.  The package system is a great concept, but there is no information other than a cryptic sentence on what a package does.  There is no list of prerequisites, which is mostly needed to calculate size for device storage.  There are a lot of functions that the device can do, but it's not documented.  Search with Google for outside threads.

All the config files you need are in one folder \etc\config\  Yes, there are other places that you will find and eventually maintain (hack), but this is the brunt of it.

Guys, he doesn't even know how to get to /etc/config/network...

1) if not already done, telnet to your router (use putty), set password, exit
2) ssh to your router (use putty)
3) After successful ssh login, enter 

ls -l  /etc/config/network

-> that's the file you are looking for

4) Now enter 

/etc/init.d/network restart

-> that's how to execute this command


For all the rest: see the wiki.

cd /etc/config
vi network
