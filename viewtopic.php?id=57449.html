Post #2
mk24
20 May 2015, 16:33

Your router will not have an IP address unless it is connected (via a lan port) to a DHCP server.  The easiest way would be to temporarily set up another router and use it as a DHCP server.  Log into that router to see what address your 1200 obtained.

Or look up failsafe mode for your model.  If you can boot into failsafe mode you can telnet to 192.168.1.1 and then either edit the config files or run firstboot to reset everything to defaults.

mk24

Connect the LAN of the other router to one of the LAN ports on the OpenWRT router, not the WAN port.  This is if you really have set its LAN by mistake as a DHCP client.  The WAN defaults to being a DHCP client, but access to the admin page from the WAN side is by default blocked.

You may just have set the LAN interface to a different IP, or otherwise disabled it. Your best bet is to get OpenWRT into failsafe mode so that it resets its configuration back to 192.168.1.1. Use this procedure:

http://wiki.openwrt.org/doc/howto/generic.failsafe

If there's no way to do that, you'll have to resort to getting to the console with a USB to TTL cable (involves opening the box and voiding the warranty) as described here: http://wiki.openwrt.org/toh/linksys/wrt1200ac and then re-flashing OpenWRT or the original firmware to reset the beast.

(Last edited by Steltek on 22 May 2015, 11:10)
