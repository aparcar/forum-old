Reboots loop TP-TL-WDR3600
Hello, everyone,
I'm new here and I don't speak English. I translated the text with deepl. com.
I have problems with my WDR3600, after using the router as a free-radio router (based on OpenWRT) for 1 year, I installed the OpenWRT version openwrt-15.05.1-ar71xx-generic-tl-wdr3600-v1-squashfs-factory. Except for the activation of the WLAN, there were no problems. After a router restart, the WLAN did not start automatically, even though it was configured this way.

Can someone help me? Thank you very much!

Hello, everyone,
I got it right, thanks to kgoerbig (see below). I only had to execute the command "mtd -r erase rootfs_data" in the filesafe mode, then the router ran again and - after restoring the backup - now even the WLAN starts automatically!

kgoerbig wrote:

Try:

1. Power off
2. Power On, press the reset button  on the back very, very fast over and over until you see the 'asterisk' light blinking rapidly. When the asterisk light is rapidly blinking the router is in failsafe mode.
3. Go to a wired PC that is connected to router.
4. Manually set IP/Mask/Gateway to 192.168.1.2/255.255.255.0/192.168.1.1
5. Telnet in, 192.168.1.1
6. mtd -r erase rootfs_data (this resets openwrt back to factory defaults, erasing all settings). Router will reboot.

7. Reflash openwrt or reinstall packages.
