First take a vlan tagged and then untagged. The order of the network vlan configuration is very important.
If you make the wrong order, not all vlan are loaded correctly or not will just work.

config 'switch'
    option 'name' 'rtl8366rb'
    option 'reset' '1'
    option 'enable_vlan' '1'
    option 'enable_vlan4k' '1'

config 'switch_vlan'
    option 'device' 'rtl8366rb'
    option 'vlan' '50'
    option 'ports' '0t 1 2 3 4  5t'

config 'switch_vlan'
    option 'device' 'rtl8366rb'
    option 'vlan' '1'
    option 'ports' '1 2 3 4 5t'

config 'switch_vlan'
    option 'device' 'rtl8366rb'
    option 'vlan' '2'
    option 'ports' '0 5t'

or

config 'interface' 'loopback'
    option 'ifname' 'lo'
    option 'proto' 'static'
    option 'ipaddr' '127.0.0.1'
    option 'netmask' '255.0.0.0'

config 'interface' 'vlan50'
    option 'ifname' 'eth0.50'
    option 'proto' 'dhcp'
    option 'gateway' '0.0.0.0'

config 'interface' 'lan'
    option 'ifname' 'eth0.1'
    option 'type' 'bridge'
    option 'proto' 'static'
    option 'netmask' '255.255.255.0'
    option 'ipaddr' '192.168.1.1'

config 'interface' 'wan'
    option 'ifname' 'eth0.2'
    option 'proto' 'dhcp'

config 'switch'
    option 'name' 'rtl8366rb'
    option 'enable_vlan4k' '1'
    option 'reset' '1'
    option 'enable_vlan' '1'

config 'switch_vlan'
    option 'device' 'rtl8366rb'
    option 'vlan' '50'
    option 'ports' '0t 4 5t'

config 'switch_vlan'
    option 'device' 'rtl8366rb'
    option 'vlan' '2'
    option 'ports' '0 5t'

config 'switch_vlan'
    option 'device' 'rtl8366rb'
    option 'vlan' '1'
    option 'ports' '1 2 3 5t'

I wrote the order as entries in the network is very important.

Add dhcp yet

config dhcp vlan50
    option interface    vlan50
    option ignore

And add firewall file

#definicja dla vlan50 z nat
config 'zone'
        option 'name' 'vlan50'
        option 'input' 'REJECT'
        option 'output' 'ACCEPT'
        option 'forward' 'REJECT'
        option 'masq' '1'
        option 'mtu_fix' '1'
        option 'network' 'vlan50'

config 'forwarding'
        option 'src' 'lan'
        option 'dest' 'vlan50'

config 'rule'
        option 'name' 'Allow-DHCP-Renew-vlan50'
        option 'src' 'vlan50'
        option 'proto' 'udp'
        option 'dest_port' '68'
        option 'target' 'ACCEPT'
        option 'family' 'ipv4'

config 'rule'
        option 'name' 'Allow-Ping-vlan50'
        option 'src' 'vlan50'
        option 'proto' 'icmp'
        option 'icmp_type' 'echo-request'
        option 'family' 'ipv4'
        option 'target' 'ACCEPT'

Add multicast route

config 'route'
        option 'interface' 'vlan50'
        option 'target' '224.0.0.0'
        option 'netmask' '240.0.0.0'
        option 'gateway' '0.0.0.0'

and add multicast firewall

config 'rule'
       option 'src' 'vlan50'
       option 'proto' 'igmp'
       option 'target' 'ACCEPT'

config 'rule'
       option 'src' 'vlan50'
       option 'proto' 'udp'
       option 'dest_ip' '224.0.0.0/4'
       option 'target' 'ACCEPT'

read more link http://eko.one.pl/forum/viewtopic.php?id=3296&p=3
