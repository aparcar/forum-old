Try using the multiwan package to manage both of your links. The logic in multiwan should handle the gateways between the two modems and allow for a more robust backup/failover behavior.


What kind of VPN are you running? OpenVPN? PPTP/L2TP? IPSec?
When I had a dual WAN setup and an IPSec VPN, I had to write a little hotplug script to restart the IPSec tunnel whenever the links changed from primary to backup, or backup to primary. I'm sure there are better ways to handle a VPN with a dynamic IP (road-warrior setup) but for me this was the easiest and most reliable solution with the least amount of work
