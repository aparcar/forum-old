I'm wondering if there are plans to support 802.11ac and hardware like Netgear R8000?

I've seen hints in various places that OpenWRT has lost some developer momentum, and looking through lists of supported hardware I don't see many recent models.

My ISP provides support for MLPPP pair bonding on the dry loops that I have, and I'd like to put some good wifi in front of two Sagemcom 2864 DSL modems. I think I could manage to put together an OpenWRT build with the features I need, but I'm not up to writing drivers, etc.

Is there an unsurmountable problem getting source code from Netgear, or from the chipset manufacturers that it is using?

Are there builds of OpenWRT for Tri-Band Routers like the R8000?

Post #2
drawz
21 Oct 2014, 06:20

Broadcom doesn't release wireless driver source, so routers like the R8000 are not well supported in OpenWrt. The best-supported 802.11ac router today is the TP-Link Archer C7 v2, with Qualcom/Atheros SoC and wireless chipsets. The Zyxel NBG6716 is a similar chipset, but the experience is more limited as it is not as widely available and is a bit more expensive.
