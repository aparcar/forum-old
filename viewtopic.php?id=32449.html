I was struggling with the problem of "Broken pipe" when using ssh https://forum.openwrt.org/viewtopic.php?id=27484. Connection es reseted. I've found that the problem only happens when ssh connections are outbound. I mean *only* when packets traverse out Cisco PIX (OS Version 7.0(1)) .

Post #2
krisk84
17 Oct 2011, 23:24

More than likely the PIX is configured to drop "idle" TCP and/or SSH sessions.  Idle as determined by the PIX; usually some time without activity.  It probably sends a TCP RST to close the connection.  This could be verified by looking at the configuration on the PIX and/or analyzing a TCP dump to look for the source of the RST packet.
