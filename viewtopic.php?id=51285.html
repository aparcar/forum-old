Hey, I started on dd-Wrt, welcome! BTW, OpenWRT is much better!!

They are UCI commands, and are the way you make changes to the /etc/config/xxx configuration files (rather than edit them directly. See:wiki.openwrt.org/doc/uci

If you executed those command I gave you, you would have something like:

root@OpenWrt:~# cat /etc/config/network
config interface 'rndis'
    ifname usb0
    proto dhcp

root@OpenWrt:~# cat /etc/config/firewall
config zone
        option name 'wan'
        option input 'REJECT'
        option output 'ACCEPT'
        option forward 'REJECT'
        option masq '1'
        option mtu_fix '1'
        option network 'wan rndis wwan'

Can I ask, have you got tethering working from router through android at all?  I mean even without VPN?

Anyway, going back to my original suggestion, this is a better piece of code:

    opkg update; opkg install kmod-usb-net-rndis   # will add: kmod-usb-net kmod-usb-net-cdc-ether

### 1/3 Configure RNDIS interface - essential (base) configuration
    uci set network.rndis=interface
    uci set network.rndis.ifname=usb0
    uci set network.rndis.proto=dhcp

#   uci set network.rndis.dns='208.67.222.222 208.67.220.220'
#   uci set network.rndis.peerdns=0

    uci commit network; /etc/init.d/network restart

### 2/3 Configure firewall...
#   uci set firewall.@zone[1].network='rndis wan'

    ZONENET=$(uci -P/var/state get firewall.@zone[1].network | sed -e 's/rndis//;s/wan//')
    ZONENET=$(echo $ZONENET | sed -e 's/ */ /g;s/^ //;s/ $//') # remove spaces
    uci set firewall.@zone[1].network="wan rndis $ZONENET"

    uci commit firewall; /etc/init.d/firewall reload

Now all you need to do is set up the VPN tunnel, either on the Android smartphone, or on the router - depends which is easier for you.
