Yes it looks daunting but I think it's ok if taken step by step.  I don't have the model you have but instead a TP-Link archer C7, but the install/de-install instructions look similar and here's how I would tackle this problem.

1) Obtain factory firmware.  The http://wiki.openwrt.org/toh/tp-link/tl- … l_firmware makes reference to the firmware image file containing 'boot' or not.  If it contains 'boot' then the file has a bootloader which needs to be removed.

2) Removing boot loader from image 'dd if=orig_firmware_file.bin of=tplink.bin skip=257 bs=512'  - this unix command copies the original file to 'tplink.bin' except for the first 257 blocks of data.  tplink.bin is now the file that you need to get into the router's flash chip.

3) Flashing the 'cut' image to flash memory. 'mtd -r write tplink.bin firmware'.  After a successful flash the '-r' option means reboot.

An alternative approach that I've had to use, is to use the router's 'emergency tftp recovery' mode.  This is a bit more 'fun' because it requires a tftp server to serve your 'cut' tplink.bin file as created above *and* you must set your recovery tftp serving machine to IP address 192.168.0.66 - all using a wired connection to a LAN port on the router. On windows 7 I have used 'opentftpserver'.  Place your cut firmware image in C:\opentftpserver making sure to rename it to 'wdr4300v1_tp_recovery.bin'.

Check your IP address is 192.168.0.66.  Fire up opentftpserver.  Fire up the router whilst holding the reset button for 10 seconds or so.  Invoke lucky charms as required.  You should see opentftpserver say that it's uploading a file 'wdr4300v1_tp_recovery.bin' to 192.168.0.88.

Good luck.
