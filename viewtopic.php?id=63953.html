hnyman
5 Apr 2016, 17:19
kevin144446 wrote:

Nothing in System>Software>Configuration

That indicates that you have likely messed the package feed config, (and thus the package lists can't be fetched).

Which Openwrt release you are using? 15.05.1 or trunk?
The contents of the /etc/opkg.conf and possible /etc/opkg/distfeeds depend on your Openwrt version...

(anything older than 15.05.1, then please update first to 15.05.1 and the update will fix the opkg config, too)

EDIT:
you might try manually fixing things with:

cp /rom/etc/opkg.conf /etc/opkg.conf
cp /rom/etc/opkg/distfeeds.conf /etc/opkg/distfeeds.conf

opkg update
opkg install <your_upnp_package>

(the second line "distfeeds.conf" works only in 15.05.1 and trunk, and is not needed in old versions like 15.05, 14.07, 12.09 etc.)

You should test console login with both ssh and telnet in case the pssword was reset to be empty and you need login with telnet.

Yep. Naturally you can't have the same IP address (192.168.1.1) as the router...
Set a fixed IP address from 192.168.1.x (x is not 1 or 255). 192.168.1.2 is fine.

You use you ssh/telnet client like Putty to try first with telnet as root/<empty>, root/<your old password>, then with ssh root/<empty>, root/<your old password>

Which router do you have? And which image you flashed on it?
