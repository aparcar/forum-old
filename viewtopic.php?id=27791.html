router and wireless functions, such that the 2-Wire acts only as a DSL modem. Connected to the 2-Wire
is a Ubiquity RouterStation Pro, on which I've installed OpenWRT Backfire 10.03, r24064, cross-compiled
from source on a Linux PC. In back of the Ubiquiti is my LAN, with several PC's. The Ubiquiity connects to
TelMex using PPPoE.

Believe it or not, this setup works fine, except for a problem I will describe, which is associated with
my occasional need to use an (Open)VPN connection to a VPN service in the USA, which service
then provides a proxy so that when accessing the internet I look like I'm in the USA. The (Open)VPN
connection also works fine, except as described following.

The DSL shares my Telmex telephone line. I use a TelMex-provided DSL filter between the TelMex
line and the TelMex 2-Wire modem. IF, and ONLY IF, I'm using the (Open)VPN connection and proxy,
every time I make or receive a telephone call, I lose my internet connection. If (Open)VPN is NOT
in use, telephone calls in or out do not break the internet connection.

It's a strange problem, but I'm hoping someone on this list might have an idea what's wrong and/or
how to fix???                Thanks, Gus
