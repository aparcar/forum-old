Easiest is probably to use the TFTP method (push reset during boot at a correct moment).
It allows you to install any of the firmwares without going back to OEM in between (as the functionality is in the Uboot boot manager).
http://wiki.openwrt.org/toh/netgear/wnd … lsafe.mode

You need to install the "factory.img" version of Openwrt with that. ("sysupgrade.bin" will not work.)

Ps. If you want everything working out of the box, you might test my community build for WNDR3700:
https://forum.openwrt.org/viewtopic.php?id=28392

hnyman
8 Oct 2014, 20:21

dd-wrt should have no role, as the recovery flash tool is in boot manager, and I don't believe that dd-wrt would overwrite that.

After entering the recovery mode (and waiting for TFTP transfer), the power LED should blink rather slowly. Blinking will stop after an accepted TFTP transfer and the LED is either lit or blank during flashing (takes 3-5 minutes), after which the router reboots automatically.

It may take some attempts to enter the recovery mode.

Most likely your tftp transfer attempt failed (or alternatively, you tried to transfer a firmware for v1).
I usually use the TFTP2 tool with Windows GUI for the transfer (linked on dd-wrt page: http://www.dd-wrt.com/wiki/index.php/Tf … #Windows).
