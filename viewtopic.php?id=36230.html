I am noticing a performance regression on copying a large file from the router to the computer via CAT5e ethernet cable using SAMBA Protocol.
I can achieve a fairly constant SAMBA copy from the router to the computer via Wireless @ ~ 4MB/s
I used to be able to achieve a fair constant SAMBA copy from the router to the computer via ethernet cable @ ~ 9MB/s
Now I am getting hanging progress bar during the copy process and miserable speed of 1MB/s using ethernet cable.
The weird part is that I am still able to achieve a fairly constant SAMBA copy from the router to the computer via Wireless @ ~ 4MB/s.
Obviously it can't be Samba's fault since there is no regression for wireless copy performance.
The reason I didn't report any bug is because I can't point it out.
I am using the latest trunk
