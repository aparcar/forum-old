I recently acquired a DIR825 Rev C1, and my first thought was to load OpenWRT on it. Based on the DIR825 page, it is supported with the Barrier Breaker trunk. It was running stock v3.00 firmware, so I flashed the openwrt .bin file from the web interface. After about ten minutes, the power light finally stopped blinking orange and was a steady orange. None of the ethernet ports would light up though. I got a serial cable attached to it, and I'm posting the boot messages below.

What's interesting to note is that the boot loader shows the mac address as 00:00:00:00:00:01. My guess is that some nvram/uboot setting got wiped somewhere. I was able to get wireless going on it without too much difficulty, and thought "I'll just flash it back to stock". I did a sysupgrade with the latest v3.01 firmware from DLink's website, and I still have the same behavior - wireless works, but none of the ethernet ports will light up. The web interface shows a cable connected, but no signal. Very odd. Also, the bootloader shows the same zeroes mac address with stock now.

I'm guessing that I need to reprogram the mac address back into the boot loader, but everytime I went to go into the boot loader, it would automatically boot the kernel.
