FriedZombie
30 Apr 2013, 21:57

Amazingly enough the other thread https://forum.openwrt.org/viewtopic.php?id=43859 did allready answers the question on how to install php packages.
Also the --nodpeps option is really idiotic to use, because it doesn't install the dependencies php needs to run properly and php will not run. As mentioned in that other thread, you most likely don't have enough memory in your device availlable to install php because of the large footprint (see the last message).

Moreover I do find it interestengly that you do have access to install packages but can't connect to the router. So I think some info is missing or you are not telling us. (I wish I am psychic, but I'm not).

My guess is at this point you do have other devices in your network and that is the reason you want to disable the dhcp server.
see: http://wiki.openwrt.org/doc/recipes/dumbap#step.3disable.dhcp.server
just follow step3, or you can set it under the luci gui as well, under 'network' --> 'interfaces' (and click edit).

Changing the ip address can also be done under the same tab in luci (web gui), or via the command line (ssh)
see: http://wiki.openwrt.org/doc/uci/network

If you do get errors it will also help enormously if you share them. (like the errors you get while installing certain packages).

p.s. Don't change quoted text, not everybody will notice the remarks you put there and some get irritated by it.
