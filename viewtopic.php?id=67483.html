Disabled means it's turned off permanently. So you'll never have up: true without enabled: true.

The reverse is important with DFS, where a 5 GHz radio will scan for radar interference before starting. That takes a bit. Not sure if OpenWrt has 'pending' as well. Trunk no doubt has it.

The answer:
/sbin/wifi status output at least 2 section: radio0 and interfaces
radio0 section is the hardware (emitter/receiver) and interface is set-up by LuCI enable/disable button. When disable the interface section is empty. Both section are independent. I did a mistake in my first post.

In radio0 section, up: true is set by /sbin/wifi and up: false by /sbin/wifi down
disable is the hardware wifi switch in the modem.
So disable: true => up: false
but disable: false => up: false or true depending of /sbin/wifi and /sbin/wifi down

Hope this will help other

Regarding the wifi-iface section, using the enable disable button in Luci will not remove the section.  It adds a line

option disable '1'

Technically different. Practically the same.

The same can not be said for the wifi-device 'radio0' section.  There is no way (that I know) to set the 

option disable '1'

in Luci, but it can be done in UCI or text editor and disables the radio itself. 

https://wiki.openwrt.org/doc/uci/wireless
