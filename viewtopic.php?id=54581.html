When traveling with my family on vacation I wanted to connect back to my home using OpenVPN and allow the kids tablets and smartphones to access the internet over the VPN using wifi. I eventually succeeded in doing this with an SLBoat modded WR703n and a bunch of custom shell scripts to fixup the routing after connecting via Luci but it was clunky, overcomplex, and took too long to set up after arrival :-(

Then I discovered that windows 7/8 comes with a builtin SoftAP that can re-serve any Windows network connection - including OpenVPN - and has builtin WPA2 security. I used a freeware application called Virtual Router ( http://virtualrouter.codeplex.com/ ) to configure it rather than messing with DOS commands.

Now,  when we arrive I can just set up the wife's laptop to connect to the hotel wifi, fire up OpenVPN, then Virtual Router, and all the kids tablets and phones automatically connect. Wife doesn't even know its running when she uses her laptop. Only down sides are that reconnecting after disconnection is manual (thats normal for hotel wifi anyway), and you have to leave the PC on.

Same set up even works with my Dell Venue Pro 8 tablet which runs Windows 8.1.

I'm a Linux guy and I really wanted to do this with OpenWRT but the Windows way was much simpler.
