I have router TP-LINK MR3420 v2 and want to use it like print-server.

After some time printer goes to sleep mode, then auto turn off.

After this the printer stops working until router reboot.

"usb 1-1: new high-speed USB device number 21 using ehci-platform

Is it possible to catch such event ("unable to enumerate") in openwrt scripts
and reboot router automatically to workaround the problem?

I tried to do this in "usb" and "usb-printer" scripts in /etc/hotplug.d/,
but it seems that they even does not fired in this case.
