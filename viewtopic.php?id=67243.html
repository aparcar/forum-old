Without knowing why you feel bound to that old software I would recommend to research two options:

1) migrate an image of the working system onto a virtual machine like oracle's virtualbox, that might allow to pass on the new machines US3 ports as USB2 ports to the old VM.

2) figure out whether you can not actually upgrade the openwrt version to something that has not accumulated > 5 years of unfixed kernel and user space bugs (assuming you do not run this old kamikaze server in complete isolation, but since it is a server I expect clients...).

While you don't mention it explicitly, it sounds a lot like you're using bog-standard x86 hardware, in which case simply upgrading to the current firmware version(s) should be trivial (especially as x86, even for 5-10 year old specimens, tends to have a much larger performance envelope (RAM, CPU, disk), making the upgrade much less painful than on typical embedded devices with merely 16-32 MB RAM and 4-8 MB flash).
