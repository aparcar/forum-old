Post #3
silentcreek
12 Feb 2016, 18:01

1) */1 is the same as just *
2) I think /etc/ is for config files and init scripts, not really for custom scripts. A better location might be /usr/bin/ for example.
3) You don't need to cd into the directory where the script is located. Simply write: 

* * * * * /usr/bin/myscript.sh

4) If you don't use the command 'crontab -e' to make changes in your crontab file, but edit it directly, you need to restart cron to make it aware of the changes: 

/etc/init.d/cron restart
