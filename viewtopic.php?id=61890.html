Hey All,

I wanted to create a set of Ethernet zones using a router and iptables/openvpn.

By using some low-price consumer router, I would want to use the upstream and typically four gigabit Ethernet ports to say things like:
Ethernet port 2 cannot communicate with Ethernet port 3
This ip and port on Ethernet port 1 can connect to that ip and port on Ethernet port 3, but only using this protocol.

What $50 router can I use which will give me:
- Continuously up-to-date OpenWrt
- Running sshd, openvpn and iptables
- Enough storage/usb overlay file system to save say a megabyte of certificates and configuration data
- Ability to configure itself to the desired state automatically at power on

Is it possible?
What router should I use?

Hi,

Any router that is supported with as much ram / flash with usb port.

A supported router will remain supported with daily (trunk updates) as long as there are no 'hardware related reasons for it not to.
sshd, openvpn, iptables will run, openvpn might be 'expensive' in terms of load and might not be the fastest though.
You can attach any memory stick, mount it and use as storage to any router with USB port.
As this is linux, you can configure it as you want, configuraion will remain after reboot, also you can write custom scripts that can be run at startup to do what ever you want.
