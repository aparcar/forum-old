I think you have the wrong idea/approach. You don't need to "integrate wifi and lan with wan". You don't need an wan interface at all.
If you want to brigde the LAN(+wifi) of dir-615 with the LAN behind your monowall/switch just do so. Disconnect the WAN-port of the dir-615, set an IP for the LAN-Interface of the dir-615 which is in the same subnet as the monowall, and than just connect a LAN-port of the dir-615 to the switch. make sure the wan-interface has no static ip assigned in the same subnet, and reboot the dir-615
If the monowall should do all the dhcp- and firewallstuff, you need to disable it on the dir-615:

/etc/init.d/dnsmasq disable
/etc/init.d/firewall disable

If you want to use the "unused" wan-port as a 5th LAN-port look at the switch/vlan-docs orca posted
