That is easy
Router A IP 192.168.0.1
Router B IP 192.168.0.2
Disable WiFi and DHCP on Router B
Enable WiFi and DHCP on Router A
Connect ethernet cable from LAN port of router A to WAN port of Router B

Post #5
mk24
1 Jan 2017, 16:18

Make router B a simple single AP and turn off its DHCP server, then have router A connect to B as one of its STAs.  Use relayd on A to bring this into your lan network.

It's the reverse of how you usually think about it.  Make sure that AP isolation in B is turned off, since this is a case of wireless clients needing to connect to each other.

This is a klugey way to press an old router that can only run stock firmware into service as a wireless range extender.  Connection by wire (which as the others noted, can be done with stock firmware, but requires running a cable) or a wifi mode that has 4 address capability (which usually isn't in stock firmware) works much better.
