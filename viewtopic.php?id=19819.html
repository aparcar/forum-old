I have a Check Point SBXW-166LHGE-4 that is doing nothing but collecting dust.  It's got some nice hardware features so I'd love to get it working with OpenWRT.  For reference, this is designed by a division or partnership organization of Check Point's called "SofaWare".  By the way, they have similar hardware marketed under the ZoneAlarm brand (http://www.sofaware.com/general.aspx?bo … ;objId=219).  My Check Point branded hardware is at this link (http://www.sofaware.com/general.aspx?bo … ;objId=163).

It has the following hardware features:

- 2 USB 2.0 ports
- 1 serial port
- Ethernet ports: 4-port LAN swith, 1 DMZ port, 1 WAN port
- MiniPCI form-factor Gigabyte GN-WIAG01 wireless NIC (Atheros chipset, I believe)
- 64 MB RAM
- Unknown flash, but it is at least 8 MB since the recent image I downloaded with 6+ MB

(The FCC ID is p6xsbxw-166lhge-4 where you can see the pictures of it opened up.  Here's the link to the FCC search site: http://www.fcc.gov/oet/ea/fccid/)

I've used some of the pointers in the OpenWRT documentation to glean the following information:

- I've done an nmap scan against it and it reports it runs a Linux 2.4 kernel.
- I've run strings against the most recent firmware that I've found on the Check Point site and I've seen a couple of references to the file "ELF.mips32.gzip", so I'm guessing it has a MIPS processor.

While I'd obviously rather not brick this thing, the fact is I'm not doing anything with it, so I'm willing to take a chance if anyone can suggest an OpenWRT firmware build that has a good chance of working.

Thanks!

- 1 x serial port
- ucLinux
- VIA VT6212L USB Controller
