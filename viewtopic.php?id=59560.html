I think you should be very happy with that performance.  It's a 720Mhz CPU, which is about as fast as a consumer router will run.  It's 1.5 to 2 times as fast as my PC-engines ALIX. 

Check the TOH for faster processors: http://wiki.openwrt.org/toh/views/toh_e … =cpu%20mhz
The  link is to the SUPPORTED devices, but you should all check ALL, as the TOH is new and not fully updated.

The bottleneck for OpenVPN is encryption speed.  The faster the processor generally the better the performance.  You probably want a CPU that performs hardware encryption, as opposed to software encryption, which I expect the TP-Link does.  It's likely to push you to using a X86 variant on a PC or embedded system. 

PC-Engines makes an APU, which has a 1000MHZ dual core CPU, and up to 4GB of memory but this does not have hardware AES, and I have not seen any performance data for VPN.  It would run a 64bit x86 version which you would probably want to build.
