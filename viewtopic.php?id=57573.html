On my 2 routers, D-Link DIR-825 rev. C1 and D-Link DIR-835 rev. A1.  I can only get HT40 to work on the channels 153 and 161.  I believe it works on the lower channels as well, but their transmit power is lower so I'd rather not use them.  153 is free, so that works well.  161 on the other hand is occupied by a few other networks.

149 - Sets the channel as 153
153 - Works as expected
157 - Sets the channel as 161
161 - Works as expected
165 - Doesn't work at all

If I set HT mode to HT20 I can choose the channels above and they are set as expected.  If I use the Force HT mode checkbox then 149-161 work as expected, 165 however still doesn't work at all.

Is this expected behavior or am I missing something?

Post #2

AFAIK: HT40 uses two channels at the same time, so HT40+ at channel 165 needs a channel higher than 165, and that is not possible; try using HT40-, or a lower channel.

Post #3

If you manually configure you can specify + or -, don't know about the GUI.  Realize that HT40- on 165 would put the secondary signals on 161, which the neighbors are using, so you don't want that in your situation anyway.
