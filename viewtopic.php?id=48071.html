Hi guys,

I am using "OpenWrt Barrier Breaker r39161 / LuCI Trunk (svn-r9945)" for my WR1043ND router to allow it to deal with IPv6 correctly.

My configuration looks like:

- AVM Fritzbox 6340 (provided by my cable-internet provider due to DS Lite. This means taht I get my own IPv6 address but no own IPv4 address. The IPv4 address will be routed by my provider. It's similarly managed like in LANs)
- TP-Link TL-WR1043ND is set up as "exposed host". At least for IPv4 but it's actually useless because via IPv4 it's not reachable.

The Fritzbox holds the internet connection. It also assigns an IP to the WAN-Port of the TP-Link router. The WAN port is configured as DHCP client. The router's WAN6 interface is configured as static address. This was the default value. I didn't change it and it works...

Now, my problem is that I cannot reach the OpenWRT router by its hostname via IPv6.

My Win7 computer allows me only to ping the router if I disable IPv6 protocol in Win7. Then the router name will be resolved correctly.

- IPv4 standardgateway is OpenWRT
- IPv4 DNS server is OpenWRT
- IPv6 standardgateway is OpenWRT
- but IPv6 DNS server is FB6340

Is it possible to change the IPv6 DNS server? If so, how? Does it make sense? At least I will be allowed to reach the luci interface by its hostname.

TIA!

Well, immediately after posting you'll typically find your answers ;-)

As described in the OpenWrt Wiki, I added hostnames for IPv6. Further information can be found here:

http://wiki.openwrt.org/doc/howto/ipv6. â€¦ on.the.lan

Without this prework there won't be resolved any IPv6 related hostname....

And the most important point for me was how to change the IPv6 DNS server which will be assigned via DHCP.
In the luci web interface just go to:

- Network
- DHCP and DNS
- Server Settings
- General Settings

There'll be a cell called "DNS forwardings"(per default it's empty). I entered there both (my IPv4: 192.168.x.y and IPv6 DNS: fd00::xxxx:yyyy:zzzz:aaaa) server IPs.

And finally it's working....
