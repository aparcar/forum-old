You wish your OpenWrt router to connect to the wireless network, request an address from DHCP, and then let clients behind OpenWrt use this as internet access, while the wireless network believes there is only one host (the OpenWrt router itself)?

Then this is where you should start: Routed client using MASQUERADE

This will provide the wireless network A as WAN access. Clients connected via Ethernet should get access directly. With this setup, the first host that connects to your OpenWrt router will need to authenticate the web login. Furthermore you can setup wireless network B on your OpenWrt router, either by a second SSID on a different VLAN, or in case of a dual-radio router, use one radio to connect to network A and the other to provide network B.
