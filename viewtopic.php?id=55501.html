I'm not aware of any way to do 802.1x authentication on wired switch ports currently with OpenWRT. I don't think it would be impossible (though maybe it is, I haven't dug into it enough on the development side to know) but I think it would involve a substantial increase in the complexity in the way the switch is managed though OpenWRT. I would call OpenWRT switches "semi-managed" in that you can create vlans and tagged (trunk) ports but that's about it. None of the other features of true enterprise managed switches are there. Some features could undoubtedly be added through a major rewrite of the OpenWRT switch management routines but others may require changes to the switch hardware chip itself which would present a problem.
