You can set up public key authentication as described here:
http://wiki.openwrt.org/DropbearPublicK â€¦ ationHowto

Then, set up a cron script to rsync.  If you want to rsync to a remote computer (using ssh) you do:
rsync -a /folder/to/backup/ root@192.168.1.1:/mnt/usb
Since you have public key authentication this won't ask you for your password so you can just put this in a cron script.

The way rsync works is it should only send the changed files.  As long as you have a fast network this should be pretty quick (though it helps to have more ram/disk cache on your openwrt box.)

There's also a "-v" option for verbose--probably should use this while testing the script out.
Also check you got the slashes at the end of each path correct.  I think if you put a slash at the end, rsync will not create a directory, but you might want to check the manpages or just try something.

Edit: forgot to mention that for rsync to work you need to have rsync on both computers... so you need to do "opkg install rsync" on openwrt.
