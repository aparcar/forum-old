On AA 12.09 stable release on TP Link WR842ND using ath9k wireless driver, I'm trying to change chanbw setting from the default of 20 to the alternative 10 or 5 Mhz, but when I do the AP stops working.   iwconfig and iwinfo show it is there, but there is actually no signal.

Same on AR23 build with ath5k.

I am changing the value in 'option chanbw' in /etc/config/wireless 'wifi-device' section, and the value stored in
     /sys/kernel/debug/ieee80211/phy0/ath9k/chanbw
is being set to the correct hex value.

But after a restart, the AP does not work unless the value is set to the default of 20.

I see there have been posts on this subject for earlier AA versions, but no real resolution.

Am I missing something?

Has anyone else got this working?

Or is it a bug?

Post #2
tlg
19 May 2013, 02:08

Fixed

I forgot that the AP signal from a device with non-default chanbw setting will not be visible to another wifi device unless that device is also set to the same chanbw setting.
