here is an update:

i got the wrt54gl to talk to my netgear router in client mode via:

lan_ifname=vlan0
wifi_ifname=eth1
wl0_mode=sta
lan_proto=static
lan_ipaddr=192.168.2.2
wifi_proto=dhcp

then i scan for routers with iwlist and issued these command

ifdown wifi
nvram set wl0_ssid=NETGEAR
nvram set set wl0_channel=11
/sbin/wifi
ifup wifi

all the above worked and the wrt54gl was able to communicate with the
netgear router in client mode/non-bridged

the netgear is on the 192.168.1.0  network
the wrt54gl is on the  192.168.2.0 network

the gateway is 192.168.1.1
the wrt54gl client router is fulling connected to the internet.

the problem is that my linux box which is connected via ethernet to the lan port is getting a gateway address of 192.168.2.2 which is not a gateway to the internet.
how can i get the linux box connected to the 192.168.1.1 gateway? how can
this be transparent with dhcpcd to the wrt54gl router (client router)?
