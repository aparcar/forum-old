Howto get busybox onto it (get precompiled busybox-mips from http://www.busybox.net/downloads/binaries/latest/ ) :

On a linux box or VM get the latest openssl binary or source and compile.

Create a privkey.pem and server.pem as explained http://www.openssl.org/docs/HOWTO/keys.txt and http://www.openssl.org/docs/HOWTO/certificates.txt

openssl genrsa -out privkey.pem 2048

openssl req -new -x509 -key privkey.pem -out server.pem -days 1095

Run the openssl server on linux box/vm as follows:

cat privkey.pem | openssl s_server -quiet -ssl3 -key privkey.pem

Login to the router and type sh to get a shell and then create a tmpfs or remap /var

mount -t tmpfs -o remount,size=1700k tmpfs /var

On the router in the remapped /var or new tmpfs dir

openssl s_client -quiet -connect ip_of_linux_box_or_vm:4433 > privkey.pem

Press Ctrl^C to break the client and server processes after a few seconds to ensure data transfer is over.

Similarly get server.pem also into the tmpfs

Then on the router, run openssl in server mode:

openssl s_server -quiet -key ./privkey.pem > busybox-mips

On the linux box/vm

cat ./busybox-mips | openssl s_client -quiet -ign_eof -connect ip_of_router:4433

Verify the Md5 of the busybox-mips with

openssl md5 busybox-mips

on both the router and the linux box/vm and see that it matches.

Then chmod +x busybox-mips on the router and busybox is up and ready.
