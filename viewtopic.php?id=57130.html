It's not really that it would be impossible but it would be a lot of development work to implement as you really need support from the switch chip. If the switch chip in the router supported 802.1x (and some do) it would theoretically be possible to add wired 802.1x configuration support (really just passing of configuration through to the switch chip including the RADIUS server IP) but that would need to be implemented differently for each switch chip supporting 802.1x and at the end of the day you're expending a lot of effort for a typically 4 port switch.

In most cases if you need wired 802.1x you're better off to combine OpenWRT as a router/AP with a dedicated managed switch which would support 802.1x.
