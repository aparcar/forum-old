Essentially the ROOter firmware is an attempt to bypass the 25Mbps limit of traditional ppp connections for 4G LTE modems, which are capable of much higher speeds, up to 150Mbps for cat 4 usb modems.

The ROOter has scripts which recognise the modem type when inserted and use the appropriate driver for the LTE connection. This so far includes Sierra modems, QMI, cdc-ncm, phones, and some of the newer wifi/hotspot/modem devices, and probably some others I have forgotten. All without using ppp, and getting native 4G LTE speeds.

The QMI connection is all controlled via a lua script that does not need the > 1MB libqmi files, hence the whole system can fit in a 4MB flash router like an MR3420.

Other very useful functionality includes being able to see your signal stats in the web interface, sending AT commands to the  modem from the web interface and bandwidth monitoring. Much more info can be found at http://whirlpool.net.au/wiki/router_openwrt
and firmware can be downloaded from here http://www.rfactory.org/OpenWRT/index.html

All of these modifications are contained in the scripts, which can be added to your build env in the /files dir. These scripts are also available on the above site.
