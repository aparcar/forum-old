Hello.  I have a WRT54GL v1.1 running 10.03.1-RC6.  I have always configured OpenWRT from the command line but this time I decided to give LuCI a try.  I ran into problems though.  When I make changes via LuCI, the router intermittently loses the settings.  For instance, I could change the admin password and setup ssh and hit "Save and Apply" only to have it take me back to the login screen with a message that I needed to set a password and configure ssh.  Sometimes I could get the password and ssh stuff to apply correctly but when making another change, such as the WAN or WIFI settings, the router would lose everything I had put in and start displaying the message that I needed to set my password and configure ssh again.  I've noticed that this usually occurs when the "Applying Changes" animation fails to finish.  It just sits and spins and never seems to finish saving the changes.  When I finally give up after several minutes and reset the router, all the settings are gone.  It's been very frustrating.  I did run across the recent shadow password changes and made the appropriate fixes in my menuconfig settings but it didn't seem to help.

I get this behavior whether I am running my own custom-built firmware or the stock 10.03.1-RC6 firmware.  I've been running OpenWRT for several years and have flashed a lot of routers but haven't seen this intermittent problem before.  Can anyone offer any advice?  Is my router on the blink?
