Hi all.

I have a Netgear WGT634U, was running a recent Kamikaze image for the first time, and then rebooted... it's now bricked due to wrong NVRAM settings.
But the real issue is that I can't seem to get an interactive CFE prompt by serial cable, but instead get a prompt like this:
CFE> FE> FE> FE> FE> FE> FE> FE> FE>
...
(actually there are 75 "FE>" prompts following the one "CFE>" prompt)
I can't seem to get my keyboard input (ctrl-c or otherwise) to have any effect... and no keystrokes are echoed.

I originally set up OpenWRT with the "screwdriverless" installation, and after the issue cropped up, ordered an FTDI serial cable.
I've connected gnd-gnd, rx-tx, tx-rx, and have alternately powered the router via the vcc pin of the serial connection or from the regular power connector.  For the console, I'm using Hyperterminal for windows, with ANSI terminal, 115200 8-N-1, software flow control.

The only other reference I have seen to this issue was in this forum, for the post with subject
"Netgear WGT634U CFE not booting properly?", posted by carlosmag on 2007-10-01 15:39:58.
There were no responses.

So... any ideas now... two years later?  Thanks!

Solved!

Bad serial cable, indeed.  It's not the fault of the factory, though; it was the little floppy power connector I used to get the serial port header out of the router's case.  And maybe scotch tape and rubber bands aren't so great as insulators.
