It's a problem with pid file. By default TinyProxy is trying to create file by location /usr/sbin with permission nobody:nogroup.
It could be resolved by adding next row to

/etc/config/tinyproxy:
    option PidFile '/tmp/tinyproxy.pid'
and /etc/init.d/tinyproxy:
    proxy_string "$1" PidFile >> $CFGFILE

or just to /etc/init.d/tinyproxy:
    proxy_string "$1" PidFile >> "/tmp/tinyproxy.pid"
