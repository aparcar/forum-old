Hi there,

I am a new member. I am not sure if I am posting this in the right place. So please guide, if this is not the right place to post this question. I have a router running openwrt(ver. attitude adjustment), I have enabled ipv6 on the WAN side of the router but not on the LAN side. I have a webserver running behind the NAT(192.168 subnet/private ip)-in ipv4 only LAN. I have configured my router with a unique global ipv6 address. I then created a firewall redirect rule to forward a http request to this ipv6 address to the webserver running in the IPV4 LAN using a firewall redirect rule as below. I have tried this, it doesnt seem to work. Please let me know if I am doing anything wrong. Also, I am just wondering if there is any mechanism/way to redirect request recived to ipv6 address of router to hosts in the ipv4 LAN?

uci add firewall redirect
uci set firewall.@redirect[-1]=redirect
uci set firewall.@redirect[-1]._name="HTTP Request"
uci set firewall.@redirect[-1].src=wan
uci set firewall.@redirect[-1].proto=tcp
uci set firewall.@redirect[-1].family=any         <--can be ipv4, ipv6 or any
uci set firewall.@redirect[-1].src_dport=80
uci set firewall.@redirect[-1].dest_ip=192.168.1.112
uci set firewall.@redirect[-1].target=DNAT
uci set firewall.@redirect[-1].dest=lan
