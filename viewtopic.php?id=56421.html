Hello, I am trying to help my local town with a project. I would like to provide public WiFi access on a ~1 mile stretch on our Main St. I would like to use something open source for the software, something like OpenWRT. One concern of mine is that open source software generally does not come with any warranty or guarantees that everything will perform correctly. While that does not bother me personally, it may be a problem to the town leaders, as I need to sell them this idea. I am seeking advice as to:

1) What kind of hardware (that can run alternate firmware) would you suggest for deploying a public wifi system? I am looking for cost effective solutions, and of course the access points need to be weather resistant as they will likely be mounted outdoors.
2) What firmware would you run on it? Is there something like OpenWRT made specifically for open public wifi systems?
3) Is it possible to provide security (encryption of some sort) to the end user, while leaving the system open for all to access? Perhaps some sort of captive portal that assigns a temporary username and password to the user? If so, how would I configure it so that the log-in process is straight forward and easy to use?
4) Any other pointers/tips/suggestions for a project like this?

Any ideas would be very appreciated. Thanks in advance.

While I personally don't have experience with this, you should search for "freifunk". It's a community who, among other things, provide openwrt-based firmware for exactly what you want: public and open wifi, the goal being that of providing mesh communications (i.e. not requiring each AP to be a gateway to Internet).

Please note that an open AP will have no encryption. This is supposed to be a good thing, given that you want to be, huh, open :). I personally dislike captive portals and anything that requires some manual action. These days everybody should be aware about what you can and what you cannot do on a public network.
