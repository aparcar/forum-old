hi.
I've installed Attitude-adjustment firmware on my tp-link mr3020.
Then i configured wi-fi network with no encryption on it.
I have four devices connected to this network:
1. Windows 7 desktop
2. Windows 7 notebook
3. Ubuntu tv stick mk908
4. Android phone.

mk908 carries an apache web server.
notebook also carries web server for test purposes.

Just after connecting to wireless network i have no problems with retrieving pages from mk908 web server to any of my devices.
But when windows devices (desktop and notebook) stay idle for some time (just a few minutes) i have 'host unreachable' error, when accessing to mk908 web server. Meanwhile at the same moment accesssing from android phone and from router(via ssh) to mk908 works well, also accessing from desktop to notebook web server works good too at this moment.
After some retries accessing to mk908 web server restores and pages refreshes good until next period of inactivity on windows devices. I have disable power saving on wireless modules in windows devices.

Is this problem in router or in windows?
If i try to ping mk908 from windows after few minutes of inactivity, i receive answer:'host unreachable' or 'no route to host' for some first ping requests (2 or 3) after some tries ping works ok.

p.s. sorry for my english

Upd.: Access to mr3020 router itself (ping, connect to ssh etc) works well always on windows computers.
