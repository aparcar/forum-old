Forgive me for re-surrecting this old topic, but is there any new development regarding collecting traffic stats on OpenWRT ?

From the options suggested above, it seems that only softflowd and fprobe are currently available as packages, although both seem quite outdated.

Currently OpenWRT offers softflowd v0.9.8 (note: at OpenWRT patchwork there's a pending update to v0.9.9 which was released 2+ years ago). The last update to softflowd was a few months ago, see https://code.google.com/p/softflowd/source/list.

The fprobe software was last updated in 2005.

It seems that in addition to softflowd there's a new actively developed package: ipt-netflow

"ipt-netflow is high performance NetFlow exporting module for Linux kernel (up to 3.11). Designed for Linux router with heavy network load. This is netfilter/iptables module adding support for -j NETFLOW target. Designed to work efficiently w/o conntrack. Supporting NetFlow protocols v5, v9, and IPFIX. Accounting for IPv4, IPv6 traffic, and NAT translation events (NEL)." -- http://sourceforge.net/projects/ipt-netflow/

After tons of reading and testing of packages I found the (quite bad documented) solution most people seem to end up with.

You can collect fine graded statistics using iptables and analyze/visualize them using luci-app-statistics and collectd-mod-rrdtool and collectd-mod-iptables...
This seems to be the OpenWRT way of doing statistics...

If you really want to use the netflow protocol, you will need to do the visualization on a different host.
