Hi,

in the default /etc/config/firewall I see the folowing option:

option limit            1000/sec

The wiki states:

Maximum average matching rate; specified as a number, with an optional /second, /minute, /hour or /day suffix. Example: 3/hour.

Source: http://wiki.openwrt.org/doc/uci/firewall#rules

Would 1000/sec and 1000/seconds both be valid values?
.. before I enter wrong values into the wiki.

Post #2
jow
10 Oct 2014, 11:07

The firewall code performs a substring match so "s", "sec", "seconds" will all be interpreted as "seconds".
