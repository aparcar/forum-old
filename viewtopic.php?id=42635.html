I have a working Seagate DockStar unit running on a self-built OpenWRT + FreeSWITCH firmware and is connected to my NAT/Firewall router. What I would like to experiment is to turn another Seagate DockStar unit into an IPv4 and/or IPv6 NAT/Firewall router by adding a USB/RJ45 dongle. So, I bought some USB 2.0 to RJ45 10M/100M Lan Ethernet Adapters. It is based on a Davicom DM9601 USB 10/100 Ethernet device. It works just fine on my Seagate Dockstar to connect to my main NAT/Firewall router. According to this network configuration (OpenWRT wiki doc), I don't need include a switch section mainly because a Seagate DockStar simply lacks some capabilities of a switch. Then, I just added a route section to forward packets between the two RJ45 ports. So, I concocted a simple /etc/config/network file to route packets between the two RJ45 ports, i.e. eth1 (USB/RJ45 dongle) to eth0 (RJ45 on a Seagate DockStar), as shown below and it doesn't work.

config interface 'loopback'
        option  ifname          'lo'
        option  proto           'static'
        option  ipaddr          '127.0.0.1'
        option  netmask         '255.0.0.0'

config interface 'lan'
        option _orig_ifname     'eth0'
        option _orig_bridge     'true'
        option  type            'bridge'
        option  ifname          'eth0'
        option  proto           'static'
        option  ipaddr          '192.168.5.1'
        option  broadcast       '192.168.5.255'
        option  netmask         '255.255.255.0'
        option accept_ra        '1'

config interface 'wan'                               
        option _orig_ifname     '@wan'               
        option _orig_bridge     'false'              
        option  proto           'dhcp'               
        option  ifname          'eth1'               
        option accept_ra        '0'                  
        option send_rs          '1'                  
        option broadcast        '1'

Does anyone have any idea why this doesn't work and have a solution to it?
