Hi, I have a multimodem from my isp that runs openwrt and I want to set in bridge mode, i.e connect another router to it via ethernet cable so that the extra router obtains the wan-ip adress and becomes the wifi-accesspoint. However I need some advice how to do the configuration.

On the openwrt-multimodem the interface atm1.1 has the public ip-adress(dynamic IP from ISP-not pppoe) and the interface br-lan acts as the dhcp gateway with ip 192.168.2.1.
I gather I must disable dhcp on multimodem and of course turn off wifi on it and in /etc/config/network add something like:

config interface 'wan'
      option proto 'dhcp'
      option type 'bridge'
      option ifname 'atm1.1 eth0'
      option auto 1

and ifconfig br-lan down to disable the wifi-dhcp server on the modem.
On the multimodem there are also the interfaces atm0 atm0.1 atm1bsmscw dsl0 gre0 ifb0 ifb1 wl01 wl0.1 of which only wl0.1 holds a an ip-adress(local). Can these just be ignored?

There are many additional configurations in the default file /etc/config/network/,must these be commented out?
