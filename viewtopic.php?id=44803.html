Edit:  swapped posts about PirateBox

kirschwasser wrote:

Hm, GPLv3, so why isn't it available in the OpenWrt-Repository?

I couldn't find a link to the source-code.

PirateBox - https://github.com/cr/mkPirateBox
LibraryBox - https://github.com/griffey/LibraryBox

Yes and no. That is not 100% of the story. Because of changing the responsibility and the lack of support by CR, I forked the mkPirateBox stuff. The current source for the OpenWRT-Package can be found here: https://github.com/MaStr/mkPirateBox
But since version 0.3 it shares the same "core" code like running on RaspberryPI or Laptops. This core code is an image-file mounted to /opt/piratebox ; the sourcecode for the core-scripts version in v0.6 and for the most recent mkPirateBox can be found here: https://github.com/MaStr/PirateBoxScripts_Webserver


I'm currently working on improving the Makefiles and so on, to get it better fit into the OpenWRT environment.
I'm extracting the source-code for the external-filesystem "management" to an extra package, so it's getting easier to reuse. The new packages (alpha) are here ( https://github.com/PirateBox-Dev/openwrt-piratebox-feed ) and are really alpha version. This github feed can be included in the build-environment (like described on the wiki for building OpenWRT for your own).

The current images are built based upon AA-stable and the stuff for building thos is located here: https://github.com/PirateBox-Dev/openwrt-image-build
On the trunk Branch is the Makefile I use for building based on the trunk-tree.

Why did I created those images? Well, when the MR3020 was supported by OpenWRT it was via trunk only. Because of the "instable" state of the repository, there were a lot of problems with not fitting kernel modules. That was the reason why I did that.

With the new images, I published a small repository covering all the target-system, because of no binaries. Because final ipk file is not exactly lilke the ipkg-make-index.sh needs it, it copied and modified it. It is only temporary, so please don't bother me because of that. https://github.com/PirateBox-Dev/openwrt-temp-repository
This repository can be used in opkg.conf, and I created an ipk file for the imagebuilder, that inserts that repository into it. (add-openwrt-opkg.conf).

I'm sorry, that jmac raided the OpenWRT wiki with all the entries about PirateBox. I think a small page about that is a good solution and I aready wrote a few lines more about it this morning.

opkg-package Repository: http://stable.openwrt.piratebox.de/all/packages/
custom images on AA-stable: http://stable.openwrt.piratebox.de/ar71xx_AA/

And LibraryBox = it what? and why do they use the piratebox-logo: https://github.com/griffey/LibraryBox/blob/master/piratebox-logo.png

LibraryBox was born out of Piratebox, that was the reason Jason reused the image those days. For a few days he released a complete new logo: http://librarybox.us/logos.html
Currently they still use the old piratebox-0.3 codebase with a heavily modified droopy and a python directory-listing. They are going to move over toward a normal webserver soon (hopefully).

Matthias,
who maybe forgot to update links at the wiki post. only wanted to answer some questions.
