I ran into the same problem and have a theory. Mount would not mount it but ntfs-3g would. Mount it with ntfs-3g then do a mount to check what file system is listed as. According to ntfs-3g documentation it should be ntfs-3g but mount would not take that file system. Doing a mount showed a filesystem other than ntfs-3g, it was fuseblahblah or something. Figure out what it lists as after mounting with ntfs-3g then try mounting it using that as the file system and see what happens. I haven't had time to do this with the samba3 issue i was working on.
