Maybe you screwed something in /etc/config/network...

If you have access to router serial console, you're done! Garbage output usually means wrong serial line settings (often speed), if it either doesn't respond to key pressing, the router is still booting and hanging somewhere because of wrong /etc/config/network?

Right serial settings are 115200,8,n,1

Don't worry about the router, you can either flash bootloader too.
