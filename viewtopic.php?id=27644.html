as far as i know, the gspca driver works with video4linux 1.0, while openwrt (and other linux distros) provide video4linux 2.0.
in theory, you can install libv4l package to provide a compatibility layer. i have successfully done it in desktop linux distributions by typing:

export LD_PRELOAD=/usr/lib/libv4l/v4l2convert.so

...before calling motion or mjpg_streamer. then the camera will be recognized.

unfortunately it doesn't work for me on openwrt backfire.
