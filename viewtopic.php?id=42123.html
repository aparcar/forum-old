I wonder if anoyone has tried this before, as I can't find the specific setup in here or in the wiki, and available packages lack features, but ...

Benefits:
* Obsoletes dumb port forwarding on firewall.
* Allows unlimited web host-names on standard ports 80 and 443 on single public IPv4 address.
* Easy to extend.
* Allows clean dualstack IP based virtual host configuration on internal web servers.
* Allows all of the above also on multiple internal devices.

Requirements:
* Should receive all external IPv4 HTTP/HTTPS connections.
* Should forward to internal IPv4 and IPv6 HTTP/HTTPS servers.
* Should be able to forward according to host-name.
* Should have SNI support to be able to forward SSL/TLS connections to appropriate hosts.

Remarks:
* Not needed for IPv6 connections.
* No need for caching on router.
