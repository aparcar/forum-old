Regarding the legality of creating such a firmware, unfortunately they probably are entitled to do so.

The GPL distinguishes between combining GPL code with your new code in one program (must be released under GPL) and "mere aggregation" of different programs together on one medium (can mix GPL with non-free).

OpenWRT is an operating system, and the firmware image is the system drive. So putting non-free programs into the firmware image is equivalent to installing non-free applications onto a linux desktop PC's hard drive, which of course is allowed. If they give you the code to build the GPL parts, they're legally in the clear - unless they are actually linking GPL code into their proprietary stuff, which would take some deep analysis to confirm.

FSF on "Mere Aggregation"
