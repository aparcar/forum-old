What exactly is "a upgrade method like CoreOS or pfsense"?

That you have two instances of the system installed at the same time.
The one you run and an old one. (That you can fall back to).
When you want to upgrade the system that is not used is deleted and written over with the new upgraded firmware.
You restart and you then use the new install.
If the install fails and you cant start the new system after reboot you just boot back in to the old system that is still there and worked before the upgrade.

Lots of things would be cool but aren't possible. Flying pigs for example.

Nonetheless. Most routers barely have enough flash to support one full working copy of Openwrt, let alone two. On >=16MB routers you might have a chance, if you either replace the bootloader or develop a custom interjecting boot redirector.
