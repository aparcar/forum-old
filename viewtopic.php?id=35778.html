I've been trying to get reasonable throughput on TL-MR3020 and an AR9285 (btw that's the chip you'll find in TL-MR3220) based laptop. Both running ath9k.

The tx/rx rates seem set to half the capacity the router is capable of (130 MBit/s). Don't see any disconnects, the link is very much stable.

With HT40-/HT40+ the router flics modes including 130 MBit HT mode but the client nor the router never settled on 130. Since there is no additional antenna, been told only HT20 makes sense.

The router is running the latest trunk r30919.

The number of tx failures may come from the fact the router is running in an AP+CLIENT mode BUT for the sake of testing switched it to AP mode only. No other client were connected.

Anyone else observe low throughput on this router/ath9k based hardware?

With HT20 and one antenna, 65 Mbps is maximum. With HT40, you can get 135 Mbps (or 150 with SGI). If you have other wlans nearby, you can't really use HT40 due to interference. The real throughput with 65 Mbps should be around 30 Mbps, or, 4 MB/s.
