almost so, especially if you try to install kmods, kernel modules. They have a checksum that requires to install kmods built at the same time as the main kernel.

If you use a snapshot firmware, you can install all modules from the same snapshot right after the firmware install. But the next day's snapshop modules will again be incompatible.

If you compile the mainfirmware by yourself, included the needed kmods into it.

Sysupgrade is normal way how you can upgrade most openwrt devices. I'm not sure you can do sysupgrade on your router, maybe you need to do the full upgrade and configure everything again.

This is what I'd do:
1. download http://downloads.openwrt.org/snapshots/ … uashfs.trx
2 follow http://wiki.openwrt.org/toh/asus/rt-n16 … ftp.method

If you want to try sysupgrade, it works like this:

cd /tmp
wget http://downloads.openwrt.org/snapshots/trunk/brcm47xx/openwrt-brcm47xx-squashfs.trx
sysupgrade openwrt-brcm47xx-squashfs.trx

If you will brick the router, use the tftp method.
