Hi everyone,

I have been trying for the past 5 hours or so to assign multiple IP addresses (each with their own interface labels) to a single port (eth0) on my Raspberry Pi2B (running openwrt obviously).

All I want to do is have these three ip addresses and labels:

eth0 - 192.168.10.2 /24, GW=192.168.10.1 (this is facing my DSL router, which has an IP of 192.168.10.1)
eth0:0 - 10.0.0.100 /24 (this is facing my client ranges)
eth0:1 - 192.168.100.1 /24 (this will have squid listening for some http redirects using iptables)

My issue is not so much putting this into /etc/config/network but rather, it seems that I cannot ever reach any alias IP addresses. Even if I set my laptop to 10.0.0.50 /24 and ping 10.0.0.100 /24, I don't get any response. Additionally, ifconfig doesn't show the aliases. I have done this on ubuntu without breaking a sweat but I am stuck and would appreciate some help.

Why can I not reach these aliased IP addresses (I can easily access whatever eth0 is set to) and why are they not showing ip in ifconfig. Am I missing something, can openwrt not handle aliases properly?

Thanks so much!
