I tried everything in here http://wiki.openwrt.org/doc/uci/samba to get user level access working. But Samba only works on a share with the "guest_ok" option. As soon as I remove that option on a share it becomes inaccessable.

In the (Windows 8) client all shares are visible. The error I get when opening the ones with user level access:
\\OPENWRT\shares is not accessible. You might not have permission to use this network resource. (...)

But I did all of the following:
- added a new user in the /etc/passwd file
- gave this user a Samba password using smbpasswd -a smbuser
- set correct permissions on the share: set owner of /shares to smbuser with read/write permissions.

Did I miss something? I've been on this for a long time...
I'm using trunk r37629.

It seems that Windows gets confused when you switch  between including the "guest_ok" and leaving it out on the same share. It correctly asks for your password on the second share (with no guests allowed), but in that case it somehow triggers a permissions issue. A reboot solves it. Although I don't know for how long it will last, requiring another reboot. We'll see.
