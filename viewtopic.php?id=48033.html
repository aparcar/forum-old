Hey there.

I do know how to do this by iptables in general, and I do know how to do this with the web gui. But I have absolutely no clue which specific iptables strategy OpenWRT has. So I stick to the web gui. I hope that's what you want.

Go to the "General Settings" tab of the "Firewall" page.
Usually you have a zone "from LAN to WAN" which has "Forward" activated. Uncheck this. That should block all outgoing traffic that's not caused by your OpenWRT device itself (that would be "Output". Leave "Output" checked, there is no need to disable the OpenWRTs own outgoing traffic.
Now go to the "Traffic Rules" tab.
For each port you want to open, go to the "New forward rule" panel and add a new one. After saving ("add and edit"), you need to adjust the "destination port" and make sure the "Action" is set to "allow".
That's basically it.

If you really want to have the traffic allowed in general, create a all-catching deny rule and spcific allow rules, then you need to have your all-catching deny rule being the last rule. Because rules are executed in a very specific order, the first matching rule counts. So if your first says "deny", then everything is denied.

Regards,
Stephan.
