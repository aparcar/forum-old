Hi, community experts!
I have some troubles with my TP-Link TL-WR1043ND (HW v.2.0) router.
I have installed there OpenWrt Barrier Breaker r39211 some time ago, it was working flawlessly.
But now I can't remember root password, so I've decided to reset it in failsafe mode.

I've set up my IP from DHCP to manual 192.168.1.2/24, read carefully documentation to my router and found out that it is necessary to press any button on system load right before SYS_LED starts blinking - and volia! I've seen that SYS_LED started blinking much more faster!
But no way to connect to the router from LAN 1 (also not possible from WAN port - I've checked it out also).

If I start router in normal mode - I can see following:

# /sbin/arp -a
? (192.168.1.1) at XX:XX:XX:XX:XX:XX [ether] on eth2
# telnet 192.168.1.1
Trying 192.168.1.1...
telnet: connect to address 192.168.1.1: Connection refused

That's correct, in normal mode my router has also IP 192.168.1.1 and I've disabled telnet

and if I'm in failsafe mode

# telnet 192.168.1.1
Trying 192.168.1.1...
telnet: connect to address 192.168.1.1: No route to host

# /sbin/arp -a
? (192.168.1.1) at <incomplete> on eth2

Could anyone give me an advice (except disassemble the router and connect via it's serial) how can I reset root password?
