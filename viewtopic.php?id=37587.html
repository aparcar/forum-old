I'm working on a custom build of OpenWRT for the RL-750GL and the RL-750UL (UL has PoE and a USB port) based off trunk.

The process of getting OpenWRT on this device is currently as follows:

* Compile initramfs and .tar.gz
* Use reset button on device to boot via TFTP into the initramfs image
* Telnet/SSH into device
* Mount the kernel partition and copy the new kernel to it
* Mount the rootfs partition, remove files, extract tar.gz over it
* Reboot the device (it successfully boots)

This leaves me with root (/) on a yaffs partition and it seems to be rather hard to cleanly deploy new builds.

Instead of untaring to rootfs, I've briefly tried flashing the jffs and squashfs images onto the flash, but they don't seem to do anything (the system mounts with the older filesystem and no changes).

How hard should it be to get JFFS (perhaps with SquashFS) on the flash storage?  Does it take more than just enabling the target in the menuconfig? Is there a specific command I should be using to get one of the files to persist on the mtd? Does it matter that /proc/filesystems doesn't show jffs2?

mtd1: 003c0000 00004000 "kernel"
mtd2: 03c00000 00004000 "rootfs"
mtd3: 039a8000 00004000 "rootfs_data"
