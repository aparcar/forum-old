Hi,

I recently upgraded from 5Mbps/1Mbps ADSL to 40Mbps/10Mbps FTTC so have a lot more bandwidth available.

I'm using a Buffalo WZR-G300NH as a router and wifi for my office, but I have a Linksys WRT54GS in the kitchen (middle of the house) providing wifi to the rest of the house. Both run OpenWRT.

The WRT54GS is running bcrm-2.4 as I need multi-ssid and is obviously only g, with no n wireless - so for both of those reasons, i'm looking to replace it with something better.

So, i'm looking for an OpenWRT capable, multi-vlan, multi-ssid wireless access point and think i've narrowed it down to the TP-Link TL-WR1043ND, D-Link DIR-825 or Netgear WNDR3700.

Obviously primary concern is wireless performance (signal strength) and stability (the Buffalo wireless is quite unstable - sometimes won't connect at all).

Looking at the spec's, the D-Link and Netgear have faster CPU's, the D-Link 5Ghz radio and the Netgear more RAM. The TP-Link has 3x removable antennas, the D-Link has 2, but apparently 1x for 2.4Ghz and 1x for 5Ghz - the Netgear only has internal antenna's - is that a negative?

How do they perform in the real world though?

Anyone have any thoughts, experience or comparisons?

Thanks,

Ian

Well, TP-Link TL-WR1043ND is practically the same as your Buffalo, so you are bound to have the same problems. The wireless stability is a lot better in RC6 compared to RC5, though.

Hi,

some corrections:

the new revision (V3) of WNDR3700 is another architecture (broadcom) so make sure you get V1 or V2. the WNDR3800 is the successor based on atheros and is supported in trunk.
And the WNDR3700(V1, V2), the WNDR3800 and the DIR-825 have all two physical wifi-devices (one for 2.4GHz, one for 5GHz).
The DIR-825 uses both antennas for 2.4GHz and 5Ghz, this possible because it another spectrum.

400MHz of the WR1043ND should be enough for 40Mbit. But if you want to use qos, layer7-filtering, vpn-(encryption) you should get a faster cpu. If you have problems with the signal-strengh of the signal (too much walls between) don't expect 5GHz to work better. The loss through walls is higher on 5Ghz, there are some problem with the legal regulation-limits in the 5Ghz so you probably stuck at 17dBm on 5GHz if running openwrt. The one big advantage of 5Ghz is that this spectrum isn't as crowded as the 2.4GHz spectrum, but on any other aspect 2.4GHz may be superior.
