Post #4
bolvan
24 Mar 2016, 10:00

Driver for network card is absent in the system
Boot x86 from linux livecd/liveflash. Run terminal, execute lspci "-k", find network adapter, remember module name.
Then search for kmod-<modname>-*.ipk  at http://downloads.openwrt.org
Download module for correct openwrt version, put it to usb drive using linux (because ext4 cant be read from windows), boot openwrt from the stick, install ipk with "opkg install"

remlei wrote:

to be honest, they should just include all the drivers from ethernet and wireless cards, since we actually had gigs of space available in x86 environment.

Unlike traditional linux that use modprobe and udev  openwrt does not have them. Modules are preloaded unless manually removed from /etc/modules.d.
Not good to have every possible module always resident
But I agree - good to have all widespread nic modules such as intel and realtek. They are not so numerous. Also good to have latest drivers for realtek because r8169 always caused panics and restarts in some situations and they were unable to fix it for years. Happily it was recently fixed.
