Hi all,

Set-up:

ASRock ION330HT
- Cisco/Linksys WUSB600N v2 USB wireless adapter, Ralink rt3572
- Linux Ubuntu 10.10
- kernel 2.6.32-23 with Ralink driver (version 2.5?)
- wpa_supplicant using wext driver

D-Link DIR-825, OpenWRT

Dell Precision M4500, Intel 6200 AGN wireless adapter, XP SP3

The router is running with 2 both radios on, one SSID on 2.4GHz, another on 5GHz (channel 36). The ASRock is connected via channel 36 to the router. The laptop is connected on the same channel 36, no other device is connected to the router on the 5GHz band. The ASRock is around 4 meters away from the router, the laptop about 2 meters. Both of them show that they are connected at 300Mbit/sec

I'm getting really slow transfer rates between the laptop and the ASRock (my XBMC media centre btw). I'm talking 2.5MB/sec *max*. I've tested this transfer speed both using an SMB mount between windows and Linux (samba) as well as using plain simple FTP.

In order to eliminate the laptop from the equation, I've added a USB flashdrive to the router, and done an SFTP from the Linux box to the router. Same results with respect to the transfer speed: somewhere between 1.5 and 2.5 MB/sec on average. Will probably retest that tonight with a USB HDD that's known to perform well, to rule out any slow-down due to the flashdrive.

So thes transfer rates are really disappointing... to be honest I'm hoping for something in the range of 50-60MB/sec, and based on a few google searches I believe that that is possible with wireless-N@300Mbit/sec connections.

Does anybody have any ideas about:
- is this a hardware limitation somewhere?
- is this a software limitation?
- is this misconfiguration? If so, is there any configuration information I can supply that would help analyze this? I can post more details tonight if requested.

Kind regards, Maarten

ps. as a comparison, previously I used the internal Atheros wireless card in the ASRock on the 2.4GHz band, I think it connected at 150Mbit/sec, and I got transfer rates on that of under 1MB/sec....
