trunk doesnt actually work on 4/16MB machines like the DIR300A because mainly of RAM memory

Failsafe never worked for me, neither does sysupgrade ( low memory) or opkg, use serial connection instead

For fresh flash i do use fon-flash (Gargoyle et.al)

CAVEAT:  AA images  contain LUCI by default, so same lowmem problem.

So your are left with little choice :
   Image generator
   buildroot

If you tell me what you need included in the image i can make one

I do use alot of Foneras 2100 and DIR300A1 in batman-adv mesh and they are working quite nice, even with adhoc-mesh+AP

regards
3zl

3zl

Try as described in http://wiki.openwrt.org/de/doc/howto/generic.sysupgrade ( german english mix )


rm -r /tmp/opkg-lists/

caches clear:

echo 3 > /proc/sys/vm/drop_caches

stop IEEE802.11-driver loading on next Reboot

rm /etc/modules.d$/*80211*
rm /etc/modules.d$/*ath9k*
reboot

should give some memory headroom to sysupgrade -n

Still an option is to try one of the flash programs (Windows / Linux)

regards
3zl
