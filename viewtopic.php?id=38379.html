There are two level loopback test:mac level and phy level.Some days ago,i use the atheros ar8021 giga phy and do the loopback test on it,i set the ar8021 regiser let it enter the loopback mode,and then write a loop,in the loop,first tx a packet and then rx a packet,what you rx is what you tx,and no tx packet lost.you can define the loop count or loop time,if no packet lost and no packet error,you can think the loopback test pass.

But if you wan to test a switch,i think you can do it by devide vlan,different port in different vlan,work as different interface in linux,and then connect port 1 and port 2 with etherent cable,port 3 and port 4 with ethernet cable.then write a loop tx/rx packet between different interface,in this way you don't need care the loopback register in the switch.

i don't know which test is better,maybe the "vlan" way is good than the phy level loopback test,but just maybe ...
