Hi

I have a rather big lan network with multiple ethernet switches in a building.
I would like to cover this large building with 2 or 3 WLANs on different SSIDs with a lot of access points.
Of course, each SSID must be connected to a single VLAN to enable roaming.
One of the SSID will be a public hotspot operated with a single coova chilli program.
The other SSID will be private.
The access points will be accton mr3201 routers with openwrt. They have an atheros chipset which support multiple SSID, but they have a single ethernet port.
If there is a single ethernet port, does this mean that the router have no hardware switch inside?
If I create networks like eth0.0, eth0.1, eth0.2, will the ethernet frames on the port be automatically VLAN tagged?
If there is an internal switch, do I have to configure it like this?

#### VLAN configuration 
config switch eth0
    option vlan0    "0t 5*"
    option vlan1    "0t 5*"
    option vlan2    "0t 5*"

0t should mean that port 0 must be tagged?

I don't know well how tagged frames are handled by SOHO ethernet switches. What does a standard ethernet switch do with tagged ethernet frames? are they forwarded as is to all other access points?

Can I bridge eth0 (untagged) to one ssid and eth0.1 (vlan1) to another ssid?

Thanks in advance for your answers

The references I found to this device suggested it was most like a Fonera, and that has no switch inside it; you just configure the ethernet interface normally.  Make sure the ethernet interface supports oversize frames, or reduce the MTU on the untagged networks.

Perhaps opening it up and taking some high-res pictures might help.
