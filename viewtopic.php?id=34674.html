Hello!

I have a problem using WPA2 EAP-TLS with FreeRADIUS in AP mode on my TP-Link WR-1043ND router, under OpenWrt 10.03.1.

I use the following config on the router (it's is from 10.03, but is use the same values on 10.03.1 through the web interface) :

config 'wifi-iface'
    option 'device' 'radio0'
    option 'network' 'lan'
    option 'mode' 'ap'
    option 'ssid' '<my ssid>'
    option 'encryption' 'wpa2'
    option 'key' '<secret key>'
    option 'server' '<server-address>'
    option 'port' '<server-port>'
    option 'hidden' '1'

The clients are connecting to the router with wpa_supplicant, using the following configuration:

ctrl_interface=eth1
ctrl_interface_group=nogroup
network={
    ssid="<my ssid>"
    scan_ssid=1
    key_mgmt=WPA-EAP
    pairwise=CCMP
    group=CCMP
    eap=TLS
    identity="<my identity>"
    ca_cert="/etc/certs/cacert.pem"
    private_key="/etc/certs/key.p12"
    private_key_passwd="<my password>"
}

And failing with:

Authentication with xx:xx:xx:xx:xx:xx timed out.

The configuration above works fine with Openwrt 10.03, but fails with 10.03.1.

The FreeRADIUS server shows no error messages in the logs nor with "freeradius -X", only "Ready to process requests."


Thank your for your help.

I tried the 10.03.1-RC versions of OpenWrt, and they produced the same symptoms, but it looks like it was a client problem, because I tried the configuration with another computers and it worked flawlessly under both the RC versions and the 10.03.1 final.

Seems like with a combination of a specific version of a Broadcom wifi-chip and the wpa-supplicant the configuration fails (the authentication request didn't reached the Radius server, the logs always showed 'Ready to process requests.', and no error messages), but with other devices it was ok.
