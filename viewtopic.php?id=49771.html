Slightly late, but I stumbled across this post while trying to do the same thing myself.

I managed to get OpenVPN to use Google Authenticator and a Yubikey configured for TOTP... on my Raspberry Pi. And having got it working on the Pi I wanted to do the same thing on my OpenWRT box.

The stumbling block is the PAM authenticator plugin for OpenVPN. There isn't one. At least, there isn't one conveniently already compiled for OpenWRT.

There is a Google Authenticator package for OpenWRT, but all that provides is the OATH framework. Individual services (e.g. SSH, OpenVPN, etc.) must be able to talk to Google Authenticator before you can authenticate. And without a suitable PAM plugin compiled for OpenWRT, OpenVPN can't talk to Google Authenticator.

I've never compiled for OpenWRT before (the most I've ever done is fiddle with Image Builder to get OpenWRT on a 4MB device) so I've absolutely no idea what might be involved in a) finding the source code for the OpenVPN PAM plugin, and b) compiling it for OpenWRT.
