I have also an DIR-652 Version A1. Since I flashed the newest FW (Firmware Version: 1.06) I have no more possibility for static routes on LAN-side.

There is a possibility explained here:

http://www.hyllander.org/content/d-link … -route-lan

But it will not work for me. There is an error message: "Destination and Gateway should not reside on the same subnet 255.255.255.0"

I try  to tell this gateway, where to give IP-Packets for network: 192.168.220.0/24 to 192.168.178.99. This is my Lan-Gateway for Packets in this network.

Background: I try to build an multiple-WAN-Gateway. At the moment I try to use it out of my Desktop-Ubuntu.

Unfortunately it does only submit every second packet (average) into the 192.168.220.0/24 network. I tried other examples on other devices, so now I am pretty sure, the problem is, that DIR-652 route packets for network 192.168.220.0/24 into WAN insted into LAN-Side to 192.168.178.99.

So this is an big issue for me and let me try since to days and night to build an customize fw.

It is really impressive to see what you can chosse in the menu of uClinux und OpenWRT for this device. There is a README of DLink which describe how to get an definied own FW. Finally I got this step and have a ready compiled fw. But I can't try it at the moment, so perhaps some brave person try to test it?

It is for a DIR-652 Revision A1 with Ubicom 7150 CPU. When you compile D-LINK says you should choose DIR855 Layout (as described) because there is no specified DIR-652 Layout.

I would love to get OpenWRT work. Actually I saw there is even a driver for the LAN-Switch and Atheros-WIFI-Card in the GNU GPL attached.

Meanwhile I think to fix every error, but ther is still an one I can't find to compile OPENWRT and a customized uClinux.

I would love find some other people intersted to get it work! This machine seems to be really fast! Faster than the Nanostation M2 and mor flexible! The Chipset is also from Ubicom.

So why don't we develop a good customized Image? The original Image seems to have 6 MB and the flash can store 8 MB. So there is a little bit flexibility.
