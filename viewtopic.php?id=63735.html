I'm late to the party, but I can confirm that I got the same key from keys.gnupg.net (using gpg --recv-keys 0xbea8f6e25378aaf8) and that this *subkey* is used to sign the sha256sums and md5sums files that I've downloaded from the OpenWRT download page. 

Note however, this doesn't necessarily prove that 5378AAF8 is the correct OpenWrt signing key.  I think it may be possible that anyone can create a new primary key that includes the subkey 8775B24C used to sign the sha256sums and md5sums files, and that the owner of the primary key is not necessarily the same person/group that controls the subkey. 

If someone of authority can publish the OpenWrt signing key's fingerprint on the website, that would make this easier to check.
