May I ask why we do not have any comments in any configuration files located in the /etc/config/* folder? It would make it a whole lot easier to read and understand what is happening. I agree that Wiki is a great source of information, but it is incomplete and sometimes even incorrect, always un-accessible without internet connection. Here is an example from /etc/config/firewall that demonstrates how a single comment can clarify lots of guesswork for people not familiar with the OpenWRT default configuration.

#Allows router to respond to pings
config 'rule'
        option 'src' 'wan'
        option 'proto' 'icmp'
        option 'icmp_type' 'echo-request'
        option 'family' 'ipv4'
        option 'target' 'ACCEPT'

Reading *.config files without comments is like reading raw, un-commented source code.

1. Comments will make it easier to maintain configuration files
2. Prevent obvious configuration mistakes
3. Warn about specific settings
4. Make config files easier to understand for new users

Please let me know what you think?
