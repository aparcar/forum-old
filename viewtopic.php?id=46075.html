One idea for you:
My router rebooted randomly due to a memory leak in a program. Router ran out of memory and rebooted. That kind of error is hard to notice, as after the reboot there is no trace, as all logs are normally only on ram and disappear during reboot. Developers fixed that leak after I had identified the culprit module (see https://dev.openwrt.org/ticket/10630#comment:3 onward )

You might use htop or some other similar tool to monitor the amount of available memory.
