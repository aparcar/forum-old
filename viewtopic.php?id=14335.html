I have run Kamikaze on WRT54GL 1.1 and I did not notice any problems.  I did not use Kamikaze for long on that router, though.  I just tried it out.  You should probably stick to the 2.4 version of the kernel, though, since I think the 2.6 wireless drivers for the WRT54GL are still a little experimental.  (i.e. the Kamikaze image that comes with kernel version 2.4.)

By the way, Kamikaze does not use nvram.  It stores the settings in files in /etc/config.  You can use the "uci" command (similar to the nvram command) or edit the files directly.

Kamikaze does not come with a Web interface, although the X-Wrt people are working on that.  I haven't tried X-Wrt, but according to their web page it looks like it should mostly work on Kamikaze.

A tip for searching the Wiki:  If you don't find something by clicking on the "title" button, try the "text" button.

Sammy, unless you want 2.6 kernel with no wireless I'd suggest staying with Whiterussian.

While I got the 2.4 kernel of Kamikaze running I've never put it into "production" use because it lacks many simple configurations (simple AP mode has no cionfig for txpower, no ability to run both tkip and aes... to name a few).

I made my own unofficial bug fix for Whiterussian router client mode if anyone needs it.

Broadcom wireless (the b43 project) is getting close but not there yet.  That should change things (the 2.6 kernel on free firmware/drivers).

Basically, Kamikaze 7.09 2.4 kernel gives NO benefit over Whiterussian 0.9 on a WRT54GL I can think of at this time.

HTH
