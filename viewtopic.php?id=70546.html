The "dumb AP" requires a wired connection to the main router.  If you try to use wifi with a standard AP--Client  connection from a main router as the AP, you can't readily bridge back multiple users making everything on one LAN.  This is because standard AP implies that a client is only one MAC address (one device) at the end of the link.

The relayd package attempts to work around this.  It doesn't always work very well.

If you can run OpenWrt on your main router you can use AP-WDS mode or mesh.  These allow seamless bridging over a wifi link, because an additional MAC address of the final destination is passed back.
