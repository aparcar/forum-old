Hi,

1st of all you only need openwrt-DSL2740B-squashfs-cfe.bin and not the others.
2nd thing is that by default the 'web interface' or the GUI is not installed, this is called Luci and should be installed using telnet and opkg.

so do the following:
1. Flash the router with openwrt-DSL2740B-squashfs-cfe.bin.
2. Set your ip to manually be 192.168.1.2 255.255.255.0
3. Telnet to 192.168.1.1

Now comes the harder part, which is to manually configure the interfaces thru terminal (telnet) until you are able to connect online, once online you can do the following:
4. opkg update
5. opkg install luci

once luci is install you should reboot the router and you can now access the web interface by typing 192.168.1.1 in your fav. browser.

again these are only high level instructions so you understand the process.

I've spent around 3 hours to install the openWRT on this device, and to avoid further unnecessary confusion, I'm writing this tiny tutorial for those who own the same router as mine. My DSL-2740B has a C3 firmware version.

Prerequisite firmware:
C3 "trunk" from http://downloads.openwrt.org/snapshots/ … fs-cfe.bin
or
C2 "trunk" from http://downloads.openwrt.org/snapshots/ … fs-cfe.bin

For fail-save firmware, I used the one downloaded from ftp://files.dlink.com.au/products/DSL-2740B/

The only reason I spent hours on installing is that I failed to login using the C3 version, while the C2 version works like a charm. So if you have the same problem as mine, try the C2 version instead.

PuTTY for the telnet session: http://www.chiark.greenend.org.uk/~sgta … nload.html

For D-Link DSL-2740B C3 installation methods:

1. I'm not sure if this is necessary step, but I first do the Factory Reset on the original firmware, under the Advance / Maintenance tab.

2. Power off the router, and using a pin gently press the reset button at the rear. Power on the router now and keep pressing the reset button until the power indicator turn the color from orange to red. Then you may release the reset button. Plug in your ethernet cable and connect it to your computer if it's not connected yet. (Yes, the firmware is uploaded while the power indicator light still red, so don't restart the router. And no, you would not be able to use the update firmware menu on the regular web interface, because it will simply tell you the firmware is outdated blablabla...)

3. Change your computer IP address manually to 192.168.1.2, netmask: 255.255.255.0, gateway 192.168.1.1. Then using your fave browser open the page to http://192.168.1.1
Choose the downloaded firmware, and upload it to your router.
Leave it for around 2 minutes until you will see the router reboots. No need to disconnect the ethernet cable during these process.

4. Wait for the reboot process to finish (power indicator green light stable, not blinking anymore). Use PuTTY to "telnet" into your router, the address would be 192.168.1.1
If it fails you to connect via telnet, first try to power off and on the device and wait 2-3 minutes then try again.
If it still fails to connect, then repeat procedure number 2 and 3 with the C2 version of the firmware. Chances are it will work like mine.
If you want to revert back to the original firmware, exactly do the same thing as number 2 and 3, while choosing the original firmware to be uploaded.
It's that easy and practically I think it's a minimum chance you would brick this device, since you can always have the engineering page to reupload the original firmware.

For the post installation stuffs, you may go on and have a further reading on the forum.
The only hassle is that when your router don't have the internet connection to download the rest of the packages to make the router works the way you want it. (I don't have a DSL line, so I'm downloading the packages manually and installing it offline now.)

That's it. Have fun.
