Hey,

So I really like having the serial console on my Mikrotik device, but I also want to be able to use that port for a GPS receiver.

I was able to comment out the relevant lines in my /etc/inittab so that the console was disabled altogether, but now my only access to the device is through ssh... if I somehow mess up the networking on it, I won't be able to get back in.

What I'd like to be able to do is have a 1-2 second window after bootup, in which I can press a specific key and get the serial console, but after which it will be killed and I can use the port to talk to my GPS.

Does this sound reasonable? What would be the best way to set up such a thing?

Thanks,

Mike

This isn't a perfect solution, but what I've found I can do is:

1. In /etc/inittab, change the "askfirst" actions to be "once" instead.
2. In the startup for the component which talks to the GPS, wait an appropriate period of time and then just use a combination of ps and grep to find the process number of the ash instance, and kill -9 it. In a more advanced version, this could also be triggered by a network ping or some other external indicator that it's okay to go ahead with the GPS activity.
