Well I can't say I know what Homegroup uses because... I don't use it. However I have seen it working behind a bridge many times, but those are "true" bridges not pseudo-bridges like relayd. In order to work, any wireless network or ethernet switch is a bridge.

Personally though I was brought up in the days before Homegroup existed and learnt to share files using... file sharing. Which works perfectly fine behind any bridge or even over the internet. I don't actually know anyone who uses Homegroup personally.

Plus we do already have a version of relayd that supports ipv6 - perhaps the developer of homegroup should spend more time in supporting non-standard networks. Or perhaps the owners of said networks should spend more time making their network behave properly, without using hacks like pseudo-bridges-but-not-real-bridges-pretending-to-be-bridges-and-breaking-various-protocols-by-lying-about-device-ids-and-not-being-a-real-bridge.

Rampler, this is an old topic, may I ask what your dhcp and or relayd settings are to make ipv6 work over the pseudo bridge?  My lan stanza currently looks like:

config dhcp 'lan'
        option interface 'lan'
        option ignore '1'
        option ra_management '1'
        option dhcpv6 'relay'
        option ra 'relay'
        option ndp 'relay'

This is on a WZR-600DHP running Barrier Breaker and otherwise the pseudo bridge is working very well for me.
