Hello,
So, I was trying to install openwrt on my Rev D router. I started with upgrading redboot using pyjungo following the instructions on http://wiki.openwrt.org/toh/actiontec/mi424wr. I then followed the instructions on https://forum.openwrt.org/viewtopic.php?id=30545. All seemed to go well.

However on the reboot I couldn't connect to the router on the lan ports. I tried the snapshot and latest backfire release to no avail. I thought that I'd try to restore the image I'd backed up and see if I could telnet in to verify the mac addresses. I followed the instructions on the wiki page to reload the backup image from redboot. It seemed to go Ok. However when I rebooted the router I end up with a red power led. If I power up with the reset button held in I end up with an orange power led.

I saw some postings implying it was in the redboot stage with the orange led. With a static ip I can ping 192.168.1.1 on the lan but not on the wan. I tried telnetting to different ports including 80 and 443 in both the red and orange state but with no luck. Is there a different port I should try?

What thoughts are there on my next step? Firstly I'd like to get a router that I can interact with, then I'd like to get openwrt on it.

Well I didn't brick it then - but maybe I have now ...

After searching various things on the net I found from the dslreports.net forum that with the orange power on led that it was running a tftp server. I was able to put an Actiontec firmware back on the router.

Hopefully this helps someone else.

However, I think in my tired state I hosed it again. First of all I thought I was enabling telnet. However I enabled it on the WAN and took me a while to realize that I hadn't enabled it on the lan ports. Doh.

After doing that I went back to the jungo-image and meant to put the revised red-boot. However, I foolishly installed the ram-image. Now I don't seem to have any boot loader.

I get no power LED, I  get green LED for the wireless, wan and lan port. But I have no dhcp, no 192.168.1.1. I've tried the 30/30/30 but that makes no difference.

Before I found the tip about the tftp server, I was looking at installing a serial connection. Maybe I still need to try that. Maybe I've just really f'ed it up now.

Why are some things so difficult!!!
