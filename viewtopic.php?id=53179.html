Hi there,

For a script I'm writing, I need to delete all the VLANs in the /etc/config/network file. Let's say I have:

config switch_vlan
    option device 'switch0'
    option vlan '1'
    option vid '1'
    option ports '0t 1'

config switch_vlan
    option device 'switch0'
    option vlan '2'
    option vid '2'
    option ports '0t 2'

config switch_vlan
    option device 'switch0'
    option vlan '3'
    option vid '3'
    option ports '0t 3'

For which I know I could do:

uci -q delete network.@switch_vlan[2]
uci -q delete network.@switch_vlan[1]
uci -q delete network.@switch_vlan[0]

The problem is that I don't know beforehand the number of switch_vlan sections to remove.

What would be the polite way to remove all the switch_vlan sections, whether if there's one or five? I mean, something like "uci -q delete network.@switch_vlan[_all_]"

Thank you very much!
