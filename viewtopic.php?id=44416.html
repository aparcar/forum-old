I think you should connect to the GPIO using a TTL device for you to interface with Openwrt in raspberry pi. I have tried using the GPIO with raspbian and other OS for the pi and it works beautifully. In my experience, for you to be able to configure the Openwrt system, you should connect a USB-to-serial adapter to it. That is how most of the stuff here are being done.

With regards to the PI not requesting an ip, that might be because of the default setting in the network which you should configure further.

I haven't tried Openwrt with my pi yet so these are just my assumptions.
