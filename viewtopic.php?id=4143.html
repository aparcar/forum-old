If you're so desperate you should try compiling and getting it to run, just skip the packaging step (regarding the earlier cross-compiling post).
I do agree with the replies on the other thread.. running mysqld on an embedded router is not exactly the greatest idea, but when has that ever stopped anyone .

To get some idea about the complexity of the code compilation seemed to take longer than kernel and the toolchain combined (didn't time it though).
Didn't get around to actually package the binaries and probably won't have the time until weekend, so here's the basic walkthrough about the compile process.

First remove line --without-server from package/mysql/Makefile, compile libmysqlclient, go into builddir and run make.. in theory that's basically it . But there are few caveats as we're cross-compiling the host system can't run target binaries (although in theory it should be aware of that and deal with it).. it tries to run extra/comp_err to regenerate some header files, but they're already fine, do..

touch include/mysqld_error.h include/sql_state.h

The second one is trickier, sql/gen_lex_hash want's to generate lex_hash.h for seeking hash algorithms, it should be possible to run from target device but it didn't succeed immediately so I replaced the header file with a googled one for the time being to get forward.
Then iirc there was a small issue of untranslated autoconf variable of @pstack_libs@  in mysqld_LDADD of sql/Makefile, just remove it.

Didn't have enough suitable usb mass-storage at hand so went off to compile cifs.ko and appropriate mount-util at this point, in order to run it over diskshare..

On server mkdir some setup dir (and share it over the lan).

cd ../toolchain_build_mipsel/gcc-3.4.4-final/
cp gcc/libgcc_s.so.1 mipsel-linux-uclibc/libstdc++-v3/src/.libs/libstdc++.so.6.0.3 ../../build_mipsel/mysql-5.0.18/scripts/mysql_install_db.sh $OLDPWD
cd $OLDPWD
mkdir data share
cd share; ln -s ../../build_mipsel/mysql-5.0.18/sql/share/ mysql
cd ..; cp ../build_mipsel/mysql-5.0.18/extra/resolveip bin
mv scripts/mysql_create* bin

Setup the openwrt side.

root@OpenWrt:/# echo daemon:x:2:2:daemon:/sbin:/bin/false >>/etc/passwd
root@OpenWrt:/# mount.cifs //server/openwrt /mnt/smb -o user=..,rw,uid=daemon
root@OpenWrt:/# cd /mnt/smb/somedir; umask a+rw
# chown -R daemon on server
root@OpenWrt:/mnt/smb/somedir# LD_LIBRARY_PATH=. ./scripts/mysql_install_db --datadir=data --basedir=. -user=daemon --socket=/tmp/mysql.sock --bootstrap --skip-grant-tables --skip-innodb --skip-bdb --skip-ndbcluster

It doesn't seem to be able to create the databases so I copied other ones from another server.. guess there was some fooling with the scripts at this point as well, don't have the shell history to check it out, but /etc/my.cnf should at least have socket = /tmp/mysql.sock.

Then try it out.

root@OpenWrt:/mnt/smb/somedir# LD_LIBRARY_PATH=. ./mysql -u root -p
Enter password:
Welcome to the MySQL monitor.  Commands end with ; or \g.
Your MySQL connection id is 9 to server version: 5.0.18

Type 'help;' or '\h' for help. Type '\c' to clear the buffer.

mysql> \s
--------------
./mysql  Ver 14.12 Distrib 5.0.18, for unknown-linux-gnu (mipsel) using readline 5.0
