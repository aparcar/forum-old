Run ifconfig to see if interface is up.
I tried default x86-64 compilation on vmware and it didnt work because required ethernet driver was missing. Had to install additional kmod : kmod-pcnet32. Had to copy required ipks to openwrt filesystem because network was unavailable. Boot from linux livecd in the VM and mount virtual hard disk filesystem, transfer files.
Next - you need correct configuration of /etc/config/network. By default vmware offers NATed network with dhcp and openwrt defaults to static 192.168.1.1/24 which will not work.
And what wifi connection are you talking about ? How are you going to have wifi adapter within VM ? Its easy in case of USB wifi stick, but if its not usb then its not possible to bring wifi adapter to VM.
Even if you assign USB stick to VM then you need correct kmod for it. Not all chipsets are supported, you can find linux does not support your adapter. I'd first boot to a linux livecd with new kernel and look if a kmod assigned to the stick. Command is called "usb-devices"
