Hello, I recently found a way to root my NVG510 U-Verse modem. This modem is notoriously known as being a really bad and buggy. I had an extra one laying around, so I opened it up, soldered some stuff to get to a serial port and had my first exercise in reverse-engineering. And I figured out how to fix nearly all of the problems with the modem as well, and got to a root shell.

When looking at the code for it's web user interface, I came across a huge security flaw that allows for rooting without opening the modem.

The modem uses a BCM6362 SoC, the same kind of "series" OpenWrt aims at. Some technical information(including GPL source links) can be found here: http://www.wikidevi.com/wiki/Motorola_NVG510

I've documented all that I've found out so far.
The solder-free web vulnerability method to get to a root shell can be found here: http://lastyearswishes.com/blog/view/4f … 385420c706
And more information about it's configuration and opening-it-up method can be found here: http://lastyearswishes.com/blog/view/4f … 385420c705

I would really love if there was an alternate OS for this modem, though I'm not for sure it's possible. It seems to be a complete other beast from what OpenWrt usually tackles. This is a modem that can't have an equivalent of PPPoE because there is no "username" or "password" to send.

Anyway, I just thought I would enlighten the community to this development. I know I've seen quite a few people having trouble getting the NVG510 to bridge to OpenWrt routers. While it'd be awesome to see a port of OpenWrt to the NVG510, I'll settle with just providing the community with a "true" bridge mode for the router detailed in my two blog posts.

If anyone has any questions or anything to test, I have an extra NVG510 that I can test experiments on

Note: Most U-Verse customers are forced to use the NVG510. There is no option for another modem. I'm sure a stable port would receive a lot of attention
