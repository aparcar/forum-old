I am trying to run so that clients connecting behind dnsmasq request for www.mywebsite.com get an IP address of the router, while the router (and code running on it) can use the real DNS.

Is there a way to configure dnsmasq so a client connecting use this entry from /etc/hosts:

www.mywebsite.com 192.168.1.1

while the router can use the real IP of www.mywebsite.com?

Post #2
eduperez
25 Jul 2015, 00:25

You can add an entry such "address=/www.mywebsite.com/192.168.1.1" at "/etc/dnsmasq.conf", and it will work for all clients.
But OpenWRT also uses dnsmasq as a DNS cache, so it will also affect the router; not sure how to achieve what you need.
