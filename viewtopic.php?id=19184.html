I'm discussing with a programming partner of mine building a crude website that would let you build custom firmware images and download them.  I've also got server resources and some bandwidth to offer.

The main desire is to avoid having to use package manager and to get people testing the newer Trunks on the 802.11N platforms routers.

My early thoughts are for version 1.0 is to frontend the normal make menuconfig system with a web form something like:

Pick Target System: [Atheros AR71xx/AR913x (2.6)]
Pick Target Profile: [TRENDNet TEW-652BRP]

Modules you want added: [ kmod-ipv6 kmod-ip6tables ip6tables-utils kmod-ip6-tunnel kmod-iptunnel4 kmod-iptunnel6 ]

some key design aspects to work out:

1) The system would come up with a unique identifier for each:  SVN_version + target_system + target_profile + module_list  .  Such an identifier would be used to name the output file and present description of the build.
2) All requests would go into a queue and generate one at a time.  It isn't like OpenWrt is _that busy_ at this point
3) Every 24 hours the build tree is deleted and fresh checkout from SVN on latest Trunk

Obviously this could evolve... but I would rather get something primitive up than overperfect it.
The overall goal is to make it similar to the normal make menuconfig system so that user can transition their experience with a particular combination of packages to their own local system.
