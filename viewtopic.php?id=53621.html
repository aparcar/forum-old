I have installed syslog-ng3 and defined a remote syslog server.

However, the remote syslog server says, "Unable to open UDP socket on port 514".

So I think that the router's port 514 is closed. I defined the following rule in the firewall:

config rule
        option target 'ACCEPT'
        option src 'lan'
        option name 'syslog'
        option family 'ipv4'
        option proto 'udp'
        option src_port '514'
        option dest_ip '192.168.10.99'
        option src_ip '192.168.10.1'
        option dest_port '514'

Why is the port 37902? Shouldn't it be 514?
