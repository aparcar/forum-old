rafkeduck
4 Feb 2013, 02:46

Based on that error, I would try to confirm that putty is really using public key authentication, and is using the right key.

/etc/dropbear/authorized_keys should look something like:
ssh-rsa AAAAB3NzaC... <name of key>

What dropbear messages do you see (logread). Successful authentication will look like:
dropbear: Pubkey auth succeeded for 'root' with key md5 ...

This is easier to troubleshoot with the OpenSSH client. You would do 'ssh -v openwrt' and it would provide plenty of information about the authentication methods and keys offered. I do not know if putty has a similar option.
