Broadcom uses its own buses that we call "ssb" and "bcma". On bus there are multiple devices, take a look at this random log:

bcma: bus1: Found chip with id 43217, rev 0x00 and package 0x08
bcma: bus1: Core 0 found: ChipCommon (manuf 0x4BF, id 0x800, rev 0x27, class 0x0)
bcma: bus1: Core 1 found: IEEE 802.11 (manuf 0x4BF, id 0x812, rev 0x1E, class 0x0)
bcma: bus1: Core 2 found: PCIe (manuf 0x4BF, id 0x820, rev 0x14, class 0x0)
bcma: bus1: Bus registered

Bus device = core.
Every core provides some function, for wireless you need a bus with "IEEE 802.11" core.

As Broadcom keep developing new wireless cores (new standards, performance, etc. ,etc.), there are a lot of "IEEE 802.11" cores, they vary by a revision (and few more details too).

"Core revisions to include" is probably an option to include firmware files for selected cores only.
