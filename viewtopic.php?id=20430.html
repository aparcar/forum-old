Hello everybody,

I have a problem about my OpenWrt box, an Asus WL-520gU with Kamikaze r13325 and kernel 2.4. When someone in our house uses a P2P software (either Emule or Bittorrent), Web navigation suddenly becomes quite difficult: I need to click Reload several times on our browsers to successfully (or even partially) get one page.
What I can see, either from my PC, from my colleagues' PC and from the router, is that ICMP pings go trough flawlessly; instead, any TCP connection (tested via wget -O- www.google.com > /dev/null) has something like 20-30% probability to end up with a timeout and a "connection refused" error.
I also see that this behaviour is probably correlated with the number of active TCP/UDP connections (wc /proc/net/ip_conntrack): when this number goes higher than 300 the web navigation becomes annoying; when it returns to lower values, everthing is OK.
This behaviour doesn't affect existing TCP connections' bandwidth, only newly created ones. There's no difference in using the wireless or wired interface, and it isn't related to our ISP's connection since when not using OpenWrt box, everything goes well.
The load factor keeps always on low values: 0.10, 0.05, 0.00.

Any help is very appreciated!

Best regards,
Enrico
