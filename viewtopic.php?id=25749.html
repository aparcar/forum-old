I've installed backfire (10.03) and add the package Coovachilli (1.2.1). Wifi, dhcp by coova and so are ok.
When a client connect to the hotspot, he's got an ip (10.1.0.x/24), arrives on the "redirecting" coova page. Then coova tries to redirect him to "http://10.1.0.1:3990/www/login.chi"

There is no login.chi (or any others miniportal files) in the package ; /etc/chilli/www/ contains a few files only. I've checked an RPM of Coovachilli for Mandriva and also the source from cova.org. There are many more files (including login.chi).
Is there any reason for the difference ?

Also I've of course tried to copy all the files to /etc/chilli/www, check with netstat -an that openwrt is listening on 80, 3390 and 4490 on ip 10.1.0.1, but the redirection is still not working.

Hello,

auto-answering.
So indeed the Coovachilli package for Backfire is using /etc/chilli/www/login.chi in the config files, and does not provide /etc/chilli/www basic files.
You can copy files from coovachilli sources or from an older installation.
My previous test was not working with copy due an error during transfert.
