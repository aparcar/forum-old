Stanley Pinchak
24 Sep 2011, 18:21

e3000 and WRT-610N-V2.0 are identical hardware wise, just a different CFE and nvram config from what I gather.  They are not very usable at the moment as the support for 2.4 kernel has been dropped from Trunk and there is no 2.6 support for the WIFI SoC and stand alone chip, or even the switch.  There is a new 2.6 based linksys firmware, so hopefully a GPL release will be coming along soon.  It is my understanding that e4200 uses the same WIFI SoC and switch, but a different secondary WIFI chip, so it is in a similar situation.

Post #5
Stanley Pinchak
25 Sep 2011, 03:30

I stand corrected.  There is a patchset for E3000 and WRT610n-V2.0 routers which enables part of their capabilities.  The SoC WIFI (2.4GHz) and switch are now working with the proprietary broadcom drivers on a 3.0.4 based build.  The stand alone WIFI chip which provides A and N (5GHz) capability does not appear operational yet.  The patchset is described:
http://www.znau.edu.ua/temp/asus-rt-n16 … README.txt

Some of these findings have been included in the TOH entry for the E3000.  Unfortunately, it still appears that these routers will not satisfy your current need until the other radio is enabled.
