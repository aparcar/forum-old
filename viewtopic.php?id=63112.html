I have been messing around with OpenDNS recently. My IP is dynamic and the No-IP script works great for No-IP. I was looking for a way to update my IP with OpenDNS as well. I tried using the dnsomatic.com option, but the problem is that the script uses [HOSTNAME] to check before updating, which is fine, but the dnsomatic.com API needs the [HOSTNAME] value to be all.dnsomatic.com when the update request is sent. If I set [HOSTNAME] to all.dnsomatic.com it doesn't resolve and breaks the script. I found a simple workaround I wanted to share with the community.

Edit /usr/lib/ddns/services. Change the value [HOSTNAME] in the URL dnsomatic.com to all.dnsomatic.com. See below:

"dnsomatic.com" "http://[USERNAME]:[PASSWORD]@updates.dnsomatic.com/nic/update?hostname=all.dnsomatic.com&myip=[IP]"

That's it. Now the script still checks based on the [HOSTNAME] value specified in /etc/config/ddns for dnsomatic, but when the update happens it uses the all.dnsomatic.com so the dnsomatic API is happy.

BTW dnsomatic also updates my No-IP IP as well so I just have the one ddns service configured. Nice and easy!
