PulseAudio firmware for TP-Link WR703N

This firmware is primarily aimed at those wants to easily setup multi-room sound system or wireless audio/speakers system. It is based on openwrt trunk r28854 and loads of cutomisation. In order to be able to make any sound you will need either a usb soundcard or a usb speaker(s). The core sound streaming/driving engine is provided by the latest and greatest pulseaudio 1.1.

WiFi in client mode is enabled by defult and it's exepcting SSID/KEY of Pulse/UEC7wzdu with WPA/WPA2 authentication. If you can change your AP's SSID and KEY then you will never need to touch WR703N's setting at all. Otherwise you should attach a cable to the WR703N and wait for the network card to be assigned an ip and then either use the web browser or ssh (putty) to manage the WR703N.

Three network setup templates are provided: AP, STA (client/station which is default) and WDS. You can found them in /etc/config folder on the WR703N. For example to switch to WDS mode all you need is to copy the two files over and reboot the router

cp /etc/conifg/network-wds /etc/conifg/network
cp /etc/conifg/wireless-wds /etc/conifg/wireless

Features:
1) Universal USB sound card support (USB 1.0/2.0)
2) pulse audio 1.1 with a few tweaks so it works smoothly over wifi
3) sound card detection script implemented for easy plug and play experience. You can pulg/unplug the usb sound card/speakers at any time
4) web gui powered by x-wrt and password protection disabled by default
5) WIFI in client mode on by default and it expects SSID/KEY of Pulse/UEC7wzdu with WPA/WPA2 authentication
6) Management IP 172.16.0.1 with netmask 255.255.255.0. Default root password is root
7) Optimised for space and speed

Limitations:
Due to the storage limit I have to make a decision of which to include and which not. Firewall (iptable userland and kernel modules), 3G support etc are not of the concern of those who will use this firmware and therefore has been excluded from this firmware.

It does not work well with multiple usb sound cards/speakers and I believe most users will only have one usb card/speakers attached to WR703N at any time. If you for any reason want to try a different usb sound card/speakers then you will need to reboot the router. However you don't need to do so if you unplug and then plug back the same usb sound card/speakers. It should just work, i.e. plug and play.


Battery Performane:
WR703N can be powered by any usb power source (5v with min. 500mA current). I hooked it up with 2 years old 4x rechargeable Ni-Mh batteries (2300mAh). WR703N runs in client wifi mode. The USB speakers I used is from Sweex (SP102 Go Stereo Speaker Bar). Music (resampled to 48K at server side) is played continously with volume set to 50%. The test started at 10AM in the morning and it still plays at 18:00 in that evening!! That's more than 8 hours continusly playback time via wifi and lound speakers.

Download:
The firmware can be downloaded from http://xbox-remote.googlecode.com/files … actory.bin

You can install it via web upgrade or mtd if you are running openwrt already.
