Hello.

I want to create a user that would be used for establishing ssh tunnels only. Such user should not have access to any shell.

I have added a new user by modifying /etc/passwd /etc/shadow and /etc/group.

And to disable shell, I set the shell to "/bin/false" in /etc/passwd. When I do that, dropbear starts failing my password authentication and I get a "Permission denied, please try again." from ssh client (I did use the -N switch to not try starting a shell). And dropbear logs this message: authpriv.warn dropbear[2378]: User 'somebody' has invalid shell, rejected.


Now through experimentation I discovered that if add "/bin/false" to /etc/shells file, then I can sucessfully authenticate to dropbear again, establishing a tunnel without a shell.

My question is whether this is the right solution? Might this introduce any problems? I googled a bit (click clack) and in general the idea of putting "/bin/false" in /etc/shells is discouraged but never explained with any examples - what and how might break?

Thanks for any input on this matter!


(Using 14.07-rc3, r42056)
