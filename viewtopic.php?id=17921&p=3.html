I found the solution for the stuck beacon message in another thread.

The madwifi-driver of 8.09 has a problem with channel 1!!!!

After switching to channel 5 the problem of disconnets disapears.

Here my config for a stable link and a goot reception rate (with some testing but no long time test):

config 'wifi-device' 'wifi0'
        option 'type' 'atheros'
        option 'disabled' '0' 
        option 'diversity' '0'
        option 'txantenna' '2'
        option 'channel' '5'
        option 'rxantenna' '2'
        option 'softled'    '0'

config 'wifi-iface'
        option 'device' 'wifi0'
        option 'network' 'lan'
        option 'mode' 'ap'
        option 'ssid' 'myssid'
option 'encryption' 'psk2'
        option 'key' 'XXXXXXXXXX'
        option 'txpower' 16
        option 'bursting' '1'
        option 'bgscan' '0'
