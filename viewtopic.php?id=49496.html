I want to assign different IP range according to vendor class identifier (option 60) provide by dhcp client. E.g. 192.168.1.10 to 192.168.1.100 for Android phone, 192.168.1.100 to 192.168.1.200 for ios.

Searched in the forum, seem it is not possible in dnsmasq.

I installed isc-dhcp-server, but don't know how to configure it.

Another question is that when using dnsmasq, the clients use the router's IP as dns server, e.g. 192.168.1.1. How to implement this in isc-dhcp-server?

Thanks for help!

In your dhcpd.conf file, set "option domain-name-servers 192.168.1.1;" to have them use the router as a DHCP server/proxy.

HOWEVER, you will need to make sure that you *actually have* something handling DNS on your router, which means either keeping dnsmasq running with dhcp disabled, or replacing dnsmasq with something else (like bind), or routing port 53 tcp/udp to an external DNS server, or just hand off a public dns server to the client (like 8.8.4.4).

Now since you've probably got some kind of special routing scheme in mind with breaking up your IP addresses like this (otherwise why waste your time with it?), you'll probably give your equipment an easier time by breaking your different allocations properly along subnet boundaries... i.e. 1-->63, 64-->127, or 1-->127, 128-->255
