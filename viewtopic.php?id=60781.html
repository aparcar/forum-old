USB traditionally connect a "host" and a "device". An ethernet adapter is a "device". The Chromecast is a "host" when it connects to the ethernet adapter.  A host can power a device, but a device is not allowed to power a host.  So what you describe violates the model.  And it looks like that is exactly what Google does here.

Answering your question: For OpenWRT to do the same, you will need
a) a router with an USB port that can function as the device end.  An OTG port would be fine
b) a gadget driver emulating one of the USB ethernet devices supported by the Chromecast. If this doesn't include any of the USB-IF standard protocols (and it might not), then you have a problem....
c) an USB cable allowing you to violate the power flow requirements - your OTG port will not allow this.
