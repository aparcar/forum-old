While luci doesn't show 5GHz channels in the config interface, i got the following results while playing with radio0 (5GHz radio):
- default trunk image with b43, self compiled, produces a Kernel oops (ticket 13012)
- trunk image with brcmsmac, self compile, b43 unselecteed - Scanning detects 5GHz channels, working 5GHz client mode

Since the board isnt recognized (see " Found board: "Unknown Board" " above), maybe some driver selection is wrong.
Anyhow i am a little bit confused with all those broadcom issues/which driver to use on brcm47xx platform
b43 should support the radio (but maybe its another hw revision?) - see linuxwireless-wiki ; with both AP and Client (STA) mode
brcmsmac ( linuxwireless ) only supports Client (STA) mode
both driver support the same chipset BCM43224 that is used in the WNDR3400v1 but there are apparently hardware (revision) differences
