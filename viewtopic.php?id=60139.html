Hello, I have a TP-Link TL-WR842N/ND v2 with OpenWRT version "OpenWrt Barrier Breaker 14.07 / LuCI Trunk (0.12+svn-r10530)" installed on it.  It has been setup as a repeater ("Routed Client with relayd (Pseudobridge)", as described in the wiki here: http://wiki.openwrt.org/doc/recipes/relayclient ).  The configuration files are identical (with the exception of ESSIDs) to those described in the wiki.

My issue is that when the repeated network goes down, the access point also goes down.  I am in a situation where the repeated network is sporadically available (buggy Verizon MiFi device that cannot be fixed with a software update or hardware replacement) but I still want the home network to be accessible over WiFi.  Is there any way configure the router so that the Access Point will stay up even if the repeated network goes down?

Post #2
dziny
6 Oct 2015, 04:47

Your problem is not solvable if you only use 1 radio. Your solution is to have 2 radios - one for AP other one for the incoming internet connection, that way AP can stay up even if the second radio becomes unavailable. The easiest solution of implementing this is to plug in USB wifi adapter into your router. You might get one with RP-SMA for stronger signal.
