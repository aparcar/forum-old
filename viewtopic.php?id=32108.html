Setup: dir825 with 8gb usb stick running 10.03.rc5
          usb mounts to /mnt/usb (ext4)
          opkg.conf reads: dest usb /mnt/usb

Scenario:  /~$ opkg -d usb install vsftpd
                installs successfully all files in correct places, but won't start on reboot.
                no link is made in /etc/rc.d/ (not sure if thats intentional or not, made the link all was good).

                /~$ opkg -d usb install umurmur-openssl
                installs successfully but some things are messed up, also won't start on reboot.
                problem is none of the files made it to /etc/.
                found directory /mnt/usb/etc containing all missing files for umurmur
                as a sidenote i also found some files for ettercap and another i forget which i have not tested yet.
                until i figure this out i put everything properly in /etc/ for now and then delete /mnt/usb/etc and
                symlink it to the real /etc/.

Question: which is the proper behaviour?

Summary: some things go into /etc/ and some go into the (-dest)/etc directory (which is fail for init.d scripts).

This is correct behaviour, using the "-d" option only changes the install location. If the package contains files to be placed in /etc/rc.d they will now be in /path/to/storage/etc/rc.d. If you want links, you'll need to install them yourself.

You could just make a script that overlays your external storage to the root filesystem using symlinks.
