I am using openwrt on a number of devices.  The menuconfig stuff (packages,etc) is the same for all the devices.  What is different is the files I include in each build.  As I do it right now I copy the files in for one device, run make, wait, copy the .bin files where they need to be, change the contents of files/, make again, etc.  This is a pretty cumbersome process. Is there a way to have this done automatically so I get something like device-sysupgrade-files1.bin, device-sysupgrade-files2.bin, device-sysupgrade-files3.bin?  Alternatively is there a way to do "make repackage" so that all the compile steps are skipped?

Thanks for the help!

Yes there is:

TOPDIR=$PWD make -C target/linux install

Will repackage without recompiling.
