I just dug it out now since I actually have wireless devices with AC and I'm disappointed with the cheap AC WAP I just bought (11 MB/sec booo!), so I'd like to resurrect this baby, and restore the stock firmware (or use Open-WRT's, whichever has better 5 GHz AC) to take it's place.

To my delight it still boots up normally and actually grabs an IP via DHCP, but then it won't respond to pings while connected to a LAN interface :-/

The TFTP server asks for ArcherC7v2_tp_recovery.bin on 192.168.1.66 (which is the non-routed interface of my desktop, connected directly to LAN port 1 on the C7), but the server never seems to serve up the file, so it just continually asks for it. The server itself works because I was able to grab the file using the Windows TFTP client. I tried using the stock Open-WRT firmware, the stock uncut firmware and the stock cut firmware. None of them download. I have no idea what the issue is since this is my first time ever doing this.

[Dr. Frankenstein] IT'S ALIVE!!!!!! [/Dr. Frankenstein]

For some reason when I had it connected directly from my pc to the C7 TFTP didn't wan't to work for the C7, even though it knew where the server was. Just for the hell of it I connected the C7 to my switch and the NIC configured for 192.168.1.66 to the switch also and disabled my main NIC, then started up the TFTP server on my desktop and INSTANTLY it pulled down the image, flashed it, rebooted and I'm back at the login page with Chaos Calmer greeting me!
