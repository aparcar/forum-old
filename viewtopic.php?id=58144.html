The D option you did may work, for example to connect to a web server on the 49.x remote machine, point your web browser to http://<router LAN IP>:1080.  In other words you have opened port 1080 on the router to go to the remote host.  The ssh system will try to guess the appropriate port to use remotely.

If that does not work you can use the L option to explicitly specify the port on the remote end, instead of -D 1080 use -L 1080:localhost:80

This site is useful for basic documentation on Linux command line stuff:
http://linux.die.net/man/1/ssh
