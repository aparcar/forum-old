Be careful with 45066 trunk build. It might be very reboot happy...
45061 and 45064 broke WiFi causing constant reboots for me. Then 45069 fixed the problem.
https://dev.openwrt.org/ticket/19329

EDIT:
Two additional duplicate bugs have already been reported about the same reboot loop issue:
https://dev.openwrt.org/ticket/19343
https://dev.openwrt.org/ticket/19337

The issue only affects wifi driver, so turning wifi off may enable recovery and flashing another firmware version.
