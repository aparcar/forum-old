Well, I managed to make this work - but I had to go to DD-WRT, use their much clearer tutorials, understand what I was doing, and then come back and apply to my openWrt settings.
For the benefit of others, I'll post my settings here - since there's some weird stuff I still don't understand fully but maybe some will be able to explain.

Context (again):

- my router is plugged into another router (router #1). Router #1's IP address took 192.168.1.1, so I had to set mine (#2) to a different IP
- I'm using a WRT54GL v.1.1
- cables : router #1 LAN port into router #2 LAN port 4 (I have set 4 as a switch later on...)

Configs :

- (Network -> Wifi) configured an SSID with normal settings but chose "LAN" for the "Network" option
- Disabled encryption for the time being, also disabled the firewall for now

- (Network -> Interfaces -> LAN) : Static IP (192.168.1.10) Netmask (255.255.255.0) Gateway (192.168.1.1 - the #1 router's address) DNS Server (192.168.1.1 - same here)
- Disable DHCP server (checked)

- (Network -> Interfaces -> WAN) : Static IP (192.168.1.201 - having an IP outside of the main router's DHCP pool is important here, apparently) Netmask (255.255.255.0) Gateway (192.168.1.1 - the #1 router's address) DNS Server (192.168.1.1 - same here)
- Disable DHCP server (checked)

- (Network -> DHCP and DNS) "This is the only DHCP in the local network" -> unchecked, since router #1 is serving the addresses

- (Network -> Switch) VLAN ID-> 1, PORT 1 -> off, PORT 2 -> off, PORT 3 ->off, PORT 4 ->TAGGED, PORT 5->off, CPU->off
- I'm not convinced this does anything, will test...

Hoping this can clear up things for others...
