yes you need to change some values in the binary file in order to upgrade.  I have successfully built a trunk for a DIR-615-E4 and upgraded my E3 device, or you can choose the DIR-600-A1 build of an older backfire RC build.  What is nice of these routers is that you can unbrick them with the recovery mode (hold reset button until power flashes once a second AMBER).

the best way is to get a hex editor (I use HHD Hex Editor Neo), and at the end of the firmware file, modify the identifier to:

AP99-AR7240-RT-091105-01

you only need to do this for the change from factory firmware to OpenWRT.  afterwards you can use the sysupgrade image.

It seems that the linked firmware .bin is a bit too long for the DIR-615/E3's liking. I just did this dd command to make it the same length as the stock firmware (since it's all padded with 0x00 anyways):

dd if=openwrt-ar71xx-dir-600-a1-squashfs-factory2.bin of=openwrt-ar71xx-dir-600-a1-squashfs-factory.615e3.bin bs=1 count=3538968

and added the AP99-AR7240-RT-091105-01 as the last bytes of the file. It even updated using the stock D-link web interface.

It might be that this one has extra MTD partitions, thus limiting the size of the firmware image a bit.

Edit: It only leaves you with 676KB left in your jffs2 after installation, you can safely remove kmod-usb-core (which will also remove kmod-usb-ohci and kmod-usb2).
I'd imagine you could probably knock out a few of those extra partitions, though it would leave you without the ability to just reload the stock firmware.

I was able to update firmware of a DIR-615 HW E3 directly when using attitude adjustment.

What I did was download the E4 openwrt firmware both factory squash and generic E4. I opened in Hex edit to verify the identifier needs to be: AP99-AR7240-RT-091105-01.

I then when to the Dlink site and downloaded the latest copy or the E3/E4 firmware here:

http://support.dlink.ca/ProductInfo.aspx?m=DIR-615

I downloaded the firmware and opened it in Hex Edit, went to the end of the binary and confirmed the identifier was  this value:

AP99-AR7240-RT-091105-01.

Then I updated the router with the latest Dlink firmware I downloaded from above link file named:

DIR-615_REVE_FIRMWARE_5.12B05.bin.

Then I just did update as normal, booted router to recovery mode then pointed to file:

openwrt-ar71xx-generic-dir-615-e4-squashfs-factory.bin

and did update. Changed PW and then did next update this file:

openwrt-ar71xx-generic-dir-615-e4-squashfs-sysupgrade.bin

Unit is running fine, I can access and create folder in the internal memory now
