Hello,

I have cloned the 15.05 / CC OpenWrt repository via git some time ago, gone through the 'make menuconfig' process where I have selected all the modules and packages that I wanted to include in my image and successfully built the firmware for my router model.

I would now like to build a new image for my router, but this time from the main development tree (bleeding edge, if you will).

How would one proceed in that case without having to go through the make menuconfig process from scratch and manually select everything again?

Is this possible? Can I just copy my .config file without risk of losing or damaging something (I'm guessing no, which is why I ask)?

What's the best practice?

Thanks!

Post #2
makro
23 Jan 2016, 21:04

Go to your CC tree, run 

./scripts/diffconfig.sh > ~/myCCconfig

It outputs the difference between the default config and your own config, instead of the complete config. Copy the myCCconfig file in your home directory to your trunk directory, save it as .config. Run make defconfig, it will take your specific config choices and build a full config based on it. Then run make to build it.

It might not work completely, packages may have renamed config options and stuff like that, but you can try it out.
