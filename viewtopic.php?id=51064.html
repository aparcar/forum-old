Hi there,

In the ECB600 firmware files list, there is a file that does contain a little hash code. You can download the latest firmware from their site and continously extract the files via 7zip to a new folder, you'll end up with a file directory similar to Downloads\ECB600 v1.5.2\ecb600-int-v1.5.2\ecb600-int-v1.5\openwrt-senao-ecb600-root. Within the final folder, there is a file within the bin/ folder, named login.sh. Open that file up in notepad++ or some sort of simple text editor of your choice. Scroll down in the file until you see something like:

console()
{
    mkdir -p "$download"
    trap "killall eval.sh 2>&- >&2; echo" 2

    if [ "$1" = "1d68d24ea0d9bb6e19949676058f1b93" ]; then
        exec /bin/ash --login

  • Now, on the ECB600 device, login to the telnet console.

  • Once you are logged into the telnet console, put in this hash code: 1d68d24ea0d9bb6e19949676058f1b93

  • Press enter.

  • It should toss you into the busybox terminal window now, which is pretty much root access on the device (at last check).

Happy browsing, editing, tweaking the device.


For whatever it's worth, I am unsure if this voids the warranty or anything of the sorts - if it does, not my fault. I too, was limited by the basic terminal and the basic web UI, so this was what I found after hours and hours of poking around with the firmware files. Killing off the telnet daemon and enabling ssh was one of the first things that I did on the device (among vlan and general additional poking around to see the full functionality of the device).
