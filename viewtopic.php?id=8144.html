snowyowlster wrote:

I don't think you can expect to build kamikaze natively on a Mac.

You can. In fact, it's one of the supported platforms for building OpenWrt.

I haven't built Kamikaze on OS X in a while, so things may be a little different now. Here's what I had to do: First, you need a case-sensitive file system to build OpenWrt, but the default file system on Macs isn't case-sensitive. The easy way around this problem is to use an appropriately formatted disk image or separate disk partition. You'll also need to install Xcode, which is available free from Apple. This will get you gcc and a few other development tools. You'll still need to install or upgrade a few more easily obtainable tools. When you try to build Kamikaze, it will tell you what's missing. In addition, you will need to get a new version of sed. Some of the scripts don't work with the version of sed included with OS X.

That ticket is almost two years old. Kamikaze 8.09 builds cleanly on OS X. (I just did it.)

VMware Player doesn't run on a Mac. It's for Windows and Linux only. There's VMware Fusion, but that costs money. In any case, I think the main reason to build natively is to avoid the overhead of having another OS running alongside OS X, or the nuisance of dual booting.

Anyway, it looks like the easiest thing to do is to install MacPorts, and use it to install the ports coreutils, findutils, gawk, getopt, wget, and md5sha1sum. I'm not entirely certain sed is needed, but I installed the port gsed +with_default_names anyway.
