For client bridge to work in OpenWrt, you must create WDS link between the two ap's. I think you can install OpenWrt to the master ap to evaluate the result and compare it to DD-WRT.

Given the diagram, it doesn’t even look like WDS is needed, because the second unit (labeled Client Bridge, .2) doesn’t need to act as an AP. It should be enough to put it into STA mode and have it associate with the first (labeled AP, .1). Be sure to disable dnsmasq on .2.

BBQmyNUTZ,

I had the same problem with trying to Bridge a WNDR3700 with dd-wrt. Let's just say that the connection was inconsistent (some packet loss and routing issues). I ended up setting up a WDS system instead.

Instead of setting the network up like this:
AP (wndr3700v1 5ghz stock firmware) -------> Client Bridge (wndr3700v2 5ghz dd-wrt)
I used WDS:
WDS AP (wndr3700v1 5ghz stock firmware) -------> WDS Station (wndr3700v1 5ghz dd-wrt)

And it works flawlessly. Instead of intermittent request timed out and destination host unreachable the connection is steady. I didn't get around to trying openwrt but from what I've read setting up a client bridge on an atheros based chipset is no cake walk. This may present a better solution for you. Good luck.
