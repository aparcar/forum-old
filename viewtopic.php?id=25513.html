Hi all,

Running Kamikaze r20367.

I am trying to backup my OpenWrt system to a networked drive.  The problem is that tar completely ignores the "exclude" switches.  I tried putting the excludes on the tar command line without success.  I then tried using an exclude file and using the -X switch.

Example command line:

tar -cvpzf /mnt/nas-smb/hollis-nginx/$HNAME.tar.gz -X /scripts/backup.exclude /

Contents of /scripts/backup.exclude:

/mnt/*
/overlay/*
/proc/*
/rom/*
/tmp/*
/dev/*

I also tried it without the wildcard (for example: "/mnt").

No luck at all.  Does anyone have an idea of what's going on?

Thanks

The tar command built into busybox has somewhat limited functionality.  Try:

# opkg update
# opkg install tar

To get a more full featured tar binary.
