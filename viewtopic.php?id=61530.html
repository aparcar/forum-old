Have you tried something like ip scanner plus or "arp -a" to see if the ip address might be other than what you expect?

Or look in your dhcp server to confirm what ip address it was given.

Did you flash it using sysupgrade to retain your settings, or put the old config files in your compiled firmware?

For a few devices, I have found that serial console input doesn't work properly unless I allow the device to boot up fully before plugging the serial in.  You might try that. Obviously since you are seeing the boot log, you don't have TX and RX swapped, but are you sure you have RX properly connected (from the device point of view)? Sure your serial uses 3.3 volt levels and not 5V levels?
