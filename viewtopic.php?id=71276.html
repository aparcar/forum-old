Hi guys, sorry if this isn't the right section to post in.

I'm currently trying to capture an open wireless access point that I do not have hardware/admin access to, providing free (legal) internet. I'd like to run a long cable inside my house, then provide another AP for devices to connect to the internet.

I have been using Gargoyle 1.9.2 on a WNDR3800 with some success as a totally wireless repeater, but I was wondering if openWrt might be better suited/have more options for the full system.

I intend to use an old router-modem inside the house. I am assuming this can be simply bridged without needing to install openWrt.

If this sounds possible so far, what specific modes/configurations would you recommend for openWrt?

Use a routed client on the outdoor unit.  All routing decisions will be made there.  Then as you said, you can connect an Ethernet cable and bridge in almost any router with stock firmware (you need to be able to turn off DHCP) as a dumb AP.

Hey there.

Of course you can do as mk24 said, adding the internal router as "dumb AP" and let the external router do everything.
But from a security point of view, I'd rather not have an unsecured rj45 wire across my backyard which belongs to the "Windows Home Zone" and all my personal devices respond to pretty much unrestricted.

I'd suggest to have the external device just as mk24 suggested and add an internal router that uses the rj45 uplink coming from the backyard as regular WAN connection. You can avoid NAT since you control both OpenWRT devices, but I'd definitively not put my primary gateway out in the open ragen.

And yes, I hate wifi cameras mounted outside of my house for the very same reason: They usually provide link access to a network that should not be exposed.
