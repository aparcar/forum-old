Did you try to connect using Ethernet (not wireless)?

Also, when unpluging/repluging the router, you say all LEDs turned on, then only Power LED. No WPS LED flashing? This is the characteristic "failsafe" indication, so if you don't have that, then your only chance is by flashing over the U-Boot serial console.

For this, you need to solder a TTL UART to USB adapter (like a cheap CA42 Nokia cable with one end cut).

Then follow the detailed instructions for flashing over serial console from the TL-MR3020.

Don't take it badly: reading the Wiki is a ALWAYS a good thing to do: a lot of people spent a lot of efforts to put it together, so others can find information rapidly and learn from it. This is an invaluable source of information, and you should try to find information in it yourself, BEFORE posting into the forum, or there will be a lot of useless duplicate posts there, for a large  loss of time and energy.

So, please, take the time to read it carefully: it is really worth it, you will learn a lot about OpenWrt in there!

Back to the cable, you can take any UART TTL <=> USB cable, but you will have to figure out the wiring by yourself.

All you need are the RX/TX/GND signals, so only 3 wires. The best way to find out for an unknown cable is to take a multimeter: the GND is the lowest voltage and is stable. The TX from the cable that should be wired to the router's RX will have varying voltage when you type in characters in the PC terminal emulator. The router's TX (cable RX) is the latest one if you have only 3 wires, or you may have to experiment...
