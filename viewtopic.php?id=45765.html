Those units of TP-Link Archer C7 that were sold/produced at least until "now" - or some months ago
contain 802.11ac card with hw1.0 ("AR1A" printed on chip)

that chip was only produced in "small" quantities and is not getting support in mainline linux/ath10k for now because it needs different handling to hw2.0 that is made now

hw1.0 with drivers will only print:

[   12.290000] ath10k_pci 0000:01:00.0: BAR 0: assigned [mem 0x12000000-0x121fffff 64bit]
[   12.300000] PCI: Enabling device 0000:01:00.0 (0000 -> 0002)
[   12.300000] ath10k: ERROR: qca988x hw1.0 is not supported
[   12.310000] ath10k: Unsupported chip id 0x043200ff
[   12.310000] ath10k: could not register driver core (-122)
[   12.320000] ath10k_pci: probe of 0000:01:00.0 failed with error -122

see: https://forum.openwrt.org/viewtopic.php?id=44201

with hw2.0 you should possibly get those drivers to work

There is some hw2.0 already in the market but:
- Unclear how to differentiate TP-Link Archer C7 because atm you need to open your case to check
- Unclear if newer unit gets the same name or how to differentiate them
- Has TP-Link produced units with new hardware already ? - Where will they sell those first? ; It can take some time to restock to "new" units that are usable in OpenWrt.

Afaik ath10k / atheros is the only one providing OpenWrt usable AC driver in Linux atm
no Realtek (RTL8812AR), no Broadcom(BCM4360), no Mediatek
binary/closed drivers are used by DD-WRT project don't  know about support on that side
