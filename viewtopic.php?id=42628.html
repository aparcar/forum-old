what you DO with SSH is that you connect to your router and debug and configure it in various ways as described in the pages that i linked to above.   OpenWRT is a (somewhat limited) linux system and you're logged into a "shell" on it.  You are using a "command line interface" (CLI) that is the predecessor of  "graphic user interfaces" (GUIs).

Presumably you are already doing so since you at least tried to install luci and such things.


You have some existing network.  Does that network not have DHCP working on it ?

Your new router/AP (the d-link) will by default just act as a client on that network.  If dhcp is working on the network the d-link would just grab an address like any other computer that you plug into the network.... you shouldn't need to set it at all.

For example, on a standard set-up it would look like this:
root@OpenWrt:~# uci show network.wan
network.wan=interface
network.wan.ifname=eth1
network.wan.proto=dhcp

---
You seem to have set the IP manually.  if you go that route you'll also have to set a gateway and dns servers (as shown wiki.openwrt.org/doc/howto/internet.connection  under "Ethernet")

reading between the lines of what you said above you can probably set both the gateway and the DNS server to
192.168.1.1,  but if you have DHCP on the existing network it's probably easiest to just revert the d-link's wan port to using DHCP so you don't have to worry about it.

anyway to see if you have networking  try using traceroute from your ssh session

traceroute -n 4.53.158.42

then to see if your  dns is working just
nslookup nytimes.com

Hmm... you've kind of jumped into the deep end trying to do openwrt. 

Anyway you need to understand:
IP address ,  netmask and gateway (they work together).    (and "gateway" is usually short for "default gateway")

and what DNS does and why you need it.

and what DHCP does.

And you also need to understand what  NAT is ("network address translation").   And realize that at least one of the boxes you already have (probably the cisco "switch" is probably not a switch but a home "router" that is doing NAT already.  no guarantee of that, but it's prob. what's going on especially since you mentioned something about it doing QoS). 

For your first experiments you're probably best off plugging in the d-link into your "cisco switch"

(btw, setting up QoS in a reasonable manner to achieve your wifi throttling goals is about 2 levels of understanding above what it takes to get  things basically working)
