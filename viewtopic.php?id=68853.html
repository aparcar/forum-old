Unplug and replug the wifi dongle then check dmesg (status -- kernel log) you should see "New high-speed USB device" at the bottom.  This means the basic USB system is working on your VMl, the kernel connects to the device but doesn't know how to drive it.  Thus you need to install the driver for your chip.  Usually this is rt2800.

The next milestone is to use the iw list command.  If it shows a bunch of info about your device, the driver is working.  Then you just need to use LUCI to set up a wifi AP.

Determine which chip is in your device.  The two four digit "vendor" and "device" hex numbers are unique to the chip type.  Go to wireless dot kernel dot org and find the driver that matches.  Then look in the kmod section of openwrt package list and install driver.

Drivers for nearly all chips are already in the mainstream kernel.  Compiling one from the manufacturer is unnecessary.
