Its possible

Downloaded OpenWRT trunk r27255
build target:
CONFIG_TARGET_ar71xx=y
CONFIG_TARGET_ar71xx_generic=y
CONFIG_TARGET_ar71xx_generic_DIR600A1=y

Build it as a DIR-600-A1.

When it comes time to upgrade from factory firmware, I use the DIR-615's bricked-router recovery (hold the reset for 10-15 seconds when you boot).   In order to upload the firwmare via this interface, some hex bits at the end of the image need to be changed (using a hex editor, I used Neo).  I believe this is the version or hardware revision check that D-link does for firmware images.  I modified [openwrt-ar71xx-generic-dir-600-a1-squashfs-factory.bin]

(mine had): AP91-AR7240-RT-090223-00

TO:  AP99-AR7240-RT-091105-01  (this was in a D-Link factory firmware file I had)

Once uploaded, you can upgrade to other images using the regular openwrt-ar71xx-generic-dir-600-a1-squashfs-sysupgrade.bin files.


PROS: you can run OpenWRT on the DIR-615revE3

CONS: the LEDs don't work (other than the power LED). 

But other than the leds, wireless speeds are good, interface is snappy and the whole system seems reliable.


Thanks to a gentleman in another thread, I was able to build a AR71xx build supporting PPPoE+Wireless+DHCPv6+radvd for both the DIR-615revE3 and the TP-LINK WR741NDv1.9 that fits in 4MB of flash.  So I finally have a cheapish (30$) IPv6 router that supports DHCP-PD, PPPoE, and has a stateful-firewall-aware 2.6 Linux kernel.  The problem was that radvd compiles to a humungeous 400k, with a sanitization of the regexps in scanner.l, the daemon now compiles to 75k, allowing for dhcp6c/dhcp6s to fit in 4MB.  Stellar!!!!!!!!!!!!!!!!!!
