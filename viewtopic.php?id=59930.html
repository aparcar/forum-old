I have managed to setup a ipv6 tunnel in openwrt barrier breaker using hurricane electric services. One of my goals was to setup a webserver in a linux machine that could be reached using an ipv6 address. Initially I couldn't manage that, until on firewall settings, i set all the zones to acept input, forward, etc. Unfortunately this opened undesirable ports, for example my openwrt webgui is visible everyone if you type the ip address of such device.

How can i open specific ports in openwrt, for example port 80 and forward it to my ipv6 server behind the openwrt router ? How can i block undesired ports ?

i want to open or close specific ports without making my whole router or my servers fully exposed to ipv6 internet.

soyuz88 wrote:

How can i open specific ports in openwrt, for example port 80 and forward it to my ipv6 server behind the openwrt router ? How can i block undesired ports ?

When using IPv6 you usually can avoid forwarding ports since all devices typically have globally routable addresses. If you want to do port forwards (aka DNAT) it should be possible, but you may need to write custom rules (I am not sure).

You should be able to allow traffic to your webserver by adding a rule to Firewall - Traffic Rules:

config rule
    option target 'ACCEPT'
    option src 'wan'
    option family 'ipv6'
    option dest 'lan'
    option proto 'tcp'
    option name 'webserver'
    option dest_ip '<ipv6 address>'
    option dest_port '80'
