The problem is most likely that you messed up the lan interface and so it has no address. I suggest that you reset it using the TFTP method and try again. You'll want to bridge LAN/WAN and set it to get an address from your modem through DHCP.

Sounds like you did something funny with the network config. It is e.g. possible that you have configured the router to use another subnet (defaul 192.168.1.x) than the one for which your PC gets address via DHCP from the modem.

Use Open failsafe mode to get access to it, or use Netgear TFTP recovery mode to flash a new firmware and revert to default settings.

http://wiki.openwrt.org/toh/netgear/wnd … lsafe.mode
http://wiki.openwrt.org/toh/netgear/wnd … lsafe.mode

dosch wrote:

3. I set both LAN and WAN to DHCP client mode.

Correct?

Wrong.
Set a fixed IP address for the router, but from the correct subnet (defined by your modem).
You want to know what is the exact address of the router.

I am not quite sure how you intend to connect the router & the modem, but easiest might be to start with the vanilla config, meaning that you connect a LAN port of the router to the modem (and leave router's wan empty). After you get that to work, you can then start experimenting with wan settings.
