Squonk

The "dd if=/dev/mtd0ro of=/tmp/mtd0" above reads the raw r/o "/dev/mtd0" partition as input file (if=) and write it to the "/tmp/mtd0" output file (of=).

So the resulting file is "/tmp/mtd0".

Now, you have to find a way to transfer it from the router to your PC or another machine using http, ftp, tftp or scp file transfer.

If you have the uhttpd web server running on the router, you can just create a symlink in the web root directory that points to this file, e.g.

ln -s /tmp/mtd0 /www/mtd0

Then, fetch the file using your PC browser at http://192.168.1.1/mtd0.

It is not very easy, but possible.

The real problem is that you must have the uboot partition writable. This can be done by modifying the "drivers/mtd/tplinkpart.c" in the Linux kernel like described in:
https://forum.openwrt.org/viewtopic.php … 92#p162992

Now, the easiest way is to take an older 1.6 bootloader binary file and modify the MAC address at offset 0x1fc00 in it to match yours (see http://code.google.com/p/wr703n-uboot-w … -failsafe/ to see a programmatic way to do this, you can also edit the file using a good hex editor )

It looks like TP-Link changed the way they initialize the Ethernet switch in U-Boot 1.1.4 (Sep 25 2012 - 09:04:47) in TL-WR703N v1.7 compared to previous U-Boot 1.1.4 (Sep 26 2011 - 11:13:45) from TL-WR703N v1.6.

We worked on this problem with nebbia88, but we don't have a working solution yet.
