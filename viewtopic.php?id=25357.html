Can you name some of the "killer features" of OpenWRT?

- writeable filesystem
- ability to easily (re)compile firmware images
- full shell access
- lots of packaged 3rd party software
- advanced routing
- advanced firewalling

One thing I look for is to have different WPA2 pre shared keys for different MAC addresses. Does OpenWRT feature this?

No. The brcm-2.4 target (used for the wrt54gl) relies on a proprietary, binary only wireless driver which certainly does not support that. I've not seen such a feature in mac80211 either.

Is there a WWW based demo that shows what the router configuration of OpenWRT looks like and what options there are?

None that I'm aware of. You can search for screenshots though. The both gui projects for OpenWrt are X-Wrt and LuCI. You can also run OpenWrt in VMWare or Virtualbox to test it.

What is the difference between OpenWRT and dd-wrt.com?

DD-Wrt has its focus on readymade, binary firmwares and is more geared towards typical enduser requirements. Many claim its easier to use (at least for the basics) but it is almost impossible to compile it yourself or to add other software (there is optware but it is always hackish). OpenWrt follows the DYI approach, focus is on open sources and customization.

X-Wrt on OpenWrt has had PPTP and OpenVPN setup through the web-interface for years.

The killer features are excellently summed up by jow, but for a new user it is hard to understand the implications. If you are familiar with linux-distributions it is probably easier to grasp. OpenWrt is a full blown linux-distribution with a package manager and repository. For space constrained devices like routers it makes an enormous difference. It means that you can easily tailor the software stack to your needs, and even on an entry level wrt54gl obtain enterprise grade functionality. This is in stark contrast to DD-Wrt's predefined images (although work is under way it seems: http://www.dd-wrt.com/wiki/index.php/Ipkg).

I can take a wrt54gl I have here as an example of the differences making OpenWrt unique:
-I changed from the default web-interface after installation to X-Wrt because I like X-Wrt better (easily installed through the package system)
-I installed ntp to sync the time on the router (critical feature for business usage, couple of clicks installation and config  in the web-interface)
-I doubled the transmit power for wireless, making the wireless available throughout the building
-I installed matrix tunnel for encrypted SSL connection to the web-interface (as everything else, one-click installation in web-interface)
-I made the router accessible from the big bad internet, so that I can access and modify my network from anywhere through the router which is always on (and rock stable I might add)
-I installed wake-on-lan so that I can turn on any machine in the network from the router (combining it with cron is invaluable if you want regular back-up of your PCs)
-I set up dnsmasq as the DNS server on the network (admittedly available on many routers)
-Needless to mention I guess is the numerous firewall rules for various needs, worth mentioning though is that no web-interface I have seen has been more user-friendly in this respect than X-Wrt

Of course, if you opt for a router with more power and a USB port, only your imagination puts a limit to what you can do with it. Before you know it you will start being interested in RAM, storage and CPU capacity of your router ;-)

Message? There is only one linux-distribution for routers, and it is OpenWrt. It will grow with you, others will not. For me, that is all I need to know. For you I think the fact that you made it to this forum is enough to assume that you are better off with OpenWrt. I also want to take this opportunity to thank everybody who have contributed to make OpenWrt what it is today.

I would also add:
-An open development process. You can see what bugs are open. Bugs also tend to be fixed (thanks to our developers). With linksys you are out of luck, except if fixing your bug will bring them more money than not fixing it. If your product is no longer sold... you understand what I mean.
-An up to date linux system. Many other 3rd party firmwares for the WRT54G use outdated kernels which are probably unsecure or lack features.
-A very clean build system. You can compile your own firmware much more easily than with DD-WRT.
-It is free. You don't have to donate for a "pro" version with added features. Nothing is obfuscated by the developers so that you can't change it.

Then, what you want to do with it is your choice, as you can install packages. For me, I wanted to run an Asterisk PBX (VoIP phone server) as well as OpenVPN. OpenWRT allows me to do that.
