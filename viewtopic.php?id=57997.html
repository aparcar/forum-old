I have noticed the same when switching between CC versions or from trunk to cc &back.
/etc/opkg.conf gets preserved as a config file, while it should change to reflect the new download repo.

I would classify that as a feature, although I am not quite sure how long it has been that way.
Earlier the possible location error was not that visible, but the new package list signing key system highlights the possible error more prominently.

The proper opkg.conf can be found as /rom/etc/opkg.conf


EDIT:
Well, it has been that way (preserved as a config file at sysupgrade) since r12319 in 2008...
