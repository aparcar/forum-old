Hi everybody,

I'm not sure this is related to openwrt kamikaze or not, but I wanted to obtain your view on this as I'm out of options.

The problem is that my dreambox 500 (a satellite receiver with ethernet port, running linux kernel 2.6.9), is unable to communicate with the Buffalo running Kamikaze (8.09, r14511). What happens is that:
- Buffalo running OK, all ports operational, any cable connected notebook can obtain IP address with straight or cross ethernet cable
- Dreambox is connected and powered off (mac address begins with 00:09)
- Dreambox is powered on, ethernet lights go on for 2-4 seconds, then go off completely
  - During this phase, openwrt arp command recognizes the MAC address of Dreambox, then dissapears or becomes 00:00:00:00:00:00 then dissapears
- Dreambox ethernet lights are now off, no IP received from DHCP

If I put a dummy 10 mbps HUB between the dreambox and buffalo and give dreambox a manual IP, it's able to communicate with Buffalo, but with extremely slow connection and 80% packet loss on 32 byte ping.

I looked for:
- Cable problems, none
- DHCP problems, none
- ethtool settings (changes did not work)
- vlan settings (did not work or I could not do it)

Current settings of Buffalo:

   config 'switch' 'eth0'
           option 'vlan1' '0 5'
           option 'vlan0' '1 2 3 4 5*'

I also tried Buffalo's own firmware, it also does not work. However, if I plug the Dreambox to another ADSL router, it works! This is what makes me crazy. There's something unexplainable between the Buffalo and the Dreambox.

Thanks for any insight,
