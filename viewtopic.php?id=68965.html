OpenWrt's default configuration use 192.168.1.x for the LAN network, if the main router also uses 192.168.1.x for its LAN network, then your OpenWrt device is configured as 192.168.1.x on both the LAN and WAN networks, and that is going to cause trouble.

You can configure the OpenWrt device to use 192.168.0.x for the LAN, or you can even configure it as a dumb access point.

slow_speed wrote:

Thank you for the responses.  I thought the same thing.  However, I could not see how to set it up as a dumb access point, nor could I see how to change the address range to 0.x instead of 1.x.

Dedicate fixed IP on main router. Let it be 192.168.1.5. Make sure static lease is present or "1.5" is not in the dynamic assignment range.

Edit LAN interface. Change protocol to "Static address". Fill the fields.
In DHCP server section check "Ignore interface".
In DHCP server - ipv6 settings - disable "Router advertisement service", dhcpv6, NDP proxy.
Save&apply.

Edit WAN interface, uncheck "Bring up on boot", save&apply, stop interface. WAN port is no more used.

Connect any lan port of the main router to any lan port of the secondary router. Thats it. Secondary router now serves as additional switch + wifi bridge. Static address is useful in case of main router failure.
