I got a TL-WDR3600 a few weeks ago and 2 days ago I flashed Openwrt AA 12.09.1 (r41803 built from source) on it.

Before Openwrt I was using the stock firmware. Stock is really cool, it supports the "hardware NAT" feature on the switch (not big deal unless you have more than 300mbps internet), works as NAS, includes a media server and it also supports printing and scanning but only with compatible printers (mine isn't on the list).

My printer not being supported and the fact that I want custom DNS stuff and OpenVpn is what made me decide to flash openwrt in it. I can live without hardware NAT, by the time 300mbps+ internet service becomes available at affordable prices down here we'll probably have developed telepathy and a collective mind already.

I'm only using 2.4Ghz wifi because I have only 2 devices that support 5Ghz and one of them is most of the time powered off, but I did test 5Ghz a little bit and didn't notice anything weird.

So far it's been working alright but since I've been only running Openwrt for a few days and booting it a lot to test different wifi settings and stuff I can't tell you how stable it will be on the long run.

The 12.09 image on the downloads page doesn't seem to have the "wifi patch" my v1.5 needs according to the supported versions list here: http://wiki.openwrt.org/toh/tp-link/tl- … d.versions
That's why I compiled it myself, which is not hard to do, it was pretty easy and fun (experience compiling linux kernels helps a lot).

It also looks like BB 14.07 is close to be released (RC3 was uploaded this week) so I'll wait for 14.07 final before I start messing with the USB ports and my plans to connect printer/scanner and HDD to this little monster.

Overall I'm satisfied with Openwrt on my WDR3600.

Cheers.

@winbond which OpenWRT version did you install?
If it's the AA 12.09 in the downloads page keep in mind it may not have the "wifi patch"mentioned here: http://wiki.openwrt.org/toh/tp-link/tl- … d.versions

Quoted from r39392 (note: wdr3600 and wdr4300 are pretty much the same):

Message:

    ar71xx: fix bad RX sensitivity on newer TL-WDR4300 boards

    On recent TL-WDR4300 boards the external LNAs of the 2.4GHz
    interface are connected to GPIO lines. Because these GPIO
    lines are disabled by default, the RX sensitivity of the
    device is quite bad.

    Setup the GPIOs of the external LNAs to fix the issue.

    Signed-off-by: Gabor Juhos <juhosg@…>

That is why I compiled my own firmware from source.
The BB release candidate already has that patch so it should be ok.

I've been streaming video from a PC (wired) to my Raspberry Pi (wifi) without any issues. Even 1080p played without a glitch.
