To have fully functional firmware for DIR-300 you have to compile trunk yourself, as the 8.09 release lacks support for the switch chip used in this router. I don't know if the trunk is fully functional at this time, a month ago it didn't work for me. My solution was to compile 8.09 release with the patches for IC-175 switch, as described in the Wiki. Works good since then, notably better than stock firmware. I have one advice though - don't use LuCI on an active duty router, it's too heavy both on memory and CPU time. Use it for initial setup only. Otherwise response time will be very slow and the roter will reboot sooner or later. Instead, install mc (Midnight Commander) and use it to edit configuration files directly. I even think it should be included in the pre-built images, because FS navigation and text editing is MUCH handier than using ash command line and vi editor.
