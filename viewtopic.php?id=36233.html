Hello,


I want to start a discussion about the idea of adding support for /etc/profile.d/

The idea goes that /etc/profile.d/ is a directory to drop shell files that are sourced near the end of the /etc/profile.


[...]

if [ -d /etc/profile.d/ ]
then for FILE in /etc/profile.d/*.sh
     do . ${FILE}
     done
fi

The above is just an example implementation, and obviously could be achieved other ways.
The idea is to source all the .sh files in the /etc/profile.d/.
Other shells, namely [t]csh, could have another extension, say *.csh.


What this brings is the ability of people to leave the version of /etc/profile unmodified from trunk/stable, and simply drop their customization into the profile.d dir.

This is already being done on my major Linux distro's and is generally considered a good idea.

So for people who build their own custom version of trunk they would not need to modify the default profile.
They would be able to globally override the defaults early in the files they drop in profile.d.



Please discuss.

Thanks :-)
