Basic system logging should work out of the box.
dmesg shows kernel log
logread shows system log

/sbin/logd is the log daemon, which should be visible in the process list.

Aside from the wiki being incorrect in a number of areas my real issue was a buildroot configuration issue. After realizing that logd was already loaded as part of ubox it becomes obvious there is no need to select ANY logging daemon options in busybox or elsewhere except logread in busybox. One wouldn't know that without digging through revision notes and the sources .

Anyhow, I hope this info helps someone else.
