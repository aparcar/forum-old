I have been having an issue where nmap only resolves hostnames on some networks, and not others. I initially suspected a network's router was to blame, but apps like fing are able to resolve hostnames no problem, so I think there is something iffy about my Openwrt configs. Can anyone points me in the right direction as to what parameters are involved in resolving hostnames?

Initially, "boguspriv" was set to 1, and that caused hostname resolution to always fail, so changing it to 0 has gotten resolution to work on some networks. Any other options that look suspicious?

EDIT: Will adding "option dns "Gateway IP" (192.168.0.1 for example) force Openwrt to use my main router as the DNS sever? Because the network's router shows hostnames in its DHCP list. I don't have access to a network where reverse lookup is NOT working, so it's a little difficult to test at the moment. 


Here is my /etc/config/dhcp:

config dnsmasq 'setup'
    option domainneeded '1'
    option boguspriv '0'
    option filterwin2k '0'
    option localise_queries '1'
    option rebind_protection '0'
    option rebind_localhost '1'
    option local '/lan/'
    option expandhosts '1'
    option nonegcache '0'
    option readethers '1'
    option leasefile '/tmp/dhcp.leases'
    option resolvfile '/tmp/resolv.conf.auto'
    option authoritative '0'
    option dhcpleasemax '40'
    option domain 'openrouter-net'

config dhcp 'lan'
    option interface 'lan'
    option leasetime '120m'
    option start '200'
    option limit '40'
    option dhcp_option '1,255.255.255.0  3,10.130.1.1  6,8.8.8.8,8.8.4.4'
    option dhcpv6 'disabled'
    option ra 'server'

config dhcp 'apspot'
    option interface 'apspot'
    option leasetime '120m'
    option start '200'
    option limit '40'
    option dynamicdhcp '1'

config dhcp 'wan'
    option interface 'wan'
    option ignore '1'

config odhcpd 'odhcpd'
    option maindhcp '0'
    option leasefile '/tmp/hosts/odhcpd'
    option leasetrigger '/usr/sbin/odhcpd-update'

/tmp/resolv.conf.auto:

# Interface apspot
nameserver 8.8.8.8
# Interface wan
nameserver 192.168.0.1
search local.tld
