ok i got 2.4 for wrt54g to compile. you need to to change the following under "Advamnced configuration options (for developers)->toolchain options->

binutils must be 2.18

For anyone finding this thread using Google, I have come across this problem exactly at the shader.c location.

This is probably where the kernel compilation firstly involves the compiler binary.

So, go check your toolchain:

$ du -s staging_dir/toolchain-mipsel_gcc*
24 staging_dir/toolchain-mipsel_gcc3.4.6
113356  staging_dir/toolchain-mipsel_gcc4.1.2

You see the toolchain using gcc3.4.6 (kernel 2.4) was empty whereas gcc4.1.2 (kernel 2.6) was available. (I had compiled a kernel 2.6 before).

To remedy this situation, first perform a ''make toolchain/install'' before ''make''-ing.
This will create a another toolchain with the needed version of gcc:

$ du -s staging_dir/toolchain-mipsel_gcc*
95900   staging_dir/toolchain-mipsel_gcc3.4.6
113356  staging_dir/toolchain-mipsel_gcc4.1.2

In any case, cleaning up tmp (inside the svn checkout) can also help remedy certain problems.
