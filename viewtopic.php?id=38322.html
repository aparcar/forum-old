every router supported by openwrt should be able to run this.
Maybe you should look into a router with detachable antennas, but this depends on the size of your cafe, and the spatial arrangement.
The WNDR3800 is a nice device but might be an overkill, unless you want to run (and can run) your private network on 5GHz only. Many clients especially smartphone only support 2.4GHz.

But you can run multiple ssids (with seperate subnets and firewall-zone) on just one physical wifi without any issues, on allmost any device.
So even a device like the TL-WR741ND which you will get for 15euros will work for this. if your internet-access has more than 50Mbit you might get a device with a 300Mbit Wifi e.g. TL-WR841ND or TL-WR842ND. But there is really no need to spend 80euro for this, and you can't upgrade the antennas of the WNDR3800 if you have issues with signalquality

For newbies I recommend the D-Link DIR-825.  It supports concurrent 2.4Ghz and 5Ghz traffic and has decent CPU horsepower and RAM (64MB).  It is supported in the stable branch (Backfire).

Personally I use the the TL-WDR4300.  It better specs than the DIR-825 but is only supported on the main trunk currently and requires text-mode provisioning without the LuCI GUI.   Not recommended for newbies (yet).

A word of warning.  There's a possible problem with backfire with regard to this.  I've had to disable my guest wireless SSID since updating from kamikaze to backfire 10.03.1.  On my system, as long as the second wireless network existed, it would always eventually stop routing between the (primary) LAN and (primary) wireless after about a half an hour. It acts as if "isolate" is on, even though it works fine for a while (and I've explicitly set isolate to '0').  The only way to get it back is to restart networking or reboot. 

I had the obvious straightforward setup involving a bridged lan+wifi pair for the main local network and another bridged lan+wifi pair for the "guest" network.  Both were identically configured except for SSID, interface names and passwords.

Deleting the extra wireless network made the problem go away, where no amount of messing about with the firewall or wireless did.  (I even tried explicitly allowing lan-lan forwarding, but it made no diffference.)  I haven't diagnosed this problem in detail though I want to get it solved eventually.

Multiple SSIDs worked great for me in kamikaze.   This is with an Alix 2D13 (i586) + DCMA-82 MiniPCI (Atheros AR5006) and madwifi.  (Had too many other problems with ath5k, including poor throughput.)  Your mileage may vary.
