puma14
21 Nov 2015, 20:46

The direct download without opening the device would be very nice and fast, but the documentation seems to have not the final state. And uboot might not work with the ethernet. There are some newer versions, but until now I did not find any confirmation that the fast way would work at all.

Therefore I recommend to flash uboot and the first image with the serial method. It is slower and you have to prepare more. But once you have installed uboot with an openwrt image, you can update it later over ethernet.

I have an Easybox 802 running now with an uptime of 38 days.
LuCI, Samba for sharing a USB stick, two WLAN segments (if I need just the this one IP range for tftp) and DSL of course.
POTS and ISDN not needed/neglected.
The box has (eye of the beholder) nice design.  I only would have expected the red plastic strip between the halves
strip to glow.

But it was a stony path, ending with Barrier Breaker (Chaos Calmer is not working because of PCI problems causing wifi errors, see #18369, #19856, #20997):
- flashed uboot according to https://wiki.openwrt.org/toh/arcadyan/arv752dpw (first load uboot with the brn-file, then flash the nor-file),  with the files from my own build,
- kernel/dts needs other partition boundaries on flash to fit a newer and bigger uboot (as it is in ARV752DPW22.dts, lesson learnt
kernel command line does not work for it, only in the device tree it can be changed),
- lzma on flash crashes, lzo works
(see https://wiki.openwrt.org/toh/arcadyan/arv7510pw for patches, I have also applied the DSL patch)

Opening the case for getting access to the serial port requires a mix of violence and patience.
There are screws at the bottom. These have to be removed of course. But they have more or less a decorative role.
A dozen of hook/tab pairs still hold the halves together.
However I have milled a porthole, when the box was open (no tab broken during opening),
on the backside, on the same height as the upper edge
of the recess for the TAE ports, between the recess and the type plate (sticker).
On a closed box I would make the hole with a hot nail or something like a heated knive (no chips falling inside).

Mihai wrote:

Well... I bricked this router, guys. I was trying to install uboot via serial and after that I mannaged to kill it somehow.

The nice thing about Lantiq based routers is the fact, that it is almost impossible to brick a device. You can hand over the u-boot directly to the cpu with UART as described by https://wiki.openwrt.org/toh/arcadyan/arv752dpw#uart.

You will have to build OpenWrt from source and enable the creation of the ram boot image by your self (menuconfig => "Boot Loaders" => uboot-lantiq-arv752dpw_ram)
