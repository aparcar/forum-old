Hello,

I'm not a networking expert and I'm having a problem connecting using telnet...

The set up is sort of complicated....

I have openwrt running on MR3040 router, and I have it set up with WiFi enabled.

I also have ANOTHER wireless home network for internet, that is in Access Point Mode, so I actually have TWO wireless networks.

The openwrt is in WiFI Access Point Mode with static IP address 192.168.1.99, and also openwrt is  simultaneously connected wirelessly to my home network SSID in sort of Bridge mode with DHCP assigned IP address of 192.168.2.32, address assigned to openwrt by my home DHCP WiFi router.

To re-cap... so the openwrt router is in Access Point with its own SSID and wireless network, and its simultaneously connected to my home WiFi network SSID in some kind of a bridge mode.

So openwrt router actually has TWO IP addresses, the static one 192.168.1.99 and the DHCP one assigned by my home WiFI network 192.168.2.32.

If I connect my computer DIRECTLY to the openwrt WiFi network SSID, I can browse the internet through openwrt WiFI, which is connected to the internet through the bridged connection of my home network WiFi SSID. I can also telnet to openwrt with static IP 192.168.1.99 no problem.

However the problem happens when I connect my computer to my home WiFi network SSID, and then try to telnet to the openwrt using the DHCP assigned IP address 192.168.2.32 of the openwrt, the telnet connection is REFUSED!

I can ping openwrt at 192.168.2.32, but telnet wont connect at that ip address when I'm connected to my home WiFi network SSID.

Seems telnet works one way, but not the other, and I need it to work through my home network.

Since I can ping it, that means something else is the problem, some kind of configuration.

Anybody have a clue why the connection is refused?

Thanks
rough

Well it is not completely the proper way, but the easiest (also missing the config for the other router to have an other answer).

under
config zone
    option name        wan

change option input REJECT
to option input ACCEPT

At this point assume this router is behind an other router and not directly connected to the internet. (otherwise this is a terrible idea).
