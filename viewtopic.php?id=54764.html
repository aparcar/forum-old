It works exactly like on regular computer.  You use the ssh comand with -L or -R options to create port tunnels.

For security reasons it is designed to not be possible to store the password.  Set up public key authentication if you need to establish connections non-interactively.

If you are being immediately logged out, the issue is likely on the remote end, not yours.

There is no web GUI (or is one possible) - by definition, ssh is a command-line tool.  As indicated before, you can certainly set up a public key relationship to prevent being asked the password, making it a "one-line" connection.

Your question isn't really OpenWRT specific and is more of a "how do I use command-line SSH?", which may be better suited for a more appropriate forum, BTW.

Rodney

The remote machine did not open a shell for you.  Try explicitly launching a remote shell by using 

ssh <user>@<host> sh

or other command instead of 'sh' that you have permission to run on the remote.
