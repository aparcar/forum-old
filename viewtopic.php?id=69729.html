config wifi-device 'radio0'
        option type 'mac80211'
        option channel '11'
        option hwmode '11g'
        option path 'platform/bcm2708_usb/usb1/1-1/1-1.4/1-1.4:1.0'

        option country 'GB'

config wifi-iface
        option device 'radio0'
        option network 'lan'
        option mode 'ap'

This is the error message that i get when i try to start the wlan0 again

root@OpenWrt:~# ifconfig wlan0 up

Are you sure that your device really uses the ath9k_htc driver?

The AR9271/ AR7010 wlan chipset has been EoL for quite a while now and newer specimens of the TL-WN722N apparently use Realtek's rtl8192eu (rtl8xxxu) chipset instead.

Does dmesg show the driver loading?  Does iw list show any wifi active?

You need to be sure to always plug it into the same physical USB socket.  That is part of the path that wifi detect finds.

If you erase or rename /etc/config/wireless, wifi detect will again look for it.
