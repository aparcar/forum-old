Once you change the password for the root user, the TELNET connection gets closed, and you need to use SSH instead. However, the permissions you get using SSH are exactly the same you had using TELNET. Now, if you cannot edit files at "/etc/config" with the "root" user, then probably something went wrong during the installation.
